# 线性表 - 栈和队列 (Stack & Queue)

### 1. **概念名称**

> **一句话核心概念总结:** 栈遵循“先进后出 (LIFO)”，队列遵循“先进先出 (FIFO)”，双端队列支持两端插删，常用于顺序控制、缓冲和任务调度。

### 2. **详细解释**

- **栈 (Stack):** 只在一端操作，`push` 入栈，`pop` 出栈，`peek` 取栈顶。典型用于“回退/撤销”“嵌套/配对”场景。
- **队列 (Queue):** 一端入队、一端出队，操作通常为 `offer/enqueue`、`poll/dequeue`、`peek`。用于排队、缓冲、调度。
- **双端队列 (Deque):** 两端都可插删，既能当栈也能当队列；例如 Java `ArrayDeque`。
- **实现方式：**
  - **顺序/数组实现：** 下标或环形数组管理头尾，缓存友好；需要扩容或容量约束。
  - **链式实现：** 动态伸缩、不需连续内存；指针开销、缓存不友好。
- **时间复杂度：** 在头尾操作均可做到 O(1)；如需按索引访问则退化为 O(n)（队列/栈通常不支持随机访问）。
- **线程安全：** 单线程用 `ArrayDeque`/`LinkedList`；多线程可用 `ConcurrentLinkedQueue`、`LinkedBlockingQueue`、`ArrayBlockingQueue` 等。

### 3. **代码示例**

```java
import java.util.ArrayDeque;
import java.util.Deque;
import java.util.Queue;

public class StackQueueDemo {
    public static void main(String[] args) {
        // 栈：用 Deque 代替过时的 Stack
        Deque<Integer> stack = new ArrayDeque<>();
        stack.push(1); // 压栈
        stack.push(2);
        System.out.println(stack.peek()); // 2
        System.out.println(stack.pop());  // 出栈 2

        // 队列：用 ArrayDeque 或 LinkedList 实现 Queue
        Queue<String> queue = new ArrayDeque<>();
        queue.offer("A"); // 入队
        queue.offer("B");
        System.out.println(queue.peek()); // A
        System.out.println(queue.poll()); // 出队 A

        // 双端队列：两端都能操作
        Deque<Character> deque = new ArrayDeque<>();
        deque.addFirst('x'); // 头插
        deque.addLast('y');  // 尾插
        System.out.println(deque.removeLast()); // y
    }
}
```

### 4. **常见应用场景**

- **栈：** 函数调用栈/递归展开，表达式括号匹配与语法解析，撤销/回退，深度优先搜索。
- **队列：** 广度优先搜索 (BFS)，任务排队与调度（线程池工作队列），生产者-消费者模型，IO/网络缓冲。
- **双端队列：** 滑动窗口最值（单调队列），LRU 双向链表，既当栈又当队列的双端任务调度。

### 5. **优缺点/注意事项 (可选)**

- **数组实现：** 快、缓存友好；需处理扩容或容量限制，环形数组避免搬移。
- **链式实现：** 动态伸缩、插删 O(1)；指针占用额外空间，局部性差。
- **API 选择：** Java 中首选 `ArrayDeque` 代替 `Stack`/`LinkedList` 作为栈/队列；并发场景使用专用并发队列。
- **越界与空队/空栈：** `pop/remove` 可能抛异常，`poll`/`peek` 返回 `null`，选用合适接口。
