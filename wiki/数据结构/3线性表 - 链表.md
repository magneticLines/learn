# 线性表 - 链表 (Linked List)

> **一句话核心概念总结:** 链表是一种**链式存储**的线性表，它通过一系列**节点 (Node)** 串联而成，每个节点不仅存储数据，还存储了指向下一个节点的**指针 (Pointer)**，从而在**非连续的内存空间**中组织数据。

## 1. 详细解释

- **节点 (Node):** 节点是链表的基本组成单位。一个最简单的节点包含两个部分：
    1.  **数据域 (Data):** 用于存储实际的数据元素。
    2.  **指针域 (Next):** 用于存储下一个节点的内存地址。通过这个指针，我们将分离的节点“链接”成一个链条。
- **内存布局:** 这是链表与数组最根本的区别。数组需要一块**连续**的内存空间，而链表的节点可以散布在内存的任何位置。这就像一个寻宝游戏，你找到当前的宝箱（节点），它会告诉你下一个宝箱的藏宝图（指针）。
- **头节点 (Head) 与 尾节点 (Tail):**
    - **头节点:** 是整个链表的入口，我们通过头节点才能开始遍历整个链表。通常会有一个 `head` 变量专门指向第一个节点。
    - **尾节点:** 链表的最后一个节点。它的指针域通常为 `null`，表示链表的结束。

## 2. 链表的类型

### 2.1 单向链表 (Singly Linked List)
这是最基本的链表形式。每个节点只有一个指针域，指向其后继节点。因此，你只能从头到尾单向遍历。

### 2.2 循环链表 (Circular Linked List)
与单向链表类似，但其尾节点的指针不再指向 `null`，而是指向**头节点**。这形成了一个环形结构，从任何一个节点出发都可以遍历整个链表。常用于需要循环处理的场景，如轮询调度算法。

### 2.3 双向链表 (Doubly Linked List)
这是功能更强大的链表。每个节点有两个指针域：
- **`next` 指针:** 指向后继节点。
- **`prev` 指针:** 指向前驱节点。

这使得链表可以双向遍历，并且在进行删除操作时更加高效，因为可以轻松找到待删除节点的前驱节点。Java 中的 `java.util.LinkedList` 就是一个双向链表。

## 3. 数组 vs. 链表 (Array vs. Linked List)

| 特性         | 数组 (ArrayList)                               | 链表 (LinkedList)                                      |
| :----------- | :--------------------------------------------- | :----------------------------------------------------- |
| **数据结构** | 线性表，顺序存储                               | 线性表，链式存储                                       |
| **内存空间** | **连续**的内存空间，必须是整块的               | **非连续**的内存空间，可以分散在各处                   |
| **随机访问** | **O(1)**，通过 `地址 = 首地址 + 索引 * 元素大小` 计算 | **O(n)**，必须从头节点开始逐个遍历才能找到目标元素     |
| **插入/删除** | **O(n)**，需要移动后续所有元素                 | **O(1)**，只需要修改目标位置前后节点的指针链接         |
| **查找元素** | **O(1)** (按索引) / **O(n)** (按值)               | **O(n)**，无论按索引还是按值，都需要遍历                 |
| **内存开销** | 较小，主要是数据本身                           | 较大，除了数据外，每个节点都有额外的指针开销         |
| **适用场景** | 读多写少，数据量固定，需要频繁的随机访问       | 写多读少，数据量不固定，需要频繁的插入和删除         |


## 4. 代码示例 (Java - 手写一个单向链表)

```java
/**
 * 手动实现一个简单的单向链表来理解其工作原理
 */
public class SimpleLinkedList {

    // 内部节点类
    private static class Node {
        int data;   // 数据域
        Node next;  // 指针域

        Node(int data) {
            this.data = data;
            this.next = null;
        }
    }

    private Node head; // 头节点

    // 构造函数
    public SimpleLinkedList() {
        this.head = null;
    }

    // 1. 在链表头部插入节点
    public void addFirst(int data) {
        Node newNode = new Node(data);
        newNode.next = head; // 新节点的 next 指向旧的 head
        head = newNode;      // head 更新为新节点
    }

    // 2. 在链表尾部插入节点
    public void addLast(int data) {
        Node newNode = new Node(data);
        if (head == null) {
            head = newNode;
            return;
        }
        Node current = head;
        while (current.next != null) { // 遍历直到找到最后一个节点
            current = current.next;
        }
        current.next = newNode; // 将原尾节点的 next 指向新节点
    }

    // 3. 删除头部节点
    public void removeFirst() {
        if (head != null) {
            head = head.next;
        }
    }

    // 4. 打印链表
    public void printList() {
        Node current = head;
        while (current != null) {
            System.out.print(current.data + " -> ");
            current = current.next;
        }
        System.out.println("null");
    }

    public static void main(String[] args) {
        SimpleLinkedList list = new SimpleLinkedList();
        
        System.out.println("在头部添加元素 10, 20:");
        list.addFirst(10);
        list.addFirst(20);
        list.printList(); // 输出: 20 -> 10 -> null

        System.out.println("在尾部添加元素 5:");
        list.addLast(5);
        list.printList(); // 输出: 20 -> 10 -> 5 -> null

        System.out.println("删除头部元素:");
        list.removeFirst();
        list.printList(); // 输出: 10 -> 5 -> null
    }
}
```

## 5. 常见应用场景

- **实现其他数据结构:** 栈和队列都可以用链表来实现，以获得动态扩展的能力。Java 的 `LinkedList` 同时实现了 `List` 和 `Deque` (双端队列) 接口。
- **LRU 缓存淘汰算法:** Least Recently Used (最近最少使用) 算法可以用一个双向链表+哈希表来实现。链表保证了元素的顺序，哈希表保证了 O(1) 的查找效率。
- **内存管理:** 操作系统中的动态内存分配，可以用链表来维护未被使用的内存块（空闲链表）。
- **软件中的“撤销/重做”功能:** 用户的操作可以被存储在一个双向链表中，方便前后移动。
- **邻接表:** 在图论中，表示稀疏图时，使用“数组+链表”组成的邻接表，比使用邻接矩阵更节省空间。

## 6. 源码/底层实现分析 (Java)

`java.util.LinkedList` 是 Java 集合框架中链表的官方实现。

- **底层结构:** 它是一个**双向链表**。其内部的 `Node<E>` 静态内部类包含三个字段：`E item` (数据), `Node<E> next` (后继指针), 和 `Node<E> prev` (前驱指针)。
- **核心属性:** `LinkedList` 类本身维护了 `size`, `first` (头节点引用), 和 `last` (尾节点引用) 这几个关键属性。
- **高效的头尾操作:** 因为直接持有 `first` 和 `last` 引用，所以在链表头尾添加或删除元素 (如 `addFirst`, `addLast`, `removeFirst`, `removeLast`) 都是 O(1) 的操作，只需修改指针即可。
- **低效的索引查找 (`get(int index)`):**
    - `LinkedList` 的 `get(index)` 方法是其性能痛点。它无法像数组那样进行数学计算定位，只能遍历。
    - 但其源码中有一个**优化**：它会先判断 `index` 是靠近链表的前半部分还是后半部分。
        - 如果 `index < (size >> 1)` (即小于长度的一半)，它就从 `first` 节点开始向后遍历。
        - 否则，它就从 `last` 节点开始向前遍历。
    - 这个优化可以将最大遍历次数减半，但时间复杂度**依然是 O(n)**。
