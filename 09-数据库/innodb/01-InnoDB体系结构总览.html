存储引擎内核

InnoDB 体系结构总览

前面几个 Tab 讲的 SQL 执行、事务 ACID、MVCC，全部要靠 InnoDB 这套存储引擎在底层落地。这一篇先画一张"全景地图"，把内存和磁盘上的各个部件摆清楚，后面每一篇再逐个深挖。

为什么要先建立全景图

事务 Tab 里讲过：持久性（Durability）要求事务提交后数据不能丢，MVCC要求能读到"历史版本"的数据（参见 ../transaction/06-MVCC原理.html）。这些能力都不是抽象概念，而是由 InnoDB 内部一组具体的内存结构和磁盘文件配合实现的。 如果不知道这些部件叫什么、各自负责什么，后面讲 Redo Log 为什么能保证持久性、Undo Log 为什么能实现多版本，就成了"知其然不知其所以然"。

类比：把 InnoDB 想象成一个仓库管理系统。内存里的 Buffer Pool 是"前台货架"（常用货品放这里，取用飞快）；磁盘上的表空间是"仓库总库存"（容量大但存取慢）；Redo Log 是"出入库流水账"（先记账后搬货，保证账目不丢）；Undo Log 是"退货凭证"（记录改动前的样子，随时能退回去）。

整体架构：内存结构 + 磁盘结构

InnoDB 的体系结构官方文档分为两大块：In-Memory Structures（内存结构）和On-Disk Structures（磁盘结构）。内存结构负责加速，磁盘结构负责持久化，二者通过刷盘（flush）和恢复（recovery）机制联系在一起。

InnoDB 存储引擎
  ├─ 内存结构（In-Memory）
  │    ├─ Buffer Pool（缓冲池：缓存表数据页 / 索引页，减少磁盘 I/O）→ 详见 02
  │    ├─ Change Buffer（写缓冲：缓存对非唯一二级索引的修改）→ 详见 05
  │    ├─ Adaptive Hash Index（自适应哈希索引：为热点页自动建哈希）
  │    └─ Log Buffer（日志缓冲：暂存 Redo Log 记录，减少刷盘次数）→ 详见 03
  └─ 磁盘结构（On-Disk）
      ├─ 表空间 Tablespace（系统表空间 / 独立表空间 .ibd：存放表数据、索引、Undo 等）
      ├─ Redo Log 文件（ib_logfile：物理日志，保证持久性）→ 详见 03
      ├─ Undo Log（回滚日志：保存修改前的旧版本，用于回滚与 MVCC）→ 详见 04
      └─ Doublewrite Buffer 文件（双写缓冲：防止页写入过程中损坏）→ 详见 06

和事务 Tab 的呼应

事务 Tab 讲 ACID 时（见 ../transaction/01-事务的ACID特性.html），"持久性"和"原子性"听起来像是两条抽象的保证，但落到 InnoDB 内部，其实分别对应两个具体的日志：

  • 持久性（Durability） 由 Redo Log 保证：只要日志刷盘成功，即便数据页还没落盘、进程崩溃，重启后也能靠日志"重做"出正确的数据页。
  • 原子性（Atomicity） 很大程度上由 Undo Log 保证：事务失败或主动回滚时，靠 Undo Log 把已经做的修改撤销，让事务要么全做要么全不做。
  • MVCC（见 ../transaction/06-MVCC原理.html）读取"历史版本"时，走的正是 Undo Log 链——这也是为什么 Undo Log 不能被无限制地及时删除，必须等所有可能用到它的事务都结束。
面试高频："MySQL 的 InnoDB 引擎，一条 UPDATE 语句执行时都发生了什么？" 标准答案的骨架就是把这张全景图串起来：先在 Buffer Pool 里定位/加载数据页 → 记录 Undo Log（为回滚和 MVCC 做准备）→ 修改内存中的数据页（此时是脏页）→ 记录 Redo Log 到 Log Buffer → 事务提交时按 innodb_flush_log_at_trx_commit 策略刷盘 Redo Log → 脏页由后台线程或 Checkpoint 机制异步刷回磁盘。

常见误区

误区："数据是不是每次 COMMIT 都会立刻写到 .ibd 数据文件里？" 不是。真正立刻（同步）落盘的是 Redo Log（在默认配置下），而承载真实数据的表空间文件是异步、批量、延迟刷盘的，由后台线程根据 Buffer Pool 的脏页比例、Checkpoint 进度等条件决定何时刷。这正是 WAL（Write-Ahead Logging）机制的精髓：把"随机写数据页"转换成"顺序写日志"，具体原理在 03 篇详细展开。

核心价值总结

  • 内存结构负责快：Buffer Pool、Change Buffer、Adaptive Hash Index 都是为了减少磁盘 I/O 这一最大的性能瓶颈。
  • 磁盘结构负责稳：表空间存真实数据，Redo Log 和 Undo Log 分别为持久性和原子性/MVCC 兜底。
