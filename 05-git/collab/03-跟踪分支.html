跟踪分支:让 git pull / push 知道"去哪"

为什么有时候 git push 一条命令就够,有时候却要写全 git push origin main?为什么 git status 会告诉你"领先远端 2 个提交"?答案都在"跟踪分支(tracking branch)"这个机制里。

📖 来源:《Pro Git》 第 3 章「Git 分支」·「远程分支」

一、什么是跟踪分支 / 上游分支(upstream)

跟踪分支是一个和某个远程分支建立了对应关系的本地分支。这个被对应的远程分支,就叫它的上游(upstream)。

比如你的本地 main 跟踪 origin/main,那么:

  • 在 main 上直接敲 git pull / git push,Git 就知道默认去 origin/main,不用每次写全。
  • git status 会告诉你 "Your branch is ahead of 'origin/main' by 2 commits"——因为它知道跟哪个远程分支比。
为什么需要这层关系:Git 有本地分支和远程分支两套,数量可以完全不对等。"跟踪关系"就是显式地告诉 Git:"我这个本地分支,平时就是和那个远程分支同步的",省去每次都指明 remote 和分支名。

二、跟踪关系从哪来:clone 会自动建立

git clone 时,Git 会自动:创建本地 main,并让它跟踪 origin/main。这就是为什么你 clone 完直接 git pull 就能用——跟踪关系已经替你建好了。

查看跟踪关系
git branch -vv          # 每个本地分支后面 [origin/xxx] 就是它的上游,还带 ahead/behind
# 输出示例:
# * main   a1b2c3d [origin/main] 修复登录 bug
#   dev    e4f5g6h [origin/dev: ahead 2] 新功能开发中
#   tmp    99xxaa1 本地实验分支    ← 没有 [] 说明没设上游

三、手动设置上游:git branch -u

你自己 git checkout -b feature 新建的分支,默认没有上游。这时你直接 git push 或 git pull,Git 会报错说不知道推/拉到哪。设置上游有几种等价写法:

三种设置上游的方式
# 方式一:给"当前分支"设置上游(-u 是 --set-upstream-to 的简写)
git branch -u origin/feature

# 方式二:切分支时顺带设(--track,常在从远程分支切出时)
git checkout --track origin/feature   # 建本地 feature 并跟踪 origin/feature
git checkout feature                  # 简写:远端有同名分支时 Git 自动 --track

# 方式三:第一次推送时用 -u 一并建立(最常用!)
git push -u origin feature            # 推送 + 设 origin/feature 为上游,一步到位
最实用的一条:新建本地分支后,第一次推送就用 git push -u origin 分支名。-u(即 --set-upstream)在推送的同时把上游关系建好,之后这个分支上就能光敲 git push / git pull 了。这是日常最高频的用法。

四、@{upstream}:引用上游的简写

Git 提供了特殊引用 @{upstream}(可简写 @{u})代表"当前分支的上游",省得写全远程分支名:

@{upstream} 的妙用
git log @{u}..            # 我比上游多了哪些提交(还没推的)
git log ..@{u}            # 上游比我多了哪些提交(还没拉的)
git diff @{u}             # 我和上游的差异
git reset --hard @{u}     # 把本地强制对齐到上游(慎用,会丢本地改动)

这些命令不用写死 origin/main,换个分支照样能用,写脚本时尤其方便。

五、本地分支与远程分支不必一一对应

跟踪关系是可以自定义的,本地分支名和远程分支名不必相同:

本地名 ≠ 远程名
# 本地叫 my-fix,但推到远端的 hotfix 分支,并跟踪它
git push -u origin my-fix:hotfix

# 从远程 release-2.0 切一个叫 rc 的本地分支来跟踪
git checkout -b rc origin/release-2.0
易错点:删除本地分支(git branch -d)不会删远程分支;远程分支要用 git push origin --delete 分支名 删。另外,远端分支被别人删了之后,你本地的 origin/xxx 跟踪引用不会自动消失,要 git fetch -p(--prune)清理陈旧的跟踪分支。

小结

  • 跟踪分支 = 与某个远程分支建立对应关系的本地分支;那个远程分支叫它的上游(upstream)。
  • 有了上游,git pull/git push 才知道默认去哪,git status 才能报 ahead/behind。
  • clone 会自动为 main 建立跟踪;自建分支默认没有上游。
  • git branch -u、checkout --track、push -u 都能设上游;push -u 最常用。
  • @{upstream} / @{u} 是"当前分支上游"的简写,配合 log/diff 很方便。
  • 本地名和远程名可不同;删远程分支用 push origin --delete,清陈旧跟踪用 fetch -p。
