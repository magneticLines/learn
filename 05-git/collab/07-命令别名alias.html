命令别名 alias:把常用命令缩到几个字母

每天敲几十次 git checkout、git status,手指很累。Git 内置的 alias 让你把它们缩成 co、st,还能封装一整串复杂命令。这是提升效率最立竿见影的一步。

📖 来源:《Pro Git》 第 2 章「Git 基础」·「Git 别名」

一、别名是什么:git config 里的一段映射

Git 别名不是什么魔法,它就是存在配置文件(~/.gitconfig 的 [alias] 段)里的一条映射:输入某个短词,Git 把它替换成对应的完整命令再执行。

用 git config 定义别名
# --global 写进用户级配置,对所有仓库生效
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit
git config --global alias.st status

# 之后就能这样用了:
git st              # 等价于 git status
git co main         # 等价于 git checkout main
本质理解:别名只是"文本替换"。git co 执行时,Git 发现 co 是别名,把它换成 checkout 再跑。所以别名后面照样能接参数(git co -b dev),它们会原样拼到展开后的命令后面。

二、常用别名清单

别名展开作用
cocheckout切换分支 / 恢复文件
brbranch分支操作
cicommit提交
ststatus查看状态
unstagereset HEAD --把文件移出暂存区
lastlog -1 HEAD看最近一次提交
带参数的别名
# 别名可以包含参数,把常用组合固化下来
git config --global alias.unstage 'reset HEAD --'
git config --global alias.last 'log -1 HEAD'

git unstage file.txt   # 展开成 git reset HEAD -- file.txt,把它撤出暂存区

三、明星别名:漂亮的 lg(图形化 log)

原生 git log 又长又难读。几乎每个老手都会配一个 lg,把提交历史压成带颜色、带分支图的一行式视图:

经典的 git lg 别名
git config --global alias.lg "log --graph --oneline --decorate --all"

# 或者更华丽的一版(带作者、相对时间、颜色):
git config --global alias.lg "log --graph --abbrev-commit --decorate \
  --format=format:'%C(bold blue)%h%C(reset) - %C(green)(%ar)%C(reset) %s %C(dim white)- %an%C(reset)' --all"

git lg   # 从此看历史像看地铁图:分支合并一目了然
为什么值得配:--graph 画分支拓扑、--oneline 每提交一行、--decorate 显示分支/标签名、--all 显示所有分支。这四个组合起来才是真正好用的历史视图,但每次全敲一遍不现实——正是别名的用武之地。

四、! 前缀:执行任意外部命令

别名默认只能是 git 子命令。但如果别名以 ! 开头,Git 会把后面的内容当成普通 shell 命令来执行——于是你能调用外部程序、组合多条命令,甚至调用别的工具。

! 前缀:跳出 git 子命令的限制
# 不加 ! 的话,Git 会去找名为 "git-visual" 的子命令,找不到就报错
# 加了 ! ,整段被交给 shell 执行:
git config --global alias.visual '!gitk'            # 启动外部 GUI 工具 gitk

# 用 ! 组合多条命令:一键 add 全部并提交
git config --global alias.ac '!git add -A && git commit'

# ! 里能写 shell 函数,处理参数更灵活
git config --global alias.pushf '!git push --force-with-lease'
注意:带 ! 的别名是在仓库根目录执行 shell 命令的,写路径要留意。另外 ! 别名脱离了 git 子命令的安全边界,能力大也更容易写出意料之外的行为,复杂逻辑建议写成脚本文件而非塞进一行别名。

五、直接编辑配置文件

别名多了,一条条 git config 敲很烦。可以直接打开 ~/.gitconfig 编辑 [alias] 段,一次写完:

~/.gitconfig 的 [alias] 段
# ~/.gitconfig
[alias]
    co = checkout
    br = branch
    ci = commit
    st = status
    lg = log --graph --oneline --decorate --all
    ac = !git add -A && git commit    # ! 开头才走 shell
作用域联系:回忆 Git 三级配置——--system(全机器)、--global(当前用户,即 ~/.gitconfig)、--local(单个仓库)。别名通常放 --global,一次配置处处可用;个别项目专属的别名可放 --local。

小结

  • 别名就是 [alias] 段里的文本映射,输入短词、Git 替换成完整命令再执行,后面可继续接参数。
  • 常用:co/br/ci/st/unstage/last,把高频命令缩到两三个字母。
  • lg = log --graph --oneline --decorate --all,把历史变成好读的分支图,值得必配。
  • ! 前缀让别名执行任意 shell 命令,可调外部程序、组合多命令,但要注意执行目录和安全。
  • 别名可直接编辑 ~/.gitconfig 批量写;通常放 --global 层级处处生效。
