.gitignore:让 Git 假装看不见某些文件

编译产物、依赖目录、密钥、IDE 配置……有些文件天生不该进版本库。.gitignore 就是那份"别管这些"的清单。

📖 来源:《Pro Git》 第 2.2 节「忽略文件」

一、为什么需要 .gitignore

并非项目里所有文件都值得进版本库。以下几类几乎永远该被忽略:

  • 编译/构建产物:Java 的 target/、*.class,前端的 dist/——它们能由源码重新生成,进库只会制造无谓的 diff 和冲突。
  • 依赖目录:node_modules/、Maven 本地缓存——体积巨大且能靠依赖描述文件还原。
  • 敏感信息:密钥、.env、数据库密码——进了库就等于泄露(还会永远留在历史里)。
  • 本地/IDE 配置与临时文件:.idea/、.DS_Store、日志——因人而异,不该强加给队友。

你把这些模式写进项目根目录的 .gitignore 文件,Git 就不再把匹配的文件当作 untracked 提示你,git add . 时也会自动跳过它们。

二、语法规则

.gitignore 示例(Java 项目)
# 以 # 开头是注释
target/            # 结尾带 / 只匹配目录
*.class            # * 通配任意个字符(不跨目录),忽略所有 .class
*.log              # 忽略所有日志
!important.log     # ! 取反:在被忽略的前提下,重新纳入这个文件
/config.local      # 开头带 / 锚定到根目录,只忽略根下的 config.local
doc/**/*.pdf       # ** 匹配任意层级目录
.env               # 忽略环境变量文件
写法含义
*匹配任意个字符(但不跨越目录分隔符 /)
**匹配任意层级的目录
?匹配单个字符
结尾 /只匹配目录,不匹配同名文件
开头 /锚定到 .gitignore 所在目录,不再递归子目录
开头 !取反,把已被忽略的重新纳入

三、匹配优先级:后面的规则能覆盖前面的

规则按从上到下匹配,后出现的规则会覆盖先前的。!(取反)常用来"先忽略一整类,再放行其中一个例外":

取反的正确姿势
logs/*.log      # 忽略 logs 下所有日志
!logs/keep.log  # 但保留 keep.log
取反的坑:如果一个文件的父目录已经被整体忽略(如 logs/),那你无法用 ! 单独把里面的某个文件重新纳入——Git 为了性能压根不会进入被忽略的目录去看里面。正确做法是像上面那样忽略目录内的文件模式(logs/*.log)而不是整个目录,再取反。

四、最大陷阱:对已跟踪文件,.gitignore 无效!

核心规则:.gitignore 只对"尚未被跟踪(untracked)"的文件生效。如果一个文件已经被 git add / 提交过,那它已进入跟踪状态,之后你再把它写进 .gitignore,Git 依然会继续追踪它的改动——忽略规则对它形同虚设。这是让无数新手抓狂的"gitignore 不生效"事件的真相。

解法:先把它从暂存区/跟踪里移除(但保留工作区的本地文件),再提交:

让已跟踪文件重新被忽略
# --cached 关键!只从 Git 跟踪中删除,磁盘上的文件保留
git rm --cached config.local
# 若是目录,加 -r 递归
git rm -r --cached target/
# 然后把它写进 .gitignore,再提交这次"停止跟踪"的改动
git commit -m "停止跟踪本地配置与构建产物"
为什么加 --cached:不加 --cached 的 git rm 会连同磁盘上的真实文件一起删掉。你只是想让 Git "别再管它",文件本身要留着,所以必须 --cached——它只把文件从 index(暂存区/跟踪) 中移除。

五、全局 ignore:属于"你"而非"项目"的忽略

像 .DS_Store(macOS)、编辑器临时文件、你个人 IDE 的配置,这些是你的机器/习惯产生的,和具体项目无关——不该塞进项目的 .gitignore 强加给所有队友。用全局 ignore:

配置全局忽略
git config --global core.excludesFile ~/.gitignore_global
# 然后在 ~/.gitignore_global 里写你个人的忽略规则,如 .DS_Store
实践建议:别从零写 .gitignore。GitHub 官方维护了 github/gitignore 仓库,按语言/框架(Java、Node、Maven…)提供现成模板,IDE 新建项目时也常自动生成。项目专属的忽略放项目 .gitignore(进版本库、全队共享),个人习惯放全局 ignore。

小结

  • .gitignore 让 Git 忽略构建产物、依赖目录、敏感信息、IDE/本地配置等不该进库的文件。
  • 语法:*(不跨目录)、**(跨层级)、结尾 /(仅目录)、开头 /(锚定根)、开头 !(取反)。
  • 规则从上到下、后者覆盖前者;但父目录被整体忽略后,无法用 ! 单独放行其中文件。
  • 最大陷阱:对已跟踪文件 .gitignore 无效,需 git rm --cached(保留磁盘文件)停止跟踪后再提交。
  • 个人习惯性忽略用全局 core.excludesFile,别塞进项目 .gitignore 强加给队友。
