创建仓库:git init 与 git clone

获得一个 Git 仓库只有两条路:把现有目录变成仓库(init),或者把别处的仓库整个拷下来(clone)。

📖 来源:《Pro Git》 第 2.1 节「获取 Git 仓库」

一、git init:把现有目录变成仓库

当你从零起一个新项目、或想给一个已存在的本地目录加上版本控制时,用 git init。

初始化仓库
cd my-project
git init
# 输出:Initialized empty Git repository in .../my-project/.git/

这一步做的唯一事情,就是在当前目录创建一个 .git 子目录——里面装着仓库需要的所有骨架(对象数据库、refs、HEAD、config 等)。此刻还没有任何文件被跟踪,你需要 git add 现有文件再 commit,才算真正开始记录历史。

.git 就是整个仓库。把 .git 目录删掉,这个目录就退回成一个普通文件夹(工作区文件还在,但历史全没了);把 .git 完整拷走,就等于拷走了整个仓库和它的全部历史。这再次印证"Git 是分布式的"——仓库不过是一个目录。

二、裸仓库(bare repository)简述

普通仓库有工作区(能看到实际文件)和 .git 目录两部分。裸仓库(git init --bare)只有 .git 的内容,没有工作区——你不能在里面直接改文件。

它用来干嘛?裸仓库专门用作共享/中心仓库(比如你自己搭的 Git 服务器上的仓库)。原因:如果中心仓库带工作区,别人 push 过来的提交和它自己工作区的状态会打架。所以"用来被 push 的仓库"一律做成 bare。你在 GitHub 上的远程仓库,本质就是一个裸仓库。文件名通常带 .git 后缀,如 project.git。

三、git clone:把远程仓库整个拷下来

如果项目已经存在于某个服务器(GitHub、GitLab、公司内网),用 git clone 获取它的一份副本。

克隆仓库
git clone https://github.com/user/repo.git
git clone git@github.com:user/repo.git my-dir   # 最后一个参数:自定义本地目录名

四、clone 到底拷了什么?——不只是最新文件

这是理解 Git 的又一个关键点。git clone 不是只下载最新的文件快照(那是 SVN 的 checkout 干的),而是:

  1. 把服务器上仓库的几乎全部数据都拉下来——包括每个文件的每一个历史版本、所有分支、所有提交历史。你本地这份是一个功能完整的仓库。
  2. 自动把远程仓库记为一个叫 origin 的 remote(远程别名)。
  3. 把默认分支(如 main)检出到工作区,于是你在目录里看到的是最新版本的实际文件。
常见误解:"我 clone 下来只想要最新代码,历史那么大干嘛都下?"——正因为把完整历史都下来了,你才能离线查看任意历史、离线切换任意分支、离线对比任意版本,而且服务器挂了你这份也是完整备份。这是分布式的核心红利,不是浪费。若确实只要最新一层,可用 --depth 1 做浅克隆(节省流量,但会牺牲部分历史操作能力)。

五、init 之后连接远程:两条路殊途同归

先本地 init 起项目,想推到 GitHub 上的空仓库,只需手动补上 clone 帮你做的那步——加个 remote:

本地仓库关联远程
git init
git add .
git commit -m "initial commit"
git remote add origin git@github.com:user/repo.git   # 手动加上 origin
git push -u origin main                                # 首次推送并建立跟踪关系

看出来了吗——clone 其实等价于 init + 加 remote + 拉全部数据 + 检出默认分支。理解这一点,init 和 clone 就不再是两个孤立命令,而是同一件事的两个入口。

小结

  • git init:在当前目录建 .git,把它变成仓库;此时还没跟踪任何文件,要 add+commit 才开始记历史。
  • .git 目录 = 整个仓库;删掉它退回普通文件夹,拷走它等于拷走全部历史。
  • 裸仓库(--bare)没有工作区,专门用作被 push 的中心/共享仓库,GitHub 远程仓库本质就是裸仓库。
  • git clone 拉下的是完整历史 + 所有分支(不是只有最新文件),并自动设好 origin、检出默认分支。
  • clone ≈ init + 加 remote + 拉全量数据 + 检出默认分支,二者是同一件事的两个入口。
