安装与初次配置:三级作用域的 git config

装完 Git 第一件事是配置身份。理解 config 的三级作用域,能避免"公司项目提交显示成个人邮箱"这类尴尬。

📖 来源:《Pro Git》 第 1.5 / 1.6 节「安装 Git · 初次运行前的配置」

一、验证安装

确认 Git 是否就绪
git --version      # 输出版本号(如 git version 2.43.0)说明已装好

Windows 装 Git for Windows(自带 Git Bash);macOS 常随 Xcode Command Line Tools 附带,或用 Homebrew;Linux 用包管理器(apt install git / yum install git)。装法不是重点,配置才是。

二、git config 的三级作用域:为什么要分级

Git 的配置项来自三个层级的配置文件,范围越窄、优先级越高——窄的会覆盖宽的。这套设计是为了让你"一次设定全局默认,再对个别仓库单独覆盖"。

作用域命令参数配置文件位置影响范围
系统级--system/etc/gitconfig(需管理员权限)本机所有用户的所有仓库
用户级(全局)--global~/.gitconfig当前用户的所有仓库
仓库级(本地)--local(默认)该仓库的 .git/config只影响当前这个仓库
优先级: local > global > system。不加作用域参数时,git config 默认写的是 local(当前仓库)。用 git config --list --show-origin 能看到每个配置项到底来自哪个文件——排查"配置怎么不生效"时特别有用。

三、必做:配置身份(name 与 email)

每一次 commit 都会把作者的姓名和邮箱永久写进提交对象。所以装完 Git 第一件事就是设身份,否则第一次提交时 Git 会拒绝或警告。

设置全局身份
git config --global user.name "Shea Yin"
git config --global user.email "shea@example.com"
高频真实场景:公司项目要用公司邮箱、个人开源要用私人邮箱。做法是——全局设个人邮箱作默认,进到公司仓库后用不带 --global 的命令单独覆盖:git config user.email "shea@company.com"(写入该仓库的 .git/config)。这正是三级作用域存在的意义。

四、其他常用初始配置

编辑器 / 默认分支名 / 换行符
# 指定 commit / merge 时弹出的编辑器(默认可能是 vim,不熟就换掉)
git config --global core.editor "code --wait"   # 用 VS Code,--wait 让 git 等你关窗口

# 新仓库默认分支名(现代约定用 main 而非 master)
git config --global init.defaultBranch main

# 换行符处理:Windows 建议 true,提交转 LF、检出转 CRLF;macOS/Linux 用 input
git config --global core.autocrlf true
换行符陷阱:Windows 用 CRLF、Unix 用 LF。跨平台团队若不配 core.autocrlf,会出现"整个文件每一行都显示被修改"的诡异 diff——其实只是换行符不同。这是跨平台协作的经典坑。

五、别名(alias):config 的隐藏福利

别名让你把常用的长命令缩写成短命令,本质就是往 config 里写一条映射。它是你手速的第一次飞跃。

常用别名雏形
git config --global alias.st status
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit
# 一条漂亮的历史图别名(后面「查看历史」会再讲)
git config --global alias.lg "log --oneline --graph --all"

# 之后就能:git st / git co main / git lg

六、查看与检索配置

查配置
git config --list                    # 列出所有生效的配置(合并三级后的结果)
git config --list --show-origin       # 同时显示每项来自哪个文件
git config user.name                  # 查单个配置项当前的值

小结

  • config 三级作用域:system(全机) < global(当前用户) < local(当前仓库),窄的覆盖宽的,不加参数默认写 local。
  • 必配 user.name 与 user.email——它们会永久写进每个 commit;公司/个人邮箱用 local 覆盖 global。
  • 常配 core.editor、init.defaultBranch main、跨平台的 core.autocrlf(避免整文件换行符 diff)。
  • alias 就是往 config 写命令缩写,能大幅提速;--list --show-origin 是排查配置来源的利器。
