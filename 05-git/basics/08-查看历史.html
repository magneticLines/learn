查看历史:git log 的正确打开方式

裸 git log 一屏塞满没人看。真正好用的,是那几个把历史压缩、画图、过滤的参数组合。

📖 来源:《Pro Git》 第 2.3 节「查看提交历史」

一、git log:沿着 parent 指针回溯历史

git log 从当前 HEAD 出发,顺着每个 commit 的 parent 指针一路往回列出提交,默认最新的在最上面。它展示的正是「暂存与提交」里说的那条由父指针串起来的历史链。

裸 git log 输出
commit a1b2c3d4e5f6... (HEAD -> main)
Author: Shea Yin <shea@example.com>
Date:   Mon Jun 30 10:12:00 2026

    修复登录空指针异常

二、四个最该记住的参数

参数作用
--oneline每个提交压成一行(短哈希 + 摘要),快速浏览历史必备
--graph用 ASCII 线条画出分支/合并的拓扑结构,看清"谁从哪分出、又并回哪"
-p(--patch)每个提交连同它的完整 diff 一起显示,追查"哪次改动引入了这行"
--stat每个提交显示改了哪些文件、各加删几行的统计摘要
黄金组合
git log --oneline --graph --all
# 一行一提交 + 画分支图 + 显示所有分支——日常最常用的一条

* a1b2c3d (HEAD -> main) 修复登录空指针
* 9f8e7d6 添加用户注册接口
| * 3c2b1a0 (feature) 试验新缓存策略
|/
* 7654321 初始化项目
做成别名:把 git config --global alias.lg "log --oneline --graph --all --decorate" 设好,以后一个 git lg 就能看清整个仓库的分支全貌。这是老手和新手手速差距最直观的地方之一。

三、自定义输出格式

--pretty=format
git log --pretty=format:"%h %an %ar : %s"
# %h=短哈希 %an=作者名 %ar=相对时间(如 2 days ago) %s=摘要
# 输出:a1b2c3d Shea Yin 2 hours ago : 修复登录空指针

四、过滤历史:按数量 / 时间 / 作者 / 路径

常用过滤
git log -5                          # 只看最近 5 条
git log --since="2 weeks ago"       # 按时间过滤(还有 --until)
git log --author="Shea"            # 只看某人的提交
git log --grep="登录"              # 按提交信息关键字过滤
git log -- src/App.java             # 只看某个文件/路径的相关提交
git log -S"loginUser"            # "镐"搜索:找出增删了这段代码的提交
易混点 -- 的位置:按路径过滤时,路径前那个 -- 是分隔符,告诉 Git "后面是文件路径,不是分支名或参数"。当文件名和分支名可能重名时,不加 -- 会让 Git 犯迷糊报错。-S(俗称"pickaxe 镐子")非常适合定位"某个函数/变量到底是哪次提交加进来又哪次删掉的"。

五、联系:log 看到的历史 = 提交对象图的遍历

git log 本质是对提交对象组成的有向无环图(DAG)做遍历:每个 commit 是一个节点,parent 指针是边。--graph 画出来的就是这张图的形状。理解这一点,后面学分支、merge、rebase 时就会明白——所谓"分支"不过是指向某个 commit 的一个可移动指针,而 log 只是从某个指针出发把能回溯到的节点列出来。

小结

  • git log 沿 parent 指针回溯,展示由提交对象组成的历史 DAG。
  • 四大常用参数:--oneline(压行)、--graph(画分支图)、-p(带 diff)、--stat(改动统计)。
  • 黄金组合 log --oneline --graph --all,建议做成别名 git lg。
  • 过滤:-n 数量、--since/--author/--grep 条件、-- 路径 按文件、-S 镐子搜代码增删。
  • 路径前的 -- 是分隔符,消除文件名与分支名的歧义。
