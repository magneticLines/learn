打包文件 packfile:松散对象、增量压缩与 gc

"Git 存快照不存差异",那为什么我 1GB 的历史仓库 clone 下来才几十兆?秘密就在 packfile——存储层的差异压缩。

📖 来源:《Pro Git》 第 10.4 章「包文件」、10.7 章「维护」

一、松散对象:朴素但浪费

回顾对象模型:每次你 add/commit,Git 就把新的 blob/tree/commit 各自单独 zlib 压缩,存成 .git/objects/ 下一个以哈希命名的文件。这种"一个对象一个文件"的形式叫松散对象(loose object)。

bash · 松散对象的存放位置
# 对象哈希 a0b34bb... → 前 2 位做目录名,后 38 位做文件名
.git/objects/a0/b34bb1f2...   ← 一个松散对象文件(zlib 压缩)

松散对象胜在简单、写入快,但有个大问题:它按对象独立存储,同一个文件改了一行就产生一个全新的完整 blob。一个大文件被改 100 次,就存了 100 份几乎相同的完整内容——极其浪费空间,传输时也要传一大堆独立文件。

二、packfile:把松散对象打包 + 增量压缩

为解决这个问题,Git 会在合适时机(gc、push、clone)把大量松散对象合并进一个 .pack 文件,并在打包时做一件关键的事——增量压缩(delta compression):对于内容相似的对象(比如同一文件的多个版本),只完整存一份基准,其余的只存相对基准的差异(delta)。

这里才用"差异存储":注意区分两个层面——逻辑/语义层Git 存的是完整快照(上一篇讲的对象模型不变);物理/存储层packfile 用 delta 把相似对象压成"基准 + 差异"以省空间。两者不矛盾:你 checkout 时 Git 会自动把 delta 还原成完整对象,你感知不到差异存储的存在。
破除误区:常有人把"packfile 用了 delta"当成"Git 底层其实是差异存储,快照说法是骗人的"。错。快照是数据模型,delta 是压缩优化——就像 JPEG 压缩不改变"图片是像素矩阵"的事实一样。面试若被追问,能把这两层讲清楚是加分项。

delta 的方向:新版本指向旧版本?反过来

有个反直觉的细节:Git 倾向于让较新、较小的对象作为基准,较旧的对象存成指向它的 delta(因为最新版本被访问最频繁,让它离"完整"最近、还原最快)。这是性能与空间的权衡。

三、.pack 与 .idx 配对

文件作用
pack-xxxx.pack真正的数据:所有打包对象(含 delta 链)的压缩内容
pack-xxxx.idx索引:记录每个对象的哈希 → 在 .pack 中的偏移量,让 Git 能 O(log n) 快速定位对象,不必扫全包

它们都放在 .git/objects/pack/ 下,成对出现。没有 .idx,.pack 就只是一坨无法随机访问的字节。

四、git gc:触发打包与清理

bash · gc 相关
git gc                # 打包松散对象、合并 refs 到 packed-refs、清理过期 reflog 与无用对象
git gc --aggressive   # 花更多时间寻找更优 delta,压得更狠(偶尔手动跑一次)
git count-objects -v   # 查看松散对象数量、打包情况、占用空间

Git 通常自动触发 gc(git gc --auto),比如松散对象攒够一定数量、或 push/fetch 后。所以你一般不用手动跑。gc 同时会做「上一篇」提到的 packed-refs 打包,以及「reflog 篇」提到的过期悬空对象清理。

为什么 clone 那么小又那么快:服务端把对象打包成 packfile 用 delta 压缩后再通过网络传输,你收到的是高度压缩的包,而非成千上万个松散文件。这也是为什么带巨大历史的仓库,clone 后 .git 目录往往比你想象的小得多。
陷阱——大文件:delta 压缩对文本很有效,但对二进制大文件(图片、视频、编译产物)几乎无能为力——它们改一点整个 delta 就失效,仓库会越滚越大且不可逆(历史里的对象一直都在)。这类文件应交给 Git LFS 管理,而不是直接塞进仓库。

小结

  • 松散对象:每个对象独立 zlib 压缩存成一个文件,简单但相似内容重复存储、浪费空间。
  • packfile:gc/push/clone 时把松散对象打包进 .pack,并用增量压缩(delta)把相似对象存成"基准 + 差异"。
  • 快照(数据模型)与 delta(存储压缩)是两个层面,互不矛盾;checkout 时 delta 自动还原,用户无感。
  • .pack 存数据、.idx 存哈希→偏移索引;git gc(常自动触发)负责打包、合并 refs、清理。大二进制文件不适合 delta,应用 Git LFS。
