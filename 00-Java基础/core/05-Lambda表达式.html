05. Lambda 表达式

Java 8 最重要的语法革命——把「行为」当成「数据」传递，彻底改变了 Java 的编程风格。

为什么需要 Lambda？从匿名类说起

在 Java 8 之前，把「一段逻辑」传递给另一个方法，只能用匿名内部类。但匿名类的语法噪音极大：

Java 7：匿名类传递行为（反例）
// 给一个列表排序，需要传入「比较逻辑」
List<String> names = Arrays.asList("Charlie", "Alice", "Bob");

Collections.sort(names, new Comparator<String>() {
    @Override
    public int compare(String a, String b) {
        return a.compareTo(b);
    }
});
// 问题：7 行代码，真正有意义的只有 1 行

这就是行为参数化（Behavior Parameterization）需求：我们想把「比较方式」作为参数传入，而不是把「具体的比较对象」传入。Lambda 让这件事变得自然：

Java 8：Lambda 传递行为（正例）
names.sort((a, b) -> a.compareTo(b));
// 意图一眼就清晰了
设计哲学：Lambda 并没有给 Java 带来新的能力，它能做的匿名类都能做。它解决的是表达力和信噪比的问题——让代码的意图更清晰，让「行为」和「数据」一样可以自由传递。

Lambda 的底层支撑：函数式接口

Java 是强类型语言，Lambda 表达式本身没有类型，它必须被赋予一个函数式接口（Functional Interface）类型。函数式接口是只有一个抽象方法的接口。

记忆口诀：一个抽象方法 = 函数式接口 = 可以用 Lambda 赋值。`default` 方法和 `static` 方法不算抽象方法，可以有多个。
自定义函数式接口
@FunctionalInterface  // 可选注解，帮助编译器检查
interface Transformer<T, R> {
    R transform(T input);  // 唯一的抽象方法

    default String describe() {  // default 方法，不影响函数式接口
        return "a transformer";
    }
}

// 用 Lambda 实现：把字符串转成大写
Transformer<String, String> upper = s -> s.toUpperCase();
System.out.println(upper.transform("hello"));  // HELLO

JDK 在 java.util.function 包里内置了最常用的函数式接口，几乎覆盖了所有场景：

接口抽象方法签名含义典型使用场景
Predicate<T>boolean test(T t)判断，返回 true/falsefilter()
Function<T,R>R apply(T t)转换，T → Rmap()
Consumer<T>void accept(T t)消费，只做操作无返回forEach()
Supplier<T>T get()生产，无参数产出值懒加载、工厂
BiFunction<T,U,R>R apply(T t, U u)两参数转换reduce()
UnaryOperator<T>T apply(T t)同类型转换（特化 Function）replaceAll()

Lambda 语法全貌

Lambda 的语法结构：(参数列表) -> 表达式或语句块。根据参数个数和方法体的复杂度，有多种简化写法：

Lambda 语法变体一览
// 1. 无参数
Runnable r = () -> System.out.println("run!");

// 2. 单参数：可省略圆括号
Consumer<String> print  = s -> System.out.println(s);

// 3. 多参数
Comparator<Integer> cmp = (a, b) -> a - b;

// 4. 表达式体：直接返回，不需要 return 关键字
Function<Integer, Integer> square = x -> x * x;

// 5. 语句块：需要花括号和 return
Function<Integer, String> grade = score -> {
    if (score >= 90) return "A";
    if (score >= 60) return "B";
    return "C";
};

// 6. 显式声明类型（类型推断失败时使用）
Comparator<String> byLen = (String a, String b) -> Integer.compare(a.length(), b.length());

闭包与变量捕获：最容易踩坑的地方

Lambda 可以捕获（capture）外部的变量，但有一个严格限制：被捕获的局部变量必须是 effectively final（实际不可变，即便没有 `final` 关键字，也不能在赋值后再被修改）。

变量捕获规则
int threshold = 10;  // effectively final，没有被再次赋值

Predicate<Integer> check = n -> n > threshold;  // OK

// --- 反例 ---
int count = 0;
List<String> items = List.of("a", "b");
items.forEach(item -> {
    count++;  // 编译错误：count 不是 effectively final
});

// 正确做法：用数组或 AtomicInteger 绕过（因为是引用不变，内容可变）
int[] cnt = {0};
items.forEach(item -> cnt[0]++);  // OK（但多线程下不安全！）

AtomicInteger atomicCnt = new AtomicInteger(0);
items.forEach(item -> atomicCnt.incrementAndGet());  // 线程安全版本
为什么有这个限制？局部变量存在于栈帧（stack frame）上，Lambda 可能在另一个线程或方法返回后才执行。如果允许修改，栈帧已销毁，就会产生悬空引用（dangling reference）。JVM 实际上会将被捕获的变量复制一份到 Lambda 对象中，所以必须保证复制时值不会再变，否则语义不一致。

注意：实例变量（this.field）和静态变量没有此限制，因为它们存在于堆上，生命周期不受栈帧影响。

Lambda 的底层实现：invokedynamic

很多人以为 Lambda 只是匿名类的语法糖，实际上并非如此。Lambda 在字节码层面用的是 invokedynamic 指令（Java 7 为动态语言支持引入）。

Lambda 编译流程
├── 编译期：javac 将 Lambda 体编译为类的私有静态方法
│ └── 同时生成一条 invokedynamic 调用指令
├── 首次执行：JVM 调用 LambdaMetafactory.metafactory()
│ └── 动态生成一个实现了目标函数式接口的类
└── 后续执行：直接调用缓存的实现类，无需再次生成

与匿名内部类的关键区别：

  • 匿名类：编译时生成独立的 Outer$1.class 文件，每次 new 都分配堆内存。
  • Lambda：字节码层面延迟到运行时生成，JVM 可以做更多优化（如无捕获的 Lambda 可被实现为单例）。
  • 无变量捕获的 Lambda 只会创建一个实例，被重复使用（类似静态常量）。
实验验证：用 javap -p -c ClassName.class 可以看到 Lambda 编译后的私有静态方法（通常叫 lambda$main$0），以及 invokedynamic 指令。

与 Stream API 结合：Lambda 的主战场

Lambda 最大的用武之地就是 Stream API，两者共同构成了 Java 函数式编程的基础设施：

Lambda + Stream 综合示例
List<Employee> employees = getEmployees();

// 筛选薪资 > 10000 的员工，提取姓名，按字母排序，取前 5 个
List<String> result = employees.stream()
    .filter(e -> e.getSalary() > 10000)      // Predicate<Employee>
    .map(Employee::getName)                  // Function<Employee, String>（方法引用）
    .sorted(String::compareTo)               // Comparator<String>
    .limit(5)
    .collect(Collectors.toList());

// 用 Comparator 的 comparing 工厂方法（底层也是 Lambda）
employees.sort(
    Comparator.comparing(Employee::getDept)
              .thenComparing(Employee::getSalary, Comparator.reverseOrder())
);

函数组合：Function 的 compose 与 andThen

Function、Predicate 等接口提供了 default 方法，可以将多个 Lambda 组合成新的函数，实现管道式处理：

函数组合
Function<Integer, Integer> times2  = x -> x * 2;
Function<Integer, Integer> plus3   = x -> x + 3;

// andThen：先 times2，再 plus3  → (x*2)+3
Function<Integer, Integer> f1 = times2.andThen(plus3);
System.out.println(f1.apply(5));  // (5*2)+3 = 13

// compose：先 plus3，再 times2  → (x+3)*2
Function<Integer, Integer> f2 = times2.compose(plus3);
System.out.println(f2.apply(5));  // (5+3)*2 = 16

// Predicate 组合
Predicate<Integer> isEven    = n -> n % 2 == 0;
Predicate<Integer> isPositive = n -> n > 0;
Predicate<Integer> isEvenAndPositive = isEven.and(isPositive);
Predicate<Integer> isOdd              = isEven.negate();

常见陷阱

陷阱一：在循环中创建捕获变量的 Lambda

循环变量陷阱（经典面试题）
List<Runnable> tasks = new ArrayList<>();

// 反例（如果 i 是 var，Java 会报编译错误，恰好帮你规避了）
for (int i = 0; i < 5; i++) {
    final int idx = i;  // 必须用 final 副本"固定"当前值
    tasks.add(() -> System.out.println(idx));
}

// 增强 for 的循环变量每次迭代是新变量，directly 可用
for (String name : List.of("a", "b")) {
    tasks.add(() -> System.out.println(name));  // OK
}

陷阱二：Lambda 中抛出受检异常

JDK 内置的函数式接口（如 Function、Consumer）的方法签名不声明受检异常，所以 Lambda 体内如果有受检异常，必须自己处理：

受检异常处理
// 反例：直接抛出，编译报错
Function<String, Path> toPath = s -> Files.createTempFile(s, ".tmp");  // 编译失败

// 正例一：Lambda 内 try-catch
Function<String, Path> toPath = s -> {
    try { return Files.createTempFile(s, ".tmp"); }
    catch (IOException e) { throw new UncheckedIOException(e); }
};

// 正例二：封装一个 wrap 工具方法（工程实践中常见）
static <T,R> Function<T,R> wrap(ThrowingFunction<T,R> f) {
    return t -> {
        try { return f.apply(t); }
        catch (Exception e) { throw new RuntimeException(e); }
    };
}

陷阱三：this 的含义不同于匿名类

Lambda vs 匿名类中的 this
class MyClass {
    String name = "MyClass";

    void test() {
        // 匿名类：this 指向匿名类实例
        Runnable r1 = new Runnable() {
            String name = "Anonymous";
            public void run() {
                System.out.println(this.name);  // 输出 "Anonymous"
            }
        };

        // Lambda：没有自己的 this，this 指向外部类实例
        Runnable r2 = () -> System.out.println(this.name);  // 输出 "MyClass"
    }
}

面试高频考点

问题核心答案
Lambda 是匿名类的语法糖吗？ 不是。匿名类编译产生 .class 文件，Lambda 用 invokedynamic 延迟到运行时生成，JVM 可以更好地优化。
什么是函数式接口？ 只有一个抽象方法的接口。@FunctionalInterface 是可选的验证注解。
为什么捕获的局部变量必须 effectively final？ 局部变量在栈帧上，Lambda 对象在堆上，JVM 复制值到 Lambda 对象，为保持语义一致性，不允许修改。
Lambda 中 this 指向谁？ 外部类实例（与匿名类不同，Lambda 没有自己的 this）。
无捕获 Lambda 和有捕获 Lambda 的区别？ 无捕获 Lambda 只创建一个实例（类似静态常量）；有捕获 Lambda 每次执行可能创建新实例（捕获的值不同）。
Function.andThen vs Function.compose？ andThen(g) = 先 f 再 g；compose(g) = 先 g 再 f。

├── 函数式接口 → java.util.function 包 → Stream API
├── 方法引用（下一篇）— Lambda 的进一步简化形式
├── Stream API — Lambda 是每个操作符的参数
