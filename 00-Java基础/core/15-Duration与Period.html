核心类库 · 日期与时间

15. Duration 与 Period

Java 8 新日期 API 中专门表示"时间跨度"的两个类——一个数纳秒，一个数年月日，看似相似，本质迥异。

为什么需要两个类？

在老的 java.util.Date / Calendar 体系里，"两个时间点的差"只能靠毫秒相减， 既丢失了语义，又在跨月、跨年计算时充满陷阱。 JSR-310（Java 8 日期 API）从 物理时间 与 日历时间 两个维度来建模时间跨度：

  • Duration：基于 ISO-8601 秒 + 纳秒，表示精确的物理时长（与时区、日历无关）。
  • Period：基于人类日历，表示"X 年 Y 月 Z 天"，与时区和月份长度强相关。
类比：Duration 是秒表计时；Period 是翻日历数格子。
"从 2024-01-31 再过 1 个月" 在日历上是 2024-02-29（2024 是闰年）， 但 "再过 31×24×3600 秒" 永远精确——这就是两者的本质区别。

Duration — 精确物理时长

Duration 内部用 long seconds 和 int nanos 两个字段存储， 覆盖范围从纳秒到大约 292 亿年，且是不可变（immutable）、线程安全的值对象。

创建方式

JavaDuration 的各种创建方式
Duration d1 = Duration.ofSeconds(90);          // 90 秒
Duration d2 = Duration.ofMinutes(2);          // 2 分钟 = 120 秒
Duration d3 = Duration.ofHours(1);            // 1 小时
Duration d4 = Duration.ofMillis(500);         // 500 毫秒
Duration d5 = Duration.ofNanos(1_000_000);    // 1 毫秒（以纳秒表示）

// ISO-8601 字符串解析：PT1H30M15S = 1小时30分15秒
Duration d6 = Duration.parse("PT1H30M15S");

// 两个时间点之间的差（最常用）
LocalTime start = LocalTime.of(9, 0);
LocalTime end   = LocalTime.of(10, 30);
Duration d7 = Duration.between(start, end);   // PT1H30M

// 也可用于 Instant（带时区语义的时间戳）
Instant t1 = Instant.now();
// ... 做一些操作 ...
Instant t2 = Instant.now();
Duration elapsed = Duration.between(t1, t2);

常用操作

JavaDuration 的运算与查询
Duration d = Duration.ofSeconds(3661);  // 1小时1分1秒

// 各单位提取（注意：toXxx 是转换为该单位的总量）
long totalSec = d.toSeconds();   // 3661
long totalMin = d.toMinutes();   // 61（取整，不是 1 分）
long hours    = d.toHours();     // 1

// Java 9+ 新增：获取"当前单位的余数部分"（类似 % 运算）
int  secPart  = d.toSecondsPart();  // 1（余 1 秒）
int  minPart  = d.toMinutesPart();  // 1（余 1 分）
long hourPart = d.toHoursPart();   // 1（小时部分）

// 算术运算（全部返回新 Duration，不可变！）
Duration doubled = d.multipliedBy(2);
Duration neg     = d.negated();       // 负时长，用于"倒退"
Duration sum     = d.plus(Duration.ofMinutes(10));

// 加到时间点上
LocalDateTime now  = LocalDateTime.now();
LocalDateTime later= now.plus(d);

// 格式化输出（无内置"1小时1分1秒"格式，需自己拼）
String fmt = String.format("%d小时%d分%d秒",
    d.toHoursPart(), d.toMinutesPart(), d.toSecondsPart());

Period — 基于日历的时间跨度

Period 内部用三个 int 字段存储：years、months、days， 三者相互独立，不会自动进位。 "1 年 13 个月" 是合法的 Period，但你可以调用 normalized() 将月份规范化。

创建方式

JavaPeriod 的各种创建方式
Period p1 = Period.ofYears(1);            // 1 年
Period p2 = Period.ofMonths(6);           // 6 个月
Period p3 = Period.ofDays(30);            // 30 天
Period p4 = Period.of(1, 6, 15);          // 1年6个月15天

// ISO-8601 字符串解析：P1Y6M15D
Period p5 = Period.parse("P1Y6M15D");

// 两个日期之间的 Period（最常用）
LocalDate birth = LocalDate.of(1995, 3, 20);
LocalDate today = LocalDate.now();
Period age = Period.between(birth, today);

System.out.printf("你 %d 岁 %d 个月 %d 天%n",
    age.getYears(), age.getMonths(), age.getDays());

常用操作

JavaPeriod 的运算与规范化
Period p = Period.of(1, 14, 3);  // 1年14个月3天（月份未规范化）
Period norm = p.normalized();    // 2年2个月3天（月份进位到年）

// 注意：normalized() 只处理年月，不会把天数折算成月

// 算术（返回新对象）
Period p2 = p.plusMonths(1);
Period p3 = p.withYears(2);      // 替换年份部分
Period neg = p.negated();        // 所有字段取反

// 加到日期上
LocalDate d = LocalDate.of(2024, 1, 31);
LocalDate later = d.plus(Period.ofMonths(1));
// 结果是 2024-02-29（自动截断到月末，闰年 2 月有 29 天）
System.out.println(later);  // 2024-02-29

// 检查是否为零或负
boolean isZero = p.isZero();
boolean isNeg  = p.isNegative();  // 任意字段为负则为 true

Duration vs Period — 核心对比

维度 Duration Period
表示的时间维度 物理时间（秒 + 纳秒） 日历时间（年 + 月 + 日）
适用时间类型 LocalTime、LocalDateTime、Instant LocalDate、LocalDateTime
是否受时区影响 不受（绝对秒数） 受影响（夏令时可使一天 ≠ 24 小时）
内部存储 long seconds + int nanos int years + int months + int days
ISO-8601 字符串格式 PT1H30M15S（T 前缀） P1Y6M15D（无 T 前缀）
是否自动进位 是（秒/纳秒自动规范） 月可 normalized()，天不自动进月
精度 纳秒级 天级

共同接口：TemporalAmount

Duration 和 Period 都实现了 TemporalAmount 接口， 这意味着可以将它们统一传入 temporal.plus(amount) / temporal.minus(amount)。

JavaTemporalAmount 多态使用
// 方法接受任意时间跨度
static LocalDateTime addSpan(LocalDateTime base, TemporalAmount amount) {
    return base.plus(amount);
}

LocalDateTime ldt = LocalDateTime.of(2024, 3, 1, 9, 0);

// 传 Duration（加 90 分钟）
LocalDateTime r1 = addSpan(ldt, Duration.ofMinutes(90));

// 传 Period（加 1 个月）
LocalDateTime r2 = addSpan(ldt, Period.ofMonths(1));
框架开发技巧：Spring 的 @Scheduled(fixedDelay = ...) 内部就是基于 Duration 构建的；Spring Data 的 TTL 注解也接受 Duration。 理解这两个类让你读框架源码更顺畅。

实战：性能计时

性能计时是 Duration 最常见的使用场景，推荐与 Instant 搭配：

Java用 Duration + Instant 计时
Instant start = Instant.now();

// 模拟耗时操作
doSomeHeavyWork();

Instant end = Instant.now();
Duration cost = Duration.between(start, end);

System.out.printf("耗时：%d 毫秒（%d 纳秒）%n",
    cost.toMillis(),
    cost.toNanos());

// 对比：不要用 System.currentTimeMillis() 相减，精度只到毫秒
// Instant.now() 在 JVM 内部调用 System.nanoTime()，精度更高

实战：精确年龄计算

Java用 Period 计算精确年龄
LocalDate birthday = LocalDate.of(1995, 8, 15);
LocalDate today    = LocalDate.of(2026, 6, 29);

Period age = Period.between(birthday, today);
System.out.printf("%d 岁 %d 个月 %d 天%n",
    age.getYears(),     // 30
    age.getMonths(),    // 10
    age.getDays());     // 14

// 用 ChronoUnit 计算整岁数（更简洁）
long yearsOld = ChronoUnit.YEARS.between(birthday, today);
System.out.println(yearsOld);  // 30

常见陷阱

陷阱 1：Duration 不能直接用在 LocalDate 上

Java类型不匹配会抛异常
LocalDate d = LocalDate.now();

// 错误：Duration 需要时间（时分秒），LocalDate 没有时间部分
// d.plus(Duration.ofDays(1));  → 抛 UnsupportedTemporalTypeException

// 正确：对 LocalDate 用 Period 或 ChronoUnit
LocalDate tomorrow = d.plus(Period.ofDays(1));
// 或者更简洁：
LocalDate tomorrow2 = d.plusDays(1);

陷阱 2：Period.between() 的 days 字段不是总天数

JavagetDays() 不等于总天数
LocalDate d1 = LocalDate.of(2024, 1, 1);
LocalDate d2 = LocalDate.of(2024, 3, 15);
Period p = Period.between(d1, d2);

System.out.println(p.getDays());   // 14（不是总天数 74！）
System.out.println(p.getMonths()); // 2

// 若需要"总天数"，用 ChronoUnit：
long totalDays = ChronoUnit.DAYS.between(d1, d2);  // 74

陷阱 3：月末日期的"月份溢出"

Java月末加月会截断到月末
LocalDate jan31 = LocalDate.of(2024, 1, 31);

// +1 个月 → 2024-02-29（2月没有31日，自动截断）
LocalDate r1 = jan31.plus(Period.ofMonths(1));   // 2024-02-29

// 再 +1 个月 → 2024-03-29（从 2 月 29 日加，而不是从原始 31 日加！）
LocalDate r2 = r1.plus(Period.ofMonths(1));    // 2024-03-29（不是 31！）

// 如果需要每月最后一天，用 TemporalAdjusters.lastDayOfMonth()
LocalDate lastDay = r2.with(TemporalAdjusters.lastDayOfMonth());

陷阱 4：Duration.ofDays() 不等于 Period.ofDays() 的行为

JavaDuration 的 days 是固定 86400 秒
// Duration.ofDays(1) = 精确 86400 秒，不管夏令时
ZonedDateTime zdt = ZonedDateTime.of(
    2024, 3, 10, 0, 0, 0, 0,
    ZoneId.of("America/New_York")
);
// 2024-03-10 美国开始夏令时，这天只有 23 小时
ZonedDateTime r1 = zdt.plus(Duration.ofDays(1));   // 加 86400 秒 → 次日 01:00
ZonedDateTime r2 = zdt.plus(Period.ofDays(1));     // 加一个日历天 → 次日 00:00
在涉及夏令时的时区场景下，"加 1 天"请用 Period.ofDays(1)，而不是 Duration.ofDays(1)。 前者保证日历天，后者保证物理 86400 秒。

与 ChronoUnit 协同使用

ChronoUnit 提供了"直接计算两个时间点相差多少某单位"的静态方法， 是 Duration.between / Period.between 的轻量替代：

JavaChronoUnit.between 一行搞定
LocalDate d1 = LocalDate.of(2024, 1, 1);
LocalDate d2 = LocalDate.of(2025, 6, 15);

long days   = ChronoUnit.DAYS.between(d1, d2);    // 531
long months = ChronoUnit.MONTHS.between(d1, d2);  // 17
long years  = ChronoUnit.YEARS.between(d1, d2);   // 1

LocalTime t1 = LocalTime.of(9, 0);
LocalTime t2 = LocalTime.of(17, 30);
long mins = ChronoUnit.MINUTES.between(t1, t2);    // 510

类层次与关联

java.time.temporal
├── TemporalAmount（接口）

├── LocalTime ← Duration
├── LocalDate ← Period / ChronoUnit
└── LocalDateTime ← Duration 或 Period

面试高频考点

  • Duration 和 Period 的区别是什么？ — 答：Duration 是物理时间（秒级），Period 是日历时间（年月日），两者都实现 TemporalAmount 接口。
  • Period.getDays() 是总天数吗？ — 答：不是，是"天数部分"。要获取总天数用 ChronoUnit.DAYS.between()。
  • Duration 可以加到 LocalDate 上吗？ — 答：不能，会抛 UnsupportedTemporalTypeException，因为 LocalDate 没有时间分量。
  • 为什么"加 1 天"在夏令时时区用 Duration 会有问题？ — 答：Duration 加的是固定 86400 秒，而夏令时切换日可能只有 23 小时，会导致时间偏移 1 小时。
  • Period.normalized() 做什么？ — 答：将月份中超过 12 的部分进位到年（如 1年14个月 → 2年2个月），但不处理天数溢出。
  • 如何用 Duration 实现函数执行时间统计？ — 答：用 Instant.now() 前后打点，再 Duration.between(start, end).toMillis()。

小结

核心记忆口诀：
Duration 数秒，Period 数格—— Duration 是秒表，精确到纳秒，与时区无关； Period 是日历，翻格子数年月日，受日历规则约束。

选择规则：

  • 需要精确物理时长（性能计时、倒计时、超时控制）→ Duration
  • 需要日历语义（到期日、账单周期、年龄计算）→ Period
  • 只需要"总共差几天/几月"一个数字 → ChronoUnit.between()
