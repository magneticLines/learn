20. Class 对象与 getClass() / .class / Class.forName()

反射的一切从 Class 对象开始——理解 JVM 如何用一个对象表示整个类型，是打开反射大门的第一把钥匙。

为什么需要 Class 对象？

Java 是一门静态类型语言，编译期已经把类型信息固化在字节码里。但现实需求远比编译期知道的要动态得多：Spring 需要在运行时读 @Autowired、JSON 反序列化库需要按字段名赋值、IDE 要自动补全方法名……

解决方案是：让 JVM 把每个类型本身也建模成一个对象。这个对象就是 java.lang.Class<T>。它是类的"身份证"，保存了类名、父类、接口、字段、方法、注解等所有元信息。

一句话定位： Class 对象 = JVM 在方法区（Metaspace）中为每个类型创建的唯一描述符，通过它可以在运行时检查和操作任意类型。

Class 对象的生命周期

JVM 第一次使用某个类时触发类加载，ClassLoader 将 .class 字节流读入内存，在 Metaspace 中创建对应的 Class 对象，并在堆中留一个引用。此后无论 new 多少个实例，共享的都是同一个 Class 对象。

ClassLoader.loadClass("com.example.Dog") │ ├─ 解析字节码 → Class<Dog> 对象（Metaspace） │ ├─ name = "com.example.Dog" │ ├─ superclass → Class<Animal> │ ├─ interfaces → [Class<Runnable>, ...] │ ├─ fields[] → [Field("name"), ...] │ └─ methods[] → [Method("bark"), ...] │ ├─ new Dog() → Dog 实例（Heap）— 内部 _klass 指针 → 同一个 Class<Dog> └─ new Dog() → Dog 实例（Heap）— 内部 _klass 指针 → 同一个 Class<Dog>
验证"唯一性"： dog1.getClass() == dog2.getClass() 永远为 true——不管创建多少个 Dog 实例，Class 对象只有一个。

三种获取 Class 对象的方式

① 对象.getClass()

运行时从实例出发，调用继承自 Object 的 getClass() 方法。返回的是实际运行时类型，不是声明类型。

JavagetClass() 示例
Animal a = new Dog();
Class<?> clazz = a.getClass();

System.out.println(clazz.getName());   // com.example.Dog（运行时类型！）
System.out.println(clazz.getSimpleName()); // Dog

// 注意：声明类型是 Animal，但 getClass() 返回的是 Dog
// 这就是多态的运行时本质
陷阱： getClass() 返回的泛型是 Class<? extends |X|>（X 是声明类型），所以你拿到的是 Class<? extends Animal>，而非 Class<Dog>。这是 Java 泛型擦除的折中设计。

② 类名.class（字面量）

编译期语法糖，不需要任何实例，也不会触发静态初始化块（static {}）的执行。类型安全，泛型参数直接确定。

Java.class 字面量示例
Class<Dog> clazz = Dog.class;          // 泛型参数已确定为 Dog

// 基本类型和 void 也有对应的 Class 对象
Class<Integer> boxed    = Integer.class;
Class<Integer> boxed2   = Integer.TYPE;   // 等价于 int.class
Class<?>       voidClz  = void.class;    // void 也是一种类型
Class<?>       arrClz   = int[].class;   // 数组也有独立的 Class

// .class 不触发初始化：下面这行不会执行 Dog 的 static {} 块
Class<Dog> c2 = Dog.class;

③ Class.forName(String)

用字符串指定类的全限定名，在运行时动态加载类。这是三种方式中唯一能接受"编译期还不知道类名"的情况，也是框架与插件机制的核心手段。

JavaClass.forName() 示例
// 基础用法：加载 + 初始化（会执行 static {} 块）
Class<?> clazz = Class.forName("com.example.Dog");

// 三参数版本：可控制是否初始化、指定 ClassLoader
Class<?> clazz2 = Class.forName(
    "com.example.Dog",
    false,                                   // initialize=false：不执行 static {}
    Thread.currentThread().getContextClassLoader()
);

// 经典用法：JDBC 驱动注册（Driver 类的 static {} 自动注册到 DriverManager）
Class.forName("com.mysql.cj.jdbc.Driver");

// 找不到类时抛出受检异常，必须处理
try {
    Class.forName("com.example.NotExist");
} catch (ClassNotFoundException e) {
    // 必须显式 catch，这是受检异常
}

三种方式对比

方式 是否需要实例 是否触发初始化 异常 典型使用场景
obj.getClass() 需要 已初始化 无 运行时判断实际类型
Dog.class 不需要 不触发 无（编译期检查） 泛型、注解处理、编译期已知类型
Class.forName() 不需要 默认触发 ClassNotFoundException（受检） 框架、插件、配置驱动加载

Class 对象的常用 API

拿到 Class 对象之后，可以查询类的一切元信息：

JavaClass API 速览
Class<Dog> c = Dog.class;

// ── 名称 ──────────────────────────────────────
c.getName();           // "com.example.Dog"（全限定名）
c.getSimpleName();     // "Dog"
c.getCanonicalName();  // "com.example.Dog"（匿名类返回 null）
c.getPackageName();    // "com.example"（Java 9+）

// ── 层级关系 ──────────────────────────────────
c.getSuperclass();     // Class<Animal>
c.getInterfaces();    // Class[]，直接实现的接口
c.isInterface();      // false
c.isEnum();           // false
c.isAnnotation();    // false
c.isArray();          // false
c.isPrimitive();     // false（int.class 才为 true）

// ── 修饰符 ────────────────────────────────────
int mod = c.getModifiers();
Modifier.isPublic(mod);   // true
Modifier.isAbstract(mod); // false

// ── 直接创建实例（无参构造，Java 9 后推荐用 getDeclaredConstructor） ──
Dog dog = c.getDeclaredConstructor().newInstance();

// ── instanceof 的运行时等价 ───────────────────
c.isInstance(dog);        // true，等价于 dog instanceof Dog
c.isAssignableFrom(Poodle.class); // Poodle extends Dog → true

深入：类加载三阶段与初始化时机

理解 .class 不触发初始化、而 Class.forName() 默认触发初始化的原因，需要了解类加载的三个阶段：

加载（Loading） └─ ClassLoader 读取字节码，创建 Class 对象骨架 链接（Linking） ├─ 验证：字节码合法性检查 ├─ 准备：为 static 字段分配内存，赋零值（不是初始值！） └─ 解析：符号引用 → 直接引用 初始化（Initialization） ← static {} 块在这里执行 └─ 按源码顺序执行 static 字段赋值 + static 初始化块

Dog.class 只完成"加载"阶段（Class 对象已存在），不进入初始化。而 Class.forName("com.example.Dog") 默认 initialize=true，会走完全部三个阶段。

触发初始化的 6 种情形（JVM 规范）：
① new 一个对象  ② 调用静态方法  ③ 读/写静态字段（非 final 常量）  ④ 反射（forName 默认）  ⑤ 子类初始化时触发父类初始化  ⑥ JVM 启动时的主类

数组与基本类型的 Class 对象

数组和基本类型也有各自的 Class 对象，这是 Java 类型系统一致性的体现：

Java特殊类型的 Class
// 基本类型：只能用 .class，不能用 forName 的简短名
Class<?> intClz = int.class;            // 不是 Integer.class！
System.out.println(int.class == Integer.class); // false！拆装箱不等价
System.out.println(int.class == Integer.TYPE);  // true，TYPE 是 int.class 的别名

// 数组：用 forName 时名字格式很怪
Class.forName("[Ljava.lang.String;");    // String[] 的内部名（L 开头；结尾）
Class.forName("[I");                    // int[] 的内部名
Class.forName("[[B");                   // byte[][] 的内部名

// 推荐写法：直接用字面量，清晰无误
Class<?> strArrClz = String[].class;  // 更易读

// 数组 Class 的特有方法
strArrClz.isArray();                  // true
strArrClz.getComponentType();          // Class<String>（元素类型）

Class 对象与泛型擦除

Java 泛型在编译后被擦除，List<String> 和 List<Integer> 在运行时共享同一个 Class 对象：List.class。这直接导致了"无法创建泛型数组"、"instanceof 不能用带参数的泛型"等限制。

Java泛型擦除的影响
// 编译期不同，运行时相同
List<String> ls = new ArrayList<>();
List<Integer> li = new ArrayList<>();
System.out.println(ls.getClass() == li.getClass()); // true！

// 编译报错：不能对带类型参数的泛型用 instanceof
// if (ls instanceof List<String>) { }  ← 编译失败
if (ls instanceof List<?>) {           // 只能用通配符
    System.out.println("is a List");
}

// 保留泛型信息：通过 getGenericSuperclass() 获取
// 这是 Gson/Jackson 的 TypeToken 技巧（匿名子类）
class StringList extends ArrayList<String> {}  // 父类泛型信息保留在字节码里
Type t = StringList.class.getGenericSuperclass();
// t = "java.util.ArrayList<java.lang.String>"

常见陷阱

  • int.class != Integer.class：装箱类型和基本类型是两个不同的 Class 对象，反射调用方法时参数类型必须精确匹配。
  • Class.forName() 依赖当前线程的 ClassLoader：在 OSGi、Tomcat 等多 ClassLoader 环境中，直接调用可能找不到插件类，应显式传入正确的 ClassLoader。
  • 内部类的全限定名用 $ 分隔：Class.forName("com.example.Outer$Inner")，不是点号。
  • 匿名类的 getSimpleName() 返回空字符串，getCanonicalName() 返回 null，序列化时需注意。
  • .class 不触发初始化可能造成困惑：某些依赖静态初始化注册的机制（如 JDBC 驱动）必须用 forName 而不是 .class。

面试高频考点

Q1：三种获取 Class 对象的方式有什么区别？
重点答出：是否需要实例、是否触发类初始化（.class 不触发）、异常处理（forName 是受检异常）。
Q2：Class.forName() 和 ClassLoader.loadClass() 的区别？
forName 默认执行初始化（运行 static 块），loadClass 只做加载，不初始化。Spring 懒加载、Tomcat 热加载都在这里做文章。
Q3：为什么 JDBC 用 Class.forName()？
驱动类在 static 块里向 DriverManager 注册自己。必须触发初始化才能注册，所以不能用 Driver.class（不触发初始化）。
Q4：int.class 和 Integer.class 相同吗？
不同。int.class 代表基本类型，Integer.class 代表包装类。反射调用 void foo(int x) 时必须传 int.class，传 Integer.class 会找不到方法。

与其他知识的联系

  • 反射调用：Class 对象是起点，Field / Method / Constructor 都从它获取。下一篇将深入讲解。
  • 类加载器：Class 对象由 ClassLoader 创建，不同 ClassLoader 加载的同名类不是同一个 Class，也不能互相赋值（这是 ClassCastException 的来源之一）。
  • Spring IoC：BeanDefinition 内部持有 Class 对象，ApplicationContext 启动时通过 forName 加载配置类、通过 getDeclaredConstructor().newInstance() 创建 Bean。
  • 泛型与 TypeToken：Gson/Jackson 的 TypeToken<List<String>>{} 通过匿名子类 + getGenericSuperclass() 绕过泛型擦除，保留运行时类型信息。
  • 序列化：ObjectInputStream 在反序列化时用 Class.forName() 按类名还原对象，这也是反序列化攻击的入口。
