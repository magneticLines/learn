14 — 方法定义与调用

1. 为什么需要方法

想象一台咖啡机——你只需按一个按钮，机器内部完成研磨、加热、萃取的全部步骤。方法就是这个按钮：把一段有名字的逻辑封装起来，对外只暴露"怎么用"，隐藏"怎么做"。

没有方法时，你必须到处复制粘贴同一段逻辑；有了方法，只需一行调用。这带来两个核心收益：

  • 消除重复（DRY）：Bug 只需修一处，逻辑只需测一处。
  • 降低认知负担：读代码时只需理解"做了什么"，不必同时理解"如何做"。
底层视角（栈帧预告）：每次调用一个方法，JVM 会在线程栈上压入一个栈帧（Stack Frame），存放该方法的局部变量、操作数栈、返回地址。方法返回时栈帧弹出。这是第 9 节会深入的内容，先建立这个直觉。

2. 完整语法结构

// 完整签名示例
public static <T> List<T> merge(List<T> a, List<T> b) throws IllegalArgumentException {
    // 方法体
}
部分含义可省略？
public访问修饰符，控制谁能调用（public / protected / private / 包私有）不写则为包私有
static属于类本身，无需实例即可调用可省（则为实例方法）
<T>泛型类型参数（第 X 节详讲）非泛型方法可省
List<T>返回类型；无返回值写 void不可省
merge方法名，遵循小驼峰命名法不可省
(List<T> a, List<T> b)形参列表；无参数写 ()括号不可省
throws IllegalArgumentException声明可能抛出的受检异常可省
方法签名 = 方法名 + 参数列表（不含返回类型！）。编译器用签名区分重载，JVM 字节码层面则是方法名 + 描述符（含返回类型）。

3. 三种调用形式

3.1 静态方法调用

// 同类内部可以直接调用
double r = Math.sqrt(9);      // 跨类调用：类名.方法名
int max = Math.max(3, 7);

3.2 实例方法调用

String s = "hello";
int len = s.length();         // 必须有对象引用才能调用
String upper = s.toUpperCase();

3.3 链式调用

// 每个方法返回对象本身（或新对象），即可链式
String result = "  Hello World  "
    .trim()           // 返回 String
    .toLowerCase()    // 返回 String
    .replace("hello", "hi");  // 返回 String

static 方法 vs 实例方法对比

维度static 方法实例方法
调用方式ClassName.method()obj.method()
能否访问实例字段不能（无 this）能
能否被重写（override）不能（隐藏 hiding）能
典型场景工具函数、工厂方法业务逻辑、状态操作

4. 形参与实参

// 定义时：a、b 是形参（formal parameter）——占位符，没有实际值
public static int add(int a, int b) {
    return a + b;
}

// 调用时：3、5 是实参（actual argument）——真实传入的值
int sum = add(3, 5);

形参 = 函数定义里的变量声明；实参 = 调用时实际传入的值/表达式。

值传递预告：Java 永远是值传递。传基本类型传的是值的拷贝，传引用类型传的是引用（地址）的拷贝——不是对象本身。这是初学者最容易混淆的点，下一节（15 - 值传递机制）会专门深入讲解。

5. 可变参数（varargs）语法糖

反例：没有 varargs 时的痛苦

// 需要为每种参数数量写一个重载——无穷尽
public static int sum(int a, int b)              { return a + b; }
public static int sum(int a, int b, int c)       { return a + b + c; }
public static int sum(int a, int b, int c, int d){ return a + b + c + d; }
// ... 这条路走不通

正例：varargs

// ... 表示"零个或多个同类型参数"，本质是数组
public static int sum(int... nums) {
    int total = 0;
    for (int n : nums) total += n;   // nums 就是 int[]
    return total;
}

// 调用方式完全自然
System.out.println(sum());           // 0
System.out.println(sum(1, 2));       // 3
System.out.println(sum(1, 2, 3, 4)); // 10

// 也可以直接传数组
int[] arr = {5, 6, 7};
System.out.println(sum(arr));        // 18
JDK 源码关联：System.out.printf(String fmt, Object... args)、String.format()、Arrays.asList(T... a) 都用了 varargs。
规则：
  • varargs 只能是参数列表中的最后一个参数。
  • 每个方法最多只能有一个 varargs 参数。
  • 不要在重载中同时提供精确版本和 varargs 版本——编译器匹配时可能产生歧义。

6. 方法重载（Overloading）

public class Printer {
    public void print(int n)    { System.out.println("int: "    + n); }
    public void print(long n)   { System.out.println("long: "   + n); }
    public void print(double n) { System.out.println("double: " + n); }
    public void print(String s) { System.out.println("String: " + s); }
}

静态分派（Static Dispatch）原理

重载是编译期行为。编译器根据实参的静态类型（声明类型）决定调用哪个版本，生成对应的字节码指令。这就是"静态分派"。

匹配优先级（从高到低）

优先级规则示例
1精确匹配传 int → 选 print(int)
2自动提升（widening）传 byte → 选 print(int)
3自动装箱（boxing）传 int → 若无 int 版本，选 print(Integer)
4varargs传 int → 若无上述版本，选 print(int...)

重载 vs 重写

维度重载（Overloading）重写（Overriding）
发生时机编译期运行期（动态分派）
发生范围同一个类内父子类之间
方法名相同相同
参数列表必须不同必须相同
返回类型可以不同必须相同或协变
关键字无@Override（推荐加）

7. return 语句

三种用法

// 用法 1：返回一个值
public int square(int x) {
    return x * x;       // 方法立即结束，将 x*x 传回调用方
}

// 用法 2：void 方法提前退出（卫语句模式）
public void process(String s) {
    if (s == null || s.isEmpty()) return;   // 卫语句，提前退出
    // 以下逻辑保证 s 非空
    System.out.println(s.toUpperCase());
}

// 用法 3：返回表达式（常见简化写法）
public boolean isEven(int n) {
    return n % 2 == 0;  // 直接返回布尔表达式结果
}
编译器路径检查陷阱：编译器会分析所有可能的执行路径，每条路径必须有 return（或抛出异常）。 
// ❌ 编译错误：编译器认为 else 路径没有 return
public int abs(int n) {
    if (n >= 0) return n;
    // 忘了写 else！
}

// ✅ 正确
public int abs(int n) {
    if (n >= 0) return n;
    return -n;
}

8. 递归方法

// 经典递归：计算阶乘
// 关键：① 基准情形（终止条件）② 递推关系（每次向基准逼近）
public static long factorial(int n) {
    if (n <= 1) return 1;           // ① 基准情形
    return n * factorial(n - 1);   // ② 递推：规模缩小 1
}

递归 vs 迭代对比

维度递归迭代
代码可读性贴近数学定义，直观命令式，步骤显式
内存开销每次调用占一个栈帧只需常数级空间
栈溢出风险有（深度过大）无
典型场景树遍历、分治、回溯线性计算、大数据量循环
Java 无尾调用优化（TCO）：Scala、Kotlin（JVM 上通过 tailrec）等语言可以将尾递归优化为循环，但标准 Java 不做这个优化。生产代码中，对深度不可控的递归（如处理用户数据的树形结构），应改写为显式栈迭代，避免 StackOverflowError。

9. 栈帧模型（Stack Frame）

每次方法调用，JVM 在当前线程的虚拟机栈上压入一个栈帧，包含：

区域存放内容
局部变量表（Local Variable Table）方法的参数 + 方法体内声明的局部变量（基本类型存值，引用类型存引用）
操作数栈（Operand Stack）字节码指令执行时的计算工作区（类似 CPU 寄存器的作用）
动态链接（Dynamic Linking）指向运行时常量池中当前方法的符号引用
返回地址（Return Address）方法返回后，程序计数器应跳回的位置

调用栈示意

main() 调用 factorial(3)
  factorial(3) 调用 factorial(2)
    factorial(2) 调用 factorial(1)
      factorial(1) 返回 1         ← 弹出栈帧
    factorial(2) 返回 2           ← 弹出栈帧
  factorial(3) 返回 6             ← 弹出栈帧
main() 继续执行

栈（顶部 = 当前执行）：
┌─────────────────────┐
│  factorial(1)       │ ← 栈顶（正在执行）
├─────────────────────┤
│  factorial(2)       │
├─────────────────────┤
│  factorial(3)       │
├─────────────────────┤
│  main()             │ ← 栈底
└─────────────────────┘
抛出异常时，JVM 会遍历栈帧寻找匹配的 catch 块；e.printStackTrace() 打印的就是这个调用栈快照。

10. 常见陷阱与面试考点

陷阱 1：static 方法误用实例成员

public class Counter {
    private int count = 0;

    // ❌ static 方法没有 this，无法访问实例字段
    public static void increment() {
        count++;   // 编译错误：non-static variable count cannot be referenced from a static context
    }

    // ✅ 实例方法才能访问实例字段
    public void increment() {
        count++;
    }
}

陷阱 2：自动类型提升意外匹配

public static void test(long x)   { System.out.println("long"); }
public static void test(double x) { System.out.println("double"); }

test(42);    // 打印 "long"，而不是 "double"
             // 因为 int → long 比 int → double 提升步数少

test(42L);   // 明确传 long，打印 "long"
test(42.0);  // 明确传 double，打印 "double"

陷阱 3：void 方法不能用于返回值表达式

public void doSomething() { ... }

// ❌ void 方法没有返回值，不能这样用
String result = doSomething();    // 编译错误
System.out.println(doSomething()); // 编译错误

面试高频题

问题关键答案
重载和重写有什么区别？重载是编译期静态分派，方法名相同但参数不同；重写是运行期动态分派，父子类间方法签名完全相同
Java 是值传递还是引用传递？永远是值传递。传引用类型时，传的是引用的拷贝，不是对象本身
方法可以有多个返回值吗？语法上只能返回一个值。变通：返回数组、List、Map，或自定义 DTO/Record
递归一定比循环慢吗？不一定，但 Java 无 TCO，深递归有栈溢出风险，大数据量场景推荐迭代
varargs 和数组参数有何区别？varargs 是语法糖，编译后就是数组；区别在于调用方不用显式构造数组。不能同时定义 f(int...) 和 f(int[])，会冲突
