算术运算符

算术运算符对数值进行计算，行为直观，但有几处细节容易踩坑。

运算符含义示例结果
+加法 / 字符串拼接3 + 47
-减法10 - 37
*乘法3 * 412
/除法（整数截断）7 / 23
%取模（余数）7 % 21
++自增（前置/后置）++a / a++见下文
--自减（前置/后置）--a / a--见下文

整数除法与取模

Java 整数除法直接向零截断，不是向下取整。当操作数为负数时结果与数学上略有差异：

整数除法Java
int a = 7, b = 2;
System.out.println(a / b);   // 3（小数部分直接丢弃）
System.out.println(-7 / 2);  // -3（向零截断，而非 -4）
System.out.println(-7 % 2);  // -1（余数符号与被除数相同）

// 如需向下取整，使用 Math.floorDiv
System.out.println(Math.floorDiv(-7, 2));  // -4

前置 ++ 与后置 ++ 的区别

前置（++i）：先加 1，再返回新值。后置（i++）：先返回旧值，再加 1。

前置 vs 后置Java
int i = 5;
int a = i++;  // a = 5，i 变为 6（先用旧值）
int b = ++i;  // b = 7，i 变为 7（先加再用）
陷阱：在复杂表达式中混用 i++ 和 ++i 极易出错，比如 arr[i] = i++ 的行为在不同语言中都未必一致。实际项目中，单独一行写自增，不要嵌在表达式里。

+ 运算符的双重身份

当操作数之一为 String 时，+ 变为字符串拼接。这个规则由编译器在编译期转换（底层生成 StringBuilder），但在循环中反复拼接会产生大量临时对象：

字符串拼接陷阱Java
// 反例：O(n²) 内存开销，每次 + 都创建新对象
String s = "";
for (int i = 0; i < 1000; i++) s += i;

// 正例：用 StringBuilder，O(n) 追加
StringBuilder sb = new StringBuilder();
for (int i = 0; i < 1000; i++) sb.append(i);
String result = sb.toString();

比较运算符

比较运算符返回 boolean，用于条件判断。对基本类型，它们比较值；对引用类型，== 比较的是引用（内存地址），而非内容。

运算符含义示例
==等于a == b
!=不等于a != b
>大于a > b
<小于a < b
>=大于等于a >= b
<=小于等于a <= b
高频陷阱：== 与 equals
String s1 = new String("hello"); String s2 = new String("hello");
s1 == s2 是 false（两个不同对象），s1.equals(s2) 才是 true。
对所有非基本类型，比较内容请用 equals()，不要用 ==。
== vs equalsJava
String a = "hello";          // 字符串常量池中的对象
String b = "hello";          // 同一个常量池对象，所以 a == b 为 true
String c = new String("hello");  // 堆上新对象，c == a 为 false

System.out.println(a == b);       // true（偶然，因为常量池复用）
System.out.println(a == c);       // false
System.out.println(a.equals(c));  // true（比较内容）

// Integer 缓存：-128 ~ 127 会复用对象
Integer x = 127, y = 127;
System.out.println(x == y);   // true（缓存范围内）
Integer p = 128, q = 128;
System.out.println(p == q);   // false（超出缓存，新建对象）

逻辑运算符

逻辑运算符对 boolean 值进行操作。Java 提供短路版（&&、||）和非短路版（&、|）两种形式。

运算符含义是否短路说明
&&逻辑与是左侧为 false 时，右侧不计算
||逻辑或是左侧为 true 时，右侧不计算
!逻辑非—取反
&逻辑与（非短路）否两侧都会求值
|逻辑或（非短路）否两侧都会求值
^逻辑异或否两侧不同则为 true

短路求值的原理与价值

短路求值（Short-circuit Evaluation）是一种惰性计算策略：只要左侧已经能决定整体结果，就跳过右侧。这既是性能优化，也是防止空指针的常用模式：

短路求值防 NPEJava
String name = null;

// 反例：name 为 null 时抛出 NullPointerException
if (name.length() > 0) { }

// 正例：利用 && 短路，左侧为 false 时右侧不执行
if (name != null && name.length() > 0) {
    System.out.println("非空字符串");
}

// || 的短路：找到第一个非 null 值
String display = name != null ? name : "匿名";

什么时候用非短路的 & 和 |？

当右侧表达式必须执行（有副作用，比如计数器自增）时，才使用非短路版本。日常逻辑判断几乎永远用 && 和 ||。

面试考点：&& 与 & 的区别不只是"短路与否"，还有类型：&& 只能用于 boolean，而 & 既能用于 boolean（非短路逻辑），也能用于整数（位运算）。

位运算符

位运算直接操作整数的二进制位。CPU 执行位运算只需一条指令，是最快的运算之一。理解位运算是读懂 JDK 源码（HashMap、ThreadLocal 等）的必备基础。

运算符名称规则示例（8位）
&按位与两位都为 1 → 11010 & 1100 = 1000
|按位或任一为 1 → 11010 | 1100 = 1110
^按位异或两位不同 → 11010 ^ 1100 = 0110
~按位取反0↔1 全部翻转~0000 1010 = 1111 0101
<<左移向左移，低位补 01 << 3 = 8
>>算术右移高位补符号位-8 >> 1 = -4
>>>逻辑右移高位补 0（无符号）-1 >>> 28 = 15

位运算的典型用途

位运算实战Java
// 1. 判断奇偶（比 % 2 更快）：末位为 1 则奇数
boolean isOdd = (n & 1) == 1;

// 2. 2 的幂次乘除（左移 = *2，右移 = /2）
int x = 5 << 3;   // 5 * 8 = 40
int y = 40 >> 3;  // 40 / 8 = 5

// 3. HashMap 容量取模（n-1 & hash，n 必须是 2 的幂）
int capacity = 16;
int index = hash & (capacity - 1);   // 等价于 hash % 16，但快得多

// 4. 用异或交换两个变量（无需临时变量）
int a = 3, b = 5;
a = a ^ b;  // a = 6 (0110)
b = a ^ b;  // b = 3
a = a ^ b;  // a = 5

// 5. 用位掩码组合权限标志（Spring Security / Linux 文件权限同理）
int READ    = 1 << 0;  // 0001
int WRITE   = 1 << 1;  // 0010
int EXECUTE = 1 << 2;  // 0100
int perm = READ | WRITE;            // 授予读写权限
boolean canRead = (perm & READ) != 0;  // 检查是否有读权限

>> 与 >>> 的区别

两者都是右移，区别在于高位如何填充：算术右移 >> 用符号位填充（负数高位补 1），逻辑右移 >>> 始终补 0。

算术右移 vs 逻辑右移Java
int neg = -8;                     // 二进制：11111111 11111000
System.out.println(neg >> 1);    // -4，高位补 1，符号不变
System.out.println(neg >>> 1);   // 2147483644，高位补 0，变正数

// HashMap 中用 >>> 避免符号位影响哈希分布
// hash(key) = (h = key.hashCode()) ^ (h >>> 16)

实践原则：当表达式混合多类运算符时，主动加括号明确优先级，让代码意图一目了然。(a > 0) && (b < 10) 比 a > 0 && b < 10 更清晰，即使优先级相同。

数值类型自动提升

Java 在运算时有一套隐式类型提升规则，是很多编译错误的根源：

  • byte、short、char 参与运算时，自动提升为 int
  • 若操作数有 long，整体提升为 long
  • 若有 float，提升为 float；有 double，提升为 double
类型提升陷阱Java
byte a = 10, b = 20;
// 编译错误：a + b 的结果是 int，不能直接赋给 byte
// byte c = a + b;
byte c = (byte) (a + b);  // 必须强制转换

// 整数字面量 + long：结果是 long
long big = 2_000_000_000 + 2_000_000_000;  // 溢出！两个 int 相加先溢出再赋给 long
long ok  = 2_000_000_000L + 2_000_000_000L; // 正确：加 L 后缀强制 long 运算

// 整数除法：结果截断小数
double avg = 1 / 2;        // 0.0，先整除得 0，再赋给 double
double avg2 = 1.0 / 2;     // 0.5，1.0 是 double，触发类型提升

面试高频考点汇总

1. i++ 与 ++i 的区别
前置先加后用，后置先用后加。在 for 循环中单独使用效果相同；在表达式中嵌套时行为不同。
2. == 比较引用类型
对对象比较的是引用地址，而非内容。Integer 的 -128~127 缓存是经典考题：Integer a = 127; Integer b = 127; a == b 为 true；改成 128 则为 false。
3. && 短路求值
左侧为 false 时右侧不执行，常用于防 NPE。面试时被问到"如何安全调用可能为 null 的对象方法"，短路求值是标准答案之一。
4. HashMap 为什么容量必须是 2 的幂？
因为使用 (capacity - 1) & hash 代替 hash % capacity 取模，只有 capacity 为 2 的幂时两者等价，且位运算更快。
5. >> 与 >>> 的区别
算术右移保留符号位，逻辑右移高位补 0。对无符号场景（如哈希值处理）必须用 >>>，否则负数右移会一直是负数。
6. 整数溢出不报错
Java 整数运算溢出时不抛异常，直接回绕（wrap around）。Integer.MAX_VALUE + 1 == Integer.MIN_VALUE。大数运算需使用 Math.addExact() 或 BigInteger。
