23. String 常用方法

掌握 String 的核心 API，理解每个方法背后的行为契约，避开高频陷阱。

为什么 String 的方法如此丰富？

字符串是编程中最高频的数据类型之一。JDK 设计者从一开始就把 String 定位为"值对象"——不可变、线程安全、可缓存哈希。 正因为不可变，所有"修改"操作都会返回一个新对象，原字符串永远不变。 这一设计贯穿所有方法，是理解 String API 最重要的前提。

上一节讲过：不可变性 意味着每次调用 replace、substring、toUpperCase 等都会产生新的 String 对象。 在循环中大量拼接字符串时，请改用 StringBuilder（下一节主题）。

核心方法速览

按功能分组，便于记忆与查找：

功能分类 主要方法 说明
长度 / 空判断 length()、isEmpty()、isBlank() isBlank() 是 Java 11 新增，空白字符也算空
字符 / 子串提取 charAt(i)、substring(from)、substring(from,to) 区间左闭右开；越界抛 IndexOutOfBoundsException
查找 indexOf(str)、lastIndexOf(str)、contains(seq) 找不到返回 -1，不是 null
比较 equals(obj)、equalsIgnoreCase(s)、compareTo(s)、startsWith()、endsWith() 切记不要用 == 比较内容
大小写 toUpperCase()、toLowerCase() 建议传 Locale.ROOT 避免土耳其语等 locale 问题
去空白 trim()、strip()、stripLeading()、stripTrailing() strip() 是 Java 11，支持 Unicode 空白；trim() 只处理 ASCII
替换 replace(old,new)、replaceAll(regex,r)、replaceFirst(regex,r) replaceAll 使用正则，性能较低
拆分 / 拼接 split(regex)、join(delim, ...) join 是 Java 8 静态方法
转换 toCharArray()、getBytes(charset)、valueOf(obj)、format() 编码转换时必须显式指定 charset
匹配 matches(regex) 等价于 Pattern.matches(regex, str)，全串匹配

长度与空值判断

length() 返回 char 数量，对于 Unicode 增补字符（码点 > U+FFFF，如 emoji） 一个字符会占 2 个 char（代理对），此时 length() 会返回 2 而非 1。 若需要准确的字符数，应使用 codePointCount(0, s.length())。

LengthDemo.java
String emoji = "😀";
System.out.println(emoji.length());          // 2  ← 代理对占两个 char
System.out.println(emoji.codePointCount(0, emoji.length())); // 1  ← 真正的字符数

String blank = "  \t\n  ";
System.out.println(blank.isEmpty());         // false（有空白字符）
System.out.println(blank.isBlank());         // true  （Java 11+，全是空白）

子串提取：substring

substring(beginIndex, endIndex) 遵循左闭右开原则，与数组切片一致。 记忆口诀：end - begin = 截取长度。

SubstringDemo.java
String s = "Hello, Java!";
//              0123456789...

System.out.println(s.substring(7));       // "Java!"  从 7 截到末尾
System.out.println(s.substring(7, 11));   // "Java"   [7, 11)，长度 11-7=4

// 常见惯用法：截掉前缀
String url = "https://example.com";
if (url.startsWith("https://")) {
    String host = url.substring("https://".length()); // "example.com"
}
陷阱：Java 7u6 之前，substring 共享底层 char[] 数组，可能导致内存泄漏（持有大字符串的引用）。 Java 7u6 之后改为复制数组，内存泄漏问题已修复，但也意味着每次 substring 都有内存分配开销。

查找：indexOf / contains

indexOf 是底层方法，contains 内部调用它。 理解 indexOf 的返回值语义：找到返回起始下标，找不到返回 -1。

IndexOfDemo.java
String s = "banana";

System.out.println(s.indexOf('a'));          // 1  第一个 'a' 的位置
System.out.println(s.lastIndexOf('a'));      // 5  最后一个 'a' 的位置
System.out.println(s.indexOf('a', 2));      // 3  从下标 2 开始向右找
System.out.println(s.contains("nan"));       // true
System.out.println(s.indexOf("xyz"));        // -1  找不到

// 遍历所有出现位置的经典写法
int idx = 0;
while ((idx = s.indexOf('a', idx)) != -1) {
    System.out.println("found at: " + idx);
    idx++; // 移动一位，避免死循环
}

比较：equals vs ==

这是 Java 最高频的陷阱之一。== 比较引用地址， equals 比较字符内容。 两个内容相同但来自不同对象的 String，== 会返回 false。

EqualsDemo.java
// 反例 ——————————————————————
String a = new String("hello");
String b = new String("hello");
System.out.println(a == b);            // false ← 两个不同堆对象
System.out.println(a.equals(b));       // true  ← 内容相同

// 正例：防 NullPointerException 的写法
String input = getUserInput();  // 可能为 null
if ("admin".equals(input)) {   // 常量放左边，不怕 NPE
    // ...
}

// 大小写不敏感比较
System.out.println("Java".equalsIgnoreCase("JAVA")); // true

// 字典序比较（排序时用）
System.out.println("apple".compareTo("banana"));  // 负数：apple 在 banana 前面

大小写转换

toUpperCase() 和 toLowerCase() 依赖 JVM 的默认 Locale。 在某些语言环境下（如土耳其语），'i'.toUpperCase() 会变成带点的大写 İ， 而非英文的 I。处理 ASCII 标识符、协议字段时，应传 Locale.ROOT。

CaseDemo.java
// 危险写法（依赖系统 Locale）
String method = "get".toUpperCase();           // 在土耳其 Locale 下出错

// 安全写法
String method2 = "get".toUpperCase(Locale.ROOT); // "GET" 稳定可靠

// 用于忽略大小写的标准化比较
String normalized = userInput.toLowerCase(Locale.ROOT).trim();

去空白：trim vs strip

trim() 是老 API，只能去掉 ASCII 空白（码点 ≤ 32 的字符）。 Java 11 引入的 strip() 调用 Character.isWhitespace()， 能识别全角空格、不间断空格等 Unicode 空白字符，新代码应优先使用 strip()。

StripDemo.java
String s = "　 hello 　"; // 　 是全角空格

System.out.println("[" + s.trim()  + "]");  // [　 hello 　]  ← 全角空格未去除
System.out.println("[" + s.strip() + "]");  // [hello]        ← 正确去除

// 只去一侧的版本（Java 11+）
s.stripLeading();   // 去左侧空白
s.stripTrailing();  // 去右侧空白

替换：replace vs replaceAll

replace(CharSequence, CharSequence) 做字面量替换，不解析正则，性能更好。 replaceAll(String regex, String replacement) 使用正则引擎，有编译开销， 且 replacement 字符串中 $ 和 \ 有特殊含义，容易引发 bug。

ReplaceDemo.java
String s = "2024.01.15";

// replace：字面量替换，简单可靠
System.out.println(s.replace(".", "-"));       // "2024-01-15"

// replaceAll：正则替换，"." 是通配符，要转义！
System.out.println(s.replaceAll("\\.", "-"));  // "2024-01-15"（正确转义）
System.out.println(s.replaceAll(".", "-"));   // "----------"（错！"."匹配所有字符）

// replacement 中 $ 的陷阱
String price = "100";
// price.replaceAll("\\d+", "$50");  ← 会报错，$ 被当做组引用
price.replaceAll("\\d+", Matcher.quoteReplacement("$50")); // 安全写法
性能陷阱：在循环中反复调用 replaceAll 会每次重新编译正则。 若需频繁使用，应提取为 static final Pattern 并调用 pattern.matcher(s).replaceAll(...)。

拆分与拼接：split / join

SplitJoinDemo.java
// split 返回 String[]
String[] parts = "a,b,,c,".split(",");
// ["a","b","","c"]  ← 末尾空串被丢弃！

// 第二个参数 limit=-1：保留末尾空串
String[] all = "a,b,,c,".split(",", -1);
// ["a","b","","c",""]  ← 末尾空串保留

// split("|") 是正则 OR，需要转义！
String[] wrong = "a|b|c".split("|");     // 错！每个字符都拆开
String[] right  = "a|b|c".split("\\|");   // 正确转义

// join：Java 8 静态方法，比手动循环干净
String csv = String.join(", ", "Alice", "Bob", "Carol");
// "Alice, Bob, Carol"

// 从 List 拼接
List<String> names = List.of("x", "y", "z");
String result = String.join("-", names); // "x-y-z"
split 末尾空串陷阱： "a,,b,".split(",") 默认丢弃末尾的空字符串，结果为 3 个元素。 解析 CSV 等格式时若字段可以为空，必须传 limit = -1。

格式化：String.format vs 文本块

String.format() 使用 printf 风格占位符，底层创建 Formatter 对象， 比手动拼接更易读，但有一定性能开销。Java 15+ 的文本块（Text Blocks）和 Java 21 的字符串模板 进一步降低了格式化的复杂度。

FormatDemo.java
// String.format — Java 5+
String msg = String.format("用户 %s 的余额：%.2f 元", "Alice", 99.5);
// "用户 Alice 的余额：99.50 元"

// formatted() — Java 15+，链式调用更流畅
String msg2 = "Hello, %s!".formatted("World");

// 文本块 — Java 15+，多行字符串不再需要 \n
String json = """
        {
            "name": "Alice",
            "age": 30
        }
        """;

转换方法：valueOf / toCharArray / getBytes

String.valueOf(obj) 是安全的静态工厂，当 obj 为 null 时返回字符串 "null"， 而 obj.toString() 则会抛出 NullPointerException。 getBytes() 涉及字符编码，必须显式指定 charset， 否则依赖平台默认编码，在不同环境下行为不一致。

ConvertDemo.java
// valueOf 对 null 安全
Object obj = null;
System.out.println(String.valueOf(obj));  // "null"（字符串）
// System.out.println(obj.toString());    ← NPE！

// toCharArray：方便逐字符处理
char[] chars = "hello".toCharArray();
for (char c : chars) { /* ... */ }

// getBytes：必须指定编码
byte[] bytes = "你好".getBytes(StandardCharsets.UTF_8);
String back  = new String(bytes, StandardCharsets.UTF_8); // 编码一致才能还原

综合示例：解析 HTTP 请求行

把常用方法组合起来，演示实际场景中的串联使用：

ParseRequestLine.java
/**
 * 解析 HTTP 请求行，如 "GET /api/users?page=1 HTTP/1.1"
 * 提取 method、path、query
 */
public static void parseRequestLine(String line) {
    if (line == null || line.isBlank()) return;

    // 1. 按空格拆分成三段
    String[] parts = line.strip().split(" ", 3);

    String method   = parts[0].toUpperCase(Locale.ROOT);
    String fullPath = parts[1];
    String version  = parts[2];

    // 2. 拆分 path 和 query string
    int qIdx = fullPath.indexOf('?');
    String path  = qIdx >= 0 ? fullPath.substring(0, qIdx) : fullPath;
    String query = qIdx >= 0 ? fullPath.substring(qIdx + 1) : "";

    System.out.println("method=" + method);   // GET
    System.out.println("path="   + path);     // /api/users
    System.out.println("query="  + query);    // page=1
}

与其他知识的联系

String 方法调用链 ├── 底层依赖 char[]（Java 8 及之前）/ byte[]（Java 9+ Compact Strings） ├── substring → Java 7u6 后创建新数组，解决旧版内存泄漏 ├── split / replaceAll → 内部用 Pattern / Matcher（正则引擎） ├── compareTo → 实现 Comparable<String>，支持 TreeMap / 排序 ├── hashCode → 31 倍乘法公式，缓存在字段中（不可变才能缓存） └── intern() → 与常量池交互，下节会深入讲

高频陷阱汇总

  • 用 == 比较 String 内容：永远使用 equals()，常量放左边防 NPE。
  • split 丢弃末尾空串：需要保留时传 limit = -1。
  • replaceAll 中 . 匹配所有字符：字面量替换优先用 replace()。
  • replace 返回值被丢弃：String 不可变，必须接收返回值 s = s.replace(...)。
  • trim 不处理全角空格：Java 11+ 用 strip() 更安全。
  • getBytes() 不指定编码：跨平台时必须显式 StandardCharsets.UTF_8。
  • 循环中 + 拼接字符串：每次 + 产生新对象，应改用 StringBuilder（下节主题）。

面试高频考点

考点 1：String s = "a" + "b" + "c" 编译后等价于 "abc"，因为常量折叠（constant folding）在编译期完成，不会产生额外对象。
考点 2：substring(0, n) 的时间复杂度是 O(n)，因为需要复制 n 个字符。 不要误以为 O(1)。
考点 3：indexOf 是朴素字符串匹配，最坏时间复杂度 O(n*m)。 大量文本搜索应使用 KMP 或 Pattern.compile 预编译。
考点 4：String.format 内部每次都创建 Formatter 对象， 高并发热路径中频繁调用会产生 GC 压力，可改用 StringBuilder 手动拼接或使用日志框架的占位符（{}）。
面试题 核心答案
equals 和 == 的区别 == 比引用地址，equals 比内容；String 重写了 equals
split(",") 和 split(",", -1) 的区别 前者丢弃末尾空串，后者保留所有元素
replace 和 replaceAll 的区别 前者字面量替换，后者正则替换；性能与语义均不同
trim 和 strip 的区别 trim 只处理 ASCII ≤ 32，strip 支持 Unicode 空白（Java 11+）
