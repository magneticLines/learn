16. 方法重载

同一个类里，用相同的名字做不同参数签名的事——编译器帮你在调用处选择正确版本。

为什么需要方法重载

没有重载之前，C 语言里要写 absInt()、absFloat()、absDouble()—— 三个函数干同一件事，只是类型不同。程序员要记住三个名字，调用时还要对号入座，极易出错。

Java 的答案是：方法重载（Overloading）——同一个类内，允许多个方法共享同一个名字， 只要它们的参数列表不同即可。编译器在编译期（静态分派）根据实参类型自动选择最合适的版本， 调用方只需记住一个语义名称。

重载是 编译期多态（静态多态），决策发生在 javac 阶段，与运行时无关。 这与方法重写（Override）的运行期多态（动态分派）形成对比。

重载的合法规则

Java 规范要求：重载方法必须在以下三个维度中至少改变一个：

维度举例是否构成重载
参数数量不同add(int) vs add(int, int)✅ 是
参数类型不同print(int) vs print(double)✅ 是
参数顺序不同（类型不完全相同时）f(int, double) vs f(double, int)✅ 是
仅返回值不同int get() vs double get()❌ 否，编译报错
仅访问修饰符不同public foo() vs private foo()❌ 否，编译报错
仅异常声明不同void f() throws A vs void f() throws B❌ 否，编译报错
仅返回值不同不构成重载。 原因：调用方可以不接收返回值（如 foo();）， 此时编译器无法通过返回类型区分调用哪个版本，产生歧义。

基础代码示例

Calculator.java基础重载
public class Calculator {

    // 重载版本 1：两个整数相加
    public int add(int a, int b) {
        return a + b;
    }

    // 重载版本 2：三个整数相加（参数数量不同）
    public int add(int a, int b, int c) {
        return a + b + c;
    }

    // 重载版本 3：两个双精度浮点数相加（参数类型不同）
    public double add(double a, double b) {
        return a + b;
    }

    // 重载版本 4：拼接字符串（参数类型完全不同）
    public String add(String a, String b) {
        return a + b;
    }

    public static void main(String[] args) {
        Calculator calc = new Calculator();

        // 编译器静态分派，选择对应版本
        System.out.println(calc.add(1, 2));          // → 3       (int, int)
        System.out.println(calc.add(1, 2, 3));       // → 6       (int, int, int)
        System.out.println(calc.add(1.5, 2.5));       // → 4.0     (double, double)
        System.out.println(calc.add("Hello, ", "World")); // → Hello, World
    }
}

编译器如何选择重载版本

当调用一个重载方法时，javac 按照三阶段匹配策略来确定最终调用哪个版本：

阶段 1：精确匹配 ── 不允许任何自动转换，直接命中
阶段 2：宽化转换匹配 ── 允许基本类型向上提升（byte→int→long→float→double）
阶段 3：装箱/拆箱 + 变长参数匹配 ── 允许 int↔Integer，以及 varargs

编译器在每个阶段里寻找唯一最具体的方法。若在某阶段找到唯一匹配就停止； 若某阶段有多个等价匹配则报 ambiguous method call 编译错误。

OverloadResolution.java三阶段匹配演示
public class OverloadResolution {

    static void show(int x)     { System.out.println("int: "     + x); }
    static void show(long x)    { System.out.println("long: "    + x); }
    static void show(Integer x) { System.out.println("Integer: " + x); }

    public static void main(String[] args) {
        // 阶段 1 精确匹配：字面量 42 是 int，命中 show(int)
        show(42);              // → int: 42

        // 阶段 2 宽化转换：byte 没有精确匹配，向上提升到 int
        byte b = 10;
        show(b);              // → int: 10（byte→int，而非 byte→long）

        // 阶段 3 装箱：Integer 对象，精确命中 show(Integer)
        Integer i = 99;
        show(i);              // → Integer: 99
    }
}
记忆口诀：精确优先，宽化其次，装箱最后，变长参数垫底。优先级越高，编译器越倾向选它。

常见陷阱与反例

陷阱 1：null 引发歧义

NullAmbiguity.javanull 导致编译歧义
static void process(String s)  { System.out.println("String");  }
static void process(Integer i) { System.out.println("Integer"); }

// ❌ 编译错误：null 同时满足 String 和 Integer，产生歧义
process(null);

// ✅ 解决方法：显式强制类型转换消除歧义
process((String) null);   // → String
process((Integer) null);  // → Integer

陷阱 2：宽化转换 vs 装箱，宽化优先

WideningVsBoxing.java宽化比装箱优先级高
static void test(long x)    { System.out.println("long");    }
static void test(Integer x) { System.out.println("Integer"); }

// int 字面量 5：宽化（int→long）比装箱（int→Integer）优先
test(5);  // → long  （不是 Integer，这里很反直觉！）
反直觉点：很多人以为 int→Integer 很自然，但 Java 规范规定宽化转换优先于自动装箱，所以 int 会先向上提升为 long，而不是装箱成 Integer。

陷阱 3：子类继承后的重载 vs 重写混淆

OverloadVsOverride.java不要混淆重载和重写
class Animal {
    void sound(String name) { System.out.println("Animal: " + name); }
}

class Dog extends Animal {
    // 这是重写（Override）：方法签名完全相同
    @Override
    void sound(String name) { System.out.println("Dog: " + name); }

    // 这是重载（Overload）：参数类型从 String 变成了 int，是新方法
    void sound(int times) { System.out.println("Woof x" + times); }
}

重载 vs 重写：核心区别

维度方法重载（Overloading）方法重写（Overriding）
发生范围同一个类内（或子类继承）父子类之间
方法名相同相同
参数列表必须不同必须相同
返回类型可以不同必须相同或协变（子类型）
决策时机编译期（静态分派）运行期（动态分派）
多态类型静态多态动态多态
@Override不适用强烈建议加，防止写错签名
与 static 方法可以重载 static 方法static 方法只能隐藏（hide），无法重写

JDK 中的真实重载案例

理解重载最好的方式是看 JDK 源码里到处都是的例子：

  • System.out.println() ——有 10 个重载版本，覆盖 boolean、char、int、long、float、double、char[]、String、Object 等，你传什么类型就自动调对应版本。
  • String.valueOf() ——同样有多个重载，统一入口，内部分别实现。
  • Arrays.sort() ——针对 int[]、long[]、double[]、Object[] 分别有重载，避免泛型带来的装箱开销。
  • Math.abs()、Math.max()、Math.min() ——分别对 int、long、float、double 重载，保证基本类型不装箱。
PrintStream.java（JDK 简化版）println 的重载
// java.io.PrintStream 内部（简化）
public void println(boolean x) { ... }
public void println(char    x) { ... }
public void println(int     x) { ... }
public void println(long    x) { ... }
public void println(float   x) { ... }
public void println(double  x) { ... }
public void println(String  x) { ... }
public void println(Object  x) { ... }
// 设计意图：基本类型有专属版本，避免每次都装箱成 Object

构造器重载

构造器不是普通方法，但同样支持重载。这也是 Java 中最常见的重载场景之一， 提供多种初始化方式，同时可以用 this(...) 进行构造器间的调用委托，避免重复代码。

Person.java构造器重载 + this() 委托
public class Person {
    private final String name;
    private final int    age;
    private final String email;

    // 最完整的构造器，真正执行初始化
    public Person(String name, int age, String email) {
        this.name  = name;
        this.age   = age;
        this.email = email;
    }

    // 没有 email 时，委托给上面的构造器，传入默认值
    public Person(String name, int age) {
        this(name, age, "");  // this() 必须是构造器第一行
    }

    // 只有名字时，再次向上委托
    public Person(String name) {
        this(name, 0);
    }
}
设计建议：多个构造器重载时，应让"参数最少的"委托给"参数最多的"，所有初始化逻辑集中在一处。这正是 Joshua Bloch 《Effective Java》Item 1 中静态工厂方法模式的前身思想。

面试高频考点

  • 重载和重写的区别？ ——必考，记住"编译期 vs 运行期"、"同类 vs 父子类"、"参数不同 vs 参数相同"。
  • 能否只靠返回值不同来重载？ ——不能，原因是调用方可以忽略返回值，编译器无法消歧。
  • println(null) 会调用哪个版本？ ——若有多个引用类型重载，编译报歧义；若有 String 和 Object，选 String（更具体）。
  • int 参数同时存在 long 和 Integer 重载时选哪个？ ——选 long，宽化优先于装箱。
  • 泛型擦除后会不会产生重载冲突？ ——会！void f(List<Integer>) 和 void f(List<String>) 擦除后同为 void f(List)，编译报错。
GenericErasure.java泛型擦除导致重载冲突
// ❌ 编译报错：泛型擦除后两个方法签名相同，无法共存
void process(List<Integer> list) { ... }
void process(List<String>  list) { ... }
// 擦除后都变成 void process(List list)，冲突！
