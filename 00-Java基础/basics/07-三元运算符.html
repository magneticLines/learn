07. 三元运算符

Java 中唯一一个接受三个操作数的运算符，本质是 if-else 的表达式形式——理解它的类型推断规则，能帮你写出更简洁、更安全的条件赋值代码。

基本语法与执行流程

三元运算符（Ternary Operator）又称条件运算符，格式为：

条件表达式 ? 值A（条件为 true） : 值B（条件为 false）

整个表达式会求值为 值A 或 值B 中的一个，并作为表达式的结果返回。与 if-else 语句不同，它是一个表达式，有返回值，可以直接赋给变量或嵌入更大的表达式里。

Java最简示例
int score = 85;
String result = score >= 60 ? "及格" : "不及格";
// score >= 60 为 true，所以 result = "及格"

// 等价的 if-else 写法（语句，无返回值）：
String result2;
if (score >= 60) {
    result2 = "及格";
} else {
    result2 = "不及格";
}
为什么叫"三元"？
运算符按操作数个数分类：一元（-x、!b）、二元（a + b）、三元（cond ? a : b）。Java 中三元运算符是唯一的三元运算符，符号为 ?:。

为什么这样设计：表达式 vs 语句

在函数式编程语言（如 Haskell、Scala）里，if-else 本身就是表达式，可以直接求值。Java 的 if-else 是语句，没有值，不能用于赋值右侧或方法参数中。

三元运算符填补了这个空白：它让"根据条件选一个值"这件事可以内联表达，而无需额外的临时变量或 if 块。这在以下场景尤其有价值：

  • 方法调用时动态决定传参：log(debug ? "详细" : "简要")
  • 初始化 final 变量时更简洁（if-else 也能做到，但要多写几行）
  • 在 lambda / Stream 管道里内联条件逻辑
Javafinal 变量场景——if-else 也能做到，三元运算符更简洁
// if-else 写法：可以给同一个 final 变量在不同分支各赋值一次
// （只要保证只被赋值一次，编译器就认可，这是"确定赋值"分析）
// final String label;
// if (score >= 60) label = "P"; else label = "F";

// 三元运算符：一行搞定，更简洁
final String label = score >= 60 ? "P" : "F";

// Stream 管道内联
list.stream()
    .map(x -> x > 0 ? "正" : "非正")
    .forEach(System.out::println);

核心：结果类型如何推断

三元运算符最容易踩坑的地方是结果类型推断。Java 编译器需要把 值A 和 值B 统一成一种类型，规则如下：

值A 类型 值B 类型 结果类型 说明
int int int 相同类型，直接使用
int long long 数值类型自动提升（numeric promotion）
int double double 同上，向更宽的类型提升
Integer int int 触发自动拆箱，可能 NullPointerException！
String Integer Object 找公共父类型
null String String null 可匹配任何引用类型
Java类型推断示例
// 1. 数值提升：int 与 double 混用 → 结果是 double
double d = true ? 1 : 2.0;   // d = 1.0，注意不是 1（int）

// 2. 两侧字面量赋给 byte/short（编译器特殊处理常量折叠）
byte b = true ? 1 : 2;      // OK，字面量范围内可窄化

// 3. 拆箱陷阱
Integer a = null;
int x = true ? a : 0;
// 结果类型是 int（因为 0 是基本类型），a 被自动拆箱 → NullPointerException!

// 安全写法：让两侧都是包装类
Integer safe = true ? a : Integer.valueOf(0);  // 结果是 Integer，不拆箱
陷阱：Integer 与 int 混用导致 NPE
当三元运算符两侧一个是 Integer（包装类），另一个是 int（基本类型）时，编译器会把结果类型定为 int，并对包装类那侧做自动拆箱。如果那个 Integer 为 null，就会在运行时抛出 NullPointerException。

短路求值：只求值一侧

三元运算符具有短路特性：根据条件，值A 和 值B 只有一个会被求值，另一个不会执行。这和 &&、|| 的短路逻辑类似，但维度不同——这里是"二选一执行"。

Java短路求值——副作用示例
int count = 0;

// 条件为 true，只执行 ++count，右侧 count-- 不会被执行
int val = true ? ++count : count--;
// count = 1, val = 1

// 实际开发中不要在三元运算符里写有副作用的表达式
// 上面只是为了说明"只求值一侧"这个特性

嵌套三元运算符：慎用

三元运算符可以嵌套，模拟 if-else if-else：

Java嵌套示例：成绩分级
int score = 75;

// 嵌套写法（可读性差，不推荐超过 2 层）
String grade = score >= 90 ? "A"
             : score >= 75 ? "B"
             : score >= 60 ? "C"
             : "D";
// grade = "B"

// 可读性更好的等价写法（超过 2 分支推荐改用 if-else 或 switch）
String grade2;
if      (score >= 90) grade2 = "A";
else if (score >= 75) grade2 = "B";
else if (score >= 60) grade2 = "C";
else                   grade2 = "D";
经验法则：嵌套三元最多 2 层，超过 2 层请改用 if-else if 或 Java 14+ 的 switch 表达式（同样是表达式，有返回值，更清晰）。

三元运算符 vs if-else 对比

维度 三元运算符 ?: if-else 语句
性质 表达式，有返回值 语句，无返回值
可赋值 可以直接赋给变量 需要提前声明变量
可嵌套于表达式 可以（方法参数、Stream 等） 不可以
执行多条语句 不支持 支持（花括号内多行）
可读性 简单条件时更简洁 复杂逻辑时更清晰
字节码差异 与 if-else 完全等价，编译器生成相同字节码 同左
字节码层面完全等价
用 javap -c 反编译可以发现，score >= 60 ? "及格" : "不及格" 和对应的 if-else 生成的字节码完全一致。三元运算符不是"语法糖"，它只是在语法层面给了表达式形式，运行时没有任何额外开销。

Java 14+ Switch 表达式：更现代的替代

Java 14 引入了 switch 表达式（正式特性），它同样是表达式、有返回值，多分支场景下比嵌套三元运算符更清晰：

Java 14+switch 表达式 vs 嵌套三元
// 嵌套三元（多分支时可读性差）
String day = num == 1 ? "周一" : num == 2 ? "周二" : "其他";

// switch 表达式（Java 14+，更推荐）
String day2 = switch (num) {
    case 1  -> "周一";
    case 2  -> "周二";
    default -> "其他";
};

// 三元运算符适合"二选一"；switch 表达式适合"多选一"

常见陷阱汇总

陷阱 1：将三元运算符当语句用

Java误用为语句
// 编译错误：三元运算符的结果必须被使用
// condition ? doA() : doB();  ← 某些情况下可以，但若返回值不用会有警告

// 如果只是为了执行副作用，直接用 if-else 语句，更清晰
if (condition) doA(); else doB();

陷阱 2：拆箱引发 NullPointerException（重要！）

JavaNPE 陷阱复现
Map<String, Integer> map = new HashMap<>();
// map.get("key") 返回 null（Integer），与 int 字面量 0 混用
int val = map.containsKey("key") ? map.get("key") : 0;
// containsKey 为 true，map.get 返回 null，拆箱时 NPE！

// 正确写法：用 getOrDefault 或显式处理 null
int safe = map.getOrDefault("key", 0);

陷阱 3：优先级低，与其他运算符混用时需加括号

Java优先级陷阱
int a = 2, b = 3;

// 意图：(a > 1) 时取 a，否则取 b，再乘以 10
int wrong = a > 1 ? a : b * 10;
// 实际解析为：a > 1 ? a : (b * 10)，因为 * 比 ?: 优先级高
// a > 1 为 true，结果是 a=2，而非 (b*10)=30，这里没问题

// 但下面这种情况就需要括号：
int x = 5 + (a > 1 ? a : b);   // 括号明确意图，避免歧义

最佳实践

  • 只用于"二选一赋值"：条件简单、两侧表达式简短时用三元；逻辑复杂、分支超过 2 个时改用 if-else 或 switch 表达式
  • 避免副作用：不在三元运算符的值A、值B 中写 ++/-- 或调用有副作用的方法
  • 警惕包装类与基本类型混用：只要有一侧是基本类型，另一侧包装类就会被拆箱，存在 NPE 风险
  • 嵌套不超过 2 层：超过 2 层可读性急剧下降，考虑提取方法或改写为 if-else if
  • 格式化多行对齐：多个嵌套时，将 ? 和 : 对齐在同一列，提升可读性

面试高频考点

Q1：三元运算符和 if-else 有什么区别？
核心区别是：三元运算符是表达式，有返回值，可以内联使用；if-else 是语句，无返回值。字节码层面两者等价，没有性能差异。
Q2：以下代码会抛异常吗？为什么？ 
Integer i = null;
int result = true ? i : 0;
会抛 NullPointerException。因为 0 是 int，编译器将结果类型推断为 int，对 i 执行自动拆箱，null.intValue() 抛出 NPE。
Q3：三元运算符可以不使用返回值吗？
形式上可以写成语句（如 flag ? a++ : b++;），但这是将表达式当语句用，代码意图不清晰。如果只需要副作用，应该用 if-else 语句。某些 IDE/Checkstyle 规则会对此给出警告。
Q4：condition ? null : "hello" 的类型是什么？
结果类型是 String。null 没有具体类型，编译器会将结果类型设为另一侧的类型 String。

与其他知识点的联系

  • 运算符与类型提升：三元运算符的结果类型推断规则，本质是上一节"数值类型自动提升"规则在两个分支之间的应用，理解了 int 与 double 混用会提升为 double，才能理解三元运算符为什么会出现"意料之外的类型"。
  • 包装类与自动装箱：三元运算符两侧类型不一致（一个 Integer、一个 int）时触发的自动拆箱 NPE，是包装类"看起来像基本类型，实际是对象"这一设计权衡最典型的暴露场景之一。
  • switch 表达式（JDK 14+）：两者都是"有返回值的条件分支"，三元运算符适合二选一，switch 表达式适合多选一；JDK 14 引入 switch 表达式的一个动机正是解决嵌套三元运算符可读性差的问题。
  • Stream API / Lambda：三元运算符因为是表达式而非语句，天然适合嵌入 map()、filter() 等函数式接口的 Lambda 体中，这是它在现代 Java 代码里最高频的使用场景。
  • Optional：map.containsKey(k) ? map.get(k) : default 这种三元写法本质上是在手动实现 Optional.orElse() 或 getOrDefault() 的逻辑，理解三元运算符的短路特性有助于理解 Optional 链式方法内部的求值时机。
