20. Arrays 工具类

JDK 内置的数组瑞士军刀——排序、搜索、填充、比较、复制，一行代码搞定，背后原理值得深挖。

为什么需要 Arrays 工具类

数组（int[]、String[]…）是 Java 最底层的容器，但它本身只提供下标访问，没有任何业务方法。 想排序？想搜索？想打印内容？你要么自己写算法，要么借助工具。

java.util.Arrays 是 JDK 自带的静态工具类，诞生于 JDK 1.2，随后在 JDK 5 / 8 / 9 持续扩充。 它的设计遵循 "让常见操作零成本" 原则：所有方法均为 static，无需实例化，直接 Arrays.xxx() 调用。

类比：Arrays 之于数组，就像 Collections 之于 List——都是 JDK 帮你包装好的"瑞士军刀"，避免重复造轮子。

全景速览：常用方法一张表

方法 作用 时间复杂度 备注
Arrays.sort(arr) 升序排序 O(n log n) 基本类型用双轴快排，对象用 TimSort
Arrays.sort(arr, from, to) 区间排序 O(k log k) 左闭右开 [from, to)
Arrays.binarySearch(arr, key) 二分查找 O(log n) 必须先排序，否则结果未定义
Arrays.fill(arr, val) 全部填充为 val O(n) 有区间重载版本
Arrays.copyOf(arr, len) 复制并指定新长度 O(n) 超出部分补零/null
Arrays.copyOfRange(arr, from, to) 区间复制 O(k) 左闭右开
Arrays.equals(a, b) 逐元素比较 O(n) 只比较一维；深比较用 deepEquals
Arrays.deepEquals(a, b) 多维数组深度比较 O(n) 仅对 Object[][] 有意义
Arrays.toString(arr) 一维数组转字符串 O(n) 打印调试首选
Arrays.deepToString(arr) 多维数组转字符串 O(n) 嵌套格式 [[1,2],[3,4]]
Arrays.asList(arr) 包装为固定 List O(1) 不可增删，慎用！
Arrays.stream(arr) 转为 Stream O(1) JDK 8+，支持并行流

sort：排序背后的算法选择

Arrays.sort() 并不是只有一种算法——JDK 根据元素类型做了分支优化：

  • 基本类型（int/long/double…）：使用 双轴快速排序（Dual-Pivot QuickSort），JDK 7 引入，由 Vladimir Yaroslavskiy 设计。平均比经典快排少约 10% 的比较次数，且对已接近有序的数据有特殊处理。
  • 对象类型（Object[]）：使用 TimSort，一种归并排序 + 插入排序的混合算法，稳定排序（相等元素保持原始相对顺序）。
为什么对象用 TimSort 而不是快排？
快排是不稳定排序。当你对 Order[] 先按金额排、再按时间排时，稳定性能保证"金额相同的订单仍然保持时间顺序"。基本类型没有"相等但不同"的概念，所以可以用更快的快排。
Sort 示例Java
import java.util.Arrays;

public class SortDemo {
    public static void main(String[] args) {
        // 基本类型：双轴快排（不稳定，但基本类型无所谓）
        int[] nums = {5, 1, 9, 3, 7};
        Arrays.sort(nums);
        System.out.println(Arrays.toString(nums));  // [1, 3, 5, 7, 9]

        // 区间排序：只排索引 [1, 4) 即下标 1、2、3
        int[] partial = {9, 5, 1, 3, 7};
        Arrays.sort(partial, 1, 4);
        System.out.println(Arrays.toString(partial));  // [9, 1, 3, 5, 7]

        // 对象类型：TimSort（稳定）+ 自定义 Comparator
        String[] words = {"banana", "apple", "cherry"};
        Arrays.sort(words, Comparator.comparingInt(String::length));
        System.out.println(Arrays.toString(words));  // [apple, banana, cherry]
    }
}

binarySearch：二分查找的前提与陷阱

Arrays.binarySearch(arr, key) 返回找到时的下标，找不到时返回 -(插入点) - 1（负数）。 这个设计让你可以从返回值同时得到"在哪里"和"该插在哪里"两个信息。

致命前提：必须先排序。对未排序的数组调用 binarySearch 结果是未定义行为——JDK 不会抛异常，但返回值可能完全错误。这是面试和生产中最常见的 bug 之一。
binarySearch 示例Java
int[] arr = {1, 3, 5, 7, 9};  // 已排序

int idx = Arrays.binarySearch(arr, 5);
System.out.println(idx);   // 2（找到了，下标为 2）

int miss = Arrays.binarySearch(arr, 4);
System.out.println(miss);  // -3（未找到；插入点=2，返回 -(2)-1 = -3）

// 如何还原插入点：
if (miss < 0) {
    int insertPoint = -miss - 1;  // = 2，4 应插在下标 2 处
}

copyOf 与 copyOfRange：安全的数组复制

直接赋值 int[] b = a 只是复制了引用，修改 b 会影响 a。 Arrays.copyOf 会创建一个全新的数组，是真正的浅拷贝（基本类型则是深拷贝，因为基本类型本身就是值）。

copyOf 示例Java
int[] origin = {1, 2, 3};

// 复制并扩容：新长度 5，多余位置填 0
int[] bigger = Arrays.copyOf(origin, 5);
System.out.println(Arrays.toString(bigger));  // [1, 2, 3, 0, 0]

// 复制并截断：新长度 2
int[] smaller = Arrays.copyOf(origin, 2);
System.out.println(Arrays.toString(smaller));  // [1, 2]

// 区间复制：[1, 3) 即下标 1、2
int[] slice = Arrays.copyOfRange(origin, 1, 3);
System.out.println(Arrays.toString(slice));   // [2, 3]
底层实现：Arrays.copyOf 内部调用的是 System.arraycopy()——这是一个 JVM 内置的 native 方法，直接操作内存块（类似 C 的 memcpy），性能极高。自己写 for 循环逐个赋值远不如它快。

equals 与 deepEquals：别踩引用比较的坑

数组直接用 == 比较的是引用地址，用 .equals() 也是（数组没有重写 equals）。 必须用 Arrays.equals 才能逐元素比较。

equals 对比示例Java
int[] a = {1, 2, 3};
int[] b = {1, 2, 3};

// 错误方式：比较引用地址，必然 false
System.out.println(a == b);             // false
System.out.println(a.equals(b));       // false（未重写 equals）

// 正确方式：逐元素比较
System.out.println(Arrays.equals(a, b));  // true

// 多维数组必须用 deepEquals
int[][] m1 = {{1,2},{3,4}};
int[][] m2 = {{1,2},{3,4}};
System.out.println(Arrays.equals(m1, m2));      // false！只比较外层引用
System.out.println(Arrays.deepEquals(m1, m2));  // true

toString 与 deepToString：调试必备

直接打印数组会输出类似 [I@7852e922 这样的地址字符串，毫无意义。 Arrays.toString() 是调试时最快速的"一键打印"方案。

toString 示例Java
int[] arr = {1, 2, 3};
System.out.println(arr);                        // [I@7852e922（无意义）
System.out.println(Arrays.toString(arr));       // [1, 2, 3]

int[][] matrix = {{1,2},{3,4}};
System.out.println(Arrays.toString(matrix));    // [[I@..., [I@...]（内层仍是地址）
System.out.println(Arrays.deepToString(matrix)); // [[1, 2], [3, 4]]

asList：包装成 List 的三个陷阱

Arrays.asList(arr) 返回的是一个固定大小的 List 视图，背后直接引用原数组，不是真正的 ArrayList。

asList 三大陷阱Java
// 陷阱 1：不能增删元素
List<String> list = Arrays.asList("a", "b", "c");
list.add("d");  // UnsupportedOperationException！

// 如需可变 List，手动包一层：
List<String> mutable = new ArrayList<>(Arrays.asList("a", "b"));

// 陷阱 2：修改 List 会反映到原数组（共享存储）
String[] arr = {"x", "y"};
List<String> view = Arrays.asList(arr);
view.set(0, "z");
System.out.println(arr[0]);  // z（原数组被修改了！）

// 陷阱 3：基本类型数组会被整体包装成一个元素
int[] nums = {1, 2, 3};
List<int[]> wrong = Arrays.asList(nums);  // size=1，不是 size=3！
// 应该用 Integer[] 或 Stream：
List<Integer> correct = Arrays.stream(nums).boxed().collect(Collectors.toList());

stream：与 Java 8 Stream API 的无缝衔接

JDK 8 起，Arrays.stream(arr) 可以将数组转为 Stream，享受 filter / map / reduce 等函数式操作， 还能开启并行流利用多核加速。

stream 示例Java
int[] scores = {88, 72, 95, 60, 81};

// 求平均分
double avg = Arrays.stream(scores).average().orElse(0);

// 过滤出及格分并排序
int[] passed = Arrays.stream(scores)
    .filter(s -> s >= 70)
    .sorted()
    .toArray();
System.out.println(Arrays.toString(passed));  // [72, 81, 88, 95]

// 并行流：大数组时有加速效果（小数组反而有线程开销）
long count = Arrays.stream(scores).parallel().filter(s -> s > 80).count();

fill：快速初始化数组

Arrays.fill(arr, val) 将整个数组或指定区间填充为同一个值，常用于动态规划初始化、棋盘重置等场景。

fill 示例Java
int[] dp = new int[10];
Arrays.fill(dp, Integer.MAX_VALUE);  // DP 数组初始化为"无穷大"

// 区间填充：只填 [2, 6)
int[] arr = new int[8];
Arrays.fill(arr, 2, 6, -1);
System.out.println(Arrays.toString(arr));  // [0, 0, -1, -1, -1, -1, 0, 0]
fill 的浅拷贝陷阱（对象数组）： Arrays.fill(objArr, new Object()) 会让所有元素指向同一个对象，修改一个等于修改全部。 对象数组需要用循环单独 new，或用 Arrays.setAll()（JDK 8+）。

与 ArrayList 的联系：Arrays 就在 JDK 内部

ArrayList 内部就是一个 Object[]，当需要扩容时，它调用的正是：

ArrayList 扩容源码片段（JDK 11）Java
// java.util.ArrayList 第 236 行（简化版）
elementData = Arrays.copyOf(elementData, newCapacity);
// 扩容 1.5 倍后，直接用 Arrays.copyOf 把老数据搬到新数组

同样，Collections.sort(list) 在 JDK 8+ 内部实际调用的是 list.sort()，最终也走到 Arrays.sort() 的 TimSort 逻辑。Arrays 是整个 Java 集合框架的底层支撑之一。

方法分类树

Arrays 工具类 ├── 排序 │ ├── sort(arr) │ ├── sort(arr, from, to) │ └── sort(arr, comparator) ← 对象数组专用 ├── 查找 │ ├── binarySearch(arr, key) │ └── binarySearch(arr, from, to, key) ├── 复制 │ ├── copyOf(arr, newLen) │ └── copyOfRange(arr, from, to) ├── 填充 │ ├── fill(arr, val) │ └── fill(arr, from, to, val) ├── 比较 │ ├── equals(a, b) │ └── deepEquals(a, b) ├── 打印 │ ├── toString(arr) │ └── deepToString(arr) └── 转换 ├── asList(T... a) ← 固定 List 视图 ├── stream(arr) ← JDK 8+ └── setAll(arr, generator) ← JDK 8+，函数式填充

面试高频考点

  • Arrays.sort 用了什么算法？ 基本类型用双轴快排（不稳定），对象用 TimSort（稳定）。追问：为什么对象要稳定排序？
  • Arrays.asList 的 List 能否增删？ 不能，UnsupportedOperationException。如何转为可变 List？new ArrayList<>(Arrays.asList(...))。
  • binarySearch 返回负数代表什么？ 未找到，返回值为 -(插入点)-1，插入点 = -返回值-1。
  • int[] 传入 Arrays.asList 会发生什么？ 返回 List<int[]>，size=1，不是你期望的 List<Integer>。
  • Arrays.equals 与 deepEquals 的区别？ equals 只比较一维，deepEquals 递归比较多维数组。
  • Arrays.copyOf 底层是什么？ System.arraycopy()，native 方法，直接内存操作，性能远高于 for 循环。

常见陷阱汇总

陷阱 1：binarySearch 前忘记排序
对未排序数组二分查找，不报错但结果随机错误，极难排查。
陷阱 2：asList 传入基本类型数组
Arrays.asList(new int[]{1,2,3}) 得到 List<int[]>，不是 List<Integer>。
陷阱 3：对象数组 fill 共享引用
Arrays.fill(arr, new int[3]) 让所有行指向同一个 int[]，修改任意一行都会影响全部。
陷阱 4：asList 视图与原数组共享存储
list.set(0, x) 会同步修改原始数组，反之亦然。
最佳实践：调试时优先用 Arrays.toString()；需要可变 List 时用 new ArrayList<>(Arrays.asList(...))；大数据量排序后查找时用 binarySearch 而非线性遍历。
