13. break / continue / label

循环体内的"紧急出口"与"跳过阀门"——以及 Java 独有的标签机制，让你在多层嵌套中精准控制跳转目标。

1. 为什么需要这三个关键字

循环的本意是"重复执行直到条件不满足"，但现实逻辑往往需要提前中断或跳过某次迭代。如果没有 break / continue，我们只能用额外的布尔标志位绕路——代码噪声极大，意图也不直接。

可以把循环想象成一条流水线传送带：

  • break：传送带紧急停机，整条线停止。
  • continue：当前这件产品不合格，直接跳过检验台，传送带继续转。
  • label（标签）：工厂有多条嵌套流水线，标签让你指定停机或跳过的是"哪一层"的传送带。

2. break — 跳出当前循环或 switch

break 会立即终止最内层的 for、while、do-while 或 switch，执行流跳到该结构之后的第一条语句。

2.1 在 for 循环中使用

BreakDemo.java
for (int i = 0; i < 10; i++) {
    if (i == 5) break;   // 找到目标，不需要再遍历
    System.out.println(i);
}
// 输出: 0 1 2 3 4（i=5 时 break，循环体后续不再执行）

2.2 在 switch 中使用（回顾）

switch 的每个 case 本质上是"跳转表"，不加 break 会发生 fall-through（贯穿）——这是 Java 从 C 继承来的设计，初学者最常踩的坑之一。

SwitchBreak.java
int day = 2;
switch (day) {
    case 1:
        System.out.println("周一");
        break;   // 必须有，否则会继续执行 case 2
    case 2:
        System.out.println("周二");
        break;
    default:
        System.out.println("其他");
}
陷阱：忘记 switch 里的 break
缺少 break 时，匹配到某个 case 后会一路执行到 switch 末尾，输出所有后续 case 的代码。Java 14+ 的 switch 表达式（-> 语法）默认不 fall-through，推荐新代码使用。

3. continue — 跳过本次迭代

continue 不退出循环，只跳过当次循环体中剩余的语句，直接进入下一轮的条件判断（while/do-while）或更新表达式（for 的第三部分）。

ContinueDemo.java
// 只打印奇数，偶数跳过
for (int i = 1; i <= 10; i++) {
    if (i % 2 == 0) continue;  // 偶数：跳过 println，i++ 照常执行
    System.out.println(i);
}
// 输出: 1 3 5 7 9

continue 在 while 中的细节

在 while 循环中，continue 会跳回条件判断处。如果更新计数器的语句在 continue 之后，会导致死循环！

反例 — 死循环
int i = 0;
while (i < 5) {
    if (i == 3) continue;  // ❌ i 永远等于 3，死循环！
    System.out.println(i);
    i++;                   // continue 跳过了这里
}
正例 — 更新在 continue 之前
int i = 0;
while (i < 5) {
    i++;                   // ✅ 先更新，再 continue
    if (i == 3) continue;
    System.out.println(i);
}
// 输出: 1 2 4 5

4. label — 跳出指定层级的循环

当循环嵌套超过一层，普通的 break/continue 只影响最内层。Java 提供了 标签（label） 语法，让你把跳转目标指向外层循环。

语法：在目标循环前写 标签名:（冒号结尾），然后用 break 标签名 或 continue 标签名。

4.1 break label — 跳出外层循环

BreakLabel.java
// 在二维矩阵中查找目标值，找到后立即跳出双层循环
int[][] matrix = {{1,2},{3,4},{5,6}};
int target = 4;
boolean found = false;

outer:                              // 给外层循环打标签
for (int row = 0; row < matrix.length; row++) {
    for (int col = 0; col < matrix[row].length; col++) {
        if (matrix[row][col] == target) {
            found = true;
            break outer;           // 直接跳出外层循环，而非只退出内层
        }
    }
}
System.out.println("found: " + found);  // found: true

4.2 continue label — 跳过外层循环的本次迭代

ContinueLabel.java
// 外层遍历行，内层遍历列；某行含负数则整行跳过
int[][] data = {{1,2},{-1,3},{4,5}};

rows:
for (int[] row : data) {
    for (int val : row) {
        if (val < 0) continue rows;  // 跳到 rows 的下一次迭代（下一行）
    }
    // 只有整行都是非负数才会执行到这里
    System.out.println(java.util.Arrays.toString(row));
}
// 输出: [1, 2]   [4, 5]   （含 -1 的行被跳过）

6. 与其他知识的联系

6.1 与 return 的区别

break 只退出循环，return 退出整个方法。在某些场景下可以用 return 替代带标签的 break，把循环逻辑抽取为私有方法往往更清晰：

更好的写法：抽方法代替 label
private boolean contains(int[][] matrix, int target) {
    for (int[] row : matrix)
        for (int v : row)
            if (v == target) return true;  // 直接返回，无需 label
    return false;
}
最佳实践：绝大多数 break label 场景都可以通过抽取方法 + return 来消除，代码可读性更好。标签语法在 Java 中合法但罕见，见到时要能读懂即可。

6.2 与 Stream / 函数式的对应

Java 8 引入的 Stream API 中没有 break/continue 的直接等价物，但提供了语义相同的终止操作：

  • findFirst() / anyMatch() — 找到即停，对应 break
  • filter() — 过滤不满足条件的元素，对应 continue

6.3 字节码层面

break 和 continue 在字节码中都被编译为 goto 指令，只是跳转目标地址不同。Java 语言层禁用了裸 goto（保留关键字但不可用），但编译器在内部大量使用它。

7. 常见陷阱

陷阱 1：break 只跳出最内层
多层循环时，break 只退出直接包含它的那一层。如果想跳出外层，必须用标签或改用方法返回。
陷阱 2：continue 在 while 中可能造成死循环
确保计数器/状态更新语句不会被 continue 跳过（见第 3 节反例）。
陷阱 3：switch 忘写 break 导致 fall-through
Java 编译器不会对缺少 break 发出警告（IntelliJ / CheckStyle 会提示）。新代码推荐使用 Java 14+ 的 switch 表达式（case X -> ...）彻底规避。
陷阱 4：标签不是 goto
break label 只能跳转到包含该语句的外层循环/块，不能任意向前跳，也不能跳进另一个代码块。

8. 面试考点

  • Q：break 和 return 的区别？
    break 退出当前循环结构，方法继续执行；return 退出整个方法并可返回值。
  • Q：continue 在 for 和 while 中的跳转目标有何不同？
    for 循环跳到更新表达式（i++）；while / do-while 跳到条件判断处。
  • Q：Java 有 goto 吗？
    goto 是 Java 保留关键字，但编译器不允许使用。替代方案是带标签的 break/continue。
  • Q：如何不使用 label 跳出双层循环？
    将双层循环抽取为独立方法，用 return 代替 break label，这是更推荐的做法。
  • Q：switch 的 fall-through 有没有合理使用场景？
    有，比如多个 case 共享同一逻辑时：case 1: case 2: doSomething(); break;。这是故意 fall-through，建议加注释说明。

9. 综合练习

用 break、continue、标签三种技术分别实现：在 5×5 矩阵中，跳过对角线元素，打印其余元素；遇到值为 13 时整体终止。

MatrixPrint.java
int[][] grid = new int[5][5];
int val = 1;
for (int r = 0; r < 5; r++)
    for (int c = 0; c < 5; c++)
        grid[r][c] = val++;  // 填入 1~25

print:
for (int r = 0; r < 5; r++) {
    for (int c = 0; c < 5; c++) {
        if (grid[r][c] == 13) break print;   // 遇到 13 终止全部
        if (r == c) continue;                  // 对角线跳过
        System.out.printf("%3d", grid[r][c]);
    }
    System.out.println();
}
