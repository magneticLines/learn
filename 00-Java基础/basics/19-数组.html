19. 一维数组与多维数组

数组是 Java 中最基础的数据结构——连续内存块、固定长度、O(1) 随机访问。理解它的内存模型，是后续学习集合、JVM 内存管理的根基。

为什么需要数组？

想象你要记录 100 个学生的成绩。如果用独立变量，需要写 score1、score2……score100，不但繁琐，还无法用循环批量处理。数组的本质是：用一个名字管理一组同类型的数据，并通过下标（索引）精确访问每一项。

更深层的原因在于 CPU 缓存友好性。数组在堆内存中连续分配，访问 arr[i] 时，CPU 会将相邻元素一并加载到 L1/L2 缓存，后续访问几乎不产生缓存未命中，性能极高。链表恰恰相反——节点散落在堆中，缓存命中率低。

设计决策：Java 数组是对象（存储在堆上），而不像 C 那样是裸指针。这带来了类型安全检查、length 属性和 NullPointerException 保护，代价是多一次堆分配。

一维数组：声明、创建、初始化

三种写法对比

Java声明 & 初始化
// ① 先声明，再用 new 创建（最常见）
int[] scores = new int[5];   // 5 个元素，默认值 0

// ② 静态初始化：已知数据时直接写值
int[] primes = {2, 3, 5, 7, 11};

// ③ 等价的长写法（匿名数组，常用于方法传参）
int[] primes2 = new int[] {2, 3, 5};

// ❌ 老式 C 风格，Java 中合法但不推荐
int scores2[] = new int[5];

遍历：for vs for-each

Java遍历对比
int[] arr = {10, 20, 30, 40};

// 普通 for：需要下标时使用（如修改元素、逆序、跳步）
for (int i = 0; i < arr.length; i++) {
    arr[i] *= 2;   // 可以修改元素
}

// 增强 for（for-each）：只读遍历，语义更清晰
for (int val : arr) {
    System.out.println(val);
    // val = 99; ← 这行只改局部变量，不影响数组！
}
陷阱：for-each 无法修改基本类型元素。val 是数组元素的副本，修改它对原数组无效。只有元素是引用类型时，才能通过 val.setXxx() 改变对象内部状态。

内存模型：数组在 JVM 中如何存放

执行 int[] arr = new int[4] 时，JVM 做了两件事：

  1. 栈帧中创建局部变量 arr，存储一个引用（reference），占 4 或 8 字节。
  2. 堆上分配一块连续内存，包含对象头（mark word + 类指针 + 数组长度）+ 4 个 int 元素（共 16 字节）。
栈 堆 ┌──────────┐ ┌─────────────────────────────┐ │ arr ────┼──────────────▶│ [对象头 | length=4 | 0 | 0 | 0 | 0] │ └──────────┘ └─────────────────────────────┘ ▲ 连续内存，下标访问 = 基地址 + i×元素大小

这也是为什么 arr[i] 是 O(1)：只需一次加法和一次内存读取。而 arr.length 没有性能代价，因为长度就存在对象头里，不需要遍历计算。

与 C 的区别：Java 数组越界时 JVM 会抛 ArrayIndexOutOfBoundsException，这是运行时的边界检查（JIT 会优化掉大部分热点处的检查）。C 数组越界是未定义行为，会踩内存，是严重安全漏洞的根源。

数组是引用类型：赋值与复制的陷阱

Java浅拷贝 vs 深拷贝
int[] a = {1, 2, 3};

// ❌ 错误：b 和 a 指向同一块内存
int[] b = a;
b[0] = 99;
System.out.println(a[0]);  // 输出 99！a 也被改了

// ✅ 正确：用 Arrays.copyOf 或 System.arraycopy 复制
int[] c = Arrays.copyOf(a, a.length);
c[0] = 42;
System.out.println(a[0]);  // 仍然是 99，互不影响

// System.arraycopy：性能最佳（底层 native 实现）
int[] d = new int[a.length];
System.arraycopy(a, 0, d, 0, a.length);
// 参数：(源数组, 源起始位, 目标数组, 目标起始位, 复制长度)

多维数组：本质是"数组的数组"

Java 没有真正的二维数组。int[][] matrix 是一个数组，其中每个元素是另一个 int[] 引用。这与 C 的二维数组（单块连续内存）有本质区别。

创建方式

Java二维数组
// ① 规则矩阵：3 行 4 列
int[][] matrix = new int[3][4];

// ② 静态初始化
int[][] grid = {
    {1, 2, 3},
    {4, 5, 6},
    {7, 8, 9}
};

// ③ 锯齿数组（每行长度不同）：先只指定行数
int[][] jagged = new int[3][];
jagged[0] = new int[1];   // 第 0 行：1 个元素
jagged[1] = new int[3];   // 第 1 行：3 个元素
jagged[2] = new int[2];   // 第 2 行：2 个元素（如杨辉三角）

多维数组的内存结构

堆内存 matrix（int[][]） │ ├─▶ int[] row0 [ 1 | 2 | 3 | 4 ] ← 独立的连续内存块 ├─▶ int[] row1 [ 5 | 6 | 7 | 8 ] ← 独立的连续内存块 └─▶ int[] row2 [ 9 |10 |11 |12 ] ← 独立的连续内存块 行与行之间在堆上不一定连续！

这意味着：按行遍历（row-major）缓存友好，按列遍历（column-major）会频繁跨内存块，性能更差。大矩阵运算时遍历顺序影响性能，这是很多面试官喜欢考的细节。

遍历二维数组

Java嵌套循环遍历
int[][] grid = {{1,2,3},{4,5,6}};

// grid.length      = 行数 = 2
// grid[0].length   = 第 0 行的列数 = 3
for (int i = 0; i < grid.length; i++) {
    for (int j = 0; j < grid[i].length; j++) {
        System.out.printf("%3d", grid[i][j]);
    }
    System.out.println();
}

// for-each 写法（每次 row 是 int[]）
for (int[] row : grid) {
    for (int val : row) {
        System.out.print(val + " ");
    }
}

实战：数组作为方法参数与返回值

Java方法传参：传的是引用
// 传入数组 = 传入引用，方法内修改会影响原数组
static void doubleAll(int[] arr) {
    for (int i = 0; i < arr.length; i++)
        arr[i] *= 2;   // 修改了堆上的实际数据
}

// 返回新数组（函数式风格，不修改原数据）
static int[] doubled(int[] arr) {
    int[] result = new int[arr.length];
    for (int i = 0; i < arr.length; i++)
        result[i] = arr[i] * 2;
    return result;
}

// 可变参数 varargs 底层就是数组
static int sum(int... nums) {   // nums 就是 int[]
    int total = 0;
    for (int n : nums) total += n;
    return total;
}
sum(1, 2, 3);       // 编译器自动打包成 new int[]{1,2,3}

数组的局限性与演进

特性数组ArrayList
长度固定，创建后不可变动态扩容（1.5 倍）

末尾插入不支持（需手动扩容）均摊 O(1)

为什么 ArrayList 不能用泛型数组？Java 泛型在运行时类型擦除，new T[10] 在运行时不知道 T 是什么，无法分配正确的内存。所以 ArrayList 内部用的是 Object[]，读取时做强制转换。

与 JDK 源码的联系

理解数组后，很多 JDK 类的实现就一目了然：

  • ArrayList：底层 Object[] elementData，扩容时调 Arrays.copyOf 生成 1.5 倍大的新数组。
  • HashMap：底层 Node[] table（哈希桶数组），每个桶挂链表或红黑树。
  • String：内部 byte[] value（Java 9+），charAt(i) 就是数组下标访问。
  • ArrayDeque：循环数组，首尾插删 O(1)，比 LinkedList 缓存友好得多。

常见陷阱汇总

陷阱 1：下标越界
Java 数组下标从 0 到 length - 1。arr[arr.length] 必然抛 ArrayIndexOutOfBoundsException。新手最常犯的错是写 i <= arr.length。
陷阱 2：引用赋值误当复制
int[] b = a 不复制数据，只复制引用。需要独立副本时务必用 Arrays.copyOf 或 System.arraycopy。
陷阱 3：多维数组 null 行
int[][] m = new int[3][] 创建后，m[0]、m[1]、m[2] 全是 null。访问 m[0][0] 会抛 NullPointerException，而不是越界异常。
陷阱 4：打印数组直接输出地址
System.out.println(arr) 输出的是 [I@1b6d3586（类型描述符 + hashCode），不是内容。要打印内容请用 Arrays.toString(arr) 或 Arrays.deepToString(matrix)。

面试高频考点

  1. 数组和 ArrayList 的区别？何时用数组？——大小固定、基本类型、性能敏感（如算法题、底层实现）时用数组；其他场景优先 ArrayList。
  2. 如何判断两个数组内容相等？——不能用 ==（比的是引用），也不能用 equals（Object 默认实现比引用）；要用 Arrays.equals(a, b)，多维数组用 Arrays.deepEquals。
  3. Java 多维数组在内存上连续吗？——不连续，每一行是独立的堆对象，这与 C 的二维数组不同。
  4. 数组能存不同类型吗？——基本类型数组不行；引用类型数组可以用父类/接口类型，如 Object[] arr 可存任意对象，但有协变陷阱（String[] s = new String[3]; Object[] o = s; o[0] = 42; 运行时抛 ArrayStoreException）。
  5. System.arraycopy 和 Arrays.copyOf 的区别？——前者是 native 方法，可以指定范围和目标位置；后者内部调前者，同时支持截断或填充默认值（常用于扩容）。
