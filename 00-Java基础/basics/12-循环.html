12. 循环：for / while / do-while

三种循环结构的设计哲学、使用场景、字节码差异与高频陷阱全解析。

为什么需要三种循环？

几乎所有编程语言都提供至少两种循环：计数驱动（知道要跑几次）和条件驱动（跑到某个状态为止）。 Java 同时提供三种，是为了让代码意图直接映射到语法，提升可读性，减少引入 bug 的机会。

循环结构 ├── for → 已知迭代次数 / 有明确的初始化+步进 ├── while → 先判断后执行，迭代次数不确定 └── do-while → 先执行后判断，至少执行一次

三种结构在图灵完备性上等价——你可以用任意一种模拟另外两种——但选错了结构，代码会让人多读一遍才懂，这就是隐性成本。

for 循环

语法结构

Javafor 循环语法
for (/* ① 初始化 */; /* ② 条件 */; /* ③ 步进 */) {
    // ④ 循环体
}

执行顺序：① → ② → ④ → ③ → ② → ④ → ③ → …，直到 ② 为 false。 注意 ① 只执行一次，③ 在每次循环体之后才执行。

典型用法：遍历数组

Java正向 / 逆向遍历
int[] arr = {10, 20, 30, 40};

// 正向：i 从 0 到 length-1
for (int i = 0; i < arr.length; i++) {
    System.out.println(arr[i]);
}

// 逆向：索引反过来，适合"从末尾删除元素"等场景
for (int i = arr.length - 1; i >= 0; i--) {
    System.out.println(arr[i]);
}

增强 for（for-each）

JDK 5 引入，专为遍历数组或实现了 Iterable 接口的集合而设计。 编译器会将其转换为迭代器调用（对集合）或普通下标循环（对数组）。

Javafor-each 与其等价展开
List<String> names = List.of("Alice", "Bob", "Carol");

// 语法糖写法 —— 清晰表达"我只是想遍历"
for (String name : names) {
    System.out.println(name);
}

// 编译器等价展开（对 Iterable）
Iterator<String> it = names.iterator();
while (it.hasNext()) {
    String name = it.next();
    System.out.println(name);
}
for-each 的限制：无法获取当前下标、无法在遍历过程中修改集合（会抛 ConcurrentModificationException）、无法从中间逆向遍历。需要这些能力时，回退到普通 for 或用迭代器的 remove()。

多变量 for / 空段

Java高级用法：多变量 / 无限循环
// 同时移动两个指针（双指针技巧）
for (int lo = 0, hi = arr.length - 1; lo < hi; lo++, hi--) {
    // 交换 arr[lo] 和 arr[hi]
    int tmp = arr[lo];
    arr[lo] = arr[hi];
    arr[hi] = tmp;
}

// 三段全省略 = 无限循环（等价于 while(true)，用于服务器主循环）
for (;;) {
    // 必须在内部用 break 退出，否则死循环
}

while 循环

结构与语义

Javawhile 语法
while (/* 条件 */) {
    // 循环体
    // 若条件一开始就为 false，循环体一次也不执行
}

while 的语义是："只要还满足条件，就继续"。 适合循环次数未知、由外部状态或输入驱动的场景。

典型用法：读取输入流

Java逐行读文件
try (BufferedReader br = new BufferedReader(new FileReader("data.txt"))) {
    String line;
    // 不知道文件有几行，用 while 表达"读到没有为止"
    while ((line = br.readLine()) != null) {
        System.out.println(line);
    }
}
设计哲学：赋值表达式本身有返回值（返回被赋的值），所以 (line = br.readLine()) != null 既完成了赋值又完成了判断，是 Java 惯用的 IO 循环写法。

while 实现二分查找

Java二分查找（不知道要折几次）
int binarySearch(int[] arr, int target) {
    int lo = 0, hi = arr.length - 1;
    while (lo <= hi) {        // 搜索空间不为空时继续
        int mid = lo + (hi - lo) / 2;  // 防止 (lo+hi) 溢出
        if (arr[mid] == target) return mid;
        if (arr[mid] < target)  lo = mid + 1;
        else                        hi = mid - 1;
    }
    return -1;
}

do-while 循环

结构与语义

Javado-while 语法
do {
    // 循环体（至少执行一次）
} while (/* 条件 */);  // 注意末尾的分号！

语义是："先做一次，再决定要不要继续"。 适合至少需要执行一次，且执行后才能得到判断依据的场景。

典型用法：菜单交互 / 输入校验

Java用户输入校验（至少要问一次）
Scanner sc = new Scanner(System.in);
int age;
do {
    System.out.print("请输入年龄（1-150）：");
    age = sc.nextInt();
    // 先拿到输入，才能判断合不合法，所以"先做后判"是自然选择
} while (age < 1 || age > 150);
System.out.println("年龄有效：" + age);
生活类比：自动贩卖机。先投币（执行一次），再判断是否足够（条件），不够就继续投。用 while 反而得写"假设初始值不合法"的哑值来凑条件，不如 do-while 直接。

三种循环对比

特性 for while do-while
条件检查时机 先检查 先检查 后检查
最少执行次数 0 次 0 次 1 次
典型场景 已知次数 / 下标迭代 条件未知 / IO 驱动 至少执行一次 / 菜单
变量作用域 初始化变量局部于 for 块 变量在 while 外声明 变量在 do 外声明
可读性信号 "迭代 N 次" "直到满足才停" "先做一次再说"

字节码视角：三种循环的底层实现

JVM 没有专门的"循环指令"，所有循环都被编译成条件跳转（goto + if_icmpXX 等）。 三种循环的字节码模式如下：

伪字节码结构对比
// for (int i=0; i<n; i++) { body }
  iconst_0          // ① 初始化 i=0
  istore_1
LOOP_START:
  iload_1           // ② 检查条件
  iload_2
  if_icmpge END     // i>=n 则跳出
  // body
  iinc 1 1          // ③ 步进 i++
  goto LOOP_START
END:

// do { body } while(cond)
LOOP_START:
  // body
  // 计算 cond
  ifne LOOP_START   // 条件为真则回跳（注意：检查在底部）
END:

do-while 的字节码比 while 少一次跳转（没有初始的 goto 到检查处）， 因此在极端性能场景下，手动将 while 改写为 do-while（加一次前置检查）是一种低层优化手段。 不过现代 JIT 通常会自动完成此优化，日常开发无需手动做。

常见陷阱

陷阱 1：循环变量溢出导致死循环

Java反例 → 分析 → 正例
// 反例：用 byte 做循环变量，byte 最大值 127，i++ 后变成 -128，永远 < 200
for (byte i = 0; i < 200; i++) {  // ⚠️ 死循环！
    System.out.println(i);
}

// 正例：改用 int，范围足够
for (int i = 0; i < 200; i++) {  // ✅ 正常终止
    System.out.println(i);
}

陷阱 2：浮点数做循环条件

Java浮点精度陷阱
// 反例：0.1 无法用二进制浮点精确表示，累加误差可能跳过 1.0
for (double d = 0.0; d != 1.0; d += 0.1) {  // ⚠️ 可能死循环
    System.out.println(d);
}

// 正例：用整数控制次数，在循环体内换算成浮点
for (int i = 0; i < 10; i++) {          // ✅ 精确控制次数
    double d = i * 0.1;
    System.out.println(d);
}

陷阱 3：for-each 遍历时修改集合

JavaConcurrentModificationException
List<String> list = new ArrayList<>(List.of("a", "b", "c"));

// 反例：在 for-each 里 remove，触发 modCount 校验 → 抛异常
for (String s : list) {
    if (s.equals("b")) list.remove(s);  // ⚠️ ConcurrentModificationException
}

// 正例 1：用 Iterator 的 remove()
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    if (it.next().equals("b")) it.remove();  // ✅ 安全
}

// 正例 2（Java 8+）：removeIf 语义最清晰
list.removeIf(s -> s.equals("b"));          // ✅ 推荐

陷阱 4：do-while 末尾忘写分号

Java语法易错点
do {
    count++;
} while (count < 10)   // ⚠️ 少了分号，编译错误

do {
    count++;
} while (count < 10);  // ✅ 必须有分号

陷阱 5：空循环体的歧义

Java空语句陷阱
// 反例：for 后面多了一个分号，循环体变成空语句，System.out 只执行一次
for (int i = 0; i < 5; i++);        // ⚠️ 这个分号就是完整的循环体（空语句）
    System.out.println("只打印一次");  // 这行在循环外！

// 正例：始终用大括号，即使只有一行循环体
for (int i = 0; i < 5; i++) {       // ✅ 大括号消除歧义
    System.out.println("打印 5 次");
}

与其他知识的联系

Stream API 与循环的关系（Java 8+）

Java 8 的 Stream 提供了一种声明式循环风格，底层仍是迭代，但把"怎么遍历"隐藏起来， 让你只表达"想要什么"。两者不是替代关系，而是各有适用场景。

Java命令式循环 vs 声明式 Stream
List<Integer> nums = List.of(1,2,3,4,5);

// 命令式：明确的 for 循环
int sum = 0;
for (int n : nums) {
    if (n % 2 == 0) sum += n;
}

// 声明式：Stream 隐藏了遍历细节，表达意图更直接
int sum2 = nums.stream()
              .filter(n -> n % 2 == 0)
              .mapToInt(Integer::intValue)
              .sum();

// 选择依据：需要 break/continue/修改外部变量/异常处理 → 用 for/while
//           纯转换/过滤/聚合且无副作用 → Stream 可读性更高

与并发的连接：volatile 与循环终止

多线程场景下，循环的退出条件如果涉及跨线程共享的标志位，必须用 volatile 修饰， 否则 JIT 可能将"反复读取的不变量"优化成寄存器缓存，导致循环永远看不到另一个线程设置的 false。

Javavolatile 保证循环可见性
// 反例：JIT 可能将 running 缓存，另一线程设 false 不生效
private boolean running = true;

// 正例：volatile 保证每次读都从主存取，禁止 JIT 缓存
private volatile boolean running = true;

public void run() {
    while (running) {  // 另一线程调用 stop() 后，此循环能正确退出
        doWork();
    }
}
public void stop() { running = false; }

面试高频考点

  • 三种循环的选择原则：能说清楚"先判断 vs 后判断"、"已知次数 vs 条件驱动"即可。
  • for-each 底层原理：数组 → 下标循环；Iterable → Iterator。能展开语法糖。
  • for-each 为什么不能删除元素：modCount 检查机制，展开后等价代码一看即知。
  • 死循环的几种写法：for(;;)、while(true)，以及它们的使用场景（服务器主循环、事件泵）。
  • 循环展开（Loop Unrolling）：JIT 会对短小的循环体做展开优化，减少条件跳转次数，无需手动操作。
  • i++ 与 ++i 在 for 循环步进中的区别：单独作为语句时无区别，两者编译出相同字节码。区别仅在作为表达式使用时（如 int j = i++）。
  • 标签（label）与嵌套循环的 break/continue（下一节详讲）。
面试陷阱题："下面代码输出什么？"
for (int i = 0; i < 3; i++) { ... } 结束后 i 的值是多少？
答：编译错误——i 的作用域仅限于 for 块内，外部无法访问。这也是 for 循环的优势之一：初始化变量不会污染外部作用域。

综合示例：打印乘法表

Java嵌套 for 循环 + 格式化输出
// 外层控制行（被乘数），内层控制列（乘数）
for (int i = 1; i <= 9; i++) {
    for (int j = 1; j <= i; j++) {       // j <= i：下三角，避免重复
        System.out.printf("%d×%d=%-3d", j, i, i * j);
    }
    System.out.println();  // 每行结束换行
}
