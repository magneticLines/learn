15. 枚举 enum

用有限、具名的常量集合替代魔法数字，同时赋予每个常量行为——enum 是披着类外衣的单例集合。

为什么需要 enum

在 enum 出现之前（Java 1.4 及更早），枚举值通常用 int 常量表示：

Java — 旧式 int 常量（反例）
public class Direction {
    public static final int NORTH = 0;
    public static final int SOUTH = 1;
    public static final int EAST  = 2;
    public static final int WEST  = 3;
}

// 调用方：传入任意 int 都合法，编译器不报错
move(999);  // ← 编译通过，运行时才出错，难以排查

这种方式有三个核心缺陷：

  • 类型不安全：任意 int 都能传进去，编译器无法拦截
  • 没有命名空间：不同枚举的值可能冲突，需要加前缀区分
  • 打印无意义：日志里只能看到 0，看不出含义

Java 5 引入 enum 关键字，本质上是 编译器帮你生成的一个继承自 java.lang.Enum 的 final 类，每个常量都是该类的一个 public static final 实例（单例），彻底解决了上述问题。

设计哲学：enum 把"有限集合"这件事从约定变成了语言层面的约束。编译器能静态检查，IDE 能自动补全，switch 能穷举——这是用类型系统解决业务问题的经典范例。

基础语法：最简单的枚举

Java — 基础枚举定义与使用
public enum Direction {
    NORTH, SOUTH, EAST, WEST
}

// 使用
Direction d = Direction.NORTH;
System.out.println(d);           // NORTH（自动调用 name()）
System.out.println(d.ordinal());  // 0（声明顺序，从 0 开始）
System.out.println(d.name());     // "NORTH"（常量名字符串）

// 字符串 → 枚举（大小写敏感，不存在时抛 IllegalArgumentException）
Direction d2 = Direction.valueOf("SOUTH");

// 所有常量数组
Direction[] all = Direction.values();

enum 继承体系

java.lang.Object
└── java.lang.Enum<E> ← 所有 enum 的父类（编译器自动加）
    └── Direction ← 你写的枚举，final class，不可再继承
        ├── NORTH ← public static final Direction 实例
        ├── SOUTH
        ├── EAST
        └── WEST

由于 Java 是单继承，enum 已经隐式继承了 Enum，所以 enum 不能再 extends 其他类，但可以 implements 接口。

带字段和方法的枚举

枚举常量可以携带数据，每个常量在声明时通过构造器传入参数。这是 enum 最强大的地方——每个常量都是一个完整的对象。

Java — 带字段的枚举（行星质量示例）
public enum Planet {
    // 每个常量调用构造器，传入质量(kg)和半径(m)
    MERCURY(3.303e+23, 2.4397e6),
    VENUS  (4.869e+24, 6.0518e6),
    EARTH  (5.976e+24, 6.37814e6);

    private final double mass;    // kg
    private final double radius;  // m

    // 枚举构造器必须是 private（或包级私有），不可 public
    Planet(double mass, double radius) {
        this.mass   = mass;
        this.radius = radius;
    }

    static final double G = 6.67300E-11;

    // 枚举可以定义方法
    double surfaceGravity() {
        return G * mass / (radius * radius);
    }

    double surfaceWeight(double otherMass) {
        return otherMass * surfaceGravity();
    }
}

// 使用：在地球上 75kg，算各星球体重
double earthWeight = 75.0;
for (Planet p : Planet.values()) {
    System.out.printf("%-8s %.2f%n", p, p.surfaceWeight(earthWeight));
}

常量特定类主体（每个常量有自己的行为）

枚举中可以定义抽象方法，每个常量必须提供各自的实现。这实际上是 策略模式的枚举版本，把多个 if-else/switch 变成多态：

Java — 运算符枚举（Strategy 模式）
public enum Operation {
    PLUS("+") {
        @Override
        public double apply(double x, double y) { return x + y; }
    },
    MINUS("-") {
        @Override
        public double apply(double x, double y) { return x - y; }
    },
    TIMES("*") {
        @Override
        public double apply(double x, double y) { return x * y; }
    },
    DIVIDE("/") {
        @Override
        public double apply(double x, double y) { return x / y; }
    };

    private final String symbol;

    Operation(String symbol) { this.symbol = symbol; }

    // 抽象方法：每个常量必须实现
    public abstract double apply(double x, double y);

    @Override
    public String toString() { return symbol; }
}

// 调用：无需 switch，直接多态分发
for (Operation op : Operation.values()) {
    System.out.printf("3 %s 2 = %.0f%n", op, op.apply(3, 2));
}
为什么优于 switch：当新增运算符时，只需在 enum 末尾加一个常量并实现抽象方法，编译器会强制你不遗漏。而 switch 是散落在代码各处的，遗漏一个 case 只有运行时才会暴露。

enum 实现接口

枚举不能继承类，但可以实现接口。这是让 enum 参与多态体系的方式：

Java — enum implements 接口
public interface Printable {
    void print();
}

public enum Color implements Printable {
    RED, GREEN, BLUE;

    @Override
    public void print() {
        System.out.println("Color: " + this.name());
    }
}

// enum 常量可以当接口类型使用
Printable p = Color.RED;
p.print();  // Color: RED

EnumSet 与 EnumMap：性能特化的集合

java.util.EnumSet 和 java.util.EnumMap 是专门为枚举设计的集合类，底层用位向量（long）实现，比 HashSet/HashMap 快一个数量级，且内存占用极小。

Java — EnumSet 与 EnumMap
import java.util.*;

enum Day { MON, TUE, WED, THU, FRI, SAT, SUN }

// EnumSet：底层是单个 long 位掩码（最多 64 个常量）
EnumSet<Day> weekend = EnumSet.of(Day.SAT, Day.SUN);
EnumSet<Day> weekdays = EnumSet.complementOf(weekend);
EnumSet<Day> all      = EnumSet.allOf(Day.class);
EnumSet<Day> mid      = EnumSet.range(Day.TUE, Day.THU);

// EnumMap：key 是 enum，底层是对象数组（按 ordinal 索引）
EnumMap<Day, String> schedule = new EnumMap<>(Day.class);
schedule.put(Day.MON, "站会");
schedule.put(Day.FRI, "回顾");
何时用 EnumSet 代替 Set<Enum>：只要集合的元素类型是枚举，就应该无脑选 EnumSet。它的 add/contains/remove 都是 O(1) 位运算，迭代按声明顺序，没有哈希碰撞的烦恼。

switch 与 enum

enum 是 switch 的天然搭档。Java 14+ 的 switch 表达式配合 enum，穷举性检查更完善：

Java — switch 表达式穷举 enum（Java 14+）
Day today = Day.WED;

// switch 表达式：必须穷举所有常量（或加 default），编译器保证
String type = switch (today) {
    case SAT, SUN -> "休息日";
    case MON, TUE, WED, THU, FRI -> "工作日";
    // 无需 default，因为已穷举 Day 所有 7 个常量
};

// 旧式 switch 语句（Java 8 也支持）
switch (today) {
    case MON:
        System.out.println("周一！打工人");
        break;
    default:
        System.out.println("其他天");
}

Enum 内置方法速查

方法返回类型说明
name()String返回常量声明时的名字，与 toString() 默认相同，但 name() 是 final，不可覆盖
ordinal()int声明顺序（从 0 起），不要用它做业务逻辑，改变顺序会破坏语义
toString()String默认返回 name()，可以覆盖以返回更友好的字符串
valueOf(String)E静态方法，字符串 → 枚举，不匹配时抛 IllegalArgumentException
values()E[]静态方法，返回所有常量的数组（每次调用都克隆一个新数组）
compareTo(E)int按 ordinal 比较大小
getDeclaringClass()Class<E>返回枚举类型的 Class 对象，比 getClass() 在匿名常量体时更可靠
陷阱：不要依赖 ordinal() 做业务！如果在数据库里存了 ordinal 值（如 0 代表 NORTH），后来在 enum 中间插入新常量，所有 ordinal 都会偏移，数据就错了。应该存 name() 字符串，或自定义 code 字段。

与其他知识的联系

1. 与单例模式的联系

每个枚举常量天然就是单例：JVM 保证在类加载时创建，线程安全，且能抵抗反序列化和反射攻击。Joshua Bloch 在《Effective Java》中推荐用 enum 实现单例：

Java — 用 enum 实现单例（最优雅的方式）
public enum AppConfig {
    INSTANCE;   // 就这一行，JVM 保证单例

    private String dbUrl = "jdbc:mysql://localhost/demo";

    public String getDbUrl() { return dbUrl; }
}

// 使用
AppConfig.INSTANCE.getDbUrl();

2. 与 Spring 的联系

Spring MVC 可以直接将 URL 参数绑定到 enum，Jackson 序列化/反序列化也支持 enum（默认用 name()，可用 @JsonValue 指定其他字段）。

Java — Jackson 枚举序列化
import com.fasterxml.jackson.annotation.*;

public enum Status {
    ACTIVE(1), INACTIVE(0);

    private final int code;
    Status(int code) { this.code = code; }

    @JsonValue  // 序列化时输出 code 数字而不是 "ACTIVE"
    public int getCode() { return code; }

    @JsonCreator  // 反序列化：从数字还原 enum
    public static Status fromCode(int code) {
        for (Status s : values()) {
            if (s.code == code) return s;
        }
        throw new IllegalArgumentException("Unknown code: " + code);
    }
}

3. 与接口 / 抽象类的关系

enum 本质上是有限成员的"封闭类层级"——这与 Java 17 引入的 sealed interface 思想一脉相承。当你的需求是"固定且有限的多态"，enum 往往比 abstract class 更简洁。

常见陷阱与误区

陷阱 1：用 == 还是 equals？
enum 常量是单例，== 和 equals() 都安全，但推荐用 ==——更直观，且能避免 NullPointerException（null == Day.MON 返回 false，null.equals(Day.MON) 直接抛异常）。
陷阱 2：序列化存 ordinal
数据库/Redis 中存枚举时，绝对不要存 ordinal()。在 enum 中间插入新常量后，所有已存储的 ordinal 含义全部错乱。应存 name() 或自定义的稳定 code 字段。
陷阱 3：valueOf() 抛异常
Day.valueOf("wednesday")（小写）会抛 IllegalArgumentException，因为大小写必须完全匹配。做防御性转换时记得 toUpperCase() 或用 try-catch 包裹。
陷阱 4：enum 不能继承，但其构造器默认 private
enum 构造器不需要也不能写 public，编译器会报错。这是故意设计的——enum 外部不能 new，只能用已声明的常量。
陷阱 5：values() 每次返回新数组
Direction.values() 每次调用都会克隆并返回一个新的数组，在高频调用路径上（如热点循环内）应将其缓存为静态常量：private static final Direction[] VALUES = values();

面试高频考点

  1. enum 的底层实现是什么？——编译为继承 java.lang.Enum 的 final 类，每个常量是 public static final 的该类实例，在静态初始化块中创建。
  2. enum 能继承类吗？能实现接口吗？——不能继承（已隐式继承 Enum），能实现接口。
  3. 如何实现线程安全的单例？——用 enum 单例，JVM 类加载保证线程安全且防止反序列化创建新实例。
  4. EnumSet 为什么比 HashSet 快？——底层用位向量（long 数组），add/contains 是位运算，O(1) 且无哈希计算。
  5. ordinal() 和 name() 分别应该在什么场景用？——ordinal 仅用于 EnumSet/EnumMap 内部实现，业务代码应用 name() 或自定义 code 字段，ordinal 不稳定，不应持久化。
  6. enum 能用在 switch 里吗，和 String/int 有什么区别？——能，且推荐。enum switch 有类型安全保证，Java 14+ switch 表达式还能做穷举性检查。

小结

enum 在 Java 中扮演的角色远不止"有名字的常量"——它是一个 拥有有限实例的类，可以携带字段、实现方法、实现接口，甚至用多态替代 switch。掌握 enum 的关键是理解：

  • 它解决了旧式 int 常量的类型不安全、无命名空间、打印无意义三大问题
  • 每个常量是单例，天然线程安全，可安全用 == 比较
  • 配合 EnumSet / EnumMap 可获得极高的集合性能
  • 不要持久化 ordinal()，用 name() 或自定义稳定 code 字段
  • 抽象方法 + 常量特定类主体是枚举实现策略模式的利器
