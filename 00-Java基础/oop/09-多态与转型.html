09. 多态与向上转型 / 向下转型

理解多态的本质：同一消息，不同对象做出不同响应。掌握向上 / 向下转型的规则与陷阱，是写出灵活可扩展代码的基础。

1. 多态是什么？为什么需要它

多态（Polymorphism）字面意思是"多种形态"。在面向对象中它的具体含义是： 父类类型的引用变量，在运行时可以指向不同子类的对象，调用同一方法时执行不同的行为。

用一个生活类比来建立直觉：

你有一个"遥控器"（父类引用），可以控制"电视"，也可以控制"空调"。 对着两者都按"开机"按钮（同一方法名），但背后执行的电路完全不同。 你写代码时只需要握着遥控器，不需要关心到底面前是哪台设备。

如果没有多态，每增加一种子类，调用方就必须加一段 if / else 分支判断，代码爆炸。 有了多态，调用方只依赖父类接口，子类随意扩展，开闭原则（OCP）得以实现。

2. 多态成立的三个前提

多态成立 = ① 继承关系 / 接口实现
          + ② 子类重写（Override）父类方法
          + ③ 父类引用指向子类对象（向上转型）

缺少任何一条，多态都不会生效。最常见的误解是：方法重载（Overload）≠ 多态，重载在编译期就已经决定调用哪个版本，属于静态分派，不算多态。

3. 向上转型（Upcasting）

把子类对象赋值给父类类型的引用，就是向上转型。因为子类 is-a 父类，这是天然安全的， 编译器自动完成，无需显式强转。

Animal.java — 向上转型示例
class Animal {
    void speak() {
        System.out.println("...");
    }
}

class Dog extends Animal {
    @Override
    void speak() {
        System.out.println("汪汪");
    }
    void fetch() {
        System.out.println("捡球！");
    }
}

class Cat extends Animal {
    @Override
    void speak() {
        System.out.println("喵喵");
    }
}

public class Main {
    public static void main(String[] args) {
        // 向上转型：Animal 引用指向 Dog / Cat 对象
        Animal a1 = new Dog();
        Animal a2 = new Cat();

        a1.speak();  // 输出：汪汪  —— 运行时绑定到 Dog.speak()
        a2.speak();  // 输出：喵喵  —— 运行时绑定到 Cat.speak()

        // 编译期只看引用类型 Animal，fetch() 不在 Animal 里，编译报错
        // a1.fetch();  ← 无法通过编译！
    }
}
关键规律：向上转型后，引用变量的编译期类型是父类， 因此只能访问父类中声明的成员。但在运行期，JVM 根据对象实际类型调用子类重写的版本。 这正是多态的核心——"编译看左边，运行看右边"。

向上转型的典型使用场景

  • 方法参数接受父类类型，实参传入任意子类对象（最常见）
  • 集合存放父类类型：List<Animal> list = new ArrayList<>();
  • 工厂方法返回父类类型，隐藏具体实现
多态参数 — 统一处理不同子类
// 这个方法不需要知道传进来的是 Dog 还是 Cat
static void letSpeak(Animal animal) {
    animal.speak();  // 多态分派，运行时决定
}

letSpeak(new Dog());  // 输出：汪汪
letSpeak(new Cat());  // 输出：喵喵
// 新增 Bird 子类时，letSpeak 无需修改 —— 开闭原则

4. 动态绑定：JVM 如何在运行时找对方法

多态背后的机制叫动态方法分派（Dynamic Method Dispatch），也称虚方法调用。 JVM 在调用一个实例方法时，不是看引用的声明类型，而是查询对象头里记录的实际类型， 再去该类的虚方法表（vtable）里找对应的方法实现。

调用 a1.speak()
  ↓ 引用类型：Animal（编译期只验证 Animal 里有 speak）
  ↓ 对象实际类型：Dog（运行期）
  ↓ 查 Dog 的 vtable → Dog.speak() ✓

这也解释了为什么 static 方法、private 方法、final 方法 没有多态：它们在编译期就已静态绑定，不进入 vtable 分派流程。

陷阱：成员变量没有多态！
字段的访问在编译期就绑定到引用的声明类型，不会动态分派。 如果父子类都定义了同名字段，通过父类引用只能访问父类字段，子类的字段被"遮蔽"（shadowing）。 这是和方法重写完全不同的行为，面试高频考点。
字段不参与多态 — 遮蔽陷阱
class Base {
    String name = "Base";
}
class Sub extends Base {
    String name = "Sub";  // 遮蔽父类字段，不是重写
}

Base b = new Sub();
System.out.println(b.name);  // 输出 "Base"，不是 "Sub"！
// 字段绑定看引用类型（编译期），方法绑定看对象类型（运行期）

5. 向下转型（Downcasting）

向上转型让我们获得灵活性，但同时也"丢失"了访问子类特有成员的能力。 当我们确定父类引用背后的真实对象类型，就可以通过显式强转把它还原为子类类型， 这叫向下转型。

向下转型 — 强转语法
Animal a = new Dog();  // 向上转型

// 向下转型：需要显式 (Dog) 强转
Dog d = (Dog) a;
d.fetch();   // 现在可以调用 Dog 特有的方法了

向下转型是潜在危险的操作。编译器只能检查两个类型是否有继承关系， 无法保证运行时对象一定是目标类型。如果转型对象根本不是 Dog 的实例， 就会抛出 ClassCastException。

ClassCastException 示例
Animal a = new Cat();  // 实际是 Cat

// 编译器不报错（Animal → Dog 有继承关系），但运行时爆炸
Dog d = (Dog) a;  // ❌ 运行时：ClassCastException: Cat cannot be cast to Dog

6. instanceof：向下转型的安全门卫

每次向下转型前，应先用 instanceof 检查实际类型，是防止 ClassCastException 的标准做法。

传统 instanceof 写法
void handle(Animal a) {
    if (a instanceof Dog) {
        Dog d = (Dog) a;  // 确认是 Dog 再转，安全
        d.fetch();
    }
}

Java 16 引入了模式匹配 instanceof（Pattern Matching for instanceof）， 合并了判断与声明两步，代码更简洁：

Java 16+ 模式匹配 instanceof
void handle(Animal a) {
    // 判断 + 自动转型 + 变量声明，三合一
    if (a instanceof Dog d) {
        d.fetch();   // d 在此作用域内直接可用
    }
}
最佳实践：如果你发现代码里充斥着大量 instanceof 判断， 这通常是设计问题的信号——说明多态没有被充分利用，应该考虑将这些分支行为下沉到各子类中去重写。

7. 综合示例：支付系统

用一个贴近实际的场景，把向上转型、多态分派、向下转型串起来演示：

Payment.java — 完整多态示例
abstract class Payment {
    abstract void pay(double amount);
    void printReceipt() {
        System.out.println("支付完成，打印收据");
    }
}

class Alipay extends Payment {
    @Override
    public void pay(double amount) {
        System.out.println("支付宝支付：¥" + amount);
    }
    public void scanQRCode() {
        System.out.println("扫描支付宝二维码");
    }
}

class WechatPay extends Payment {
    @Override
    public void pay(double amount) {
        System.out.println("微信支付：¥" + amount);
    }
}

public class Checkout {
    // 参数类型是父类 —— 向上转型，不关心具体支付方式
    static void checkout(Payment p, double amount) {
        p.pay(amount);        // 多态分派，运行时决定
        p.printReceipt();  // 调用父类通用逻辑

        // 只有 Alipay 才需要扫码逻辑，向下转型
        if (p instanceof Alipay alipay) {
            alipay.scanQRCode();
        }
    }

    public static void main(String[] args) {
        checkout(new Alipay(),    99.9);
        checkout(new WechatPay(), 49.9);
    }
}

8. 向上转型 vs 向下转型对比

维度 向上转型（Upcasting） 向下转型（Downcasting）
方向 子类 → 父类 父类 → 子类
是否自动 自动，无需显式写 必须显式 (SubType)
安全性 100% 安全，编译器保证 有风险，可能 ClassCastException
访问范围 只能访问父类中声明的成员 可访问子类特有成员
方法调用 运行时绑定到子类重写版本 同上（多态依然有效）
前置条件 存在继承 / 接口实现关系 对象实际类型必须兼容目标类型

9. 常见陷阱与面试考点

陷阱一：构造器中调用被重写的方法

父类构造器执行时，子类还未完成初始化，但此时虚方法已经绑定到子类版本，可能导致用到未初始化的字段。

构造器中多态调用的隐患
class Parent {
    Parent() {
        init();  // ⚠️ 调用被子类重写的方法
    }
    void init() { System.out.println("Parent.init"); }
}

class Child extends Parent {
    int value = 42;

    @Override
    void init() {
        // 父类构造器调用时，value 还是 0（默认值），不是 42！
        System.out.println("Child.init, value = " + value);
    }
}

new Child();  // 输出：Child.init, value = 0  ← 不是 42！

陷阱二：接口引用的多态

多态不只在继承链上有效，通过接口引用调用方法同样是动态分派。这是 Java 多态最强大的使用形式。

接口引用的多态
interface Drawable {
    void draw();
}

class Circle implements Drawable {
    public void draw() { System.out.println("画圆"); }
}

class Square implements Drawable {
    public void draw() { System.out.println("画方"); }
}

List<Drawable> shapes = List.of(new Circle(), new Square());
shapes.forEach(Drawable::draw);  // 各自调用自己的 draw()

面试高频考点汇总

  • 编译看左边，运行看右边：引用类型决定能访问哪些成员，对象类型决定实际执行哪个方法
  • 字段没有多态：同名字段是遮蔽（shadowing），不是重写，访问结果由引用类型决定
  • static / private / final 方法不参与多态，不能被重写
  • 向下转型前必须 instanceof 检查，否则可能 ClassCastException
  • 父类构造器中调用虚方法是陷阱，子类字段尚未初始化
  • 接口引用同样支持多态分派，是 Spring 依赖注入、策略模式的底层机制

10. 与其他知识的联系

相关知识 联系
方法重写 @Override 多态的必要条件，没有重写就没有动态分派
抽象类 / 接口 多态最常见的宿主，强迫子类重写，让多态更纯粹
设计模式 策略、模板方法、工厂、装饰器……几乎所有模式都以多态为核心
Spring IoC Bean 注入时返回接口类型，实际对象是代理或实现类，典型多态应用
JVM vtable 虚方法表是动态分派的底层数据结构，每个类在类加载时构建
泛型 泛型通过类型上界（<T extends Animal>）与多态协作，扩大复用范围
一句话记住核心： 向上转型 = 自动 + 安全 + 丢能力；向下转型 = 手动 + 有风险 + 找回能力。 多态 = 向上转型 + 方法重写 + 运行时动态分派，三者缺一不可。
