面向对象 · 类与对象

01. 类的定义与实例化

从「模具与产品」的视角理解 class 与 object 的本质关系，掌握 Java 定义类的完整语法，并深入理解 new 关键字背后发生了什么。

为什么需要"类"：从过程式到面向对象

早期的 C 语言是过程式的：程序 = 一系列操作步骤。当业务复杂到需要同时维护多个相关数据时，代码就会失控——比如描述 100 个学生，每人有姓名、年龄、分数，就需要 300 个变量，互相之间毫无关联。

类（class）解决的核心问题是：把「数据」和「操作数据的行为」封装在一起，形成一个自洽的单元。这和现实世界的认知方式完全一致：一台手机既有属性（品牌、电量），也有行为（打电话、充电）。

设计哲学：类是对现实事物的抽象建模。你描述的是「一类事物的共同特征」，而不是某一个具体的东西。模具（class）和用模具生产的产品（object）是不同层次的概念。

类的基本语法结构

一个最完整的 Java 类由访问修饰符、类名、字段（field）、方法（method）组成：

Dog.javaJava
// 访问修饰符 + class 关键字 + 类名（大驼峰）
public class Dog {

    // ── 字段（Field）：描述对象的「状态」 ──
    String name;      // 实例字段，每个对象各自拥有一份
    int    age;
    String breed;

    // ── 方法（Method）：描述对象的「行为」 ──
    void bark() {
        // this 指向调用这个方法的那个具体对象
        System.out.println(name + " 说：汪汪！");
    }

    String getInfo() {
        return name + "，" + age + " 岁，品种：" + breed;
    }
}

类名命名规范

  • 大驼峰（UpperCamelCase）：Dog、UserAccount、HttpRequest
  • 类名应是名词，代表一类事物，而不是动作
  • Java 强制要求：public 类的文件名必须与类名完全一致（含大小写），否则编译报错

实例化：new 关键字做了什么

写完类只是定义了「模具」，new 才是真正「生产产品」的动作。

Main.javaJava
public class Main {
    public static void main(String[] args) {

        // 1. 在堆（Heap）上分配内存，创建 Dog 对象
        // 2. 调用默认构造方法，字段初始化为默认值（null / 0）
        // 3. 返回对象在堆中的地址，赋给栈上的引用变量 d1
        Dog d1 = new Dog();

        // 通过「引用.字段」访问并赋值
        d1.name  = "旺财";
        d1.age   = 3;
        d1.breed = "金毛";

        // 通过「引用.方法」调用行为
        d1.bark();           // 旺财 说：汪汪！
        System.out.println(d1.getInfo());  // 旺财，3 岁，品种：金毛

        // 同一个类可以创建无数个相互独立的对象
        Dog d2 = new Dog();
        d2.name = "小白";
        d2.age  = 1;

        // d1 和 d2 是两个完全独立的对象，修改一个不影响另一个
    }
}

new 背后的内存模型

new Dog() 执行时，JVM 在堆（Heap）上分配一块内存存放对象的字段数据，在栈（Stack）上存放指向该内存的引用变量。引用变量本质上是一个地址值（类似 C 的指针，但 Java 不允许你直接操作它）。

栈（Stack） 堆（Heap） ┌──────────┐ ┌──────────────────────┐ │ d1 ●───┼─────────────▶│ Dog 对象 #A1F2 │ └──────────┘ │ name = "旺财" │ ┌──────────┐ │ age = 3 │ │ d2 ●───┼──────┐ │ breed = "金毛" │ └──────────┘ │ └──────────────────────┘ │ ┌──────────────────────┐ └──────▶│ Dog 对象 #3C9D │ │ name = "小白" │ │ age = 1 │ │ breed = null │ └──────────────────────┘

用 new 创建对象时，如果字段没有显式赋值，JVM 会自动初始化为以下默认值：

float / double0.0浮点类型统一为 0.0
booleanfalse
char''空字符（Unicode 0）

陷阱：局部变量（方法内声明的变量）没有默认值，使用前必须显式赋值，否则编译器直接报错 variable might not have been initialized。这与字段的行为完全不同，是初学者最常踩的坑之一。

类与对象的关系：模具与产品

可以用三个层次来理解类与对象：

概念类（Class）对象（Object / Instance）
是什么模具、蓝图、抽象描述用模具产出的具体产品
存在位置方法区（Method Area）堆（Heap）
数量同一个类只有一份可以 new 出无数个
生命周期类加载后一直存在没有引用指向时被 GC 回收

完整示例：BankAccount 类

用银行账户来演示一个「字段 + 方法」都更完整的类：

BankAccount.javaJava
public class BankAccount {

    // 字段：描述账户的状态
    String owner;
    double balance;

    // 存款：修改自身状态
    void deposit(double amount) {
        if (amount <= 0) return;  // 防御性编程
        balance += amount;
    }

    // 取款：返回是否成功
    boolean withdraw(double amount) {
        if (amount > balance) return false;
        balance -= amount;
        return true;
    }

    void printStatus() {
        System.out.printf("账户[%s] 余额：%.2f%n", owner, balance);
    }
}
Main.java — 使用 BankAccountJava
BankAccount acc = new BankAccount();
acc.owner   = "张三";
acc.balance = 1000.0;

acc.deposit(500);
acc.printStatus();   // 账户[张三] 余额：1500.00

boolean ok = acc.withdraw(2000);
System.out.println("取款结果：" + ok);  // false（余额不足）
acc.printStatus();   // 账户[张三] 余额：1500.00（未变）

引用赋值：「两个遥控器，同一台电视」

Java 中变量存的是引用（地址），而不是对象本身。把一个引用赋给另一个变量，只是让两个变量指向同一个对象，并不会复制对象。

引用赋值示例Java
Dog a = new Dog();
a.name = "旺财";

Dog b = a;   // b 和 a 指向「同一个」堆对象，并没有创建新对象

b.name = "小黑";
System.out.println(a.name);  // 小黑 ← a 也变了！因为本质是同一个对象
高频陷阱：把对象作为方法参数传递时，传入的是引用的副本，两个引用依然指向同一个堆对象。在方法内修改对象的字段，调用方可以感知到变化；但如果在方法内对参数变量重新 new，调用方的引用不受影响——这是"引用传递 vs 值传递"争论的根源，Java 严格来说是「按值传递引用」。

null 引用与 NullPointerException

当引用变量没有指向任何对象（值为 null），对它调用方法或访问字段就会抛出 NullPointerException（NPE）——Java 最臭名昭著的运行时异常。

NPE 示例Java
Dog d = null;   // d 不指向任何对象
d.bark();       // 💥 NullPointerException：对 null 解引用

// 正确做法：使用前先判空
if (d != null) {
    d.bark();
}

与其他知识点的联系

与 JVM 内存模型

类的元数据（字段名、方法字节码）存放在方法区（Method Area，JDK 8+ 称为 Metaspace），对象实例存放在堆（Heap），引用变量和局部变量存放在栈（Stack）。理解这三块内存的职责，是后续学习 GC、OOM 问题的基础。

与封装（Encapsulation）

目前字段是直接可访问的（默认包访问权限），这是不安全的。下一步会学习用 private 修饰字段 + 提供 getter/setter，把「状态」藏起来，只暴露行为接口——这就是面向对象三大特性之一的「封装」。

与 Spring 框架

Spring 的核心就是「控制反转（IoC）容器」：容器自动帮你 new 对象（Bean），并注入依赖，不需要你手动 new。理解 new 的本质，是后续理解 Spring Bean 生命周期、作用域（singleton/prototype）的前提。

面试高频考点

Q：类和对象的区别是什么？
类是对象的模板（蓝图），存放于方法区，只有一份；对象是类的实例，存放于堆，可以创建多个，互相独立。
Q：Java 是值传递还是引用传递？
Java 只有值传递。传基本类型时传的是值的副本；传对象时传的是引用（地址）的副本，两个副本仍指向同一个堆对象，所以对象内部的修改调用方可见，但重新赋值（param = new Xxx()）不影响调用方。
Q：== 与 equals 的区别？
对于对象，== 比较的是引用（地址）是否相同，即是否是同一个对象；equals 默认也是比较地址，但 String、Integer 等类重写了 equals 来比较内容。这也是"new String("a") == new String("a") 为 false"的原因。
Q：new 一个对象的完整过程？
① 类加载检查（如果类未加载，先执行类加载）→ ② 在堆分配内存 → ③ 字段初始化为零值 → ④ 执行实例初始化（字段显式赋值 + 构造方法） → ⑤ 返回对象引用。
