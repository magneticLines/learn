底层编码优化

五种类型是"对外的接口"，Redis 在内部会根据数据量大小自动切换更紧凑的底层编码——这是 Redis 省内存的关键设计，也是理解跳表为什么被选中的入口。

一句话讲清楚

同一个类型（比如 Hash 或 ZSet），Redis 在数据量小的时候用更紧凑但操作稍慢的编码节省内存， 数据量变大后再自动"升级"为更适合大数据量的编码。三种典型的紧凑编码是：ziplist（压缩列表）、 intset（整数集合）、skiplist（跳跃表）。这是一种典型的"用条件换性能"设计： 大多数业务里 Hash/List/ZSet 的元素并不多，没必要一开始就用最通用但开销更大的结构。

ziplist：小数据量下的压缩列表

ziplist 是一段连续内存，把多个元素紧凑地挨着存放，省掉了普通链表里每个节点都要额外维护的前后指针开销。 代价是：不支持随机访问（要查第 N 个元素得从头遍历），且中间插入/删除元素可能引发连锁的内存重分配（"连锁更新"）。 Redis 用它作为小 List、小 Hash、小 ZSet 的默认编码，一旦元素数量或单个元素大小超过阈值（如 hash-max-ziplist-entries）， 就转换成更通用的编码（Hash 转 hashtable，ZSet 转 skiplist）。

类比：ziplist 像把一排小盒子紧挨着摆放在一条架子上，取东西方便、不浪费架子空间， 但如果要在中间插入一个新盒子，后面所有盒子都得往后挪——数据量小的时候挪一下无所谓，数据量大了就得换成"每个盒子单独挂钩子"（哈希表）的方式。

intset：全是整数的小 Set 专用编码

如果一个 Set 里的元素全部是整数，且数量不多，Redis 会用 intset 编码——本质是一个有序的整数数组， 比通用的哈希表结构更省内存。一旦往里面塞入非整数元素，或者元素数量超过阈值，就会转换成 hashtable 编码。

redis-cli
# 全是整数、数量少 → intset 编码
SADD myset 1 2 3
OBJECT ENCODING myset  # "intset"
# 混入字符串 → 升级为 hashtable 编码
SADD myset "abc"
OBJECT ENCODING myset  # "hashtable"

skiplist：ZSet 数据量大时的选择，为什么不用红黑树

当 ZSet 元素数量超过阈值时，Redis 用跳跃表（skiplist）维护有序性，而不是像很多教科书数据结构那样选择平衡树（红黑树、AVL 树）。 跳跃表的核心思路是用空间换查询效率：在有序链表的基础上，额外建立多层"索引层"，每一层都是下一层的"稀疏采样"， 查找时从最高层开始跳跃前进，逐层缩小范围，平均时间复杂度做到 O(log N)，接近平衡树。

面试高频："ZSet 为什么用跳表而不用红黑树？" 作者黄健宏在《Redis 设计与实现》里给出的原因主要两点： ①实现简单——跳表的插入、删除逻辑比红黑树的旋转、变色平衡操作直观得多，不易出 bug，代码维护成本低； ②范围查询更方便——跳表本质是有序链表加多层索引，找到起点后直接顺着链表向后遍历即可完成区间查询（如 ZRANGEBYSCORE）， 而红黑树做范围查询需要中序遍历，实现和性能都不如链表直接。

呼应：和 09-数据库 模块讲过的 B+ 树对比

跳表和 B+ 树都是"有序结构"，但适用场景完全不同，本质是内存结构和磁盘结构的取舍差异： B+ 树为了减少磁盘 IO 次数，把每个节点设计得很"胖"（一个节点装尽可能多的 key，匹配磁盘页大小），层数很浅； 跳表是纯内存结构，不需要考虑磁盘页对齐，靠"层级索引 + 随机层数"实现近似平衡，不需要 B+ 树那种复杂的节点分裂合并逻辑。 一句话总结：B+ 树为磁盘 IO 优化，跳表为内存下的简单实现和范围查询优化，两者都追求"有序 + 高效查找"，但因为存储介质不同，选择了完全不同的路径。

三种紧凑编码速查

编码适用场景核心特点
ziplist小 List / 小 Hash / 小 ZSet连续内存紧凑存储，省空间但不支持随机访问
intset全是整数的小 Set有序整数数组，比哈希表更省内存
skiplist大 ZSet多层索引链表，O(log N) 查找，实现简单，范围查询方便

核心价值总结

  • Redis 对象的编码不是固定的，会随数据量自动"升级"，这是典型的"用复杂度换内存"设计思想。
  • ziplist/intset 是"小数据量专属捷径"，本质是牺牲部分操作效率换取更紧凑的内存布局。
  • 跳表被选为 ZSet 大数据量下的实现，核心原因是实现简单、范围查询友好，而不是查找效率比红黑树更高。
  • 理解"为什么不用某个更常见的结构"，往往比记住"用了什么结构"更有面试价值。
