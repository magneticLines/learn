IP 协议与 IP 地址分类

传输层的 TCP/UDP 决定了"数据怎么可靠或高效地发"，但真正负责"把数据包从 A 主机送到 B 主机"这件跨网络的事，是网络层的 IP 协议。它的设计哲学简单到极致：尽力而为，出了问题不负责。

IP 协议的定位：尽力而为的、无连接的数据报服务

先回顾上一个 Tab 讲过的分层：传输层的 TCP 负责建立连接、保证可靠传输（三次握手、序号确认、超时重传）。 很多初学者会想当然地认为"可靠传输"是整个网络协议栈共同的目标，于是也以为 IP 协议同样会负责重传、纠错。事实恰恰相反。

IP（Internet Protocol）提供的是一种 尽力而为（best-effort）的、无连接（connectionless）的数据报（datagram）服务，翻译成人话：

  • 尽力而为：IP 层只管尽量把包送到，送不到、送丢了、送重复了、送的顺序乱了，它都不保证，也不负责发现和纠正。
  • 无连接：每个 IP 数据包（IP datagram）都是独立处理的，包和包之间没有关联，路由器不会像 TCP 那样维护"连接状态"。
  • 不可靠：不做确认、不做重传、不做流量控制、不做拥塞控制——这些统统交给上层的 TCP 去做。
类比：IP 协议像寄明信片——你把信丢进邮筒（发送数据包），邮局尽量帮你送到，但不会给你回执，也不保证送达顺序，甚至可能弄丢。如果你需要"确认对方收到了""按顺序收到了"，那是你自己（也就是 TCP）该操心的事，不是邮局（IP）的职责。

这就是为什么可靠性要靠 TCP，而不是靠 IP：把"传输可靠"和"路径转发"这两个职责拆开，是经典的分层解耦思想—— IP 只管"怎么把包从一个网络转发到另一个网络"，这样设计足够简单，路由器才能高速转发海量数据包；如果每个路由器都要维护每条连接的可靠性状态，整个互联网的转发效率会崩溃。

IPv4 地址：32 位，点分十进制

IPv4 地址长度是 32 位（4 字节），书写时按每 8 位一组、用点分隔、转成十进制，即"点分十进制"，例如 192.168.1.1。 32 位意味着理论上限是 2^32 ≈ 42.9 亿个地址——这个数字在互联网诞生初期看起来绰绰有余，但随着联网设备爆炸式增长，早已不够用（下文会讲 IPv6 如何解决这个问题）。

分类编址：A/B/C/D/E 类

IPv4 早期设计采用"分类编址"（classful addressing），把地址空间按最高几位划分成 5 类，每类地址中"网络号"和"主机号"的划分比例不同：

类别第一字节范围网络号/主机号典型用途
A 类1 ~ 1268 位网络号 / 24 位主机号超大型网络（每个网络可容纳约 1677 万台主机）
B 类128 ~ 19116 位网络号 / 16 位主机号中型网络（每个网络约 6.5 万台主机）
C 类192 ~ 22324 位网络号 / 8 位主机号小型网络（每个网络 254 台主机）
D 类224 ~ 239—组播地址（multicast）
E 类240 ~ 255—保留，用于实验研究
面试高频但要辩证看待：分类编址早已被 CIDR（无类别域间路由，Classless Inter-Domain Routing） 取代—— CIDR 用"网络前缀长度"（如 192.168.1.0/24 中的 /24）代替死板的 A/B/C 类划分，可以按需灵活切分地址块，大幅提高地址利用率。 但面试中问"IP 地址分几类"仍然是高频送分题，A/B/C/D/E 的划分逻辑要记住，同时要能说出"现在实际用的是 CIDR"这一点，才显得懂原理而不是死记硬背。

公网地址 vs 私网地址

并不是所有 IPv4 地址都能在公网上直接路由。IETF 在 RFC 1918 中专门划出三段地址作为私网地址（Private Address），规定路由器不转发这些地址的流量到公网，只能在局域网内部使用：

私网地址段所属分类常见场景
10.0.0.0/8A 类企业内网、云厂商 VPC
172.16.0.0/12B 类Docker 默认网桥常用此段
192.168.0.0/16C 类家庭路由器、小型办公网络

公网地址（Public Address）则是全球唯一、可以在互联网上被直接路由到的地址，每一个公网 IP 在同一时刻全球只能有一台设备在用。 你家路由器对内分配 192.168.1.x 这样的私网地址给每台设备，对外则用运营商分配的一个公网 IP 统一出口——这就是下一篇要讲的 NAT 要解决的问题。

IPv6：从根本上解决地址耗尽问题

IPv4 的 32 位地址空间理论上限约 43 亿，实际因为分类编址浪费、私网地址占用等原因，公网可用地址早已在 2011 年前后被 IANA 分配殆尽。 NAT 只是权宜之计（延缓危机，见下一篇），真正的长期方案是 IPv6：地址长度从 32 位扩展到 128 位，地址空间达到 2^128 个， 数量级上足以给地球上每一粒沙子分配一个地址，从根本上解决地址耗尽问题。

IPv6 地址写法也完全不同，采用冒号分隔的 8 组十六进制数，例如 2001:0db8:85a3:0000:0000:8a2e:0370:7334（可以省略前导零、用 :: 压缩连续的全零组）。 由于历史包袱和过渡成本，目前 IPv4/IPv6 仍处于长期共存阶段，本模块后续不深入展开 IPv6 细节，理解"它解决了地址不够用的根本问题"这一点即可。

常见误区

误区一：以为 IP 层也做"确认与重传"。可靠性完全是 TCP 的职责，IP 层只管尽力转发，UDP 之上如果不加可靠性机制，数据包丢了就是丢了。
误区二：把"私网地址"和"内网穿透就访问不到"画等号——私网地址本身没有技术上的访问限制，只是路由器按约定不转发这些地址的公网流量，配合 NAT 或端口映射依然可以从外部访问到内网设备（详见下一篇 NAT）。

核心价值总结

  • IP 协议只负责转发，不负责可靠——这是分层解耦的经典案例，可靠性留给上层 TCP 处理。
  • 分类编址（A/B/C/D/E）已被 CIDR 取代，但仍是理解地址划分历史和面试题的基础。
  • 私网地址 + NAT 是应对 IPv4 地址枯竭的过渡方案，IPv6 128 位地址才是根本解法。
