网络层与链路层

交换机与集线器的区别

上一篇讲了 MAC 地址是链路层设备识别彼此的依据。那链路层的转发设备具体是怎么工作的？集线器和交换机是两种历史上真实存在过的方案，一种早已被淘汰，一种至今仍是局域网的标准配置——理解两者的差异，就是理解链路层"定向转发"设计的价值。

集线器（Hub）：工作在物理层，广播式转发

集线器是一种非常"笨"的设备：它工作在 OSI 模型的物理层，本质上只是一个多接口的信号放大和转发器。 当集线器的某个端口收到一个电信号（比特流），它完全不解析里面的内容（不知道也不关心什么是 MAC 地址、什么是帧），只是简单粗暴地把这个信号从所有其他端口原样广播出去。

类比：集线器就像一个不隔音的大通铺宿舍——只要有一个人说话，所有人都能听到，不管这句话是不是说给自己听的。想让 A 和 B 悄悄说句话，结果全宿舍的人都被迫听了一遍。

集线器的两个致命问题

这种"广播式转发"带来了两个根本性的问题，也是集线器最终被淘汰的原因：

  • 效率低下：不管目标设备是谁，每个信号都要发给所有端口，链路带宽被大量浪费在"根本不需要收到这条消息的设备"身上，接入的设备越多，无效流量占比越高。
  • 冲突域问题：集线器连接的所有设备共享同一个"冲突域（Collision Domain）"——因为大家共用同一条广播信道，如果两台设备同时发送数据，信号会在物理层叠加冲突、相互干扰，导致数据损坏，双方都需要检测冲突（CSMA/CD 机制）并等待随机时间后重传。设备越多，冲突概率越高，网络性能会随接入设备数量增加而急剧下降。

交换机（Switch）：工作在链路层，基于 MAC 地址表定向转发

交换机的设计聪明得多：它工作在 OSI 模型的数据链路层，能够解析每个数据帧的帧头，读出其中的源 MAC 地址和目的 MAC 地址。 交换机内部维护一张 MAC 地址表（也叫 CAM 表），记录"某个 MAC 地址的设备连接在哪个物理端口上"，转发时只把数据帧从目的 MAC 地址对应的那一个端口发出去，而不是像集线器那样广播给所有端口。

交换机 MAC 地址表示例
// 交换机通过监听收到的每一帧的源 MAC，自动学习并维护这张表
MAC 地址              端口
00:1A:2B:3C:4D:5E     Port 1   // 设备 A 接在 1 号口
AA:BB:CC:11:22:33     Port 3   // 设备 B 接在 3 号口
11:22:33:AA:BB:CC     Port 5   // 设备 C 接在 5 号口

// 当交换机收到一个目的 MAC 为 AA:BB:CC:11:22:33 的帧
// 只会从 Port 3 转发出去，其余端口完全收不到这个帧

这种"学习 + 定向转发"的机制带来了两个关键收益，正好对应集线器的两个缺陷：

  • 效率大幅提升：数据只发给真正需要接收的设备，链路带宽不再被无关流量浪费，多对设备之间甚至可以同时进行"全双工"通信而互不干扰。
  • 隔离冲突域：交换机的每一个端口都是一个独立的冲突域——设备 A 和交换机之间、设备 B 和交换机之间，都是独占的点对点链路，不存在争抢同一条信道的问题，CSMA/CD 冲突检测机制在全双工交换网络中基本失去了意义。
面试高频："交换机和集线器的本质区别是什么？" 标准答案：集线器工作在物理层，不解析帧内容，广播转发，共享同一个冲突域；交换机工作在链路层，基于 MAC 地址表定向转发，每个端口独立成一个冲突域。能说出"冲突域"这个词，基本能说明你理解了原理而不是死记硬背。

对比一览

对比维度集线器（Hub）交换机（Switch）
所属 OSI 层物理层数据链路层
转发方式广播（发给所有端口）基于 MAC 地址表定向转发
冲突域所有端口共享同一个冲突域每个端口独立成一个冲突域
带宽利用率低，设备越多冲突越频繁高，可实现全双工通信
安全性差，任意端口能嗅探到全部流量较好，非目标设备正常情况下收不到无关流量

为什么交换机基本淘汰了集线器

从上面的对比不难看出：交换机在效率、可扩展性、安全性上全面碾压集线器，而且随着芯片成本下降，交换机的价格早已降到和集线器相当的水平—— "既然贵不了多少，为什么不用性能更好的那个"，这是集线器在实际网络建设中几乎绝迹的根本原因。 如今除了极少数教学演示、老旧设备维护场景，几乎不会再有人主动选择集线器组网。

与上一篇的呼应

交换机之所以能"定向转发"，前提正是上一篇讲的 MAC 地址局部有效、可以标识局域网内的具体设备——如果没有 MAC 地址表这个"链路层身份证"， 交换机也做不到精确投递，只能退化成集线器式的广播。这也再次印证了分层设计的价值：链路层解决"局域网内如何精确寻址"，链路层设备（交换机）才能据此做出更聪明的转发决策。

常见误区

误区：以为交换机和路由器是一回事。交换机工作在链路层，靠 MAC 地址转发，只能在同一个局域网/广播域内工作；路由器工作在网络层，靠 IP 地址和路由表转发，负责跨网络通信。很多入门级"路由器"设备实际上是"路由器+交换机+无线 AP"三合一的产品，这也是初学者容易混淆功能边界的原因。

核心价值总结

  • 集线器工作在物理层，广播转发，共享冲突域——简单但低效，已被淘汰。
  • 交换机工作在链路层，基于 MAC 地址表定向转发，隔离冲突域——是现代局域网组网的标准设备。
