📖 Java Concurrency in Practice — Brian Goetz et al.

Java 并发领域的圣经

作者: Brian Goetz, Tim Peierls, Joshua Bloch 等

适合谁:有 Java 基础、想深入理解并发的开发者。前置知识:Java 基础语法 + 基本的多线程概念（Thread / Runnable）。

阅读建议:第 1-4 章是全书的理论核心，必须吃透；第 5-7 章讲实践工具；第 10-11 章（性能与测试）适合有项目经验后再读。不建议跳着读——前 4 章的概念在后面反复引用。

避坑:这本书基于 Java 5/6，部分 API 已被现代 Java 替代（如 Atomic 包新增了更丰富的类），但核心原理完全没过时。

  • 第 1-2 章 · 线程安全的基础
    • 什么是线程安全性核心原子性 / 可见性 / 有序性
    • 竞态条件与复合操作核心
    • synchronized 与内置锁
    • volatile 与 happens-before核心
  • 第 3-4 章 · 共享对象与组合
    • 发布与逸出this 引用逸出
    • 线程封闭与 ThreadLocal核心
    • 不变性:不可变对象是天然的线程安全
    • 安全发布的四种方式
    • 组合设计线程安全的类
  • 第 5-7 章 · 并发构建模块
    • 阻塞队列 BlockingQueue 与生产者消费者核心
    • 同步工具:CountDownLatch / CyclicBarrier / Semaphore
    • 并发容器:ConcurrentHashMap / CopyOnWriteArrayList
    • 任务执行框架:Executor / ThreadPoolExecutor核心
    • Future / Callable 与异常处理
    • CompletableFuture 与异步编排现代补充
  • 第 8-9 章 · 活跃度与性能
    • 死锁、活锁、饥饿核心高频面试
    • 死锁的检测与避免:锁顺序 / 定时锁 / 死锁检测
    • 性能权衡:上下文切换 / 内存同步 / 锁竞争
    • 锁分段与 ConcurrentHashMap 的设计
  • 第 10-11 章 · 显式锁与原子变量
    • ReentrantLock:可中断 / 定时 / 公平核心
    • ReadWriteLock 与 StampedLock
    • Condition:比 wait/notify 更灵活的等待通知
    • AQS 框架原理进阶
    • Atomic 类与 CAS 原理核心
  • 第 12-16 章 · JMM 与测试
    • Java 内存模型(JMM): happens-before 规则核心高频面试
    • 安全初始化与双重检查锁(DCL)
    • 并发测试:正确性测试 / 性能测试
    • 并发陷阱:指令重排与内存屏障
