📖 Refactoring: Improving the Design of Existing Code, 2nd Ed.

工程素养必读——比"怎么写代码"更重要的是"怎么改代码"

作者: Martin Fowler

适合谁:有 1-3 年经验的开发者。前置知识:面向对象编程基础 + 基本的设计模式常识。

阅读建议:第 1 章先读（一个完整的重构案例），建立"重构的节奏感"。第 2 章（原则）和第 3 章（代码坏味道）是全书的核心——先学会"闻到坏味道"比背重构手法更重要。重构目录（第 6-12 章）当字典翻，不要硬啃。第 2 版改用 JavaScript 示例，比第 1 版的 Java 更简洁。

  • 第 1-2 章 · 理念与原则
    • 什么是重构:不改变外部行为的前提下改善内部结构
    • 重构 vs 重写:什么时候该重构，什么时候该重写
    • 重构的前提:好的测试核心
    • 重构的节奏:小步修改 + 频繁测试
  • 第 3 章 · 代码坏味道(Code Smells)
    • 神秘命名:名字看不出意图
    • 过长函数 + 过长参数列表核心
    • 全局数据 / 可变数据 / 发散式变化
    • 霰弹式修改:一个改动要改很多类
    • 依恋情结:一个方法过分依赖另一个类
    • 基本类型偏执 / switch 滥用
    • 重复代码 / 夸夸其谈通用性
  • 第 6-7 章 · 基本重构手法
    • 提炼函数 / 内联函数 / 提炼变量核心
    • 改变函数声明 / 引入参数对象
    • 封装变量 / 以查询取代派生变量
  • 第 8-9 章 · 数据与特征重构
    • 搬移函数 / 搬移字段 / 提炼类 / 内联类
    • 隐藏委托关系 / 移除中间人
    • 以多态取代条件表达式核心
    • 以工厂函数取代构造函数
  • 第 10-12 章 · 继承与简化
    • 方法上移 / 字段上移 / 构造函数本体上移
    • 以委托取代继承 / 以继承取代委托
    • 拆分循环 / 合并条件表达式 / 以卫语句取代嵌套条件
