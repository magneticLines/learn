# 学习项目「以书为骨架」重构 · 设计文档

日期:2026-06-28

## 背景

`java_learn` 是一个全栈学习项目,含 7 个模块:`00-Java基础`、`01-数据结构`、
`02-并发`、`03-设计模式`、`04-工程实战`、`05-git`、`06-前端`。

现状:

- 已有路线图(roadmap)的模块:Java、设计模式、前端、React(`*-roadmap.html`),
  采用「路线图 + 1:1 叶子文件」结构,`06-前端` 已较完整遵循 CLAUDE.md 规范。
- 无路线图的模块:数据结构、并发、工程实战、git(目前为 `notes/*.md` + `code/`)。
- 根目录已有 `计算机抽象塔.html`(全局抽象层地图)。
- 已有统一暗色视觉风格(leaf 节点、callout、code-block、nav-footer 等)。

## 目标

把所有模块统一为「**以一本(或多本)经典书为骨架的框架文件 + 据书逐节点生成讲解文件**」
的结构,并在根目录建一个总 index 串起所有模块。

## 选书定稿

| 模块 | 骨架书来源 |
|---|---|
| `00-Java基础` | 《Java 核心技术·卷 I》(Cay Horstmann) |
| `01-数据结构` | 《数据结构与算法分析:Java 语言描述》(Mark A. Weiss) |

| `05-git` | 《Pro Git》(Scott Chacon) |
| `06-前端` | 多本:JS《JavaScript 高级程序设计》+ HTML/CSS(MDN) + React(官方文档) |

## 三层结构

```
根 index.html  ──►  各模块「框架文件」 ──►  各节点「讲解文件」
 (总目录)          (以书为骨架的节点树)      (单知识点,据书 + notes 生成)
```

### 1. 框架文件(每模块一个)

- 命名统一 `<slug>-roadmap.html`,复用现有暗色视觉风格与树形/Tab 结构。
- **改造现有**:Java(`java-roadmap`)、设计模式(`design-patterns-roadmap`)、
  前端(`frontend-roadmap`)、React(`react-roadmap`)——在顶部新增「骨架书」banner,
  必要时按书章节微调分组。
- **新建**:数据结构(`ds-roadmap`)、并发(`concurrency-roadmap`)、
  工程实战(`engineering-roadmap`)、git(`git-roadmap`)。
- 顶部「骨架书」banner:书名 + 一句话定位 + 「这本书的脉络」简介。
- 节点树按书的章节 / Part 分组;每个叶子 `<a class="leaf">` 链到一个节点文件。
- 多书模块(04 / 06)在**每个分组标注来源书**。
- 遵循 CLAUDE.md:写完一个节点文件,就把对应 `<span class="leaf">` 改为
  `<a class="leaf" href="...">`。

### 2. 节点讲解文件

- 命名 `NN-节点名.html`,放在模块下的主题子目录(沿用现有 `06-前端/css/`、
  `03-设计模式/dp/`、`00-Java基础/core/` 等;新模块新建子目录)。
- 模板沿用现有叶子文件风格(`06-前端/css/06-盒模型.html`:
  breadcrumb / hero / code-block / callout / table / nav-footer)。
- hero 下标注来源:`📖 《Java 并发编程实战》第 2 章`。
- 正文严格按 CLAUDE.md:**讲透原理 → 具体例子 → 建立联系 → 点出陷阱 → 给出对比**;
  并发 / 设计模式类用「反例 → 问题 → 正例」结构。
- 已有 `notes/*.md` 作为生成素材复用,不凭空重写。
- nav-footer:上一篇(NN-1)/ 下一篇(NN+1)/ 回框架文件。

### 3. 根 index.html

- 根目录新建 `index.html`,卡片式列出 7 个模块,每张卡显示:
  模块名 + 骨架书 + 节点进度(如 3/24),点击进框架文件。
- 保留 `计算机抽象塔.html`,作为 index 里一个特殊的「全局地图」入口。
