IoC 核心与自动配置

主启动类与 @SpringBootApplication

一个 main 方法、一个注解，就拉起了整个容器、内嵌服务器和一堆自动配置好的 Bean。拆开这个"魔法注解"，看看它到底做了什么。

一句话讲清楚

@SpringBootApplication 不是一个独立的新注解，而是三个注解的组合注解（Meta-Annotation）。 Spring 允许把多个注解打包成一个，贴在类上等价于同时贴了这几个注解，这是一种"减少样板代码"的常见手法。

@SpringBootApplication 源码结构（简化）
@SpringBootConfiguration   // ① 本质是 @Configuration，声明这是一个配置类
@EnableAutoConfiguration   // ② 开启自动配置，见第 10 篇（本组重点）
@ComponentScan(excludeFilters = { ... }) // ③ 扫描组件
public @interface SpringBootApplication {}
组成部分作用
@SpringBootConfiguration标记这是一个配置类（内部就是 @Configuration），可以在里面写 @Bean 方法，见第 8 篇
@EnableAutoConfiguration触发自动配置机制，读取 classpath 下的自动配置清单并按条件生效，见第 10 篇
@ComponentScan扫描 @Component/@Service/@Repository/@Controller 等注解标注的类，注册为 Bean，见第 6 篇

SpringApplication.run() 做了什么

DemoApplication.java
@SpringBootApplication
public class DemoApplication {
    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}

这一行代码背后按顺序做了这些事：

① 推断应用类型（Web / 非 Web），创建对应的 ApplicationContext
② 加载 application.properties/yml 等外部化配置
③ 触发组件扫描 + 自动配置，完成 IoC 容器初始化（复用 04-工程实战讲过的三阶段：扫描 → 实例化注入 → 初始化回调）
④ 若是 Web 应用，启动内嵌 Tomcat 并绑定端口
⑤ 发布 ApplicationReadyEvent，应用正式对外可用

坑：组件扫描的默认范围

@ComponentScan 不写参数时，默认扫描范围是主启动类所在包及其所有子包。这意味着主启动类的包路径位置直接决定了哪些类能被扫描到。

典型的踩坑场景
// 主启动类在 com.example.demo 包下
package com.example.demo;
@SpringBootApplication
public class DemoApplication { ... }

// 但 Controller 被放在了平级的包，不是子包！
package com.other.web;
@RestController
public class UserController { // 永远不会被扫描到，访问接口 404 ... }
新手最常踩的坑：接口写完访问一直 404，检查半天代码逻辑没问题，最后发现是 @RestController 所在包不在主启动类的子包范围内，压根没被注册成 Bean。 团队约定：主启动类始终放在包结构的最顶层（如 com.company.project），所有业务代码放在它的子包下，从源头规避这个问题。

如果确实需要扫描包外的类，可以显式指定 @ComponentScan(basePackages = "com.other.web")，但这种写法应该是例外而不是常态——依赖显式配置来补救包结构设计上的缺陷，不如一开始就把包结构放对。

核心价值总结

  • @SpringBootApplication = @SpringBootConfiguration + @EnableAutoConfiguration + @ComponentScan 的组合注解。
  • SpringApplication.run() 完成容器创建、配置加载、组件扫描、自动配置、内嵌服务器启动的全流程。
  • 组件扫描默认范围是主启动类所在包及子包，把主启动类放在包结构顶层是团队协作的隐性约定。
  • 接口 404 但代码逻辑没问题时，第一反应应该是检查包结构是否在扫描范围内。
