# 线性表(散列) - 哈希表 (Hash Table)

### 1. **概念名称**

> **一句话核心概念总结:** 哈希表通过哈希函数把键映射到桶索引，在平均 O(1) 时间内完成查找、插入、删除，冲突则用链式/开放寻址等策略解决。

### 2. **详细解释**

- **哈希函数 (Hash Function):** 将任意键转换为整数，再对桶数组长度取模得到索引；好的哈希函数应分布均匀、计算快，减少碰撞。
- **桶数组 + 冲突解决:** 底层是数组。若多个键落在同一索引，需要处理冲突。常见方法：
  - **链地址法 (Separate Chaining):** 桶中存一个小链表/树，新增元素挂到链表头/尾。
  - **开放寻址 (Open Addressing):** 冲突时在数组内寻找下一个空位（线性探测、二次探测、双重哈希等）。
- **负载因子 (Load Factor):** 已存元素个数 / 桶数。过大碰撞多，过小浪费空间。典型阈值 0.75，超过则扩容并重哈希。
- **时间复杂度:** 理想情况下平均 O(1) 查/增/删；最坏（哈希劣化或负载过高）可退化到 O(n)。
- **有序性:** 哈希表不保证键的顺序；如需按插入/排序遍历，需额外结构（例如 LinkedHashMap、TreeMap）。

### 3. **代码示例**

```java
import java.util.Arrays;

/**
 * 用链地址法实现的简易整型键 -> 字符串值哈希表
 */
public class SimpleHashTable {
    private static class Node {
        final int key;
        String value;
        Node next;
        Node(int key, String value) {
            this.key = key;
            this.value = value;
        }
    }

    private Node[] buckets;
    private int size;
    private static final double LOAD_FACTOR = 0.75;

    public SimpleHashTable(int capacity) {
        buckets = new Node[Math.max(4, capacity)];
    }

    private int index(int key) {
        return Math.floorMod(key, buckets.length);
    }

    public String get(int key) {
        Node cur = buckets[index(key)];
        while (cur != null) {
            if (cur.key == key) return cur.value;
            cur = cur.next;
        }
        return null;
    }

    public void put(int key, String value) {
        if ((size + 1.0) / buckets.length > LOAD_FACTOR) {
            resize();
        }
        int idx = index(key);
        Node cur = buckets[idx];
        while (cur != null) {          // 已存在则覆盖
            if (cur.key == key) {
                cur.value = value;
                return;
            }
            cur = cur.next;
        }
        Node newNode = new Node(key, value);
        newNode.next = buckets[idx];   // 头插法挂到链表
        buckets[idx] = newNode;
        size++;
    }

    public boolean remove(int key) {
        int idx = index(key);
        Node cur = buckets[idx], prev = null;
        while (cur != null) {
            if (cur.key == key) {
                if (prev == null) buckets[idx] = cur.next;
                else prev.next = cur.next;
                size--;
                return true;
            }
            prev = cur;
            cur = cur.next;
        }
        return false;
    }

    private void resize() {
        Node[] old = buckets;
        buckets = new Node[old.length << 1];
        size = 0;
        for (Node head : old) {
            Node cur = head;
            while (cur != null) {      // 重新分布到新桶
                put(cur.key, cur.value);
                cur = cur.next;
            }
        }
    }

    public static void main(String[] args) {
        SimpleHashTable table = new SimpleHashTable(4);
        table.put(1, "Alice");
        table.put(5, "Bob");   // 与 1 可能同槽，考验链表
        table.put(9, "Cindy"); // 触发扩容

        System.out.println(table.get(5)); // 输出 Bob
        table.remove(1);
        System.out.println(table.get(1)); // 输出 null
    }
}
```

### 4. **常见应用场景**

- **键值存储/配置表/缓存元数据：** 通过键快速定位值。
- **去重/判重：** 利用哈希集合检测元素是否出现过（如布隆过滤器扩展）。
- **计数与频率统计：** 词频、PV/UV、热点排行榜前置计数。
- **LRU/LFU 缓存：** 结合双向链表维护顺序，哈希表提供 O(1) 查询与定位。
- **编译器/解释器符号表：** 快速查变量/函数符号。

### 5. **优缺点/注意事项 (可选)**

- **优点:**
  - 平均 O(1) 查/增/删，常用场景高效。
  - 按需扩容，容量可动态增长。
- **缺点/注意事项:**
  - 依赖哈希函数质量，碰撞多会退化。
  - 最坏时间复杂度 O(n)，需控制负载因子。
  - 不保序，遍历顺序不确定；需有序请用 LinkedHashMap/TreeMap 等。
  - 扩容需要重哈希，存在瞬时开销；并发场景需线程安全实现。
