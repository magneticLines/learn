拓扑排序 Topological Sort

给一堆有依赖关系的任务排出一个不违反依赖的执行顺序——这是"先修课程"问题、构建工具依赖解析问题的通用解法。

📖 来源:《数据结构与算法分析:Java语言描述》 第 9 章「图论算法」

一、它解决的问题

现实中大量任务之间存在"必须先做 A 才能做 B"的依赖关系,把这种依赖画成一张有向图(A → B 表示 A 是 B 的前提),拓扑排序要回答的问题就是:能不能找到一个顶点的线性顺序,使得对每条边 A → B,A 都排在 B 前面?

  • 课程先修关系:学"数据结构"before "算法分析",拓扑排序给出一个合法的选课顺序。
  • 任务调度:CI/CD 流水线里,"编译"必须在"测试"之前,"测试"必须在"部署"之前。
  • Maven / Gradle 依赖解析:模块 A 依赖模块 B,构建工具要先构建 B 再构建 A——这正是拓扑排序在真实工程里最常见的落地场景。

二、基于入度的算法思路

入度(in-degree)指"有多少条边指向这个顶点",也就是"这个顶点还有多少个未完成的前提任务"。算法的核心直觉很朴素:入度为 0 的顶点,就是当前没有任何未完成前提的任务,可以立即执行。

  1. 统计每个顶点的入度,把所有入度为 0 的顶点放入队列。
  2. 从队列取出一个顶点,输出它(代表"执行这个任务")。
  3. 把它的所有出边"移除"——即把每条出边指向的顶点的入度减 1。
  4. 如果某个顶点的入度因此减到 0,说明它的前提任务全部完成,加入队列。
  5. 重复 2-4,直到队列为空。

走一遍具体例子

假设课程依赖:A→B、A→C、B→D、C→D(学 A 才能学 B/C,学完 B 和 C 才能学 D)。

手动模拟
// 初始入度: A=0, B=1, C=1, D=2
// 第1步:A入度为0,输出A;移除A的出边(A->B, A->C)
//        B入度 1-1=0,C入度 1-1=0,B、C入队
// 第2步:输出B(或C,顺序不唯一);移除B->D,D入度 2-1=1
// 第3步:输出C;移除C->D,D入度 1-1=0,D入队
// 第4步:输出D
// 最终顺序: A -> B -> C -> D (或 A -> C -> B -> D)
Java · 基于入度的 BFS 实现
List<Integer> topoSort(int n, Map<Integer, List<Integer>> graph) {
    int[] indegree = new int[n];
    for (List<Integer> nbrs : graph.values())
        for (int v : nbrs) indegree[v]++; // 统计入度

    Queue<Integer> queue = new LinkedList<>();
    for (int i = 0; i < n; i++)
        if (indegree[i] == 0) queue.add(i); // 无前提的任务先入队

    List<Integer> order = new ArrayList<>();
    while (!queue.isEmpty()) {
        int cur = queue.poll();
        order.add(cur);
        for (int next : graph.getOrDefault(cur, List.of())) {
            if (--indegree[next] == 0) queue.add(next); // 前提全部完成,可以入队
        }
    }
    return order.size() == n ? order : List.of(); // 数量不够说明有环,无解
}

三、为什么有环图不存在拓扑排序

假设存在环 A→B→C→A:要排 A 就必须先排 C(因为 C→A),要排 C 就必须先排 B,要排 B 又必须先排 A——形成了"你等我、我等你"的死循环,没有任何一个顶点能真正做到"前提全部满足"。反映到算法上,环上的顶点入度永远无法减到 0,它们永远进不了队列。

联系死锁检测:操作系统里检测死锁的经典方法之一就是画出"资源分配图",判断图中是否存在环——如果 A 等 B 持有的资源、B 又在等 A 持有的资源,就是一个环,死锁发生。拓扑排序"跑不完就说明有环"这个副产品,正好可以拿来检测有向图中是否存在环,这是它除了排序本身之外最常被考到的应用。
面试高频:"如何判断一个有向图是否有环?" 标准答案之一就是跑一遍基于入度的拓扑排序,如果最终输出的顶点数少于总顶点数,就说明有环(环上的顶点入度永远不会归零)。

小结

  • 拓扑排序给有依赖关系的任务排出一个不违反依赖的线性顺序,只对有向无环图(DAG)有意义。
  • 核心算法:反复取出入度为 0 的顶点输出,并移除它的出边(让下游顶点入度减 1)。
  • 有环图不存在拓扑排序——环上顶点入度无法归零,这也是检测图中是否有环的经典方法。
