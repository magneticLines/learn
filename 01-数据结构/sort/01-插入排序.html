插入排序 Insertion Sort

最朴素的排序算法,却是理解"稳定性"和"自适应性"的最佳起点,也是很多工业级排序算法收尾时的秘密武器。

📖 来源:《数据结构与算法分析:Java语言描述》 第 7 章「排序」

一、思路:像摸牌一样理牌

打牌时你不会把所有牌堆在一起再统一排序,而是摸到一张新牌,就把它插到手里已经排好序的牌中的正确位置。插入排序做的就是这件事:把数组分成"已排好序的前缀"和"还没处理的后缀",每一轮从后缀里摸一张牌(下一个元素),往前缀里找到合适位置插进去。

这解决的核心问题是:如何用最少的额外结构,把"局部有序"逐步扩展为"全局有序"。它不需要额外数组,只需要在原地不断把元素往后挪腾出空位,这就是它空间复杂度 O(1) 的来源。

二、走一遍具体例子

数组 [5, 2, 4, 6, 1, 3],下标 0 视为已排好序的前缀(只有一个元素天然有序),从下标 1 开始摸牌:

i=1,摸到 2:与前面的 5 比,2<5,5 后移,2 插到最前 → [2, 5, 4, 6, 1, 3]
i=2,摸到 4:与 5 比,4<5,5 后移;与 2 比,4>2,停 → [2, 4, 5, 6, 1, 3]
i=3,摸到 6:与 5 比,6>5,不用动 → [2, 4, 5, 6, 1, 3]
i=4,摸到 1:比 6、5、4、2 都小,全部后移,1 插到最前 → [1, 2, 4, 5, 6, 3]
i=5,摸到 3:比 6、5、4 小,比 2 大,插到 2 和 4 之间 → [1, 2, 3, 4, 5, 6]

可以看到:每一轮只需要跟前缀里比自己大的元素比较并挪位,一旦遇到比自己小的就停手——这正是它对"几乎有序数据"高效的原因。

三、代码实现

Java · 插入排序
public static void insertionSort(int[] a) {
    for (int i = 1; i < a.length; i++) {
        // tmp 是这一轮要"插进去"的新牌,先摘出来腾位置
        int tmp = a[i];
        int j = i;
        // 只要前一个元素比新牌大,就把它挪到后面腾位置——这是"自适应"的关键
        while (j > 0 && a[j - 1] > tmp) {
            a[j] = a[j - 1];
            j--;
        }
        a[j] = tmp; // 找到坑位,新牌落地
    }
}

四、复杂度:最坏 O(n²),但对近乎有序数据接近 O(n)

最坏情况(完全逆序):每插入一个元素都要跟前面所有元素比较、挪位,总比较+移动次数是 1+2+...+(n-1) ≈ n²/2,所以是 O(n²)。

但如果数据已经接近有序(比如只有少数几个元素错位),内层 while 循环几乎不会执行,每轮只需 O(1) 就能确定位置,整体退化为接近 O(n)。这个"自适应性"是插入排序最大的亮点,也是后面希尔排序的立足点——希尔排序本质上就是"先用大步长把数据变得近乎有序,再用插入排序收尾"。

它还是稳定排序:代码里的判断条件是 a[j-1] > tmp(严格大于才移动),遇到相等元素不会跨越,原始的相对顺序被保留。

五、联系:Java 的 Arrays.sort 混合策略

你可能觉得插入排序太"弱",但它其实活跃在 JDK 源码的关键位置。java.util.Arrays.sort(int[]) 底层使用的是双轴快速排序(见后续「快速排序」一篇),但当子数组长度小于某个阈值(JDK 中通常是 47)时,会直接切换成插入排序,而不是继续递归快排。

为什么这么设计? 快排的递归、分区在小数组上的"常数开销"(函数调用、边界判断)反而比插入排序的简单双重循环更贵。插入排序在小数组上没有递归开销、缓存局部性好、代码分支少,实测比继续递归快排更快。这是"渐进复杂度"和"真实机器上的常数因子"之间取舍的经典案例——O(n²) 在 n 很小时可能比 O(n log n) 更快。

六、易错点

误区 1:以为插入排序一定慢。 它的时间复杂度确实是 O(n²),但常数因子极小、对小规模/近乎有序数据表现优异,不能一概而论地说它"没用"。
误区 2:混淆"插入排序"和"选择排序"。 选择排序是"每轮从未排序区找最小值放到前面",比较次数固定为 O(n²) 且不具备自适应性;插入排序的比较次数会随数据有序程度浮动。二者都是 O(n²)、都原地,但插入排序更"聪明"。

对比表

算法最坏时间空间稳定性特点
插入排序O(n²)O(1)稳定对近乎有序数据接近 O(n),小数组常数因子小

小结

  • 插入排序 = 把新元素插入已排好序的前缀,原地操作,空间 O(1)。
  • 最坏 O(n²),但对近乎有序的数据接近 O(n)——这是它的核心价值。
  • 稳定排序(相等元素不跨越)。
  • JDK Arrays.sort 在快排递归到小数组(阈值约 47)时会切换成插入排序收尾。
