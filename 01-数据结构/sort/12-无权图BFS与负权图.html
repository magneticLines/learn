最短路径 · 无权图 BFS / 负权图

最短路径没有唯一算法——图的性质不同,该用的工具也不同:无权图用 BFS,含负权边用 Bellman-Ford。

📖 来源:《数据结构与算法分析:Java语言描述》 第 9 章「图论算法」

一、无权图:BFS 比 Dijkstra 更简单更快

如果所有边的权重都相等(无权图,或者说每条边权重都算作 1),Dijkstra 那套"贪心 + 堆"就显得杀鸡用牛刀了。因为广度优先搜索(BFS)天然按"离起点的边数"一层一层扩展——第一次访问到某个顶点时,走过的边数就一定是最少的,因为 BFS 不可能绕远路先到达更远的层。

Java · BFS 求无权图最短路径
int[] bfsShortest(int n, Map<Integer, List<Integer>> graph, int src) {
    int[] dist = new int[n];
    Arrays.fill(dist, -1); // -1 表示还没访问过
    dist[src] = 0;
    Queue<Integer> queue = new LinkedList<>();
    queue.add(src);
    while (!queue.isEmpty()) {
        int u = queue.poll();
        for (int v : graph.getOrDefault(u, List.of())) {
            if (dist[v] == -1) { // 第一次到达就是最短,不需要像Dijkstra那样反复松弛
                dist[v] = dist[u] + 1;
                queue.add(v);
            }
        }
    }
    return dist;
}

没有堆的 O(log V) 开销,复杂度直接是 O(V+E)——BFS 是 Dijkstra 在"所有边权相等"这个特殊场景下的简化版。

二、负权图:Bellman-Ford

上一篇说过 Dijkstra 处理不了负权边,因为它"确定即不变"的贪心假设被打破了。Bellman-Ford 换了一种更笨但更稳的思路:不贪心,而是暴力松弛——对图中所有边反复做松弛操作,松弛足够多轮之后,所有最短距离必然收敛到正确值。

  • 对每条边 u→v 执行:如果 dist[u] + weight(u,v) < dist[v],更新 dist[v]。
  • 把这个过程对所有边重复 V-1 轮。为什么是 V-1 轮?因为最短路径最多经过 V-1 条边(不会重复经过同一个顶点),每一轮松弛至少能把"最短路径的下一段"确定下来。
  • 检测负权环:如果做完 V-1 轮后,再做第 V 轮松弛仍然有顶点的距离在变小,说明图中存在负权环(环上的距离可以无限变小,根本不存在"最短路径")。

代价是复杂度变成了 O(V·E),比 Dijkstra 的 O((V+E) log V) 慢很多——这是"能处理负权边"换来的代价,没有免费的午餐。

三、三种场景怎么选

场景推荐算法复杂度核心原因
无权图(所有边权相等)BFSO(V+E)按层扩展,第一次到达即最短,不需要贪心比较
非负权图DijkstraO((V+E) log V)贪心 + 堆,已确定的最短距离不会再变,效率最高
含负权边(无负权环)Bellman-FordO(V·E)暴力松弛 V-1 轮,不依赖"贪心即最优"的假设
选择原则:先看图的性质再选算法——能用简单的就不用复杂的。权重全相等用 BFS;权重非负(现实中距离、时间、成本大多非负)用 Dijkstra;明确知道存在负权边(比如金融套利、某些差分约束问题)才上 Bellman-Ford。

小结

  • 无权图求最短路径,直接用 BFS 即可,比 Dijkstra 更简单更快。
  • Bellman-Ford 用"暴力松弛 V-1 轮"处理负权边,还能顺带检测负权环。
  • 三种场景对应三种算法:BFS / Dijkstra / Bellman-Ford,按图的权重特性选择,不要一律套用最复杂的方案。
