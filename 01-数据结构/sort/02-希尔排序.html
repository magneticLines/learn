希尔排序 Shell Sort 与增量序列

插入排序的改进版:先让元素"大跨步"移动到大致正确的位置,再用插入排序精修——用增量序列换来了远低于 O(n²) 的实际性能。

📖 来源:《数据结构与算法分析:Java语言描述》 第 7 章「排序」

一、它在解决插入排序的什么痛点

上一篇讲到,插入排序的代价来自"元素每次只能挪一个位置"。如果一个很小的元素恰好排在数组最后面,它必须经过 n-1 次相邻交换才能挪到最前面——这种"小元素"在 Weiss 书里被称为乌龟(turtle),是插入排序退化为 O(n²) 的根源。

希尔排序的核心思路是:先用较大的间隔(增量)对元素分组,组内做插入排序,让元素能一次跳跃很远的距离到达大致正确的位置;然后逐渐缩小间隔,直到间隔为 1。当间隔缩小到 1 时,数据已经"几乎有序",插入排序此时的表现接近 O(n)——这正是利用了上一篇讲的插入排序的自适应特性。

二、走一遍具体例子

数组 [9, 1, 6, 8, 3, 7, 2, 4],用增量序列 4, 2, 1(每次减半):

增量 4:分组 (下标0,4)=(9,3)、(1,5)=(1,7)、(2,6)=(6,2)、(3,7)=(8,4),各组内插入排序 → [3, 1, 2, 4, 9, 7, 6, 8]
增量 2:分组 (0,2,4,6)=(3,2,9,6)、(1,3,5,7)=(1,4,7,8),组内插入排序 → [2, 1, 3, 4, 6, 7, 9, 8]
增量 1:退化为普通插入排序,此时数据已接近有序,只需少量挪动 → [1, 2, 3, 4, 6, 7, 8, 9]

关键观察:增量 4、2 阶段虽然"看起来没排好",但它们把原本相距很远的错位元素(比如开头的 9、结尾的 2)快速拉近到正确的大致区域,减少了增量为 1 时插入排序需要挪动的距离。

三、代码实现(希尔增量:n/2, n/4, ..., 1)

Java · 希尔排序
public static void shellSort(int[] a) {
    int n = a.length;
    // 增量每轮减半,直到 1——1 就是普通插入排序,负责最后精修
    for (int gap = n / 2; gap > 0; gap /= 2) {
        // 对间隔为 gap 的每个分组做"插入排序",本质就是插入排序把步长从 1 换成 gap
        for (int i = gap; i < n; i++) {
            int tmp = a[i];
            int j = i;
            while (j >= gap && a[j - gap] > tmp) {
                a[j] = a[j - gap];
                j -= gap;
            }
            a[j] = tmp;
        }
    }
}

四、增量序列决定了复杂度

希尔排序的性能高度依赖增量序列的选择,这也是它区别于其他排序算法的独特之处——算法框架相同,换一套数字就能显著改变复杂度:

增量序列提出者最坏时间复杂度说明
n/2, n/4, ..., 1Shell(原始)O(n²)简单但效果一般,偶数增量之间互不独立,容易抵消前面的排序效果
2^k - 1HibbardO(n^1.5)相邻增量互质,避免了 Shell 增量的抵消问题
Sedgewick 增量SedgewickO(n^4/3)实践中表现更好,是更精细的数学构造

Weiss 书中用 Hibbard 增量证明了 O(n^1.5) 的上界,核心原因是:如果增量序列中相邻的两个数互质,那么用较小增量排序时不会破坏较大增量已经建立起来的有序性;而 Shell 原始的"减半"序列全是偶数关系,后面的分组会与前面重叠,效果打折。

五、陷阱:希尔排序不稳定

希尔排序不是稳定排序。 原因是:元素在大增量分组阶段可能跨越很远的距离移动,与它"跳过"的那些相同 key 的元素的相对顺序被打乱了。举例:[3a, 3b, 1](下标0的3标记为a,下标1的3标记为b),增量为 2 时会先比较 3a 和 1,1 更小被挪到前面,而 3b 没有参与这一轮——最终排序结果中 3a 和 3b 的相对顺序可能与原始顺序不一致。这是面试常考的对比点:插入排序稳定,但它的改进版希尔排序反而不稳定,因为"跳跃式比较"破坏了稳定性所依赖的"只跟相邻元素比较、不跨越"的前提。

对比表

算法最坏时间空间稳定性特点
插入排序O(n²)O(1)稳定只能一步步挪,远距离错位代价高
希尔排序(Hibbard)O(n^1.5)O(1)不稳定先跳跃式粗排,再插入排序精修

小结

  • 希尔排序 = 分治思想 + 插入排序:先用大间隔粗排,逐步缩小间隔到 1。
  • 增量序列直接决定复杂度:朴素减半是 O(n²),Hibbard 增量是 O(n^1.5)。
  • 核心陷阱:不稳定,因为元素会跨分组跳跃移动。
  • 是"用空间换时间"之外的另一种优化思路——用"移动距离"换时间。
