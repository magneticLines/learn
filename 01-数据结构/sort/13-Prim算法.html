最小生成树 · Prim 算法

用最小的总代价连通所有顶点——网络布线、道路规划里"怎么花最少钱把所有点连起来"的通用解法。

📖 来源:《数据结构与算法分析:Java语言描述》 第 9 章「图论算法」

一、最小生成树解决的问题

生成树(Spanning Tree)是"用最少的边(V-1 条)把图中所有顶点连通、且不产生环"的一个子图。如果图是带权的,最小生成树(MST)就是所有生成树里总权重最小的那一个。

典型场景:电力公司要把 N 个城镇连上电网,每两个城镇之间架线的成本不同,要求用最少的总成本让所有城镇都通电——不需要任意两点直连,只需要整体连通。这正是 MST 的应用:网络布线、道路规划、管道设计,本质都是"连通所有点,总代价最小"。

二、Prim 算法的贪心思路

Prim 的思路和"滚雪球"很像:从一个顶点开始,维护一个"已选顶点集合",每次贪心地在"连接已选集合和外部顶点"的所有边里,挑权重最小的那条加入,把新顶点纳入集合,直到所有顶点都被纳入。

  1. 任选一个顶点作为起点,加入"已选集合"。
  2. 找出所有"一端在已选集合内、另一端在集合外"的边,选权重最小的一条。
  3. 把这条边和它对应的新顶点加入已选集合。
  4. 重复 2-3,直到所有顶点都被纳入。

走一遍具体例子

4个顶点 A/B/C/D,边:A-B(4)、A-C(1)、B-C(2)、B-D(5)、C-D(3)。

手动模拟
// 已选={A}。候选边(连已选和外部): A-B(4), A-C(1)
// 选最小 A-C(1),加入C。已选={A,C}
// 候选边: A-B(4), C-B(2), C-D(3)
// 选最小 C-B(2),加入B。已选={A,B,C}
// 候选边: B-D(5), C-D(3)
// 选最小 C-D(3),加入D。已选={A,B,C,D},完成
// MST总权重 = 1+2+3 = 6
Java · 堆优化版 Prim(和Dijkstra框架几乎一样)
int primMST(int n, Map<Integer, List<Edge>> graph) {
    boolean[] inMST = new boolean[n];
    PriorityQueue<Edge> heap = new PriorityQueue<>((a, b) -> a.weight - b.weight);
    heap.add(new Edge(0, 0)); // 从顶点0出发,虚拟一条权重0的边把它送进堆
    int total = 0, count = 0;
    while (!heap.isEmpty() && count < n) {
        Edge e = heap.poll();
        if (inMST[e.to]) continue; // 已在集合内,这条候选边过期了
        inMST[e.to] = true;
        total += e.weight; count++;
        for (Edge next : graph.getOrDefault(e.to, List.of()))
            if (!inMST[next.to]) heap.add(next); // 新顶点的所有出边都是新的候选边
    }
    return total;
}

三、容易混淆的陷阱:Prim 不是 Dijkstra

Prim 和 Dijkstra 的代码框架几乎一模一样——都是"贪心 + 堆,每次弹出权重最小的候选、标记已确定、扩展候选边",这也是初学者最容易搞混的地方。但它们维护的物理意义完全不同:

核心区别:Dijkstra 的 dist[v] 表示"顶点 v 到起点 s 的最短距离"(必须是从起点出发累加的路径长度);Prim 的候选权重表示"某条边本身的权重,即把顶点 v 拉进已选集合需要付出的直接代价"——不需要关心它离起点多远,只关心这一步这条边多贵。这就是为什么 Prim 求的是"总代价最小的连通方式"(MST),而 Dijkstra 求的是"到起点的最短路径"(最短路径树),两者的树可能完全不同。

小结

  • 最小生成树:用总权重最小的边集,把所有顶点连通且不成环,应用于网络布线/道路规划等成本最小化场景。
  • Prim:从一点出发,每次贪心加入"连接已选集合与外部"的最小权重边,直到覆盖所有顶点。
  • 易混淆陷阱:Prim 维护"到已选集合的代价",Dijkstra 维护"到起点的距离"——框架相似,含义不同。

与其他知识点的联系

  • Dijkstra 算法:前面已经点出两者代码框架几乎一致,这不是巧合——都属于"贪心 + 堆维护候选集合最小值"的算法范式,区别只在于堆里存的权重语义不同(到起点的累积距离 vs 单条边的直接代价)。理解这一层"框架可复用、语义要分清"的关系,是图论贪心算法举一反三的关键。
  • 堆 / 优先队列:Prim 能把复杂度从暴力扫描的 O(V²) 降到堆优化的 O(E log V),靠的仍然是堆"反复取最小值"的特性——这是堆在图论算法里第二次以同样的方式发挥作用(第一次是 Dijkstra),说明"需要不断从动态集合里取极值"是堆的标志性应用场景。
  • 贪心算法的正确性证明(切割性质):Prim 的正确性依赖图论里的"切割性质(Cut Property)"——对图的任意一种顶点二分割,横跨两部分的最小权重边必然属于某个最小生成树。这是证明"局部贪心选择等于全局最优"的标准工具,和 Dijkstra 依赖的"边权非负保证已确定顶点不再变"是两种不同的贪心正确性证明思路,面试深挖 MST 算法原理时常会问到。
  • Kruskal 算法(下一篇):同样求最小生成树,但视角完全相反——Prim 从顶点扩张,Kruskal 从边的全局排序切入,是"同一个问题,两种不同贪心切入角度"的经典对照,类似归并排序（分治）和插入排序（增量式）解决同一个排序问题但思路不同。
