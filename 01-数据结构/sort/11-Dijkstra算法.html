最短路径 · Dijkstra 算法

解决"从一个起点到所有其他顶点的最短距离"——贪心思路 + 堆,是路由、导航类系统的核心算法。

📖 来源:《数据结构与算法分析:Java语言描述》 第 9 章「图论算法」

一、它解决的问题:单源最短路径

给定一个带权图和一个起点 s,求 s 到其他所有顶点的最短距离——这就是单源最短路径(Single-Source Shortest Path)问题。地图导航"从家到公司怎么走最快"、网络路由"数据包走哪条链路延迟最小",本质都是这个问题。

二、贪心思路

Dijkstra 的核心直觉:维护一个"已确定最短距离"的集合,每次从"未确定"的顶点里,挑出当前距离起点最近的那个,把它标记为已确定。为什么这样贪心是对的?因为图中边权非负,所以"当前离起点最近的未确定顶点",不可能再通过其他未确定的(更远的)顶点绕路变得更近——它已经是全局最优解了。

  1. 初始化:起点距离为 0,其余顶点距离为 ∞。
  2. 从未确定的顶点里,选出距离最小的顶点 u,标记为已确定。
  3. 松弛(relax):检查 u 的每条出边 u→v,如果 dist[u] + weight(u,v) < dist[v],就更新 dist[v]。
  4. 重复 2-3,直到所有顶点都被确定。

"每次选距离最小的未确定顶点"这个操作,如果每次线性扫描是 O(V),整体是 O(V²);如果换成优先队列(堆)维护"当前候选距离最小的顶点",取最小值降到 O(log V),整体优化到 O((V+E) log V)——这正好用上了之前学过的堆这种数据结构:堆本来就是为"反复取最小值"这个操作量身定做的。

走一遍具体例子

图:A→B(4)、A→C(1)、C→B(2)、B→D(1)、C→D(5),起点 A。

手动模拟
// 初始: dist = {A:0, B:∞, C:∞, D:∞}
// 第1轮:未确定中最小是A(0),确定A。松弛A的出边:
//        dist[B]=4, dist[C]=1
// 第2轮:未确定中最小是C(1),确定C。松弛C的出边:
//        dist[B] = min(4, 1+2=3) = 3  (发现走A->C->B更近!)
//        dist[D] = min(∞, 1+5=6) = 6
// 第3轮:未确定中最小是B(3),确定B。松弛B的出边:
//        dist[D] = min(6, 3+1=4) = 4  (走A->C->B->D比A->C->D更近)
// 第4轮:确定D(4)。结束。
// 最终: A到B最短距离3,A到C最短距离1,A到D最短距离4
Java · 堆优化版 Dijkstra
int[] dijkstra(int n, Map<Integer, List<Edge>> graph, int src) {
    int[] dist = new int[n];
    Arrays.fill(dist, Integer.MAX_VALUE);
    dist[src] = 0;
    // 堆按距离排序,保证每次弹出的是"当前候选中最近"的顶点
    PriorityQueue<int[]> heap = new PriorityQueue<>((a, b) -> a[1] - b[1]);
    heap.add(new int[]{src, 0});
    boolean[] done = new boolean[n];
    while (!heap.isEmpty()) {
        int[] cur = heap.poll();
        int u = cur[0];
        if (done[u]) continue; // 堆里可能有同一顶点的多条过期记录,跳过
        done[u] = true;
        for (Edge e : graph.getOrDefault(u, List.of())) {
            if (dist[u] + e.weight < dist[e.to]) { // 松弛:发现更短路径就更新
                dist[e.to] = dist[u] + e.weight;
                heap.add(new int[]{e.to, dist[e.to]});
            }
        }
    }
    return dist;
}

三、关键陷阱:不能处理负权边

Dijkstra 贪心成立的前提是:一旦一个顶点被标记为"已确定",它的最短距离就再也不会变了。这个前提依赖于"边权非负"——如果存在负权边,一个已经被确定的顶点,可能在后续被一条"负权边"绕过来,反而找到更短的路径,但算法已经不会再更新它了,导致结果错误。

反例:负权边让 Dijkstra 出错
// A->B(4), A->C(5), B->C(-3)
// Dijkstra: 先确定A(0)。松弛得 dist[B]=4, dist[C]=5
// 下一步选最小的B(4),确定B。松弛 B->C: dist[C] = min(5, 4-3=1) = 1
// 看似正确,但如果顺序是 dist[C]=5 先被"确定"(比如换一张更复杂的图),
// 之后才发现经过负权边能让C变得更短,C已经被标记为done,不会再更新——结果错误
面试高频陷阱:"Dijkstra 能不能处理负权边?"——不能。根本原因是贪心策略的正确性依赖"最短距离一旦确定就不会再变小",负权边直接破坏了这个不变量。处理带负权边的图需要 Bellman-Ford 之类的算法(见下一篇)。

小结

  • Dijkstra 用贪心策略解决非负权图的单源最短路径问题。
  • 核心操作是"取当前未确定顶点中距离最小的一个",用堆优化可以把复杂度降到 O((V+E) log V)。
  • 致命陷阱:负权边会破坏贪心的正确性前提,不能用 Dijkstra 处理含负权边的图。

与其他知识点的联系

  • 堆 / 优先队列:Dijkstra "反复取当前最小值"这个操作,正是堆这种数据结构存在的意义——之前学堆时讲的"插入 O(log n)、取最小 O(log n)"在这里直接决定了算法从 O(V²) 降到 O((V+E) log V),是"数据结构选型直接决定算法复杂度"的典型例子。
  • 贪心算法:Dijkstra 是贪心算法能给出全局最优解的经典案例——贪心不总是对的(比如 0-1 背包贪心会错),Dijkstra 之所以贪心可行,是因为"边权非负"保证了局部最优(当前最近的未确定顶点)不会被后续操作反悔。这也是判断"某问题能不能用贪心"的通用套路:先找有没有类似的"无后效性"保证。
  • Bellman-Ford(下一篇):两者解决同一个问题(单源最短路径),但 Bellman-Ford 放弃贪心、改用暴力松弛换取"能处理负权边"的能力——是同一问题在不同约束条件下的两种解法,面试常问"什么时候用 Dijkstra、什么时候用 Bellman-Ford",答案就是看图里有没有负权边。
  • 动态规划的松弛思想:"松弛"操作(if dist[u]+w < dist[v] then 更新)本质上是动态规划里"用已知的更优子结构更新当前状态"的思路,和最长公共子序列、编辑距离等 DP 问题"用子问题最优解推导当前解"是同一类思维方式,只是 Dijkstra 靠贪心顺序保证了每个状态只需要被最终更新一次。
  • 拓扑排序(上一篇):如果图是有向无环图(DAG),求单源最短路径甚至不需要 Dijkstra——按拓扑序处理顶点,每个顶点被处理时其所有前驱都已确定最短距离,一次线性扫描即可得到结果,比 Dijkstra 更快。这说明"图的额外结构信息(无环)"往往能替代通用算法换来更优的复杂度。
