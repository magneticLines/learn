排序与图

最小生成树 · Kruskal 算法

换一个角度求最小生成树:不从顶点出发扩张,而是直接对边排序、贪心挑选——配合并查集判环,是稀疏图场景下的最优解法。

📖 来源:《数据结构与算法分析:Java语言描述》 第 9 章「图论算法」

一、Kruskal 的贪心思路

Prim 是"从顶点视角"扩张——盯着已选集合,一点点往外滚雪球。Kruskal 换了个视角,直接从边下手:把图中所有边按权重从小到大排序,依次考察每条边,只要加入它不会形成环,就加入生成树,直到选够 V-1 条边为止。

  1. 把所有边按权重升序排序。
  2. 依次取出权重最小的边 (u, v):如果 u 和 v 当前不在同一个连通分量里,就加入这条边(不会成环);如果已经在同一个连通分量里,跳过(加入会成环)。
  3. 重复步骤 2,直到已选边数达到 V-1。

走一遍具体例子

沿用上一篇的图:4个顶点 A/B/C/D,边:A-C(1)、B-C(2)、C-D(3)、A-B(4)、B-D(5)。按权重排好序后依次处理:

二、如何快速判断"是否形成环"

判断"加入这条边会不会形成环",本质是判断"u 和 v 是否已经连通(在同一个连通分量里)"。如果每次都用 DFS/BFS 去查连通性,太慢了——这正是并查集(Union-Find)被发明出来解决的问题:它专门用来快速回答"两个元素是否属于同一组"和"把两个组合并"。

  • find(x):查询 x 所在集合的"代表元素"(根节点),两个元素的根节点相同就说明它们已经连通。
  • union(x, y):把 x 和 y 所在的两个集合合并成一个(把一个的根节点接到另一个的根节点下面)。
  • 配合路径压缩(查找时把沿途节点直接接到根上)和按秩合并,两个操作均摊后接近 O(1)。
Java · Kruskal + 并查集
int kruskalMST(int n, List<int[]> edges) { // edges[i] = {u, v, weight}
    edges.sort((a, b) -> a[2] - b[2]); // 按权重升序,贪心的前提
    int[] parent = new int[n];
    for (int i = 0; i < n; i++) parent[i] = i; // 初始每个点自己是一个集合

    int total = 0, used = 0;
    for (int[] e : edges) {
        int ru = find(parent, e[0]), rv = find(parent, e[1]);
        if (ru != rv) { // 根不同 = 不在同一分量 = 加入不会成环
            parent[ru] = rv; // 合并两个集合
            total += e[2];
            if (++used == n - 1) break; // 已凑够V-1条边,生成树完成
        }
    }
    return total;
}
int find(int[] parent, int x) {
    while (parent[x] != x) x = parent[x]; // 沿链找到根节点
    return x;
}

三、Kruskal vs Prim

维度PrimKruskal
视角从顶点出发,滚雪球式扩张从边出发,全局排序后贪心挑选
核心数据结构堆(优先队列)排序 + 并查集
复杂度O(E log V)(堆优化)O(E log E)(排序主导)
适合场景稠密图:边多,按顶点扩张更省事稀疏图:边少,排序成本低,并查集判环很快
实现直觉和 Dijkstra 框架相似,易混淆两者含义和"贪心 + 排序"套路一致,判环逻辑独立清晰
选择原则:图很稠密(边接近 V²)时选 Prim,顶点扩张的开销分摊到每条边上更划算;图很稀疏(边远小于 V²)时选 Kruskal,排序和并查集的开销都只跟边数相关,天然更省。

整个「数据结构与算法」路线图的总结

回顾整条学习脉络:我们从算法分析开始,学的是一门"衡量复杂度的语言"——大 O 记号让我们能在不跑代码的前提下,预判一个方案在数据量变大时会不会崩;接着进入线性结构,学习如何用数组、链表、栈、队列组织数据,理解"连续内存换随机访问速度"与"链式结构换灵活的插入删除"之间的取舍;然后是树与散列,数据不再是一条线,而是有层次的组织——二叉树、堆、平衡树、哈希表,解决的是"如何让查找、插入、删除都尽可能快"这个核心矛盾;最后来到排序与图,排序是"把一堆数据整理成有序状态"的通用工具,图则把数据结构的应用范围从"孤立的数据集合"扩展到"数据之间的关系网络"——拓扑排序处理依赖,Dijkstra/Bellman-Ford 处理最短路径,Prim/Kruskal 处理最小连通代价。这条主线其实只在做一件事:先学会衡量好坏的标准,再逐步升级组织数据的方式,最终掌握处理数据关系的算法——这正是数据结构与算法这门学科由浅入深的完整脉络。
