摊还分析的基本方法

进阶内容。有些操作偶尔很贵，但平均下来很便宜——摊还分析就是用来精确刻画这种"平均开销"的数学工具。

为什么需要摊还分析

传统的最坏情况分析回答的是"单次操作最贵能贵到什么程度"，但很多数据结构的操作代价并不均匀—— 大多数时候很便宜，偶尔一次很贵。如果只看"最贵的那一次"来估算整体性能，会严重高估实际开销。

摊还分析（Amortized Analysis）回答的是另一个问题：连续执行 n 次操作，总代价是多少，平均到每次操作是多少？ 它关注的是一系列操作的整体表现，而不是孤立地看某一次操作。

类比：一辆车平时加一次油能跑 500 公里，油费均摊到每公里很便宜；但买车、换轮胎这些大额支出虽然不常发生， 平摊到几年的使用周期里，"每公里的实际成本"依然是一个稳定、可预测的数字。摊还分析就是在算这个"平摊到每次操作的稳定成本"。

方法一：聚合分析（Aggregate Analysis）

最直接的方法：计算 n 次操作的总代价上界，再除以 n，得到每次操作的摊还代价。

这个方法不区分"哪次操作贵、哪次便宜"，只关心总量。适合操作代价有明显规律、容易求和的场景。

方法二：记账法（Accounting Method）

给每次操作预先"分配"一笔虚拟信用（credit），信用可以理解为"预付的费用"：

  • 如果某次操作的实际代价小于分配的信用，多余的信用存起来，供以后代价高的操作使用。
  • 如果某次操作代价很高，就花掉之前存的信用来支付超出摊还代价的部分。

只要能证明"信用余额始终不为负"（即前面攒的信用永远够支付后面的大额开销）， 就说明每次操作分配的摊还代价是一个有效的上界。这种方法的直觉性比数学证明更强，适合直观解释。

方法三：势能法（Potential Method）

势能法是记账法的数学化版本：定义一个势能函数 Φ，描述数据结构当前状态"蕴含的潜在能量" （类似物理学中弹簧压缩得越紧，势能越高，之后能释放的能量也越多）。

摊还代价公式势能法
摊还代价(第 i 次操作) = 实际代价(第 i 次操作) + Φ(第 i 次操作后) - Φ(第 i 次操作前)

只要能证明所有操作的摊还代价之和是总实际代价的上界（通常通过 Φ 恒 ≥ 0 且 Φ(初始) = 0 来保证）， 就能得出摊还复杂度的结论。势能法是三种方法中最严谨、最通用的，代价是需要巧妙地设计势能函数。

呼应已学内容：ArrayList 扩容的均摊 O(1) 插入

回顾 Ⅱ Tab 线性结构讲过的 ArrayList：底层用数组实现，当容量不够时会整体扩容 （通常是扩容为原来的 1.5 倍或 2 倍），这个扩容操作需要把所有旧元素拷贝到新数组，代价是 O(n)。

如果简单地说"ArrayList 的 add 操作是 O(n)"，显然是不准确的——绝大多数 add 调用只是把元素放进数组末尾， 代价是 O(1)，只有触发扩容的那一次才是 O(n)。用聚合分析可以精确回答"n 次 add 操作总共花多少时间"：

聚合分析以 2 倍扩容为例
假设扩容策略是容量翻倍，从容量为 1 开始。
第 n 次插入触发的扩容操作，拷贝代价为 2^0 + 2^1 + 2^2 + ... + 2^k（k ≈ log n）
这是一个等比数列求和，总和约为 2n（等比数列求和的性质：总和不超过最后一项的 2 倍）

n 次 add 操作的总代价 = n 次普通插入（每次 O(1)）+ 扩容拷贝的总代价（约 O(n)）
                    = O(n) + O(n) = O(n)

摊还到每次操作：O(n) / n = O(1)

这正是"均摊 O(1)"结论的来源：扩容代价虽然单次很贵，但因为扩容后能支撑的插入次数也同比例增长， 平摊下来的成本是常数。如果用记账法来理解：每次普通插入时"多付一点点信用"，这些信用攒起来， 刚好够在触发扩容时支付那次昂贵的整体拷贝。

常见误区：不要把摊还分析和"平均情况分析（Average Case Analysis）"搞混。 平均情况分析是对随机输入分布求期望（比如随机化快速排序的期望复杂度）， 而摊还分析针对的是确定性的一系列操作，不依赖任何概率假设—— 即使输入是最坏情况精心构造的，摊还复杂度的结论依然成立。
面试高频："ArrayList 扩容为什么选 1.5 倍或 2 倍，而不是每次固定增加 10 个？" 答案正是摊还分析的应用：固定增量扩容会导致每隔固定次数插入就要花 O(n) 拷贝，n 次插入的总拷贝代价是 O(n²/常数)， 摊还到每次是 O(n)，退化了；而按比例扩容能保证总拷贝代价是 O(n)，摊还到每次是 O(1)。这也是为什么此前学 ArrayList 源码时会发现它的扩容倍数是固定比例而不是固定数值。

三种方法的对比与取舍

方法思路优点局限
聚合分析直接求 n 次操作总代价，再除以 n最直观，计算简单只能给出"整体均摊"的结论，无法区分不同操作类型的摊还代价
记账法给操作分配信用，多退少补直觉性强，易于向他人解释信用分配需要经验，证明"余额不为负"有时不严谨
势能法定义势能函数，用状态变化量分析最严谨、最通用，能处理复杂数据结构（如并查集、斐波那契堆）势能函数的设计需要洞察力，门槛最高

核心价值总结

  • 摊还分析回答"平均"而非"最坏"：关注一系列操作的总体表现，避免被个别昂贵操作误导。
  • 三种方法各有侧重：聚合分析简单直接，记账法直觉性强，势能法最严谨也最通用，实际使用中常常选择最容易证明的一种。
  • ArrayList 扩容是最贴近日常开发的例子：均摊 O(1) 插入的结论，本质上就是聚合分析里等比数列求和的直接应用。
