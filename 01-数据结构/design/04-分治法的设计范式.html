分治法的设计范式

你早就用过分治法了——归并排序和快速排序就是最经典的分治算法，这一篇把它抽象成通用范式，并学会用主定理分析复杂度。

分治法的三个步骤

分治（Divide and Conquer）是一种"化大为小、各个击破"的通用设计思路，标准套路分三步：

  1. 分解（Divide）：把原问题拆分成若干规模更小、结构相同的子问题。
  2. 解决（Conquer）：递归地解决每个子问题——当子问题足够小（基准情形），直接求解。
  3. 合并（Combine）：把子问题的解组合成原问题的解。
类比：批改 100 份试卷，一个人改太慢。分治的做法是：把试卷分成两堆各 50 份（分解），交给两个人分别改（解决，如果一个人还是嫌多可以继续往下分）， 最后把两份成绩单合并成一份总表（合并）。这正是"分而治之"这个成语的字面含义。

呼应已学内容：归并排序 & 快速排序都是分治

回顾 Ⅳ Tab（或排序模块）学过的两个排序算法，会发现它们都严格遵循分治三步骤，只是"分解"和"合并"谁更重谁更轻不同：

算法分解（Divide）解决（Conquer）合并（Combine）
归并排序 从中间切一刀，分成左右两半（分解本身几乎不费力） 递归排序左右两半 重活在这里：把两个有序子数组归并成一个有序数组，O(n)
快速排序 重活在这里：选枢轴并按大小把数组分成两部分，O(n) 递归排序左右两部分 合并几乎不费力——两部分各自有序后拼在一起就是整体有序

这个对比揭示了分治算法设计的一个规律："分解"和"合并"两个阶段，往往一个轻一个重， 工作量总会体现在其中一个步骤上，设计算法时要想清楚重活放在哪一步更划算。

用主定理（Master Theorem）分析时间复杂度

分治算法的递归结构决定了它的时间复杂度可以写成一个递推式：

递推式分治算法的通用形式
T(n) = a·T(n/b) + f(n)

其中各参数的含义：

  • a：每次递归产生多少个子问题（归并/快排都是 a=2）
  • b：每个子问题的规模缩小为原问题的 1/b（归并/快排都是 b=2，从中间或按枢轴大致对半分）
  • f(n)：分解和合并这两步本身（不含递归调用）花费的时间

主定理给出了这类递推式的通用解法，通过比较 f(n) 与 n^(log_b a) 的大小关系，分三种情况：

情况条件结论
情况 1f(n) 比 n^(log_b a) 增长得慢T(n) = Θ(n^(log_b a))，复杂度由递归树的叶子主导
情况 2f(n) 与 n^(log_b a) 同阶T(n) = Θ(n^(log_b a) · log n)，归并排序正是这种情况
情况 3f(n) 比 n^(log_b a) 增长得快（且满足正则条件）T(n) = Θ(f(n))，复杂度由分解/合并阶段主导

以归并排序为例：a=2, b=2，f(n)=O(n)（归并两个有序数组）。n^(log_2 2) = n^1 = n，与 f(n) 同阶， 落入情况 2，所以 T(n) = Θ(n log n)——这正是归并排序时间复杂度的严格推导过程，而不是死记硬背的结论。

面试高频：面试官可能会给一个陌生的递推式（比如 T(n) = 4T(n/2) + n），要求你用主定理判断复杂度。 这里 a=4, b=2，n^(log_2 4) = n^2，比 f(n)=n 增长快，落入情况 1，所以 T(n) = Θ(n²)。 记住三种情况的判断逻辑比死记结论更重要，因为面试常常换递推式的参数考察理解程度。
常见误区：主定理不是万能公式，它要求子问题规模必须是"等比例缩小"（n/b 这种形式），且 a、b 是常数。 如果递推式是 T(n) = T(n-1) + n 这种"每次只减少常数规模"的形式（比如某些朴素递归），就不能直接套主定理，得用递归树法或代入法展开求解。

核心价值总结

  • 分治三步骤：分解、解决（递归）、合并，缺一不可，重活通常集中在分解或合并中的一步。
  • 归并排序和快速排序是最佳教材：同样是分治，工作量分布完全相反，体会这种设计上的对称性。
  • 主定理是分析工具：把递归结构翻译成 T(n)=aT(n/b)+f(n) 的形式，就能直接判断复杂度阶数，不用每次都展开递归树硬算。

与其他知识点的联系

  • 运行时间计算法则：主定理本质上是"递归调用复杂度分析"法则的一个专用加强版——只适用于 T(n)=aT(n/b)+f(n) 这种等比例缩小的递推式，遇到 T(n)=T(n-1)+n 这种非等比例的递归，还是要退回到展开法或递归树。
  • 动态规划：分治和 DP 都是"把大问题拆成子问题"，区别在于分治的子问题互不重叠（如归并排序左右两半完全独立），DP 的子问题会重叠（如 0-1 背包的 dp[i-1][j] 被多处复用）——这也是为什么分治不需要记忆化，DP 需要。
  • 大整数乘法 / 矩阵乘法：Karatsuba 大整数乘法和 Strassen 矩阵乘法都是分治法的经典应用，它们的价值在于把 a（子问题数）从朴素算法的更大值降下来，通过主定理可以直接看出复杂度改善的幅度。
  • 快速选择（Quickselect）：和快速排序共享"选枢轴划分"的分解步骤，但因为只需要递归一侧而不是两侧，递推式变成 T(n)=T(n/2)+O(n)，用主定理算出是 O(n) 而不是 O(n log n)——这是主定理里"情况 3"的典型例子。
  • 面试高频考点："给你一个新的递推式，用主定理判断复杂度"是算法面试的常见考法，本篇正文里 T(n)=4T(n/2)+n 得到 O(n²) 就是这类题目的标准解法示范。
