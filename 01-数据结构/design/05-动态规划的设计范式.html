动态规划的设计范式

核心考点。DP 听起来神秘，其实本质就是"带备忘录的递归"——你甚至在 Ⅰ Tab 求最大子序列和时已经无意中用过它。

DP 的两个必要条件

动态规划（Dynamic Programming）能够适用，必须同时满足两个条件：

  • 重叠子问题（Overlapping Subproblems）： 如果用朴素递归解决问题，会发现同一个子问题被反复计算多次。DP 通过"记住已经算过的结果"消除这种重复计算。
  • 最优子结构（Optimal Substructure）： 与贪心算法共享的条件——原问题的最优解，可以由子问题的最优解推导出来。区别在于 DP 不敢像贪心那样"赌一个子问题的解"， 而是枚举所有可能的子问题组合，从中选出最优的。
类比：斐波那契数列 fib(n) = fib(n-1) + fib(n-2) 朴素递归会重复计算 fib(n-2) 很多次 （计算 fib(5) 要算两次 fib(3)，四次 fib(2)……），这就是重叠子问题。如果算过一次就把结果记下来，下次直接查表， 就是 DP 最朴素的形态——记忆化递归。

两种实现方式：自顶向下 vs 自底向上

方式做法特点
记忆化递归（自顶向下） 保留递归结构，用一个数组/哈希表缓存已经算过的子问题结果 代码改动小（在朴素递归上加缓存），但有函数调用栈开销，可能栈溢出
递推填表（自底向上） 从最小的子问题开始，用循环逐步填一张表，直到填出原问题的答案 没有递归开销，通常更快更省栈空间，是面试和工程中更常用的写法

"自底向上填表"本质上和"带备忘录的递归"求的是同一个东西，只是执行顺序反过来：递归是从大问题往小问题拆，填表是从小问题往大问题推。

呼应已学内容：最大子序列和就是最简单的 DP

还记得 Ⅰ Tab 讲过的"最大子序列和"问题吗？给定一个整数数组，求和最大的连续子数组。它的 O(n) 解法（在线处理算法） 维护一个变量 curSum 表示"以当前元素结尾的最大子序列和"：

MaxSubArray.java回顾 O(n) 解法，本质是一维 DP
public static int maxSubArray(int[] nums) {
    // dp[i] 表示以 nums[i] 结尾的最大子序列和
    // 状态转移：要么把 nums[i] 接到前面的子序列后面，要么以 nums[i] 单独另起一段
    int curSum = nums[0], maxSum = nums[0];
    for (int i = 1; i < nums.length; i++) {
        curSum = Math.max(nums[i], curSum + nums[i]); // dp[i] = max(nums[i], dp[i-1]+nums[i])
        maxSum = Math.max(maxSum, curSum);
    }
    return maxSum;
}

当时讲这道题时把它当成"分治 vs 在线处理"的效率对比案例，其实这个 O(n) 解法的本质就是 DP—— curSum 就是状态数组压缩后的一维滚动变量（因为状态转移只依赖前一项，没必要保留整个数组）。 这说明 DP 不是什么全新的黑魔法，你可能早就在不知不觉中用过它。

经典例子：0-1 背包问题的状态转移方程怎么推导

回到上一篇贪心算法失效的 0-1 背包问题：n 个物品，每个物品重量 w[i]、价值 v[i]，背包容量 W， 每个物品选或不选，求能装下的最大价值。

定义状态：dp[i][j] 表示"只考虑前 i 个物品，背包容量为 j 时能得到的最大价值"。

推导状态转移方程：对于第 i 个物品，只有两种选择，取较优的一种：

  • 不选第 i 个物品：最大价值就是只考虑前 i-1 个物品、容量仍为 j 时的最优解，即 dp[i-1][j]
  • 选第 i 个物品（前提 j ≥ w[i]）：价值等于 v[i] 加上"用剩下的容量 j-w[i] 装前 i-1 个物品"的最优解， 即 v[i] + dp[i-1][j-w[i]]
Knapsack.java0-1 背包 · 自底向上填表
public static int knapsack(int[] w, int[] v, int W) {
    int n = w.length;
    int[][] dp = new int[n + 1][W + 1]; // dp[i][j]：前 i 个物品、容量 j 时的最优解

    for (int i = 1; i <= n; i++) {
        for (int j = 0; j <= W; j++) {
            dp[i][j] = dp[i - 1][j]; // 默认：不选第 i 个物品
            if (j >= w[i - 1]) {
                // 容量够，比较"选"与"不选"哪个价值更高
                dp[i][j] = Math.max(dp[i][j], v[i - 1] + dp[i - 1][j - w[i - 1]]);
            }
        }
    }
    return dp[n][W];
}

这张表正是"重叠子问题"发挥作用的地方：dp[i-1][j-w[i]] 这个子问题会被多个更大的状态复用， 如果不填表而是朴素递归，会被反复计算指数级次数。

面试高频：DP 题的标准解题四步骤——① 定义状态（dp 数组表示什么含义）；② 找状态转移方程（当前状态由哪些子状态推出）； ③ 确定初始值和边界条件；④ 确定遍历顺序（自底向上填表时，必须保证用到的子状态已经算出来）。 能清晰讲出这四步，比直接背代码更能体现对 DP 的理解。
常见误区：把 DP 和"记忆化"划等号，认为只要加了缓存就是 DP。真正的关键是能不能正确定义状态和状态转移方程—— 很多人写 DP 时直接照搬模板代码却说不清 dp[i][j] 具体代表什么，一旦题目稍微变形就无从下手。理解状态定义比记代码模板重要得多。

核心价值总结

  • DP 的本质：用空间换时间，通过记录子问题的解，避免重叠子问题的重复计算。
  • 两个必要条件：重叠子问题决定"值不值得记"，最优子结构决定"能不能通过子问题的最优解推出原问题的最优解"。
  • 最大子序列和是入门 DP，0-1 背包是进阶 DP：前者状态可以压缩成一个变量，后者需要二维状态表，理解这个演进过程有助于举一反三。

与其他知识点的联系

  • 贪心算法：DP 和贪心共享"最优子结构"这一条件，但 DP 面对"选或不选都可能是最优"的不确定性时选择枚举全部子问题（如 0-1 背包的 dp[i-1][j] 和 dp[i-1][j-w[i]] 都要算），而贪心敢直接选一个不回头——这正是上一篇 0-1 背包用贪心失败、必须用 DP 的根本原因。
  • 回溯法：回溯法穷举解空间树的所有分支，DP 则是在回溯的基础上发现"很多分支的子问题重复了"，于是加一层缓存——可以把 DP 理解成"消除了重叠子问题之后的回溯"，两者的解空间树结构往往是一样的。
  • 递归调用的复杂度分析：朴素递归解斐波那契数列是 T(n)=T(n-1)+T(n-2)+O(1)，展开后是指数级 O(2ⁿ)；加上记忆化后每个子问题只算一次，退化成 O(n)——这个对比直观展示了"重叠子问题"对复杂度的决定性影响。
  • Java 集合框架 / LRU 缓存：DP 的记忆化递归本质就是一个"键为子问题参数、值为子问题解"的缓存表，实现上常用 HashMap 或数组；LinkedHashMap 实现 LRU 缓存的思路和记忆化递归的"缓存已算结果"是同一类工程手段。
  • 面试高频考点："斐波那契数列有几种解法，复杂度分别是多少" 是考察 DP 理解程度的经典问题——朴素递归 O(2ⁿ)、记忆化递归 O(n)、自底向上填表 O(n)、滚动变量优化后 O(1) 空间，四个层次正好覆盖本篇讲的所有要点。
