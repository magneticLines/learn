树的基本概念与遍历(前/中/后/层序)

从"线性"到"层次":数组和链表只能表达前后关系,而现实世界大量数据(文件系统、组织架构、DOM)天然是层次结构——树就是为此而生的模型。

📖 来源:Weiss《数据结构与算法分析:Java语言描述》 第 4 章「树」

一、为什么需要"树"这种结构

数组和链表都是线性结构:每个元素最多有一个"前驱"和一个"后继"。但很多真实数据不是线性排列的——公司的组织架构、电脑里的文件目录、一段 HTML 的 DOM 结构,它们的共同特征是:一个节点可以有多个"下级",下级之间互不相干,但都从属于同一个上级。这种"一对多的层次关系"用链表表达不了,用数组表达也很别扭(要靠下标计算,牺牲灵活性),于是需要专门的结构——树(Tree)。

树的递归定义很简洁:一棵树是 n 个节点的集合,其中一个是根(root),其余节点被分成若干个不相交的子集,每个子集本身又是一棵树(称为根的子树)。这个"整体由局部构成,局部又是整体的缩影"的分形特征,决定了树的绝大多数操作都天然适合用递归来写。

二、基本术语

术语含义
根(root)没有父节点的唯一节点,是整棵树的入口
父/子(parent/child)直接的上下级关系,一个节点的父节点唯一,子节点可以有多个
叶子(leaf)没有子节点的节点,是树的"末端"
兄弟(sibling)拥有同一个父节点的节点
深度(depth)从根到该节点的路径长度(边数),根的深度为 0
高度(height)从该节点到最远叶子的路径长度;树的高度 = 根的高度
度(degree)一个节点的子节点个数;二叉树里度最大为 2
易混淆点:深度是"从根往下数",高度是"从叶子往上数"。只有一个节点的树,高度是 0(不是 1)——这个边界条件在写递归高度计算函数时特别容易搞错。

三、四种遍历方式:不只是"怎么走",更是"为什么这么走"

遍历(Traversal)就是按某种规则访问树中每一个节点恰好一次。核心问题是:面对"根、左子树、右子树"这三块,先访问谁?不同的顺序对应不同的应用场景,这不是随意的命名,而是每种顺序天然适配某类问题。

1. 前序遍历(Pre-order):根 → 左 → 右

先处理根,再处理子树。典型用途:复制/序列化一棵树——因为你必须先创建根节点,才能挂接它的左右子树,前序的访问顺序正好符合"自顶向下构建"的天然顺序。

Java · 前序遍历
void preorder(TreeNode node) {
    if (node == null) return;
    visit(node);              // 先处理自己,再递归子树 —— 天然适合"先建根再挂子树"的复制场景
    preorder(node.left);
    preorder(node.right);
}

2. 中序遍历(In-order):左 → 根 → 右

对二叉搜索树(BST)而言,中序遍历会按严格升序输出所有节点值。这不是巧合,而是 BST"左小右大"的性质决定的——先走完所有比自己小的(左子树),再输出自己,再走完所有比自己大的(右子树)。后面讲 BST 时会反复用到这个性质做范围查询、找第 k 小等。

Java · 中序遍历
void inorder(TreeNode node) {
    if (node == null) return;
    inorder(node.left);
    visit(node);              // 对 BST 而言,这一步输出的顺序天然是升序的
    inorder(node.right);
}

3. 后序遍历(Post-order):左 → 右 → 根

子树处理完了才处理自己。典型用途:释放资源、计算子树聚合值(如子树大小/高度/和)——想清理一棵目录树,必须先删完所有子文件才能删空目录本身;想算某节点子树的高度,必须先知道左右子树的高度。"先处理依赖,再处理自己"正是后序的本质。

Java · 后序遍历计算子树高度
int height(TreeNode node) {
    if (node == null) return -1;
    int lh = height(node.left);   // 必须先拿到左右子树的结果
    int rh = height(node.right);
    return Math.max(lh, rh) + 1;  // 自己的高度依赖子节点算完之后才能确定 —— 典型的后序场景
}

4. 层序遍历(Level-order):按层从上到下,同层从左到右

本质是广度优先搜索(BFS),前三种都是深度优先(DFS)。它需要借助队列而不是递归/栈来实现——因为要保证"同一层的节点先于下一层被访问",队列的先进先出特性正好模拟"一层处理完再处理下一层"。典型用途:按层处理的场景,比如求二叉树的最大宽度、逐层打印、图的最短路径(无权图 BFS)。

Java · 层序遍历(BFS)
void levelOrder(TreeNode root) {
    if (root == null) return;
    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);
    while (!queue.isEmpty()) {
        TreeNode cur = queue.poll();  // 队列保证先进先出,天然按层次顺序出队
        visit(cur);
        if (cur.left != null) queue.offer(cur.left);
        if (cur.right != null) queue.offer(cur.right);
    }
}
遍历方式访问顺序典型场景
前序根 → 左 → 右复制/序列化树结构(先建根再建子树)
中序左 → 根 → 右BST 得到有序序列;表达式树还原成中缀表达式
后序左 → 右 → 根释放资源、计算子树聚合值(高度/大小/和)
层序按层,从上到下逐层处理、求最大宽度、无权图最短路径
易错点/面试高频:递归实现遍历简洁但依赖调用栈,树太深(比如退化成链表的 BST,见下一篇)会栈溢出(StackOverflowError)。生产代码里对深度不可控的树,常改用显式栈的迭代写法。另外面试常考"给定前序和中序序列还原二叉树",本质就是利用"前序第一个是根""中序中根左边是左子树、右边是右子树"这两条性质递归切分。

小结

  • 树是递归定义的层次结构:根 + 若干互不相交的子树。
  • 深度从根往下数,高度从叶子往上数,单节点树高度为 0。
  • 前序适合"自顶向下构建",中序在 BST 中天然有序,后序适合"先处理依赖再处理自己",层序按层处理靠队列实现的 BFS。
  • 递归遍历简洁但有栈溢出风险,深树场景要考虑迭代实现。
