二叉树与表达式树

限制"每个节点最多两个孩子",看似是收窄了能力,实际上换来了性质稳定、易于分析——这正是"约束换性能"的经典权衡。

📖 来源:Weiss《数据结构与算法分析:Java语言描述》 第 4 章「树」

一、二叉树:为什么限制成"最多两个孩子"

一般树里一个节点可以有任意多个子节点,结构灵活但也难分析——比如很难写出"第 k 个孩子"这种统一的存取逻辑。二叉树(Binary Tree)把度限制在 0~2,并且区分"左孩子"和"右孩子"(即便只有一个孩子,也要明确它是左还是右)。这个限制带来了两个好处:

  • 结构简单统一:每个节点只需要 left、right 两个指针字段,递归函数的分支数量固定,代码好写好证明。
  • 是很多高级结构的基础:BST、AVL、红黑树、堆、表达式树,几乎所有实用的树结构都建立在二叉树之上。二叉树是"地基",不是"终点"。
联系:任意一棵一般树都可以等价地转换成一棵二叉树(经典技巧:"左孩子-右兄弟"表示法,左指针指向第一个孩子,右指针指向下一个兄弟)。这说明二叉树的表达能力并不弱,只是换了一种"看"多叉结构的方式。

二、表达式树:用树结构表示算术表达式

之前讲栈的那篇里,我们用栈解决了表达式求值和中缀转后缀的问题——本质是把运算符的优先级"压栈/弹栈"处理成线性的一步步操作。表达式树提供了另一种视角:把表达式的结构直接画成一棵树,而不是压成一串符号序列。

规则很直观:叶子节点是操作数,内部节点是操作符,一个操作符节点的左右子树就是它的两个操作数(子表达式)。比如表达式 (3 + 4) * 5 画成树是这样:

表达式树示意:(3 + 4) * 5
        *
       / \
      +   5
     / \
    3   4

这棵树把"先算 3+4,再乘 5"的运算顺序直接编码进了结构里——不需要括号,不需要记优先级规则,子树的嵌套关系天然就表达了"谁先算"。这是表达式树相比字符串表达式的核心优势:结构自解释,不依赖额外的优先级规则或括号。

三、三种遍历对应表达式的三种写法

对表达式树做不同的遍历,能直接得到该表达式的前缀、中缀、后缀写法——这也印证了上一篇讲的"遍历顺序决定用途":

遍历方式得到的表达式形式对上面例子的结果
前序(根→左→右)前缀表达式(波兰式)* + 3 4 5
中序(左→根→右)中缀表达式(需要补括号才唯一)3 + 4 * 5(丢失了原括号信息)
后序(左→右→根)后缀表达式(逆波兰式)3 4 + 5 *
陷阱:中序遍历还原出来的中缀表达式默认不带括号,如果原表达式的运算符优先级和树的嵌套关系不一致(比如例子里 + 比 * 优先级低,却因为括号被安排在更深层),直接拼接会丢失括号信息导致语义错误。要还原正确的中缀表达式,需要在遍历时额外比较父子节点操作符的优先级,必要时补括号。

四、构造表达式树的算法(基于后缀表达式,联系栈)

构造表达式树的经典做法就是复用"栈解析后缀表达式"的思路:扫描后缀表达式,遇到操作数就建一个叶子节点入栈;遇到操作符就弹出两个节点作为左右子树,组成新节点再入栈。这和用栈直接计算后缀表达式的算法几乎一模一样,只是"计算数值"换成了"构造子树"。

Java · 由后缀表达式构造表达式树
class ExpNode {
    String val;              // 操作数或操作符
    ExpNode left, right;
    ExpNode(String val) { this.val = val; }
}

ExpNode buildFromPostfix(String[] tokens) {
    Deque<ExpNode> stack = new ArrayDeque<>();
    for (String t : tokens) {
        ExpNode node = new ExpNode(t);
        if (isOperator(t)) {
            // 后缀表达式里操作符前面紧跟着的两个操作数,顺序不能颠倒
            node.right = stack.pop();
            node.left  = stack.pop();
        }
        stack.push(node);   // 无论叶子还是内部节点,都统一入栈等待被上层消费
    }
    return stack.pop();      // 扫描结束,栈里唯一剩下的就是整棵树的根
}

五、求值:后序遍历天然适配

有了表达式树,求值就是一次后序遍历:先递归算出左右子树的值,再对当前操作符应用运算——这正是上一篇讲的"后序遍历适合处理子树聚合值"的又一个例子,和计算子树高度是同一种模式。

Java · 后序遍历求值
double evaluate(ExpNode node) {
    if (node.left == null) return Double.parseDouble(node.val); // 叶子:直接是数值
    double l = evaluate(node.left);   // 必须先拿到左右子树的结果,才能对当前操作符求值
    double r = evaluate(node.right);
    switch (node.val) {
        case "+": return l + r;
        case "*": return l * r;
        // ... 其余运算符类似
        default: throw new IllegalArgumentException();
    }
}
为什么要有"栈求值"和"表达式树"两种视角:栈方法是"边扫描边算,不保留结构",省内存、适合一次性求值;表达式树是"先固化结构,可反复使用",适合需要多次求值(比如代入不同变量值)、符号化简(编译器的常量折叠、代数化简)、求导等需要"理解表达式结构"而不只是"算个数"的场景——编译器的抽象语法树(AST)正是表达式树思想的直接延伸。

小结

  • 二叉树限制度 ≤ 2 且区分左右,换来结构统一、递归好写,是 BST/堆/红黑树等的地基。
  • 表达式树用"操作符做内部节点、操作数做叶子"把运算顺序直接编码进树的结构。
  • 前序/中序/后序遍历分别对应前缀/中缀/后缀表达式;还原中缀时要注意括号信息可能丢失。
  • 由后缀表达式构造表达式树的算法和"栈求值后缀表达式"同构,只是把"算数值"换成了"建子树"。
  • 表达式树相比纯栈计算的优势在于结构可复用,是编译器抽象语法树(AST)的雏形。
