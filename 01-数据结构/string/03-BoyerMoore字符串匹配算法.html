字符串与网络流

Boyer-Moore 字符串匹配算法

进阶内容。反其道而行之：从模式串的尾部往前比较，利用"未匹配的信息"实现比 KMP 更大步幅的跳跃。

换个方向比较：从后往前

KMP 无论主串还是模式串，都是从左往右逐字符比较。Boyer-Moore（BM）算法反其道而行之： 把模式串和主串对齐后，从模式串的最后一个字符开始，从右往左比较。

这个反直觉的设计带来一个好处：一旦在模式串末尾附近就发现字符不匹配，往往能立刻判断出"这个对齐位置根本不可能匹配"， 从而一次性跳过一大段距离，而不用像从左往右比较那样，要先匹配掉前面一堆字符才能发现问题。

规则一：坏字符规则（Bad Character Rule）

从右往左比较时，如果在某个位置失配，主串上这个导致失配的字符称为"坏字符"。 坏字符规则：查看模式串中坏字符最后一次出现的位置，把模式串滑动，让模式串中的这个字符对齐到坏字符上 （如果模式串里根本没有这个字符，直接把模式串滑到坏字符后面，跳过更大一截）。

类比：在书架上找一本书名叫"数据结构"的书，你从书脊的最后一个字"构"开始看起。如果当前这本书最后一个字是"网"， 而你知道"数据结构"这四个字里根本没有"网"这个字，那么这本书、以及紧挨着后面凡是包含"网"这个字位置对应重叠的可能，都可以直接跳过， 不用一本一本翻。

规则二：好后缀规则（Good Suffix Rule）

如果失配发生在模式串中间（前面已经有一段后缀成功匹配，称为"好后缀"）， 好后缀规则：利用这段已匹配的后缀信息，找模式串中是否有别的位置也能匹配这段后缀（或它的一部分）， 将模式串滑动对齐到那个位置，跳过不可能成功的中间地带。

实际实现中，BM 算法在每一步会分别计算坏字符规则和好后缀规则各自建议的滑动距离，取两者中较大的一个， 以获得最大的跳跃步幅。

BoyerMoore.java简化版：仅演示坏字符规则
public class BoyerMooreSimple {

    // 记录模式串中每个字符最后一次出现的位置，找不到则为 -1
    private static int[] buildBadCharTable(String pattern) {
        int[] table = new int[256]; // 按 ASCII 字符集大小建表
        java.util.Arrays.fill(table, -1);
        for (int i = 0; i < pattern.length(); i++) {
            table[pattern.charAt(i)] = i; // 后出现的位置覆盖前面的，天然记录"最后一次出现"
        }
        return table;
    }

    public static int search(String text, String pattern) {
        int n = text.length(), m = pattern.length();
        if (m == 0) return 0;
        int[] badChar = buildBadCharTable(pattern);

        int shift = 0; // 模式串相对主串的偏移量
        while (shift <= n - m) {
            int j = m - 1;
            // 从右往左比较
            while (j >= 0 && pattern.charAt(j) == text.charAt(shift + j)) {
                j--;
            }
            if (j < 0) return shift; // 全部匹配成功

            // 失配：坏字符在模式串中最后出现的位置，决定滑动多远
            char badCharInText = text.charAt(shift + j);
            int lastOccurrence = badChar[badCharInText];
            // 保证 shift 至少前进 1，避免坏字符位置在 j 右边导致死循环
            shift += Math.max(1, j - lastOccurrence);
        }
        return -1;
    }
}

实际工业级实现（比如 Java String.indexOf 某些 JDK 版本、grep、文本编辑器的查找）通常会同时结合坏字符规则和好后缀规则， 这里为了讲清楚核心思路，只演示了坏字符规则部分。

KMP vs Boyer-Moore：利用信息的方向不同

维度KMPBoyer-Moore
比较方向从左往右从右往左
利用的信息利用已经匹配成功的部分（模式串自身的最长相同前后缀）利用未匹配 / 刚失配的信息（坏字符、好后缀）
跳跃步幅较小，一次通常前进 1 到几个字符较大，常能一次跳过一大段，实际应用中平均性能通常更优
最坏时间复杂度严格 O(n+m)，稳定朴素版本最坏情况可能退化到 O(n·m)，需要配合好后缀规则才能保证 O(n+m)
典型应用需要严格保证最坏情况性能的场景（如网络入侵检测的确定性匹配）文本编辑器查找、grep 等追求平均性能的场景
面试高频："为什么文本编辑器的查找功能大多用 Boyer-Moore 而不是 KMP？" 因为实际文本中大多数位置很快就能通过坏字符规则判断"根本不可能匹配"，从而实现远大于 1 的跳跃步幅， 平均性能通常优于 KMP；而 KMP 的优势在于严格保证 O(n+m) 的最坏情况，适合对最坏情况敏感的场景。
常见误区：认为 Boyer-Moore "一定比" KMP 快。这是对平均情况和最坏情况的混淆—— 如果模式串和主串的字符集非常小（比如只有 DNA 序列的 4 种碱基），坏字符规则的区分度会大打折扣， 退化情况下 BM 的优势并不明显，此时 KMP 稳定的 O(n+m) 反而更有保障。

核心价值总结

  • Boyer-Moore 的核心创新：从右往左比较，利用"未匹配"的坏字符/好后缀信息实现大步幅跳跃。
  • 与 KMP 的本质区别：KMP 利用已匹配信息稳步前进，BM 利用未匹配信息大跳步，两者是同一问题的两种不同视角的解法。
  • 取舍依据：字符集大、追求平均性能选 BM；需要保证最坏情况稳定选 KMP。
