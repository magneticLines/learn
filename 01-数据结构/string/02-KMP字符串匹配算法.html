KMP 字符串匹配算法

核心考点。失配之后不回退主串指针——把"已经匹配过的信息"利用起来，是 KMP 算法效率飞跃的关键。

朴素字符串匹配的问题

字符串匹配问题：在主串（文本）中查找模式串（要找的子串）第一次出现的位置。最朴素的做法是暴力匹配： 从主串的每个位置开始，逐字符比较模式串，一旦失配就把起始位置右移一位，重新从头比较。

NaiveMatch.java朴素匹配：失配就整体右移一位重来
public static int naiveMatch(String text, String pattern) {
    int n = text.length(), m = pattern.length();
    for (int i = 0; i <= n - m; i++) {
        int j = 0;
        while (j < m && text.charAt(i + j) == pattern.charAt(j)) j++;
        if (j == m) return i; // 完全匹配
        // 失配：i 右移一位，j 归零，之前比较过的信息全部丢弃，重新开始
    }
    return -1;
}

问题在于：一旦在模式串第 j 位失配，主串指针 i 只能右移一位，之前已经匹配成功的 j 个字符信息被完全丢弃， 最坏情况下时间复杂度是 O(n·m)——比如主串是"AAAAAAAAAB"，模式串是"AAAB"，每次都要比较到快结尾才失配，效率很低。

KMP 的核心思路：利用已匹配信息，不回退主串指针

KMP（Knuth-Morris-Pratt）算法的关键洞察是：失配发生时，主串中已经比对过的那一段字符是已知的 （因为它们等于模式串前面的一段），没必要把主串指针 i 往回退—— 只需要调整模式串指针 j，跳到一个合理的位置继续比较，让"已经匹配上的部分"尽量不被浪费。

具体跳到哪里，取决于模式串自身的结构——这正是 next 数组（部分匹配表）要解决的问题。

next 数组的核心概念：最长相同前后缀

next[j] 记录的是：模式串前 j 个字符组成的子串，其"最长相同前后缀"的长度。

"相同前后缀"指的是一个字符串的前缀子串和后缀子串内容完全相同（但不能是字符串本身）。比如字符串 "ABAB"：

  • 前缀有 "A"、"ABA"，后缀有 "B"、"BAB"、"AB"……
  • 其中 "AB"（前缀）等于 "AB"（后缀），长度为 2，是最长的相同前后缀。

这个长度的意义是：如果模式串匹配到某个位置失配了，"最长相同前后缀"告诉我们模式串自身有一段前缀和当前已匹配部分的后缀是重复的， 可以直接把模式串滑动到这个位置对齐，跳过一定重新比较那些必然会匹配成功的字符，而不用把 j 归零重新开始。

类比：假设你在拼一句歌词"我爱你我爱他"，唱到"我爱你我爱"时发现下一个字对不上（应该是"他"却唱成别的）， 但你注意到已经唱出的"我爱你我爱"里，开头"我爱"和结尾"我爱"是重复的——那么与其从头重新对齐， 不如直接把"新的一轮"对齐到那个重复的"我爱"上继续往下唱，因为你已经知道这部分肯定是对的。

next 数组的构造思路

next 数组本质上是模式串在和自己做字符串匹配——用双指针 i（当前构造到的位置）和 j（当前最长相同前后缀的长度）逐步递推：如果 pattern[i] == pattern[j]，说明前后缀可以再延长一位； 否则 j 回退到 next[j-1]（复用之前已经算出的结果），直到找到匹配或 j 归零。

KMP.java构造 next 数组 + 匹配主逻辑
public class KMP {

    // 构造 next 数组：next[j] = 模式串 pattern[0..j] 的最长相同前后缀长度
    private static int[] buildNext(String pattern) {
        int m = pattern.length();
        int[] next = new int[m];
        int j = 0; // j 表示当前已确定的最长相同前后缀长度
        for (int i = 1; i < m; i++) {
            // 失配时，j 回退到 next[j-1]，复用已经算过的匹配信息
            while (j > 0 && pattern.charAt(i) != pattern.charAt(j)) {
                j = next[j - 1];
            }
            if (pattern.charAt(i) == pattern.charAt(j)) j++;
            next[i] = j;
        }
        return next;
    }

    // 主匹配逻辑：失配时不回退 i，只把 j 跳到 next[j-1]
    public static int search(String text, String pattern) {
        if (pattern.isEmpty()) return 0;
        int[] next = buildNext(pattern);
        int j = 0;
        for (int i = 0; i < text.length(); i++) {
            while (j > 0 && text.charAt(i) != pattern.charAt(j)) {
                j = next[j - 1]; // 利用 next 数组跳转，主串指针 i 全程不回退
            }
            if (text.charAt(i) == pattern.charAt(j)) j++;
            if (j == pattern.length()) return i - j + 1; // 完全匹配，返回起始位置
        }
        return -1;
    }
}

因为主串指针 i 从头到尾只前进不回退，模式串指针 j 的跳转也有均摊上界， KMP 的整体时间复杂度稳定在 O(n + m)——构造 next 数组是 O(m)，主匹配过程是 O(n)， 相比朴素匹配的 O(n·m) 是质的飞跃。

面试高频："next 数组是什么？" 记住核心表述："next[j] 表示模式串前 j+1 个字符中， 最长相同前后缀的长度"。面试中不需要默写完整的递推证明，但一定要能解释清楚"为什么失配后可以直接跳到 next[j-1] 而不用整体重来"—— 因为这段前缀已知会和主串当前位置往前的一段字符相同，不必重新比较。
常见误区：很多人记不住 next 数组的下标含义，容易把"前 j 个字符"和"前 j+1 个字符"弄混， 导致实现时下标错位。不同教材对 next 数组的定义（是否整体右移一位、是否单独处理 next[0]=-1）略有差异， 关键是自己选定一种定义后，插入匹配逻辑时要保持前后一致。

核心价值总结

  • KMP 解决的核心问题：失配时避免主串指针回退，把朴素匹配的 O(n·m) 降到 O(n+m)。
  • next 数组的本质：预处理模式串自身的"最长相同前后缀"，把"已经匹配的信息"转化为下次比较该从哪里继续的依据。
  • 适用场景：大文本中查找固定模式串、判断字符串是否为另一字符串的循环移位等，是字符串处理的基础工具。

与其他知识点的联系

  • 字符串哈希（Rabin-Karp）：字符串匹配还有另一条完全不同的路子——给主串每个位置的子串算一个可增量更新的哈希值（滚动哈希），直接比较哈希值是否相等来判断是否匹配，平均复杂度也是 O(n+m)。它不需要 next 数组这种"内部结构分析"，代价是存在哈希冲突的理论风险；而 KMP 是确定性的，不存在冲突问题。两者是"用信息换效率"这一目标下的两种不同实现路径，面试常问"除了 KMP 还有什么方法能做到线性时间字符串匹配"。
  • 动态规划的状态复用思想：next 数组的构造过程（失配时 j = next[j-1]，复用之前算出的结果而不是重新计算）和动态规划"用已求出的子问题结果推导当前问题"是同一种思维——next 数组本身就可以看成一个一维 DP 数组，next[i] 依赖 next[0..i-1] 的结果，这也是为什么很多教材把 next 数组的推导称为"KMP 的自匹配 DP"。
  • Trie 与 AC 自动机（上一篇）：KMP 解决的是"一个模式串在文本中的匹配"，如果要同时匹配多个模式串，就需要把这些模式串先组织成一棵 Trie，再借用 KMP 的 fail 指针思想给 Trie 的每个节点建立失配跳转——这正是 AC 自动机的构造方法，是 Trie 和 KMP 两个知识点的直接结合，也说明字符串算法之间往往不是孤立的，而是能互相拼接组合。
  • Boyer-Moore 算法（下一篇）：KMP 是"从左到右扫描、利用已匹配前缀信息"的思路，Boyer-Moore 则反过来"从右到左比较、利用失配字符信息跳跃更远"，实践中往往比 KMP 更快（尽管最坏复杂度分析更复杂）。两者都是"避免朴素匹配的重复比较"这一目标下的不同技术路线，对比学习能更清楚看到"预处理信息用在什么方向上"这一设计选择的影响。
  • 面试高频延伸——字符串周期性判断：next 数组的一个经典应用是判断字符串的最小周期：如果 len - next[len-1] 能整除 len，这个值就是字符串的最小重复单元长度。这是 next 数组"最长相同前后缀"含义的直接推广，常作为 KMP 面试题的进阶考点出现。
