算法分析 · 数学基础

指数 / 对数 / 级数等数学工具

算法分析本质上是"用数学语言描述代码的行为"。这些工具不是额外负担,而是后面所有复杂度化简的基本词汇。

📖 来源:《数据结构与算法分析:Java语言描述》 第 1 章「引论」

一、为什么学算法分析要先补数学

你写一段嵌套循环,直觉上知道它"很慢",但要把这种直觉变成严谨、可比较的结论(比如证明它是 O(n²)),就必须把代码的执行次数写成一个数学表达式,再对这个表达式做化简、估算、比较。这三件事分别对应:

  • 指数律:化简形如 2^(2n)、a^m · a^n 这类表达式,常出现在递归展开和分治算法的时间公式里。
  • 对数律:算法分析里"log"几乎总表示以 2 为底(除非特别说明),换底公式让你能自由转换,证明"底数不影响大 O 量级"这一关键结论。
  • 级数求和:循环的总执行次数往往就是一个级数求和的结果,把求和公式记熟,就能秒判断一段循环是线性、平方还是对数级。

换句话说:算法分析 = 把代码翻译成表达式 + 用这几套数学工具化简表达式。这一篇就是先把"翻译词汇表"准备好。

二、指数律

核心公式(A 为正整数,X、Y 为任意实数):
· X^A · X^B = X^(A+B)
· X^A / X^B = X^(A-B)
· (X^A)^B = X^(AB)
· 2^N + 2^N = 2^(N+1)(不是 4^N,这是初学者最容易错的一条)

最后一条尤其重要:很多人下意识觉得"两个一样的东西相加要乘以指数",但指数运算的"加倍"对应的是指数加 1,不是指数翻倍。这个直觉错误会直接导致复杂度分析出错——比如把两个 O(2^n) 的算法顺序执行,总量级依然是 O(2^n),而不是 O(4^n)。

三、对数律

换底公式: log_a(N) = log_b(N) / log_b(a)
令 b = 2,a 是任意大于 1 的常数,可得:log_a(N) = log_2(N) / log_2(a) —— 1/log_2(a) 是常数,所以不同底数的对数只相差一个常数因子。

这正是为什么大 O 记号里写 O(log n) 从不标注底数——不管是以 2、10 还是 e 为底,在大 O 的世界里(忽略常数因子)它们是同一个量级。这个结论后面分析二分查找、平衡树高度时会反复用到。

常用的对数运算规则:

  • log(AB) = log(A) + log(B) —— 把乘法问题变成加法问题,这也是对数表、计算尺的原理。
  • log(A/B) = log(A) - log(B)
  • log(A^B) = B·log(A)

四、级数求和:循环次数的通用公式

一段 for 循环的总执行次数,几乎总能写成某种级数求和。把下面几个公式吃透,分析循环复杂度时可以直接"对号入座"。

级数类型公式典型场景
等差级数1 + 2 + ... + N = N(N+1)/2双重循环里内层循环次数递增(如冒泡排序、选择排序)→ O(N²)
等比级数(几何级数)1 + 2 + 4 + ... + 2^k = 2^(k+1) - 1每次问题规模翻倍/折半的递归(如归并排序的合并层数、二分查找)
调和级数1 + 1/2 + 1/3 + ... + 1/N ≈ ln(N)某些期望分析、"每隔 k 步做一次操作"类算法(如某些排序的平均情况分析)
直觉记忆法:等差级数求和 = "首末两项相加,乘以项数,再除以 2"(高斯当年的做法);等比级数求和的结果只由最大的一项决定(前面所有项加起来也不超过最后一项的量级)——这条结论在分析递归算法的时间复杂度时极其常用。

五、和后面知识点的联系

这里埋几个后面会反复用到的引子:

  • 下一篇讲递归,递归展开后的时间公式往往就是一个等比级数求和——比如归并排序展开成 log n 层,每层总工作量是 n,总和用等比级数的思路一步到位。
  • 后面讲大 O 记号时会证明"对数的底数不影响量级",直接用到这里的换底公式。
  • 再后面分析二分查找为什么快,本质就是"问题规模每次减半,最多减多少次",这是对数律的直接应用。

小结

  • 指数律的关键坑:2^N + 2^N = 2^(N+1),不是指数翻倍。
  • 对数的底数只影响常数因子,大 O 记号里从不区分底数。
  • 循环复杂度分析 = 把执行次数写成级数求和,再套等差/等比/调和级数公式。
  • 等比级数求和的量级由最大一项决定——分析递归时非常好用。
