最大子序列和:四种解法 O(n³)→O(n²)→O(n log n)→O(n)

Weiss 书中最经典的教学案例——同一个问题,四种复杂度,直观感受「算法设计」比「写对代码」更值钱。

📖 来源:《数据结构与算法分析:Java语言描述》 第 1~2 章「引论/算法分析」

一、问题是什么

给定一个整数数组(可能含负数),求连续子数组之和的最大值。例如 [-2, 11, -4, 13, -5, -2],答案是 11 + (-4) + 13 = 20。这个问题的价值在于:它简单到人人能懂,却能引出从「暴力」到「优雅」的完整思维演进。

二、解法一:暴力枚举 O(n³)

枚举所有子数组的起点 i 和终点 j,每次都重新累加一遍区间和。三层嵌套(起点、终点、累加)导致 O(n³)。

Java · O(n³) 暴力
static int maxSubO3(int[] a) {
    int best = Integer.MIN_VALUE;
    for (int i = 0; i < a.length; i++)
        for (int j = i; j < a.length; j++) {
            int sum = 0;
            for (int k = i; k <= j; k++) sum += a[k]; // 每次都重新加,大量重复劳动
            best = Math.max(best, sum);
        }
    return best;
}

三、解法二:消除重复累加 O(n²)

观察到:固定起点 i 后,sum(i, j) 可以由 sum(i, j-1) 直接加上 a[j] 得到,不必每次从头累加。去掉最内层的重复计算,降到 O(n²)。

Java · O(n²) 消除重复累加
static int maxSubO2(int[] a) {
    int best = Integer.MIN_VALUE;
    for (int i = 0; i < a.length; i++) {
        int sum = 0;
        for (int j = i; j < a.length; j++) {
            sum += a[j]; // 复用上一步的结果,而不是重新累加
            best = Math.max(best, sum);
        }
    }
    return best;
}

四、解法三:分治 O(n log n)

把数组从中间切成左右两半。最大子序列和只有三种可能:完全在左半、完全在右半、跨越中点。前两种递归求解;跨越中点的情况可以在 O(n) 内算出(从中点向左找最大后缀和,向右找最大前缀和,相加)。递推式是 T(n) = 2T(n/2) + O(n),下一篇会证明这等于 O(n log n)。

Java · O(n log n) 分治
static int maxSubDivide(int[] a, int lo, int hi) {
    if (lo == hi) return a[lo];
    int mid = lo + (hi - lo) / 2;
    int leftBest  = maxSubDivide(a, lo, mid);       // 完全在左半
    int rightBest = maxSubDivide(a, mid + 1, hi);  // 完全在右半

    int leftSum = 0, leftMax = 0; // 从 mid 向左的最大后缀和
    for (int i = mid; i >= lo; i--) { leftSum += a[i]; leftMax = Math.max(leftMax, leftSum); }
    int rightSum = 0, rightMax = 0; // 从 mid+1 向右的最大前缀和
    for (int i = mid + 1; i <= hi; i++) { rightSum += a[i]; rightMax = Math.max(rightMax, rightSum); }

    int crossBest = leftMax + rightMax; // 跨越中点的情况
    return Math.max(leftBest, Math.max(rightBest, crossBest));
}

五、解法四:在线算法 O(n)——Kadane 算法

换个角度想:扫描到位置 j 时,「以 j 结尾的最大子序列和」只有两种来源——单独 a[j] 自己起,或者接上「以 j-1 结尾的最大子序列和」。如果后者是负数,不如舍弃前面重新开始。这是只看一眼当前元素和一个累计变量就能决策的「在线算法」(不需要把数组存下来,边读边算)。

Java · O(n) 在线算法(Kadane)
static int maxSubOnline(int[] a) {
    int best = a[0], cur = a[0];
    for (int i = 1; i < a.length; i++) {
        // cur 为负,不如从当前元素重新开始——这就是"及时止损"
        cur = Math.max(a[i], cur + a[i]);
        best = Math.max(best, cur);
    }
    return best;
}

六、四种解法对比

解法复杂度核心思想空间
暴力枚举O(n³)枚举所有区间,每次重新累加O(1)
消除重复累加O(n²)固定起点,累加结果复用O(1)
分治O(n log n)左/右/跨中点三种情况递归O(log n)递归栈
在线算法O(n)只维护"以当前元素结尾的最大和"O(1)
为什么 O(n) 解法最优雅:它甚至不需要把整个数组存下来,来一个元素就能立刻更新答案(这就是"在线"的含义)。这也是后面学动态规划时会反复看到的模式雏形:定义一个"以 i 结尾的最优解"状态,用它和上一个状态的关系推导出转移方程,一次遍历求解。

七、建立联系与陷阱

陷阱:如果数组全为负数,答案应该是数组中最大的那个负数(允许只选一个元素),不能返回 0。上面 O(n) 解法用 best、cur 初始化为 a[0] 而不是 0,正是为了正确处理这种情况——初学者常见 bug 是想当然地把初值设为 0。

这个案例最大的价值不是记住某个具体算法,而是体会「同一个问题,想法上的差异能带来复杂度量级的差异」——这正是本组开篇「算法分析」想建立的核心直觉。

小结

  • 同一个问题:O(n³) 暴力 → O(n²) 消除重复 → O(n log n) 分治 → O(n) 在线,每一步都是「换个角度想问题」而不是「代码写得更快」。
  • 分治法的关键是拆出「跨越中点」的第三种情况并在 O(n) 内求解。
  • 在线算法(Kadane)是动态规划思想的萌芽:定义局部最优状态,靠递推关系逐步更新。
  • 易错点:全负数组要防止把答案错误地拉到 0。
