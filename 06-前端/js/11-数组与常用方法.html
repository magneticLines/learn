数组与常用方法（map / filter / reduce）

数组是前端最常打交道的数据结构——接口返回的列表、表格的行、下拉的选项，几乎都是数组。会不会用 map / filter / reduce 这套"函数式三剑客"，直接决定你写出的是清爽的声明式代码，还是一堆嵌套的 for 循环。

它解决了什么问题

传统遍历靠 for 循环：你要手动维护索引、手动 push 到新数组、手动累加结果。代码里充满了"怎么做"的细节（命令式），真正的"想做什么"反而被淹没。数组的高阶方法把循环这件事封装起来，让你只描述"对每个元素做什么变换"——这就是声明式编程。它的好处不只是少写代码：意图清晰、不易写错索引、天然支持链式调用，还能避免共享可变状态带来的 bug。

同一件事：命令式 vs 声明式
// 命令式：取出所有偶数的平方
const result = [];
for (let i = 0; i < nums.length; i++) {
  if (nums[i] % 2 === 0) result.push(nums[i] ** 2);
}

// 声明式：读起来就是需求本身
const result = nums.filter(n => n % 2 === 0).map(n => n ** 2);

核心分水岭：改原数组 vs 返回新数组

这是用数组方法最容易踩坑也最该先记住的一条线。一类方法会原地修改（mutating）调用它的数组，返回值往往不是新数组；另一类不碰原数组（non-mutating），而是返回一个全新的数组。混淆这两类，会导致"我以为没改，结果原数据被污染了"的隐蔽 bug。

会改原数组（mutating）返回新数组 / 新值（non-mutating）
push / popmap
shift / unshiftfilter
spliceslice
sortconcat
reverseflat / flatMap
fillreduce（返回聚合值）
高频坑：sort 和 reverse 会改原数组！很多人以为"排序"是返回一个排好序的新数组，其实它就地修改并返回同一个引用。要保留原数组，先拷贝再排：const sorted = [...arr].sort()。ES2023 新增的 toSorted / toReversed / with / toSpliced 正是不可变版本，可惜旧环境还用不上。

遍历三剑客之一：map（一对一映射）

map 把数组的每个元素经过一个函数变换，原数组多长，结果就多长——它是严格的一对一映射。回调能接到三个参数：(element, index, array)。最典型的场景就是把后端数据转成视图需要的形状。

map：变换每个元素
const users = [{ id: 1, name: 'Ann' }, { id: 2, name: 'Bob' }];
const names = users.map(u => u.name);   // ['Ann', 'Bob']

// 用上 index：生成带序号的标签
const labels = names.map((name, i) => `${i + 1}. ${name}`);
// ['1. Ann', '2. Bob']
map vs forEach 是面试常考的区别：map 必须返回值，它收集每次回调的返回值组成新数组；forEach 没有返回值（返回 undefined），它只为"产生副作用"（打印、发请求、改外部变量）而存在。如果你 map 了却忘记 return，会得到一数组的 undefined；如果你只想遍历不要新数组，却用了 map，则是白白浪费内存——该用 forEach 或 for...of。

三剑客之二：filter（按条件筛选）

filter 保留所有让回调返回真值的元素，丢弃返回假值的。结果数组长度 ≤ 原数组。注意它和 map 的本质差异：map 看回调返回什么值（拿来当新元素），filter 看回调返回值的真假（决定留不留这个元素）。

filter：留下满足条件的
const nums = [5, 12, 8, 130, 44];
const big = nums.filter(n => n >= 10);   // [12, 130, 44]

// 一个常用技巧：filter(Boolean) 过滤掉所有假值
const clean = [0, 'a', '', null, 'b', undefined].filter(Boolean);
// ['a', 'b']  —— 0、''、null、undefined 都是假值，被剔除

三剑客之三：reduce（聚合）

reduce 是三者里最强、也最让人犯怵的一个。它把整个数组归约成一个值——这个值可以是数字、字符串，也可以是对象、甚至另一个数组。理解它的关键是两个东西：累加器 acc（accumulator，每轮的"中间结果"，会传给下一轮）和初始值（第二个参数，acc 的起点）。

reduce 的签名与执行过程
arr.reduce((acc, cur, index, array) => {
  return /* 新的 acc，会带入下一轮 */;
}, initialValue);

// 求和：acc 从 0 开始，每轮加上当前元素
const sum = [1, 2, 3, 4].reduce((acc, cur) => acc + cur, 0);
// 过程：acc=0,cur=1 → 1；acc=1,cur=2 → 3；acc=3,cur=3 → 6；acc=6,cur=4 → 10

reduce 真正的威力在于聚合成复杂结构。比如按某个字段分组（相当于 SQL 的 GROUP BY），这正是它远超 map/filter 的地方：

reduce：按属性分组
const people = [
  { name: 'Ann', city: 'SH' },
  { name: 'Bob', city: 'BJ' },
  { name: 'Cara', city: 'SH' },
];
const byCity = people.reduce((acc, p) => {
  (acc[p.city] ??= []).push(p.name);  // 没有这个 key 就先建空数组
  return acc;
}, {});
// { SH: ['Ann', 'Cara'], BJ: ['Bob'] }
务必传初始值！如果省略第二个参数，reduce 会拿数组第一个元素当初始 acc，并从第二个元素开始遍历。这有两个隐患：① 对空数组且无初始值，会直接抛 TypeError；② 当 acc 类型和元素类型不同时（比如要聚合成对象），不传初始值逻辑会全错。养成习惯：永远显式给初始值。

查找与判断：find / findIndex / some / every / includes

当你不是要变换整个数组，而是想"找一个"或"问一句是非题"时，用这组方法，它们都能短路（找到/确定结果就提前停止，不跑完全程）。

方法作用返回
find(fn)找第一个满足条件的元素元素本身，没有则 undefined
findIndex(fn)找第一个满足条件的下标下标，没有则 -1
some(fn)是否至少有一个满足（逻辑"或"）boolean
every(fn)是否全部满足（逻辑"与"）boolean
includes(v)是否包含某个值（按值比较）boolean
查找与判断
const nums = [3, 8, 15, 22];
nums.find(n => n > 10);        // 15（第一个 >10 的元素）
nums.findIndex(n => n > 10);   // 2（下标）
nums.some(n => n > 20);       // true（有 22）
nums.every(n => n > 0);       // true（全为正）
nums.includes(8);             // true
对比 includes vs indexOf：都能判断是否包含，但 includes 能识别 NaN（[NaN].includes(NaN) 为 true），而 indexOf 用 === 比较，对 NaN 永远返回 -1（因为 NaN !== NaN）。判断"在不在"优先用 includes。

扁平化：flat / flatMap

flat(depth) 把嵌套数组"摊平"，默认只摊一层；flatMap 等于 map 后再 flat(1)，常用于"一个元素映射成多个"的场景，性能比分两步写更好。

flat 与 flatMap
[1, [2, 3], [4, [5]]].flat();        // [1, 2, 3, 4, [5]]（默认 1 层）
[1, [2, [3]]].flat(Infinity);          // [1, 2, 3]（彻底拍平）

// flatMap：把每句话拆成单词，再合成一个扁平数组
['a b', 'c d'].flatMap(s => s.split(' '));  // ['a', 'b', 'c', 'd']

排序：sort 与那个经典大坑

sort 接收一个比较函数 (a, b)，它的返回值符号决定 a、b 的先后：

  • 返回负数 → a 排在 b 前面
  • 返回正数 → a 排在 b 后面
  • 返回 0 → 两者相对顺序不变

所以升序记 a - b，降序记 b - a。

最经典的坑：不传比较函数时，sort 把元素当字符串按字典序排！于是 [10, 2, 1].sort() 得到的是 [1, 10, 2]——因为字符 '10' 的首字符 '1' 小于 '2'。数字排序必须传比较函数。
sort 的默认坑 vs 正确写法
[10, 2, 1].sort();                // ❌ [1, 10, 2]  按字符串字典序！
[10, 2, 1].sort((a, b) => a - b);  // ✅ [1, 2, 10]  升序
[10, 2, 1].sort((a, b) => b - a);  // ✅ [10, 2, 1]  降序

// 别忘了 sort 会改原数组，要保留原数据先拷贝
const sorted = [...arr].sort((a, b) => a - b);

splice vs slice：一字之差，行为天壤之别

这两个名字像双胞胎，却分属"改原数组"和"返新数组"两个阵营，是初学者高频混淆点。

slice(start, end)splice(start, deleteCount, ...items)
改原数组？否，返回新数组是，原地增删
作用截取一段（含 start 不含 end）删除/替换/插入元素
返回值截取出的新数组被删除元素组成的数组
slice 只读取，splice 会动手
const a = ['a', 'b', 'c', 'd'];
a.slice(1, 3);          // ['b', 'c']；a 原封不动

const b = ['a', 'b', 'c', 'd'];
const removed = b.splice(1, 2, 'X');  // 从下标 1 删 2 个，插入 'X'
// removed = ['b', 'c']；b 变成 ['a', 'X', 'd']（原数组被改！）

链式调用：把三剑客串起来

因为 map / filter 都返回新数组，可以像流水线一样接龙。下面把"统计大于 18 岁用户的平均分"一气呵成：

链式：筛选 → 映射 → 聚合
const users = [
  { name: 'Ann', age: 20, score: 88 },
  { name: 'Bob', age: 16, score: 95 },
  { name: 'Cara', age: 22, score: 76 },
];

const adultScores = users
  .filter(u => u.age >= 18)      // 只留成年人
  .map(u => u.score);          // 取出分数 [88, 76]

const avg = adultScores.reduce((a, b) => a + b, 0) / adultScores.length;
// avg = 82
链式 ≠ 越长越好。每一环 map/filter 都会生成一个中间数组，遍历 N 次。数据量大时，若同一趟能完成多步，单个 reduce 或 for...of 只遍历一次会更省。先写清晰，性能有压力再优化。

forEach 与稀疏数组的坑

forEach 是最朴素的遍历，没有返回值，无法用 break 中途跳出（需要提前结束请用 for...of + break，或 some）。另一个隐蔽点：forEach、map 这类方法会跳过稀疏数组的"空位"（holes），而不是把它们当 undefined 处理。

稀疏数组：空位被跳过
const sparse = [1, , 3];      // 下标 1 是"空位"，不是 undefined
sparse.forEach(x => console.log(x));  // 只打印 1 和 3，跳过空位！
sparse.length;                  // 3（长度还是算空位的）

// 对比：for...of 不跳过，空位读成 undefined
for (const x of sparse) console.log(x);  // 1, undefined, 3
小结：① 先分清两类——push/pop/shift/unshift/splice/sort/reverse 改原数组，map/filter/slice/concat/flat 返回新数组；② map 一对一、要 return，forEach 只为副作用、无返回；③ filter 看真假，reduce 靠累加器 acc 聚合、永远传初始值；④ find/some/every/includes 短路省力；⑤ sort 默认按字符串排，数字必须传 (a,b)=>a-b 且它会改原数组；⑥ slice（返新）别和 splice（改原）搞混；⑦ 稀疏数组的空位会被 forEach/map 跳过。
