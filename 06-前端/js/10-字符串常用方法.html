字符串常用方法

字符串是前端打交道最频繁的数据：拼接 URL、格式化展示、解析接口返回、做表单校验，处处用到。掌握它的核心特性"不可变"和那十几个高频方法，能让你的文本处理代码既简洁又不踩坑。

它解决了什么问题

几乎所有真实数据最终都要变成文字呈现给用户：金额要补零对齐、手机号要打码、接口返回的 "a,b,c" 要拆成数组、用户输入要去掉首尾空格再校验。如果没有这些内置方法，每件事都得手写循环逐字符处理，既啰嗦又易错。JS 把这些高频操作做成了字符串原型上的方法，让你一行搞定。理解它们的关键前提只有一句：字符串是不可变的，任何"修改"方法都不改原串，而是返回一个新串——这决定了你必须用变量接住返回值，否则操作等于白做。

字符串不可变（最重要的前提）

原始类型 string 一旦创建，内容就不能被原地修改。你看到的所有 toUpperCase、replace、slice 都不动原字符串，而是生成并返回一个全新的字符串。这和 Java 的 String 不可变、用 StringBuilder 拼接是同一个设计哲学：不可变带来可预测性，也方便引擎做字符串驻留（共享内存）优化。

不可变：原串纹丝不动
let s = "hello";
s.toUpperCase();        // 返回 "HELLO"，但没人接住，白算一场
console.log(s);      // "hello" —— 原串根本没变

s = s.toUpperCase();   // ✅ 用变量接住新串，这才是"修改"
console.log(s);      // "HELLO"

s[0] = "x";          // 静默失败（严格模式下报错），s 不变
新手最常见的 bug：写了 s.trim() 却发现空格还在。原因就是没接返回值。记住口诀——字符串方法的结果必须赋值才有意义，写成 s = s.trim() 或链式 s.trim().toLowerCase()。

模板字符串：拼接的终极形态

用反引号 ` 包裹，内部可以用 ${ } 嵌入任意表达式，还能直接换行写多行文本。它彻底取代了 + 号拼接的繁琐和换行 \n 的拼装，是现代代码拼字符串的默认选择。

模板字符串 vs 加号拼接
const name = "小明", age = 18;

// 反例：加号拼接，引号和空格容易漏
const a = "我叫" + name + "，今年" + (age + 1) + "岁";

// 正例：${} 内可放表达式，所见即所得
const b = `我叫${name}，明年${age + 1}岁`;

// 多行文本：换行原样保留，无需 \n
const html = `<ul>
  <li>${name}</li>
</ul>`;

高频方法速查表

下面按"用途"分组列出最常用的方法。所有方法都返回新值、不改原串，索引从 0 开始。

用途方法说明 / 示例
查找indexOf(s)返回首次出现的下标，找不到返回 -1
includes(s)是否包含，返回布尔。语义比 indexOf !== -1 更清晰
startsWith(s)是否以 s 开头
endsWith(s)是否以 s 结尾，常用于判断文件后缀
截取slice(a, b)截取 [a, b)，支持负索引（从末尾数）。首选
substring(a, b)截取 [a, b)，不支持负数，且 a>b 时会自动交换
大小写toUpperCase()转全大写
toLowerCase()转全小写，常用于忽略大小写比较
去空白trim()去掉首尾空白，表单校验必备
trimStart()只去开头空白
trimEnd()只去结尾空白
替换replace(a, b)把 a 换成 b，默认只换第一个；配合正则 /g 才全换
replaceAll(a, b)替换全部（ES2021），免去写正则
分割合并split(sep)按分隔符拆成数组："a,b".split(",") → ["a","b"]
join(sep)数组方法，反向把数组拼成串：["a","b"].join("-")
填充padStart(n, s)开头补 s 到长度 n，常用于补零："5".padStart(3,"0") → "005"
padEnd(n, s)结尾补 s 到长度 n
重复repeat(n)重复 n 次："ab".repeat(3) → "ababab"
访问charAt(i)取第 i 个字符，越界返回 ""
at(i)取第 i 个字符（ES2022），支持负索引：at(-1) 取末位

陷阱一：slice vs substring

两者都截取子串、区间都是左闭右开 [start, end)，但行为有两处关键差异，记错了结果就南辕北辙：① slice 支持负索引（从末尾倒数），substring 把负数当 0；② 当 start > end 时，substring 会偷偷交换两个参数，slice 则直接返回空串。日常优先用 slice，行为更直觉、和数组 slice 也一致。

同样的参数，结果却不同
const s = "javascript";  // 下标 0..9

s.slice(4, 10);       // "script"
s.substring(4, 10);   // "script" —— 正常情况下两者一致

// ① 负索引：slice 从末尾数，substring 当 0 处理
s.slice(-6);          // "script"（倒数第 6 个起）
s.substring(-6);      // "javascript"（负数被当成 0）

// ② start > end：substring 偷偷交换参数，slice 返回空串
s.slice(10, 4);       // ""
s.substring(10, 4);   // "script"（自动变成 substring(4, 10)）

陷阱二：replace 默认只换第一个

这是面试和实战双高频坑。str.replace("a", "b") 传字符串时只替换第一处匹配，后面的原封不动。要全部替换，要么用带全局标志 /g 的正则，要么用 ES2021 的 replaceAll。

replace 的"只换一次"陷阱
const s = "a-b-c-d";

s.replace("-", "_");      // "a_b-c-d" —— 只换了第一个！

// 方案一：正则 + 全局标志 g
s.replace(/-/g, "_");     // "a_b_c_d"

// 方案二：replaceAll（更直观，推荐）
s.replaceAll("-", "_");  // "a_b_c_d"
易错点：replaceAll 如果第一个参数传正则，那个正则必须带 g 标志，否则直接抛 TypeError。这是它和 replace 的一个微妙差别。

实战组合：拆分、处理、再拼回

split 拆成数组 → 数组方法加工 → join 拼回字符串，是文本处理最经典的流水线。配合前面的 trim、padStart 等，几乎能搞定所有格式化需求。

split / join / padStart 的组合拳
// 把 "1,2,3" 每个数字补成两位再用 - 连接
const out = "1,2,3"
  .split(",")                // ["1", "2", "3"]
  .map(n => n.padStart(2, "0"))  // ["01", "02", "03"]
  .join("-");               // "01-02-03"

// 手机号打码：取头取尾，中间填 *
const phone = "13812345678";
const masked = phone.slice(0, 3) + "****" + phone.slice(-4);
// "138****5678" —— slice(-4) 取末尾四位
小结：① 字符串不可变，所有方法返回新串，结果必须用变量接住；② 拼接首选模板字符串 `${}`，多行直接换行；③ 截取优先 slice（支持负索引），别用易交换参数的 substring；④ replace 默认只换第一个，全换用 /g 或 replaceAll；⑤ split → map → join 是文本加工流水线，配合 trim/padStart/at(-1) 应对绝大多数格式化场景。
