解构赋值与展开运算符

解构让你"按形状拆包"数据，展开让你"把容器摊开"。它们是 ES6 里改变日常写法的两个利器，看懂同一个 ... 在不同位置的两副面孔，是真正用顺手的关键。

它解决了什么问题

ES6 之前，从数组或对象里取值是一件啰嗦的事：要先拿到容器，再一个个用下标或属性名赋给临时变量。合并两个数组得调 concat，把数组当实参传给函数还得靠 apply。这些"搬运数据"的样板代码淹没了真正的逻辑。解构赋值（destructuring）让你按数据的结构形状直接"对位拆包"；展开运算符（spread）让你把一个容器原地铺开成多个元素。两者一拆一铺，配上同名的剩余运算符（rest）负责"收集",把数据搬运压缩成一行声明式表达。

数组解构：按位置对位

数组解构靠位置匹配，左边方括号里的变量按顺序接收右边数组对应下标的值。用逗号留空可以跳过不想要的项，用 = 给默认值，用 ...rest 把剩下的全部收进一个新数组。

数组解构的几种用法
const [first, second] = [10, 20, 30];  // first=10, second=20，多出的 30 被丢弃

// 用逗号占位跳过中间项
const [, , third] = [10, 20, 30];     // third=30

// 默认值：只在该位置取到 undefined 时才生效
const [a = 1, b = 2] = [100];             // a=100, b=2

// 配合 rest 收集剩余项（必须放最后）
const [head, ...rest] = [1, 2, 3, 4];        // head=1, rest=[2,3,4]

一行交换两个变量

这是数组解构最经典的应用，不再需要中间临时变量。右边先求出一个临时数组 [y, x]，再整体拆给左边。

无需临时变量的交换
let x = 1, y = 2;
[x, y] = [y, x];     // 现在 x=2, y=1

对象解构：按属性名匹配

对象解构跟位置无关，靠属性名对应。常见的进阶写法有三种：用 { 原名: 新名 } 重命名、用 = 给默认值、用嵌套结构一层层拆深层属性。

对象解构进阶
const user = { name: "Tom", age: 20, address: { city: "BJ" } };

// 按名取值；address 没被取，所以不影响
const { name, age } = user;          // name="Tom", age=20

// 重命名：把 name 取出来叫 userName（左边是源属性，右边是新变量名）
const { name: userName } = user;     // userName="Tom"，注意没有变量叫 name

// 默认值 + 重命名可叠加
const { gender: g = "unknown" } = user;  // g="unknown"（user 没有 gender）

// 嵌套解构：拆出深层的 city
const { address: { city } } = user;    // city="BJ"，没有变量叫 address

函数参数解构 + 默认值

这是 React、各类配置型 API 里最常见的写法：参数列表里直接解构对象，既能"按需取字段"，又能给整个参数和单个字段都设默认值，调用方传参的顺序也就不重要了。

带默认值的参数解构
// = {} 防止不传参时对 undefined 解构而报错；size/round 各有字段默认值
function createBtn({ text, size = "md", round = false } = {}) {
  console.log(text, size, round);
}
createBtn({ text: "OK" });   // OK md false
createBtn();                // undefined md false（靠 = {} 兜底，不报错）

展开运算符：把容器摊开

同样的 ... 用在值的位置（数组字面量里、对象字面量里、函数调用的实参处）时，它把一个可迭代对象/对象"拆散铺平"。这让数组合并、浅拷贝、把数组当实参传递都变得极简。

数组与对象的展开
// 数组合并 / 复制（复制是浅拷贝）
const a = [1, 2], b = [3, 4];
const merged = [...a, ...b];      // [1, 2, 3, 4]
const copy = [...a];            // [1, 2]，新数组，但元素仍是原引用

// 对象合并 / 复制；同名属性后者覆盖前者
const base = { a: 1, b: 2 };
const next = { ...base, b: 99 };  // { a: 1, b: 99 }

// 函数调用时把数组摊开成一个个实参
const nums = [5, 1, 9];
Math.max(...nums);          // 等价于 Math.max(5, 1, 9) → 9

核心对比：rest 收集 vs spread 展开

初学者最大的困惑：... 到底是"收"还是"放"？答案取决于它出现的位置，而不是符号本身。看它在赋值号 / 参数列表的哪一边，就知道是哪种语义。

维度rest（剩余）spread（展开）
语义把多个收集成一个把一个铺开成多个
出现位置等号左边的解构目标里 / 函数形参末尾等号右边的字面量里 / 函数调用的实参处
位置约束必须是最后一个任意位置，可多次
结果得到一个新数组/对象把元素逐个铺出去
例子[a, ...rest] = arrfn(...arr)
同一符号，两副面孔
// rest：在形参末尾，把多余实参收集成数组
function sum(first, ...others) {   // others 是真正的数组
  return others.reduce((s, n) => s + n, first);
}
sum(1, 2, 3, 4);          // first=1, others=[2,3,4] → 10

// spread：在调用处，把数组铺开成实参
const arr = [1, 2, 3, 4];
sum(...arr);             // 等价于 sum(1, 2, 3, 4) → 10
一句话记忆：在左边/形参是 rest（往里收），在右边/调用是 spread（往外放）。同一个 ...arr，fn(...arr) 是展开，function fn(...arr) 是收集。

三个高频陷阱

陷阱一：对 undefined / null 解构直接报错。解构本质是访问属性，而 null.x / undefined.x 本就抛 TypeError。函数参数解构时务必加 = {} 兜底。

陷阱一：解构空值崩溃
const { a } = null;        // ❌ TypeError: Cannot destructure property 'a' of 'null'
const { b } = undefined;   // ❌ 同样报错
const { c } = obj || {};   // ✅ 先兜底成空对象再解构

陷阱二：默认值只在值为 undefined 时生效，null 不触发。这是面试常考点。默认值的判定标准是"严格等于 undefined"，而 null 是一个真实存在的值，会被原样赋上，默认值形同虚设。

陷阱二：null 不触发默认值
const { x = 10 } = { x: undefined };  // x=10，undefined 触发默认值
const { y = 10 } = { y: null };       // y=null！null 是有效值，默认值不生效
const { z = 10 } = {};             // z=10，缺失即 undefined，触发默认值

陷阱三：展开是浅拷贝。[...arr] 和 { ...obj } 只复制最外层一层，嵌套的对象/数组仍然共享同一引用。改深层属性会"串台"。需要深拷贝得用 structuredClone() 或递归处理。

陷阱三：浅拷贝串台
const origin = { id: 1, tags: ["a"] };
const clone = { ...origin };          // 只拷了一层
clone.tags.push("b");             // 改的是共享的那个数组
console.log(origin.tags);            // ["a", "b"]！原对象被污染

const deep = structuredClone(origin);  // ✅ 真深拷贝，嵌套互不影响
小结：① 数组解构按位置、对象解构按属性名，都支持默认值、重命名、嵌套，函数参数解构记得 = {} 兜底；② ... 看位置定语义——左边/形参是 rest 收集，右边/调用是 spread 展开；③ 解构 null/undefined 会崩；④ 默认值只对 undefined 生效，null 不触发；⑤ 展开只做浅拷贝，深层引用仍共享。
