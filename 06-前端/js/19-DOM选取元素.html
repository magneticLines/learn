DOM 选取元素

想用 JS 操作页面，第一步永远是"先找到那个元素"。本节讲清浏览器如何把 HTML 变成一棵可编程的对象树，以及五种选取 API 的取舍——尤其是 HTMLCollection 与 NodeList 这对最容易让人栽跟头的"双胞胎"。

它解决了什么问题

HTML 写出来只是一段静态文本，浏览器没法直接"操作"它。于是浏览器在解析 HTML 时，会把每个标签、文本、属性都变成一个 对象（节点 Node），再按嵌套关系把这些对象组织成一棵树——这就是 DOM（Document Object Model，文档对象模型）。树根是 document，它是 JS 操作页面的唯一入口。有了这棵树，JS 才能"按地址"找到某个 <div>、改它的文字、绑事件。所以一切 DOM 操作的前提，就是先从 document 出发把目标元素选取出来。

类比：HTML 是建筑图纸（静态文本），DOM 是照图纸盖好的真实大楼（活的对象树）。你不能在图纸上开灯，只能在大楼里操作具体的房间。document 就是大楼大门。

getElementById：按 id 精确定位

最古老也最快的方法。id 在页面里应当唯一，所以它返回单个元素（找不到返回 null）。注意：参数是纯 id 字符串，不带 #——这点和 CSS 选择器不同，常被搞混。

getElementById
// HTML: <h1 id="title">你好</h1>
const el = document.getElementById("title");  // 注意：不写 "#title"
console.log(el.textContent);  // "你好"

const missing = document.getElementById("nope");
console.log(missing);  // null —— 没找到不会报错，要记得判空

getElementsByClassName / getElementsByTagName：返回动态集合

这两个方法按类名或标签名查找，返回的是 HTMLCollection——一个动态（live）的类数组对象。"动态"的意思是：它不是一份快照，而是实时反映 DOM 当前状态的视图。你后续往页面里加了一个同类元素，这个集合的 length 会立刻变长，哪怕你早就把它存进变量了。

getElementsByClassName 是实时的
const items = document.getElementsByClassName("item");  // 同样不带 "."
console.log(items.length);  // 假设是 2

const div = document.createElement("div");
div.className = "item";
document.body.appendChild(div);

console.log(items.length);  // 3 —— 没重新查询，集合自己变了！
陷阱：边遍历边删元素。因为集合是实时的，删掉一个元素后它会立刻"缩短"，索引会错位，导致漏删。下面是典型反例。
反例：实时集合 + 正序删除 = 漏删
const items = document.getElementsByClassName("item");
for (let i = 0; i < items.length; i++) {
  items[i].remove();  // 删了 [0]，原 [1] 变成 [0]，但 i 已经走到 1 —— 跳过它了
}

// 正解一：每删一个 length 就变，干脆一直删第 0 个直到空
while (items.length) items[0].remove();
// 正解二：先转成静态数组再遍历（见下文 [...]）

querySelector / querySelectorAll：现代首选

这对方法接受任意 CSS 选择器，表达力远超前面那些按单一条件查找的老 API。querySelector 返回匹配的第一个元素（无则 null）；querySelectorAll 返回所有匹配元素组成的 NodeList。

用 CSS 选择器，语法和写样式一致
document.querySelector("#title");       // id 选择器，要带 #
document.querySelector(".item");       // 类选择器，要带 . —— 第一个 .item
document.querySelectorAll("ul > li.active");  // 复合选择器，老 API 做不到

const list = document.querySelectorAll(".item");
list.forEach((el) => console.log(el));  // NodeList 自带 forEach
语法陷阱：querySelector 系列必须带 . / #（因为它吃 CSS 选择器）；而 getElementById / getElementsByClassName 不带（它们直接接收 id 名 / 类名）。两套规则混在一起写，是新手最常见的"明明有元素却选不到"的原因。

关键对比：HTMLCollection（动态） vs NodeList（静态快照）

这是本节最核心的考点。getElementsBy* 返回 HTMLCollection（live）；querySelectorAll 返回 NodeList（static）——它是查询那一刻 DOM 的静态快照，之后 DOM 怎么变，这个 NodeList 都纹丝不动。

易混点："动态"听起来更强大，其实更危险（边遍历边改会出错）；"静态快照"听起来过时，反而更可预测——这正是 querySelectorAll 成为现代首选的原因之一。但要记住：DOM 变了，已经拿到的 NodeList 不会自动更新，需要重新查询。
同一段操作，两者表现相反
const live = document.getElementsByClassName("box");   // HTMLCollection
const snap = document.querySelectorAll(".box");      // NodeList 快照
console.log(live.length, snap.length);  // 2 2

document.body.insertAdjacentHTML("beforeend", '<div class="box"></div>');

console.log(live.length, snap.length);  // 3 2 —— live 跟着变，snap 还是旧快照

转成真数组：[...nodeList] 或 Array.from

无论 HTMLCollection 还是 NodeList，都是类数组（array-like）：有 length、能用下标，但没有 map / filter / reduce。想用数组的高阶方法，先转成真数组。这一步还有个副作用好处：转出来的数组是静态的，从此不再受"实时集合"的坑困扰。

两种转换写法
const coll = document.getElementsByClassName("item");

const arr1 = [...coll];           // 展开运算符，最简洁
const arr2 = Array.from(coll);      // 等价，且能传第二个 map 函数

const texts = arr1.map((el) => el.textContent);  // 现在能用 map 了

// 转成数组后再删，索引稳定，不会漏删（对比上文反例）
[...coll].forEach((el) => el.remove());

缩小范围：在元素上再选取

querySelector / querySelectorAll / getElementsBy* 不只挂在 document 上，任何元素都有这些方法。先选到一个容器，再在它身上查，就能把搜索限定在该子树内——既精确又高效，还避免了跨组件误选同名元素。

容器内查找
const card = document.querySelector("#card-1");

// 只在 #card-1 内部找 .title，不会选到别的卡片里的 .title
const title = card.querySelector(".title");
const btns = card.querySelectorAll("button");
这其实就是 React、Vue 之前手写组件时隔离 DOM 的核心手法：拿到组件根节点，所有查询都从它出发，组件之间互不干扰。

找不到怎么办：永远记得判空

所有"返回单个元素"的方法（getElementById / querySelector）在找不到时返回 null。如果不判空就直接 .textContent，会抛出经典的 Cannot read properties of null 错误——这是初学者最高频的运行时报错之一。

判空的两种姿势
const el = document.querySelector(".maybe");

if (el) {
  el.textContent = "找到了";
}

el?.classList.add("on");  // 可选链 ?.：el 为 null 时整句安全跳过
注意区别：querySelectorAll / getElementsBy* 这类"返回集合"的方法，找不到时返回的是空集合（length 为 0），不是 null，所以可以放心 forEach，无需判空。
小结：① DOM 是浏览器把 HTML 解析成的对象树，document 是入口；② getElementById 按 id 取单个（不带 #），getElementsByClassName/TagName 返回动态 HTMLCollection；③ querySelector/All 用 CSS 选择器（带 ./#），返回静态 NodeList，现代首选；④ 动态集合边遍历边删会出错，[...coll] / Array.from 转真数组更安全且能用 map；⑤ 可在任意元素上再查询以缩小范围；⑥ 取单个元素找不到返回 null，务必判空。
