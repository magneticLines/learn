classList 操作

给元素增减 class，是前端最高频的操作之一——切换主题、激活菜单、显隐弹窗，本质都是在改 class。classList 提供了一套增删查改 class 的标准 API，让你告别手动拼接字符串的痛苦，也引出一条重要的工程理念：用切换 class 驱动样式，而不是直接改内联样式。

它解决了什么问题

在 classList 之前，操作 class 只能靠 element.className——它是一整个字符串。想加一个类，你得这么写：el.className += " active"；想删一个类，得用正则把它从字符串里抠出来。这种"字符串拼接"的方式有三个致命问题：容易覆盖（写成 el.className = "active" 会把原有所有类一笔勾销）、容易出错（忘了前面的空格，"box" + "active" 拼成了 "boxactive"）、判断繁琐（想知道"有没有某个类"得自己切分字符串再比对）。

classList 把 class 当成一个集合（DOMTokenList）来对待，提供 add / remove / toggle / contains / replace 这些语义化方法，自动处理空格、去重、存在性判断。这就像 Java 里你不会去手动拼接逗号分隔的字符串来管理一组元素，而是用 Set 调 add / remove / contains——抽象层次对了，bug 自然就少了。

classList vs className 对比

维度classNameclassList
类型字符串DOMTokenList（类数组集合）
加一个类el.className += " a"（要记得空格）el.classList.add("a")
删一个类需正则替换，繁琐易错el.classList.remove("a")
判断是否含有需 indexOf / 切分判断el.classList.contains("a")
切换需先判断再增删el.classList.toggle("a")
误覆盖风险高（= 直接清空原有类）低（方法只动目标类）
整体替换全部类适合（el.className = "x y"）不直接支持
什么时候还用 className？当你确实想一次性设置全部类（比如重置元素到某个已知状态），el.className = "card active" 反而更直接。两者不是取代关系，而是各有适用场景；但日常"增、删、切、查单个类"，一律用 classList。

核心 API 一览

所有方法都挂在 element.classList 上。下面逐个看，注意 add 和 remove 可以一次传多个参数。

classList 五大方法
const box = document.querySelector('.box');

// 1. add：添加类，可一次加多个，已存在的会自动忽略（不会重复）
box.classList.add('active', 'highlight');

// 2. remove：移除类，可一次删多个，不存在的不会报错
box.classList.remove('highlight');

// 3. contains：是否含有某个类，返回布尔值
if (box.classList.contains('active')) {
  console.log('已激活');
}

// 4. toggle：有就删、没有就加；返回操作后"是否存在"的布尔值
const isOn = box.classList.toggle('active');

// 5. replace：把旧类换成新类（旧类不存在则返回 false，不做任何事）
box.classList.replace('active', 'inactive');

toggle 的第二个参数：强制开关

toggle 默认是"翻转"，但它接受第二个布尔参数作为强制指定：传 true 表示"确保加上"（相当于 add），传 false 表示"确保移除"（相当于 remove）。这在你已经算出一个布尔状态、想让 class 跟随它时特别有用——免去 if/else。

toggle(name, force) 的妙用
const isDark = prefersDark;  // 某个算好的布尔值

// 不用写 if (isDark) add 否则 remove，一行搞定：
document.body.classList.toggle('dark', isDark);
// isDark 为 true → 加上 dark；为 false → 移除 dark

实战一：切换暗色主题

主题切换的经典做法：在 <body> 上挂一个 dark 类，CSS 里用 body.dark .card { ... } 定义暗色样式。JS 只负责开关这个类，所有具体颜色都写在 CSS 里——这就是关注点分离。

一行实现主题切换
btn.addEventListener('click', () => {
  const dark = document.body.classList.toggle('dark');
  localStorage.setItem('theme', dark ? 'dark' : 'light');
});

实战二：激活导航项（单选高亮）

导航栏点哪个高亮哪个：先把所有项的 active 去掉，再给当前项加上。这是"互斥状态"的标准套路。

点击切换激活项
const items = document.querySelectorAll('.nav-item');
items.forEach(item => {
  item.addEventListener('click', () => {
    // 先清掉所有人的 active
    items.forEach(i => i.classList.remove('active'));
    // 再给自己加上
    item.classList.add('active');
  });
});

实战三：显示 / 隐藏

显隐元素，不要去改 el.style.display，而是切一个 hidden 类，把 display: none 写在 CSS 里。这样既能复用样式，又能搭配 CSS 过渡做动画，JS 逻辑也更干净。

用 class 控制显隐
/* CSS: .hidden { display: none; } */

// 显示
panel.classList.remove('hidden');
// 隐藏
panel.classList.add('hidden');
// 或者切换
panel.classList.toggle('hidden');
核心理念：用切换 class 驱动样式，而非直接操作 style。JS 只负责"打开/关闭一个状态标记"（class），具体长什么样交给 CSS。这就是关注点分离：样式归 CSS、行为归 JS。直接写 el.style.xxx 会把样式硬编码进逻辑里，难维护、难复用、还无法用伪类和媒体查询。下一篇《CSS 样式读写》会进一步讲什么时候才真正需要动 style。

常见陷阱

陷阱一：className = 会覆盖原有所有类。写 el.className = "active" 时，如果元素本来是 <div class="card big">，执行后就只剩 active，card 和 big 都没了。要"加一个类"请用 classList.add，别用 className =。
陷阱二：toggle 的返回值与第二参。toggle 返回的是操作后该类是否存在（布尔），别误以为返回的是"是否做了切换"。另外当第二参为 false 时，它只删不加——即使原本没有这个类，调用后也仍然没有，不会变成"翻转"。
陷阱三：类名不带点，和选择器不一样。CSS 选择器写 .active，但 classList 的方法里类名不带点：是 add("active")，不是 add(".active")。写成 add(".active") 会真的加上一个名为 .active 的奇怪类（旧浏览器甚至报错）。同理类名里也不能含空格，add("a b") 会抛 InvalidCharacterError，要分开传 add("a", "b")。
小结：① 增删查切单个类一律用 classList，别拼 className 字符串；② add/remove 可传多个，contains 返回布尔，replace 换类，toggle 翻转；③ toggle 第二参强制开关、返回值是"操作后是否存在"，false 时只删不加；④ 核心理念是用切换 class 驱动样式，把样式留给 CSS，实现关注点分离；⑤ 类名不带点、不含空格，与 CSS 选择器写法不同。
