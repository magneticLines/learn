CSS 样式读写

拿到一个 DOM 元素后，怎么用 JS 读它的样式、改它的外观？这件事有两套入口——一套写行内样式，一套读最终计算值，各有各的脾气。搞混它们，是动效与交互代码最常见的踩坑点。

它解决了什么问题

页面是静态的，但交互是动态的：鼠标悬停要换色、滚动要显隐、拖拽要改位置。这些都需要 JS 在运行时读出当前样式、写入新样式。浏览器为此提供了两个工具：element.style 负责读写行内样式（即 style="..." 属性），getComputedStyle() 负责读取浏览器最终算出来的样式。关键在于：写样式有两条路（直接写 style vs 切换 class），读样式也有两条路（style 只读行内，computedStyle 读最终值），用错路就拿不到想要的结果。

element.style：读写行内样式

el.style 对应的是元素的 style="..." 属性。属性名用驼峰命名（CSS 里的 background-color 在 JS 里写作 backgroundColor，因为连字符在 JS 标识符里非法）。它最大的特点也是最大的坑：只能读到行内样式，读不到样式表（<style> 或 .css 文件）里设的值。

style 读写：注意只看得见行内
// HTML: <div id="box" style="color: red">...</div>
// CSS:  #box { background-color: blue; }   ← 来自样式表
const box = document.getElementById('box');

console.log(box.style.color);            // "red"  ← 行内有，读得到
console.log(box.style.backgroundColor);  // ""    ← 在样式表里，读不到！

// 写入：驼峰属性名，值是带单位的字符串
box.style.color = 'green';
box.style.fontSize = '16px';      // 必须带 'px'，写 16 无效
陷阱：① 写数值类属性必须带单位字符串，style.width = 100 不生效，要写 '100px'；② 属性名是驼峰，style.fontSize 而非 style.font-size（后者是减法表达式，直接报语法错）；③ el.style.color 读不到 CSS 文件/<style> 里设的颜色，因为那不是行内样式——这是新手最常困惑的地方。

getComputedStyle：读最终计算样式

想知道元素"实际长什么样"，要用 getComputedStyle(el)。它把行内样式、样式表、继承、浏览器默认值全部层叠计算后，返回最终生效的样式。注意三点：只读（不能用它赋值改样式）、返回的是带单位的字符串（如 "16px" 而非数字 16）、颜色通常被规范化为 rgb() 形式。

getComputedStyle 读最终值
const box = document.getElementById('box');
const cs = getComputedStyle(box);

console.log(cs.backgroundColor);  // "rgb(0, 0, 255)" ← 即使来自样式表也读得到
console.log(cs.fontSize);         // "16px"  ← 字符串，带单位

// 要做计算？先解析成数字
const size = parseFloat(cs.fontSize);  // 16（number）

cs.color = 'red';            // ❌ 无效，computedStyle 是只读的

批量设置：style.cssText

逐条赋值在改很多属性时啰嗦。style.cssText 可以一次性写入一整串 CSS 文本（这里属性名用 CSS 原生的连字符写法，因为它本质是字符串）。注意它会整体替换已有的行内样式，而非追加。

cssText 一次写多条
// 一行搞定多个属性（注意：会覆盖原有行内样式）
box.style.cssText = 'color: green; font-size: 16px; padding: 8px;';

// 想追加而不覆盖，用 += （但要保证前面有分号）
box.style.cssText += ' border: 1px solid #ccc;';

操作 CSS 变量（自定义属性）

CSS 变量（如 --theme-color）名字带连字符且不是标准属性，不能用 style.themeColor 这种驼峰方式读写。要用专门的方法：setProperty() 写、getPropertyValue() 读。这是做主题切换、动态换肤的常用手段。

setProperty / getPropertyValue
// 设置 CSS 变量（第一个参数是完整变量名，含 --）
document.documentElement.style.setProperty('--theme-color', '#4ade80');

// 读取：从计算样式里取，记得 trim 掉两端空格
const val = getComputedStyle(document.documentElement)
  .getPropertyValue('--theme-color').trim();  // "#4ade80"

核心建议：优先切 class，而非堆内联 style

能用 JS 直接写 style，不代表就该这么写。绝大多数场景下，更好的做法是把样式写在 CSS 里、用 JS 切换 class（呼应上一篇 classList 操作）。直接堆内联 style 有三宗罪：内联样式优先级极高、后续难覆盖；把"长什么样"和"做什么"混在 JS 里，破坏了结构与表现分离；散落在各处的 style 赋值难维护、难复用。

反例：JS 直接堆一堆内联 style
// ❌ 样式逻辑塞进 JS，难维护，且内联优先级高难被覆盖
function activate(el) {
  el.style.backgroundColor = '#4ade80';
  el.style.color = '#fff';
  el.style.fontWeight = 'bold';
  el.style.transform = 'scale(1.05)';
}
正例：样式归 CSS，JS 只切 class
/* CSS：所有外观集中在这里，方便复用与覆盖 */
/* .item.active { background:#4ade80; color:#fff; font-weight:bold; transform:scale(1.05); } */

// ✅ JS 只负责"状态"，不关心具体长什么样
function activate(el) {
  el.classList.add('active');
}
什么时候才该直接写 style？当样式值是运行时才能算出来的动态数值时——比如跟随鼠标的坐标 el.style.left = x + 'px'、按比例计算的进度条宽度。这类"无法预先写死在 CSS 里"的值，才是 style 的主场。固定的几套外观，一律用 class。
小结：① el.style 读写行内样式，属性名驼峰、值带单位字符串，读不到样式表里的值；② 读最终样式用 getComputedStyle(el)，只读、返回带单位字符串、需 parseFloat 解析；③ 批量写用 style.cssText（整体覆盖）；④ CSS 变量用 setProperty / getPropertyValue；⑤ 固定外观优先切 class，只有动态计算值才直接写 style。
