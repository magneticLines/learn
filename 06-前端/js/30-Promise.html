Promise

Promise 是一个代表"未来某个结果"的占位符对象。它把回调地狱里层层嵌套的金字塔，拉平成一条可读、可组合、可统一捕错的链条——这是现代 JS 异步编程的基石。

它解决了什么问题

上一篇我们见识了回调地狱：异步操作一个套一个，代码向右无限缩进，错误要在每层手动传递，逻辑被切碎得无法阅读。问题的根源是——回调把"结果"直接交给了调用方传入的函数，调用方没有一个可以拿在手里、传来传去的"结果对象"。

Promise 的核心思想就是：异步函数不再要求你传回调，而是立刻返回一个 Promise 对象，代表"这件事最终会有结果"。你拿到这个对象后，可以用 .then() 登记"成功了怎么办"、用 .catch() 登记"失败了怎么办"。因为它是个值，所以能 return、能传参、能组合——异步流程第一次变得像同步代码一样可以"线性书写"。

一句话类比：Promise 就像取餐小票。你点餐后不用站在窗口干等（阻塞），而是拿到一张小票（Promise 对象）。小票现在状态是"制作中"（pending），将来要么"叫号取餐"（fulfilled），要么"售罄退款"（rejected）。你可以拿着小票去做别的事，约定好"叫号了我就去取"（.then）。

三种状态：一旦改变不可逆

每个 Promise 任意时刻都处于三种状态之一，状态转移只能发生一次，且不可逆——这是 Promise 行为可预测的根本保证。

状态含义如何进入
pending进行中，结果未定初始状态
fulfilled已成功，有一个结果值调用 resolve(value)
rejected已失败，有一个原因调用 reject(reason) 或抛异常

状态只能从 pending 走向 fulfilled 或 rejected，且定型后永不再变（统称 settled / 已敲定）。这意味着一个 Promise 的成功回调和失败回调，最终只会有一个被触发，且只触发一次——你不必担心回调被重复调用。

创建 Promise

用 new Promise(executor) 创建。executor 是一个立即同步执行的函数，接收 resolve 和 reject 两个函数参数，你在异步操作完成时调用它们来敲定状态。

把基于回调的 API 包装成 Promise
function delay(ms) {
  return new Promise((resolve, reject) => {
    if (ms < 0) reject(new Error("时间不能为负"));
    setTimeout(() => resolve(`等了 ${ms}ms`), ms);
  });
}

delay(500).then((msg) => console.log(msg));  // 500ms 后打印"等了 500ms"
陷阱：Promise 不是惰性的。executor 在 new Promise 的那一刻就立即同步执行了——异步任务已经启动，并不是等到你 .then() 才开始。这点和某些"冷信号"（如 RxJS Observable）相反。所以别指望"先创建放着、需要时才触发"，它一创建就跑。

消费结果：then / catch / finally

.then(onFulfilled, onRejected) 登记回调：第一个参数处理成功值，第二个（可选）处理失败。.catch(onRejected) 是 .then(undefined, onRejected) 的语法糖，专门捕错。.finally(fn) 无论成败都执行，常用于收尾（关闭 loading、释放资源）。

三个方法配合
fetchUser(id)
  .then((user) => console.log("成功：", user))
  .catch((err) => console.error("失败：", err))
  .finally(() => console.log("无论成败，都关闭 loading"));
对比 .then(fn, errFn) 与 .then(fn).catch(errFn)：前者的 errFn 捕获不到 fn 自身抛出的错误（它俩是同一层的并列分支）；后者的 .catch 在下一层，能兜住前面 then 里抛的错。所以实践中更推荐 .then().catch() 的写法。

链式调用：扁平化的关键

Promise 真正的威力在于：.then() 总是返回一个新的 Promise。这让多个异步步骤可以串成一条"链"，而不是层层嵌套。回调里 return 的值，会成为下一个 .then() 的入参；如果 return 的是一个 Promise，链会自动等它敲定再继续。

反例：回调地狱（向右坍缩的金字塔）
getUser(id, (user) => {
  getOrders(user, (orders) => {
    getDetail(orders[0], (detail) => {
      console.log(detail);      // 缩进越来越深，错误还得每层手动传
    });
  });
});
正例：Promise 链（拉平成一条竖线）
getUser(id)
  .then((user) => getOrders(user))     // return 一个 Promise，链会等它
  .then((orders) => getDetail(orders[0]))
  .then((detail) => console.log(detail))
  .catch((err) => console.error(err));  // 一个 catch 兜住整条链的错误
陷阱一：忘记 return 导致链断裂。如果在 then 里调用了异步函数却没 return 它，链不会等待，下一个 then 会立刻拿到 undefined 执行。
陷阱二：又把 Promise 嵌进了 then。在 then 回调里再写 .then() 而不 return，等于把回调地狱换了层皮——正确做法永远是 return 出去，让它接到主链上。

静态方法：组合多个 Promise

真实场景常需要同时处理多个异步任务。Promise 提供了四个组合方法，区别就在于"什么时候敲定"和"用谁的结果"。

方法何时 fulfilled何时 rejected结果
Promise.all全部成功任一失败（立即）所有结果组成的数组
Promise.allSettled全部结束（不论成败）永不 rejected每项 {status, value/reason}
Promise.race最先敲定的是成功最先敲定的是失败最快那个的结果
Promise.any最先成功的那个全部失败第一个成功值 / AggregateError
四个组合方法对比
// all：并发请求，全部成功才拿到结果数组；一个挂了整体就挂
Promise.all([fetchA(), fetchB(), fetchC()])
  .then(([a, b, c]) => console.log(a, b, c));

// allSettled：不论成败都等齐，逐个查看每项状态——批量任务的首选
Promise.allSettled([fetchA(), fetchB()])
  .then((results) => {
    // [{status:"fulfilled", value:...}, {status:"rejected", reason:...}]
  });

// race：最快敲定者决定结果——常用于"超时控制"
Promise.race([fetchData(), timeout(3000)]);

// any：只要有一个成功就行（如多个 CDN 取最先返回的）
Promise.any([cdn1(), cdn2(), cdn3()])
  .then((html) => console.log(html));

另有两个常用的"快捷构造器"：Promise.resolve(value) 返回一个已成功的 Promise，Promise.reject(reason) 返回一个已失败的。它们常用来把普通值"提升"成 Promise，或在条件分支里保持返回类型一致。

记忆法：all = "全都要，一损俱损"；allSettled = "都给我等齐了，谁成谁败我自己看"；race = "谁快听谁的，不管好坏"；any = "谁先成功听谁的"。race 与 any 的差别就在于：race 连失败都算数，any 只认成功。

更多陷阱与考点

  • 未处理的 rejection。一个 rejected 的 Promise 若一路没有 .catch，会触发 unhandledrejection 事件，Node 里甚至可能让进程退出。每条链都要有终点的 catch。
  • .catch 只捕获它"之前"的错误。放在 catch 后面的 then 抛错，这个 catch 管不着。一般把 catch 放在链的最末端。
  • resolve 后再 reject 无效。状态一旦敲定就不可逆，后续的 resolve/reject 调用会被静默忽略——不会报错，也不会改变结果。
  • resolve 一个 Promise 会"展开"它。Promise.resolve(p) 若 p 本身是 Promise，结果会跟随 p，而不是包成"Promise 套 Promise"。
  • then 回调永远是异步执行（微任务）。即便 Promise 已经 settled，.then 里的代码也会被推入微任务队列，在当前同步代码跑完后才执行——这是事件循环的考点。
小结：① Promise 是异步结果的占位符对象，把回调地狱拉平成链；② 三态 pending → fulfilled / rejected，敲定后不可逆、只触发一次；③ .then 返回新 Promise 实现链式，return 值传给下一个 then、return Promise 会被等待；④ 组合方法记四字诀：all 一损俱损、allSettled 等齐、race 谁快听谁、any 谁先成功；⑤ 牢记五坑：忘记 return 断链、catch 只管前面、未处理 rejection、then 里再嵌套、Promise 一创建就执行。
