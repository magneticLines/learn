Map 与 Set

很长一段时间里，JS 开发者把普通对象当字典、把数组当集合凑合用。ES6 带来了真正的 Map 和 Set，它们专为"键值映射"和"唯一值集合"而生，补齐了语言长期缺失的数据结构。理解它们，能让你写出更清晰、更不容易踩坑的代码。

它解决了什么问题

在 Map 出现之前，要存"键 → 值"的映射只能用普通对象 {}。但对象当字典有三个硬伤：键只能是字符串或 Symbol（用对象当键会被悄悄转成 "[object Object]"）；没有可靠的"数量"概念（得手动 Object.keys(o).length）；原型链上的键会污染查找（比如 obj["toString"] 本不该存在却返回了函数）。同样，要存"一堆不重复的值"只能用数组手动去重，既慢又啰嗦。Map 和 Set 就是为这两个场景量身打造的：Map 是有序的键值对集合，键可以是任意类型；Set 是值唯一的集合。

Map：键值对集合

Map 的核心卖点：键可以是任意类型（字符串、数字、对象、函数、NaN 都行）；严格保持插入顺序；有现成的 size 属性；天生可迭代，遍历极其方便。

Map 基本 API
const m = new Map();
m.set('name', 'Alice');      // set 返回 Map 本身，可链式调用
m.set(1, '数字键').set(true, '布尔键');

const objKey = { id: 1 };
m.set(objKey, '对象也能当键！');  // 普通对象做不到这一点

m.get('name');        // 'Alice'
m.has(1);             // true
m.size;               // 4，无需手动数
m.delete(true);        // 删除一个键，返回 true/false
m.clear();            // 清空全部

遍历 Map

Map 本身实现了迭代器协议，可以直接 for...of。它还提供 keys() / values() / entries() 三个迭代器，以及 forEach。注意 entries() 是默认迭代器，所以直接 for...of m 拿到的就是 [key, value] 数组。

四种遍历方式
const m = new Map([['a', 1], ['b', 2]]);  // 可用二维数组初始化

for (const [k, v] of m)            console.log(k, v);  // 等价于 m.entries()
for (const k of m.keys())      console.log(k);     // 'a' 'b'
for (const v of m.values())    console.log(v);     // 1 2

m.forEach((v, k) => console.log(k, v));  // 注意：回调是 (value, key) 顺序！

// 转回数组 / 普通对象
[...m];                 // [['a',1], ['b',2]]
Object.fromEntries(m);  // { a: 1, b: 2 }（键会被转成字符串）
易错点：forEach 的回调参数顺序是 (value, key, map)，和 for...of 拿到的 [key, value] 正好相反。这是初学者最常写反的地方。

重点对比：Map vs 普通对象

这是面试和实战中都绕不开的取舍。下表逐项对比：

维度Map普通对象 {}
键的类型任意类型（对象、函数、NaN 均可）只能是 string 或 symbol，其他键被隐式转字符串
顺序严格按插入顺序整数键会被排序提前，顺序不完全可靠
大小map.size 直接读需 Object.keys(o).length
原型污染无，干净的纯数据结构有，obj.toString 等继承键会干扰判断
迭代原生可迭代，for...of 直接用需 for...in（会遍历原型）或 Object.entries
JSON 序列化不支持，JSON.stringify(map) 得到 {}原生支持
怎么选？键是动态生成的、数量频繁增删、键不是字符串、需要保序或要数量——用 Map。结构固定、需要 JSON 序列化、当作"记录/配置"用——用普通对象。一句话：当字典/哈希表用就选 Map，当数据记录用就选对象。

Set：值唯一的集合

Set 存储不重复的值，添加重复值会被自动忽略。它最经典的用途就是数组去重，以及高效地判断"某个值是否存在"。

Set 基本 API 与去重
const s = new Set();
s.add(1).add(2).add(2);  // 第二个 2 被忽略，add 可链式
s.size;          // 2
s.has(2);       // true，比 arr.includes 快（哈希查找 O(1)）
s.delete(1);    // true

// 数组去重：最常用的一行写法
const arr = [1, 2, 2, 3, 3, 3];
const unique = [...new Set(arr)];  // [1, 2, 3]

// 求交集 / 并集也很自然
const a = new Set([1, 2, 3]), b = new Set([2, 3, 4]);
const inter = [...a].filter(x => b.has(x));  // [2, 3]
陷阱：Set 去重对对象无效。new Set([{a:1}, {a:1}]) 的 size 是 2，因为两个对象字面量是不同的引用。Set 比较的是引用相等，不是内容相等。对象数组去重要按某个唯一字段（如 id）自己处理。

WeakMap 与 WeakSet：弱引用版本

WeakMap / WeakSet 是它们的"弱引用"变体，专门解决内存泄漏问题。三个关键约束：键（WeakMap）或值（WeakSet）必须是对象，不能是原始值；不可遍历（没有 size、没有迭代器）；对键持有弱引用——一旦外部不再引用那个对象，垃圾回收器就能回收它，对应的条目也随之消失。

WeakMap 缓存：对象被回收，缓存自动清理
const cache = new WeakMap();

function compute(obj) {
  if (cache.has(obj)) return cache.get(obj);  // 命中缓存
  const result = /* 昂贵计算 */ obj.val * 2;
  cache.set(obj, result);
  return result;
}
// 若用普通 Map，只要 cache 活着，obj 就永远不会被回收 —— 内存泄漏
// WeakMap 中 obj 一旦无人引用即可被 GC，对应缓存条目自动消失
典型场景：① 给 DOM 节点关联额外数据（节点移除后数据自动释放）；② 为对象做计算结果缓存；③ 给对象存"私有数据"——把内部状态存进模块级 WeakMap，外部无法访问，对象销毁后也不残留。

陷阱：键的比较规则（SameValueZero）

Map 的键和 Set 的值，判断是否"相同"用的是 SameValueZero 算法——基本等同于 ===，但有一个例外：NaN 被认为等于自身。这一点和 === 不同（NaN === NaN 是 false）。而对象则永远按引用比较，内容相同的两个对象算不同的键。

NaN 相等，但两个内容相同的对象不相等
const s = new Set();
s.add(NaN).add(NaN);
s.size;                  // 1！两个 NaN 被视为同一个值

const m = new Map();
m.set({}, 'A').set({}, 'B');
m.size;                  // 2！两个 {} 是不同引用，互不覆盖
m.get({});               // undefined！新的 {} 又是另一个引用，取不到

// +0 和 -0 在 SameValueZero 下被视为相等
new Set([0, -0]).size;   // 1
小结：① Map 是有序键值对，键可任意类型，有 size，原生可迭代，set/get/has/delete/clear 一应俱全；② 当哈希表用选 Map，当数据记录用选对象，注意 Map 不能直接 JSON 序列化；③ Set 存唯一值，[...new Set(arr)] 是去重神器，但对对象按引用比较故去重无效；④ WeakMap/WeakSet 键须为对象、不可遍历、弱引用，适合缓存与私有数据避免内存泄漏；⑤ 键用 SameValueZero 比较，NaN 等于自身，但内容相同的对象是不同的键。
