HTML · Head 元数据

meta 常用属性

<meta> 标签不渲染任何内容，但它控制着页面在搜索引擎、社交平台、移动设备上的表现。写错一行，分享出去的卡片可能完全错乱。

一、meta 是什么

<meta> 是元数据标签——描述页面自身信息的数据，放在 <head> 里， 浏览器、搜索引擎爬虫、社交平台爬虫都会读取它，但用户在页面上看不到。

它是空元素（void element），没有闭合标签，也没有子节点。

meta 的两种基本形态
<!-- 形态 1：name + content 键值对 -->
<meta name="description" content="页面描述">

<!-- 形态 2：http-equiv + content（模拟 HTTP 响应头）-->
<meta http-equiv="X-UA-Compatible" content="IE=edge">

<!-- 形态 3：独立属性（charset 是特例）-->
<meta charset="UTF-8">

二、必写的两条 meta

任何页面都必须有这两行
<!-- 第 1 行：字符编码，必须是 head 内最前面 -->
<meta charset="UTF-8">

<!-- 第 2 行：视口控制，移动端适配的关键 -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">

charset="UTF-8" 为什么必须最前面

浏览器在解析 HTML 时，遇到 charset 之前必须用某种编码猜测文件内容。 如果 charset 出现得太晚，浏览器可能已经用错误编码解析了前面的内容，导致乱码。 规范要求它出现在整个文档的前 1024 字节内（注意是从文档起始字节算起，包含 <!DOCTYPE>、<html> 等），最安全的做法是放 <head> 第一行。

viewport 各参数含义

参数含义推荐值
width 视口宽度 device-width：等于设备物理宽度，不要写固定像素
initial-scale 初始缩放比例 1.0：不缩放
maximum-scale 最大缩放倍数 不要设置为 1——会禁止用户缩放，损害无障碍性
user-scalable 是否允许用户缩放 不要设为 no——同上，无障碍问题
常见错误：user-scalable=no 或 maximum-scale=1 会阻止用户放大页面。 这对视力不好的用户是严重障碍，也会被 Google 的移动端评分降权。永远不要禁止缩放。

三、SEO 相关 meta

搜索引擎会读取的 meta
<!-- 页面描述：出现在搜索结果的摘要里 -->
<meta name="description"
      content="深入讲解 Java 并发编程，涵盖锁机制、线程池、JMM 内存模型。">

<!-- 关键词（现代 SEO 几乎已无效，Google 早已忽略）-->
<meta name="keywords" content="Java, 并发, 线程池">

<!-- 作者信息 -->
<meta name="author" content="Shea Yin">

<!-- 控制爬虫行为 -->
<meta name="robots" content="index, follow">

description 怎么写才有效

规则说明
长度120～160 个字符。太短搜索引擎会自己生成；太长会被截断
内容准确描述本页内容，包含核心关键词，但不要堆砌
唯一性每个页面写不同的 description，不要全站一样
影响不直接影响排名，但影响点击率（用户看到摘要是否想点）

robots meta 常用值

值含义
index, follow默认行为：收录本页，跟踪页面内链接
noindex不收录本页（登录页、感谢页、测试页常用）
nofollow不跟踪页面内的链接
noindex, nofollow完全屏蔽爬虫

四、Open Graph：社交分享卡片

当你把链接发到微信、微博、Twitter、Slack 时，平台会抓取页面的 Open Graph（OG）标签 来生成预览卡片。没有 OG 标签，卡片要么显示空白，要么随机取页面内容，效果很差。

有 OG 标签时，分享到社交平台的效果
图片
example.com
深入理解 JVM 内存模型
从 JMM 到 happens-before 原则，彻底搞清楚 Java 并发的底层语义。
Open Graph 核心四条（缺一不可）
<!-- 页面标题（可以比 title 标签更适合分享语境）-->
<meta property="og:title"       content="深入理解 JVM 内存模型">

<!-- 内容类型 -->
<meta property="og:type"        content="article">

<!-- 分享图片：建议 1200×630px，小于 5MB -->
<meta property="og:image"       content="https://example.com/og-jvm.png">

<!-- 页面的规范 URL -->
<meta property="og:url"         content="https://example.com/jvm-memory">

<!-- 可选：描述文字 -->
<meta property="og:description"  content="从 JMM 到 happens-before，彻底搞清楚底层语义。">
<meta property="og:site_name"    content="Shea 的技术博客">
注意：OG 标签用的是 property 属性，不是 name。 这是因为 OG 协议使用 RDFa 语法。写成 name="og:title" 是错的，平台爬虫可能读不到。

Twitter Card（X）

Twitter / X 的专属卡片标签
<meta name="twitter:card"        content="summary_large_image">
<meta name="twitter:title"       content="深入理解 JVM 内存模型">
<meta name="twitter:description"  content="从 JMM 到 happens-before...">
<meta name="twitter:image"       content="https://example.com/og-jvm.png">

<!-- twitter:card 的三种类型 -->
<!-- summary              : 小图 + 文字 -->
<!-- summary_large_image  : 大图卡片（最常用）-->
<!-- player               : 视频/音频播放器 -->

五、http-equiv：模拟 HTTP 响应头

常用 http-equiv 值
<!-- 告诉 IE 用最新引擎渲染，不要用兼容模式 -->
<meta http-equiv="X-UA-Compatible" content="IE=edge">

<!-- 设置 CSP（内容安全策略），防 XSS -->
<meta http-equiv="Content-Security-Policy"
      content="default-src 'self'; script-src 'self'">

<!-- 定时刷新/跳转（不推荐，影响体验）-->
<meta http-equiv="refresh" content="5; url=https://example.com">
<!-- 5 秒后跳转到指定 URL -->
为什么叫 http-equiv：这些值等价于 HTTP 响应头字段。 例如 http-equiv="Content-Security-Policy" 等同于服务器发送 Content-Security-Policy: ... 响应头。 区别是：HTTP 头由服务器发送，优先级更高；meta 是 HTML 降级替代，适合静态页面。 生产环境应优先用真实的 HTTP 响应头，而不是 meta。

六、一份完整的 head 模板

新项目 head 起手式
<head>
  <!-- 1. 编码：必须第一行 -->
  <meta charset="UTF-8">

  <!-- 2. IE 兼容（有 IE 需求时加）-->
  <meta http-equiv="X-UA-Compatible" content="IE=edge">

  <!-- 3. 视口：移动端必须 -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  <!-- 4. 页面标题 -->
  <title>页面标题 - 网站名</title>

  <!-- 5. SEO -->
  <meta name="description" content="120~160 字的页面描述">

  <!-- 6. Open Graph -->
  <meta property="og:title"   content="页面标题">
  <meta property="og:type"    content="website">
  <meta property="og:image"   content="https://example.com/og.png">
  <meta property="og:url"     content="https://example.com/page">

  <!-- 7. favicon 和 CSS（下一节讲）-->
  <link rel="icon" href="/favicon.ico">
  <link rel="stylesheet" href="/styles.css">
</head>

七、与 Java 的类比

meta 概念Java / 后端类比
<meta> 整体 Java 注解（@）：不影响运行逻辑，但为框架/工具提供元信息
charset="UTF-8" JDBC URL 里的 characterEncoding=UTF-8：都是"声明用什么编码解析"
http-equiv="Content-Security-Policy" Spring Security 的 http.headers().contentSecurityPolicy(...)：同一件事，一个在 HTML，一个在 HTTP 头
OG 标签 og:title / og:image API 响应里的 summary / thumbnailUrl 字段：都是给"下游消费者"（爬虫/前端）用的描述字段
name="robots" content="noindex" 接口加 @RateLimiter 或权限注解：控制"谁能访问/处理这个资源"
