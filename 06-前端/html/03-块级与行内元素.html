块级元素 vs 行内元素

这是 HTML 布局最基础的分类。搞清楚它，你才能理解为什么有些标签"自己占一行"，有些却和文字挤在一起。

一、为什么要区分这两类

HTML 把元素分为两种基本显示模式，决定了元素在页面里的排列方式和尺寸控制方式。 这个分类不是标签的样式设定，而是元素的默认行为， 由浏览器内置样式表（User Agent Stylesheet）决定。

了解这个分类，是理解 CSS 盒模型和布局的前提。

二、直观对比

块级元素：每个元素独占一整行
二级标题
这是一个段落，它独占一行，宽度默认撑满父容器
这是一个 div，同样独占一行
行内元素：随文本流排列，不换行
这是一段普通文字，其中 <strong> 加粗词语 </strong> 、<em> 斜体词语 </em> 和 <a> 超链接 </a> 都与文字在同一行流动，不会另起一行。

三、核心差异表

块级元素（Block）

  • 独占一整行，前后自动换行
  • 宽度默认 100% 父容器
  • 可以设置 width / height
  • 可以设置 上下 margin
  • 可以包含块级和行内元素

行内元素（Inline）

  • 与相邻内容在同一行排列
  • 宽高由内容决定，不可手动设置
  • 设置 width / height 无效
  • 上下 margin 无效，左右有效
  • 只能包含文本或其他行内元素
行内元素设置 width、height、margin-top、margin-bottom 都不生效——这是初学者最常遇到的"为什么 CSS 不起作用"之一。

五、行内替换元素（特殊类别）

<img>、<input>、<button>、<video> 是行内替换元素（Inline Replaced Element）—— 它们在文档流中的位置是行内的，但可以设置宽高，因为浏览器会用外部资源"替换"元素本身的位置。

替换元素 vs 普通行内元素
<!-- ✅ img 是行内替换元素，width/height 有效 -->
<img src="photo.jpg" width="200" height="150">

<!-- ❌ span 是普通行内元素，width/height 无效 -->
<span style="width:200px; height:150px">我没变大</span>

<!-- ✅ 想让 span 可设尺寸，改成 display:inline-block -->
<span style="display:inline-block; width:200px; height:150px">现在有效了</span>

六、display 属性：随时切换

块级 / 行内只是默认行为，用 CSS 的 display 属性可以随时改变：

用 display 改变显示模式
/* 把行内元素变成块级 */
a { display: block; }         /* 导航链接铺满一行 */
span { display: block; }

/* 把块级元素变成行内 */
li { display: inline; }         /* 水平导航菜单 */

/* inline-block：行内排列 + 可设宽高（两全其美）*/
span { display: inline-block; width: 120px; height: 40px; }

/* 隐藏元素（不占空间）*/
div { display: none; }
display 值行为典型用途
block 独占一行，可设宽高 将链接做成按钮块
inline 随文本流，不可设宽高 将列表项水平排列
inline-block 随文本流，但可设宽高 图标按钮、导航项
none 完全隐藏，不占空间 JS 控制显示/隐藏
flex / grid 块级容器，启用弹性/网格布局 现代布局主力（CSS 章节详讲）
记住这个规律：HTML 的块/行内是默认值，CSS 的 display 才是最终裁判。 实际开发中，你会频繁用 display: flex 把块级容器变成弹性容器——届时子元素的块/行内默认值也会被覆盖。

七、嵌套规则：不能乱放

块级 / 行内的区分还决定了嵌套合法性：

嵌套规则
<!-- ✅ 块级可以包含行内 -->
<p>这是<strong>加粗</strong>文字</p>

<!-- ✅ 块级可以包含块级 -->
<div>
  <p>段落在 div 里</p>
</div>

<!-- ❌ <p> 不能包含块级元素（p 只接受行内内容）-->
<p>
  <div>这会让浏览器自动关闭 <p> 再开 <div></div>
</p>

<!-- ❌ 行内元素不能包含块级元素 -->
<span><div>不合法</div></span>

<!-- ⚠️  例外：HTML5 允许 <a> 包含块级元素（透明内容模型）-->
<a href="#">
  <div class="card">整张卡片都可点击</div>
</a>  ← 实际开发常见写法，HTML5 合法
陷阱：把 <div> 放进 <p> 里，浏览器会自动把 <p> 提前关闭，导致你的 DOM 树比预想多出一个 <p> 节点，CSS 选择器可能因此失效。

八、div 和 span：通用容器

div 和 span 是两个没有任何语义的容器， 存在的唯一目的是为了挂 CSS 样式或 JS 钩子：

divspan
类型块级行内
语义无无
用途布局容器、组合多个元素对文本局部应用样式
典型场景 <div class="card">...</div> <span style="color:red">错误</span>
最佳实践：能用语义标签（header / p / nav）就不要用 div。 但没有合适语义标签时，div + class 命名是完全正确的做法。 "div 汤"（满屏无语义 div 嵌套）是技术债，不是错误。

九、与 Java 的类比

这个分类背后的设计思想，在 Java GUI 框架里也有对应：

HTMLSwing / JavaFX 类比
块级元素（独占一行）FlowLayout 中换行排列的组件，或 VBox 子项
行内元素（随文本流）文本组件内嵌的 Label，HBox 水平排列子项
display: flexHBox / VBox，弹性控制方向和对齐
div（无语义容器）JPanel（纯布局容器，无业务含义）
