超链接 <a>：从跳转到协议处理器

<a> 是 Web 的灵魂——"超文本"里的"超"，指的就是它。一个标签同时承担了页面跳转、锚点定位、唤起邮件/电话、触发下载等职责。看着简单，但 target="_blank" 的安全坑、a 与 button 的语义之争，都是高频考点。

一、href 的五种写法

href（Hypertext REFerence）决定"点了去哪"。它接受的不只是网址，而是各种 URI—— 协议不同，浏览器的处理方式也不同。

href 的五类取值
<!-- ① 绝对 URL：跳到另一个站点 -->
<a href="https://developer.mozilla.org">MDN</a>

<!-- ② 相对路径：相对当前页面定位（详见上一节"路径"）-->
<a href="./about.html">关于</a>
<a href="/docs/intro">根相对路径</a>

<!-- ③ 页面锚点：跳到本页某个 id 元素 -->
<a href="#section-3">跳到第三节</a>

<!-- ④ mailto：唤起邮件客户端 -->
<a href="mailto:hi@x.com?subject=反馈&body=你好">联系我们</a>

<!-- ⑤ tel：移动端唤起拨号 -->
<a href="tel:+8613800138000">拨打客服</a>
scheme 决定行为：浏览器看 href 开头的协议（scheme）来决定怎么处理。 http(s): 走网络请求，mailto: / tel: 交给操作系统注册的"协议处理器"（邮件 App、拨号 App）。 这和后端的 URLStreamHandler 按协议分发是一个思路——协议名是路由的 key。

二、target：在哪个窗口打开

target 指定链接在哪个浏览上下文（browsing context）打开。 四个保留值：

值含义
_self默认值，在当前标签页打开（覆盖当前页面）
_blank新建标签页/窗口打开
_parent在父级 frame 打开（无父级时等于 _self）
_top在最顶层窗口打开，跳出所有 iframe 嵌套
_parent 和 _top 只在 <iframe> 嵌套场景才有区别。 没有 iframe 时，整页就是顶层，三者（_self/_parent/_top）效果相同。

三、安全重点：_blank 的 Reverse Tabnabbing 漏洞

这是面试和 code review 都爱问的点。当你写 target="_blank" 打开一个新页面时， 新页面可以通过 window.opener 反向控制原页面——哪怕它是另一个域名的恶意站点。

Reverse Tabnabbing 攻击流程（未加 rel 时）
1
你的页面有一条 <a href="https://evil.com" target="_blank">，用户点击。
2
新标签打开 evil.com。此时新页面拿到 window.opener，指向你的原页面。
3
恶意页执行 window.opener.location = "https://phishing.com"，把你背后那个原标签悄悄换成钓鱼页。
4
用户看完新标签切回来，看到"还是原来那个网站"（其实已是仿冒登录页），输入账号密码被盗。

关键在于：跨域时新页面读不到原页面内容（同源策略拦着），但 window.opener.location 是允许跨域赋值的—— 它只"导航"不"读取"，绕过了同源限制。这就是攻击能成立的根源。

正确 vs 错误
<!-- ❌ 危险：opener 暴露，可被反向劫持 -->
<a href="https://外站.com" target="_blank">外链</a>

<!-- ✅ 安全：noopener 切断 window.opener；noreferrer 顺带不发 Referer -->
<a href="https://外站.com" target="_blank"
   rel="noopener noreferrer">外链</a>
好消息但别依赖它：现代浏览器（Chrome 88+、Firefox、Safari）已默认对 target="_blank" 隐式加上 noopener。但你无法保证用户用的是新浏览器，也无法保证旧 webview—— 所以显式写 rel="noopener noreferrer" 仍是规范要求的最佳实践，linter 也会强制检查。

四、rel 的其他值

rel（relationship）描述当前页与目标的关系，是给浏览器和搜索引擎看的元信息。常用：

rel 值作用
noopener切断 window.opener，防 Reverse Tabnabbing（安全）
noreferrer跳转时不发送 Referer 请求头（隐私）；同时也隐含 noopener
nofollow告诉搜索引擎"别把我的权重传给这个链接"（SEO），用于用户生成内容、付费链接
external纯语义标记，表示这是外部链接
nofollow 的本意：搜索引擎用"链接投票"算排名（PageRank 思想——A 链向 B，等于 A 给 B 投了一票）。 论坛、评论区的链接若不加 nofollow，会被垃圾广告刷外链来"借"你网站的权重。 加上后等于声明"这票不算数"。注意 noreferrer 已隐含 noopener 的防护效果，但由于 noopener 出现得比 noreferrer 晚、早期部分浏览器只认 noreferrer，为兼容这些旧浏览器，习惯上两个一起写。

五、download：把链接变成下载

默认情况下，浏览器对能渲染的资源（图片、PDF、HTML）会直接打开。加 download 属性后， 点击会触发"下载到本地"而不是导航，还能指定保存的文件名。

download 的两种用法
<!-- 触发下载，文件名用服务器返回的原名 -->
<a href="/files/report.pdf" download>下载报告</a>

<!-- 指定下载后的文件名（重命名）-->
<a href="/files/abc123.pdf" download="2026年度报告.pdf">下载</a>
关键限制：download 仅对同源资源有效。出于安全，浏览器会忽略跨域 URL 上的 download（防止恶意站点伪造他人文件下载）。要下载跨域文件并自定义文件名， 得后端代理转发，或用 fetch 拿 blob 再生成 blob: URL 触发下载。

六、锚点跳转：同页与跨页

href="#id" 让页面滚动到带该 id 的元素。它本质是修改 URL 的 fragment（哈希片段，# 后面那部分），不会触发页面刷新或网络请求。

同页锚点 + 跨页锚点
<!-- 目标：任意元素带 id -->
<h2 id="install">安装步骤</h2>

<!-- 同页跳转：滚动到 #install -->
<a href="#install">去看安装</a>

<!-- 跨页锚点：先到 guide.html，再滚到其中的 #install -->
<a href="guide.html#install">查看安装指南</a>

<!-- 回到顶部的特殊写法 -->
<a href="#top">回到顶部</a>
配合 CSS 平滑滚动：给 html { scroll-behavior: smooth; } 就能让锚点跳转变成动画滚动。 另外 scroll-margin-top 可解决"固定顶部导航栏遮住标题"的经典问题。

七、href="#" 与 a / button 的语义之争

新手常把 <a href="#"> 当按钮用，点击后挂个 JS。这有两个坑： # 会让页面跳回顶部，并在 URL 末尾留下一个 #。 通常要在事件里 e.preventDefault() 阻止默认导航。

用 a 模拟按钮的常见写法
<a href="#" id="btn">点我</a>
<script>
  document.getElementById('btn').addEventListener('click', function (e) {
    e.preventDefault()  // 阻止跳回顶部 + URL 加 #
    // ... 你的逻辑
  })
</script>
语义原则（重要）：
· 有跳转/有 URL → 用 <a>（导航到页面、锚点、外链、下载）。
· 纯页面内操作 → 用 <button>（提交、展开折叠、打开弹窗、触发计算）。
用 <a href="#"> 当按钮是反模式：它在语义、SEO、可访问性上都不对。该用 button 就用 <button type="button">，连 preventDefault 都省了。

八、陷阱速查

陷阱后果正确做法
在 <a> 里嵌套 <a> HTML 非法，浏览器会自动拆开，行为不可预期 交互元素不可嵌套，重构 DOM 结构
空 href（<a href="">） 等于链接到"当前页面"，点击会重新加载整页 当按钮就改用 <button>；占位用 href="#" + preventDefault
没有 href 的 <a> 不是链接，键盘无法 Tab 聚焦，回车也不触发 需要可聚焦的操作请用 <button>
target="_blank" 不加 rel Reverse Tabnabbing 风险 + 泄露 Referer 固定搭配 rel="noopener noreferrer"
用 <div onclick> 做链接 键盘不可用、屏幕阅读器读不出、搜索引擎抓不到 能用语义的 <a>/<button> 绝不用 div

九、与 Java 的类比

<a> 概念Java / 后端类比
href 跳转到新页面 HTTP 302 重定向 / 路由跳转——服务端告诉浏览器"去这个地址"
scheme 决定行为（http/mailto/tel） URLStreamHandler 按协议分发；类似策略模式按 key 选实现
rel="noopener" 隔离 window.opener 调用外部服务时做安全隔离——沙箱 / 不暴露内部引用，防被反向操控
mailto: / tel: 协议处理器 SPI 机制——操作系统/JVM 按协议名查找已注册的处理器实现
nofollow 不传递权重 调用链上打"不计入信任"标记，类似 trace 中标记"外部不可信节点"
a（导航）vs button（操作）的语义分工 REST 里 GET（安全幂等，查询/导航）vs POST（有副作用，操作）的区分
<a> 的核心心智模型是：它代表"一个可达的资源地址"——所以它能被收藏、被分享、被搜索引擎抓取、被键盘聚焦。 一旦你用它做"无 URL 的纯操作"，就破坏了这个契约，各种可访问性和 SEO 问题都源于此。 记住一句话：能写出 URL 的就是 a，写不出 URL 的就是 button。
