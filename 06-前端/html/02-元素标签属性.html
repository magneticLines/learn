元素、标签、属性

HTML 只有三个核心概念：元素、标签、属性。搞清楚它们的区别，后面所有内容都会顺畅很多。

一、三个概念的关系

很多人混用"标签"和"元素"，实际上它们不是同一个东西：

概念定义举例
标签（Tag） 尖括号包裹的标记符，分开始标签和结束标签 <p> 和 </p>
元素（Element） 开始标签 + 内容 + 结束标签，构成一个完整的单元 <p>Hello</p>
属性（Attribute） 写在开始标签里，为元素提供额外信息 <a href="..."> 里的 href
口诀：标签是括号，元素是整体，属性是配置。说"这个标签"通常没问题，但技术文档里"元素"更准确。

二、元素的完整解剖

以一个带属性的链接为例，拆开来看每个部分叫什么：

── 元素解剖图 ──
<a href="https://example.com" target="_blank">点我</a>
a 标签名，决定元素的类型和语义
href / target 属性名，每个元素支持哪些属性由规范定义
"https://..." / "_blank" 属性值，用双引号包裹（单引号合法但不推荐）
点我 内容（Content），可以是文本或子元素
属性值要加引号。虽然 href=https://example.com 在简单情况下不报错，但一旦值里有空格或特殊字符就会出错。始终加引号是好习惯。

三、空元素（Void Element）

大多数元素有开始标签 + 内容 + 结束标签。但有一类元素天生没有内容， 因此也没有结束标签，叫做空元素（Void Element）。

空元素写法
<!-- ✅ 正确：空元素没有结束标签 -->
<img src="photo.jpg" alt="风景">
<input type="text">
<br>
<hr>
<meta charset="UTF-8">
<link rel="stylesheet" href="style.css">

<!-- ❌ 错误：给空元素加了结束标签（浏览器会忽略，但语义上错误）-->
<img src="photo.jpg"></img>

<!-- ℹ️ 你可能见过 XHTML 风格的自闭合写法，HTML5 不要求但兼容 -->
<br />  ← 斜杠不是必须的

常见空元素速查：

标签用途
<img>图片
<input>表单输入
<br>换行（尽量少用，排版应交给 CSS）
<hr>水平分割线（语义是"主题切换"）
<meta>元信息
<link>外部资源链接（CSS、favicon）

四、属性详解

① 属性的三种写法

属性的三种形态
<!-- 1. 键值对（最常见）-->
<a href="page.html">链接</a>

<!-- 2. 布尔属性：只写属性名，值等于属性名本身，意思是"启用" -->
<input disabled>           ← 等价于 disabled="disabled"
<input required>
<video autoplay muted></video>

<!-- 3. 全局属性：任意元素都可以加 -->
<div id="container" class="card" style="color:red" title="提示文字"></div>

② 重要全局属性速查

属性作用注意点
id 元素的唯一标识 页面内不能重复；JS 用 getElementById 定位；CSS 用 #id
class 元素的分类标签，可多个 class="btn btn-primary"，CSS 用 .class；最常用的属性
style 内联 CSS 优先级最高，但难维护，一般只用于动态样式
title 鼠标悬停提示 辅助说明，不是无障碍访问的替代品
hidden 隐藏元素（布尔属性） 等效 display:none，但语义上表示"暂时不相关"
lang 元素内容的语言 可局部覆盖 <html lang>，如页面中嵌入英文段落

③ 属性的大小写问题

HTML5 对大小写不敏感，但有约定
<!-- 这三种写法浏览器都能解析 -->
<INPUT TYPE="text">
<Input Type="text">
<input type="text">  ← 现代标准：全部小写

<!-- XHTML 要求小写，且属性值必须加引号 -->
<!-- React JSX 也要求小写（但部分属性名是驼峰：className）-->
最佳实践：标签名和属性名全小写，属性值始终用双引号。这既符合 HTML5 规范，也和 XHTML / JSX 习惯兼容。

五、嵌套规则

元素可以嵌套，但有规则。最重要的一条： 标签必须正确闭合，不能交叉。

合法 vs 非法嵌套
<!-- ✅ 正确：先开后关，像括号匹配 -->
<p>这是<strong>加粗</strong>文字</p>

<!-- ❌ 错误：交叉嵌套 -->
<p>这是<strong>加粗</p></strong>  ← 浏览器会自动修正，但结果不可预测

<!-- ❌ 错误：块级元素不能放在段落里 -->
<p>
  <div>我是块级</div>  ← <p> 只能包含行内内容
</p>

<!-- ❌ 错误：<a> 不能嵌套 <a> -->
<a href="#"><a href="#">嵌套链接</a></a>
浏览器有容错解析能力，会尝试自动修复非法嵌套，所以错误的 HTML 通常也能显示，但 DOM 树可能和你预期不同，导致 CSS 或 JS 失效。

用 VSCode + HTML 语言服务 写代码，错误嵌套会有波浪线提示。

六、常见陷阱与误区

陷阱后果正确做法
用 id 代替 class 做样式 CSS 选择器优先级极高，难以覆盖；且 id 不可复用 样式用 class，id 留给 JS 定位和锚点跳转
在 class 里写空格当分隔符 class="btnprimary" 是一个类名，不是两个 class="btn primary"，空格分隔多个类名
属性值不加引号 值里有空格时截断，class=btn primary 只识别 btn 始终加引号
忘记关闭非空元素 后续内容全部成为该元素的子节点 写完开始标签就立刻写结束标签，再填内容
把 <br> 当排版工具用 间距硬编码，响应式时无法调整 间距交给 CSS 的 margin / padding

七、与 Java 的类比

如果你有 Java 背景，这个类比有助于建立直觉：

HTMLJava 类比
元素（Element）对象实例（new Object()）
标签名（如 div、p）类名（决定类型和能力）
属性（Attribute）构造参数 / 字段初始值
内容（子元素）组合关系（has-a）
DOM 树对象图（Object Graph）
全局属性（id / class）继承自 Object 的方法（所有类共有）
DOM 树本质上是一个树形对象图，浏览器通过它管理页面状态。 JavaScript 操作 DOM 就是在操作这张对象图——和你在 Java 里操作对象没有本质区别。

八、与其他知识点的联系

  • 与块级/行内元素（下一节）：本节讲的是元素的"解剖结构"，下一节讲的是元素的"显示模式"——二者是正交的两个维度，同一个元素既有标签名、属性这些静态结构，也有 block/inline 这种默认渲染行为，缺一都不足以理解一个标签的完整面貌。
  • 与 CSS 选择器：本节提到的 id、class 全局属性，正是 CSS 章节里 #id、.class 选择器的匹配依据——HTML 里怎么命名属性，直接决定了 CSS/JS 里怎么定位元素，两者是同一枚硬币的两面。
  • 与 React JSX：本节提到 JSX 要求属性名小写但 className 是驼峰——这背后的原因是 JSX 本质是 JavaScript 对象语法而非真正的 HTML，class 是 JS 保留字所以才改名，是 HTML 属性系统迁移到虚拟 DOM 世界时留下的历史痕迹。
  • 与浏览器容错解析：本节"嵌套规则"一节提到浏览器会自动修正非法嵌套，这个容错机制正是 HTML 解析器（HTML Parser）里专门设计的"错误恢复算法"，与 XML 解析器遇到格式错误直接抛异常的严格模式形成鲜明对比，也是 HTML 比 XHTML 更"宽容"的根源。
  • 与无障碍访问（a11y）：本节提到 title 属性不是无障碍替代品，这是一个重要伏笔——图片的 alt、表单的 label 这些语义属性才是屏幕阅读器真正依赖的信息源，属性选得对不对直接决定了残障用户能不能用好这个页面。
