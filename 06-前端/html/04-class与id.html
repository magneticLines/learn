class 与 id

给元素打标签。理解 class 和 id 的本质区别，是写出可维护 CSS 和正确 JavaScript 的基础。

一、它们解决什么问题

HTML 标签本身（div、p、span…）只描述结构，不描述身份。 当页面上有 20 个 div，你怎么告诉 CSS "只给这 3 个加蓝色边框"，或者告诉 JS "找到那个弹窗"？

答案是给元素打标签——用 class 和 id 属性。 它们是 HTML、CSS、JavaScript 三者之间的沟通语言。

最简单的用法
<!-- 没有标识：三个 div 无法区分 -->
<div>内容 A</div>
<div>内容 B</div>

<!-- 用 class 打上"角色"标签（可重复）-->
<div class="card">内容 A</div>
<div class="card">内容 B</div>

<!-- 用 id 打上"唯一身份证"（全页唯一）-->
<div id="main-nav">导航</div>

二、核心差异

class（类）

  • 可重复：多个元素可同名
  • 可叠加：一个元素可有多个 class
  • CSS 选择器：.card
  • JS：querySelectorAll 返回多个
  • 类比：职务（"工程师"可以有很多人）

id（标识）

  • 全页唯一：一个 id 值只能用一次
  • 一个元素只有一个 id
  • CSS 选择器：#main-nav
  • JS：getElementById 返回单个
  • 类比：身份证号（全国唯一）
记住这个口诀：class 是"角色"，可以多人担任；id 是"身份证"，全局唯一。
写 CSS 样式 → 用 class；锚点跳转 / JS 精确查找 → 用 id。

三、语法细节

3.1 一个元素可以有多个 class

多 class 用空格分隔
<!-- 同时拥有 btn、btn-primary、large 三个 class -->
<button class="btn btn-primary large">提交</button>

/* CSS：三条规则都会作用到这个按钮 */
.btn         { padding: 8px 16px; border-radius: 4px; }
.btn-primary { background: #3b82f6; color: white; }
.large       { font-size: 1.1rem; }

这就是 CSS 组合式写法的基础：把样式拆成小的、单一职责的 class，需要哪些效果就叠哪些——这是现代 CSS 框架（如 Tailwind）的核心思想。

class 叠加演示：同一个标签可以戴多顶"帽子"
btn + btn-primary + large = 最终按钮样式

3.2 id 命名规则

id 的合法命名
<!-- ✅ 推荐：小写字母 + 连字符（kebab-case）-->
<section id="about-us"></section>
<div     id="modal-login"></div>

<!-- ✅ 也合法：camelCase（JS 操作方便）-->
<div id="loginModal"></div>

<!-- ❌ 避免：数字开头（CSS 选择器需要转义）-->
<div id="1st-section"></div>  ← CSS 里用 #\31 st-section 才能匹配，麻烦

3.3 id 与锚点跳转

id 还有一个特殊用途：页面内锚点。点击链接可以跳到页面内指定位置，这是 URL 中 # 符号的来源。

锚点跳转
<!-- 点击这个链接，页面滚动到 id="contact" 的位置 -->
<a href="#contact">联系我们</a>

<!-- ... 很多内容 ... -->

<section id="contact">
  联系方式内容
</section>

<!-- 也可以跨页面跳到指定锚点 -->
<a href="about.html#team">团队介绍</a>

四、CSS 中如何使用

CSS 选择器语法
/* class 选择器：点号开头 */
.card          { border: 1px solid #e2e8f0; }
.card.featured { border-color: #f59e0b; }  /* 同时有这两个 class */

/* id 选择器：井号开头 */
#main-nav      { position: sticky; top: 0; }

/* 组合使用 */
#sidebar .card { background: #f8fafc; }  /* #sidebar 内部的 .card */
陷阱：不要滥用 id 写 CSS 样式
id 的优先级（specificity）极高，一旦用 #login-btn 写了样式， 后续想用 class 覆盖它几乎不可能——你必须再写一个 id 选择器才能压过它。 实际项目约定：CSS 只用 class，id 留给 JS 和锚点。

五、CSS 优先级（Specificity）

当多条 CSS 规则作用到同一个元素时，浏览器用优先级得分决定哪条规则生效。 id、class、标签选择器的得分差异是初学者最常踩的坑。

优先级计算方式：(id 数量, class 数量, 标签数量)，按位比较。

标签 div
(0,0,1)
类 .card
(0,1,0)
类+标签 div.card
(0,1,1)
双类 .card.active
(0,2,0)
id #main-nav
(1,0,0)
优先级冲突示例
.btn        { color: blue; }    /* (0,1,0) */
#submit-btn { color: red; }     /* (1,0,0) ← 赢，红色生效 */

/* 想用 class 覆盖 id？加 !important（强烈不推荐）*/
.btn        { color: green !important; }  /* 优先级核弹 */
工程实践：保持选择器"扁平化"，避免过深的嵌套和 id 选择器。 CSS 模块化方案（CSS Modules、BEM 命名法）本质上都是在解决"优先级战争"这个问题。

六、JavaScript 中如何使用

DOM 查询 API 对比
// ── 通过 id 查找（返回单个元素或 null）──
const nav = document.getElementById('main-nav')
const nav2 = document.querySelector('#main-nav')   // 等价写法

// ── 通过 class 查找（返回 NodeList，类似数组）──
const cards = document.querySelectorAll('.card')
cards.forEach(card => card.style.border = '1px solid red')

// ── 动态增删 class（最常用）──
const btn = document.querySelector('#submit-btn')
btn.classList.add('loading')       // 添加 class
btn.classList.remove('loading')    // 删除 class
btn.classList.toggle('active')    // 有则删，无则加
btn.classList.contains('active') // 检查是否包含

动态切换 class 是前端交互的核心模式： 用 JavaScript 控制"加不加某个 class"，配合 CSS 里对应 class 的样式， 就能实现菜单展开/收起、按钮加载状态、暗黑模式切换…… JS 负责逻辑，CSS 负责视觉，两者通过 class 名解耦。

经典模式：JS 控制 class，CSS 控制样式
/* CSS：定义两种状态的样式 */
.menu         { max-height: 0; overflow: hidden; transition: 0.3s; }
.menu.open    { max-height: 400px; }

// JS：只管加减 class，不碰样式
document.querySelector('#menu-btn').addEventListener('click', () => {
  document.querySelector('.menu').classList.toggle('open')
})

七、何时用 class，何时用 id

场景用什么原因
写 CSS 样式 class 可复用，优先级可控，不会引发优先级战争
页面内锚点跳转（href="#section"） id 浏览器锚点机制依赖 id
JS 精确定位唯一元素（弹窗、表单） id getElementById 是最快的 DOM 查询
JS 动态操作一批元素 class querySelectorAll 返回集合
关联 label 与 input id（配合 for） <label for="email"> 要指向唯一的 input
ARIA 无障碍关联 id aria-labelledby、aria-describedby 都引用 id

八、与 Java 的类比

这两个概念在 Java 后端里也有直觉对应：

HTML 概念Java 类比关键共同点
class 属性 Java @Service、@Repository 等注解
（同类型可以有多个 Bean）		 按"类型/角色"分组，可以有多个实例
id 属性 Spring Bean 的 name 或 @Qualifier("beanName") 全局唯一标识，精确指向某一个
querySelectorAll('.card') applicationContext.getBeansOfType(CardService.class) 按类型批量查找
getElementById('modal') applicationContext.getBean("modal") 按唯一名精确查找
本质上，class 和 id 都是元数据（metadata）—— 它们不影响 HTML 的内容和结构，只是给元素贴标签，供其他系统（CSS 引擎、JS 运行时、无障碍工具）识别和操作。 这和 Java 注解（@Entity、@Transactional）的本质是一样的：给代码加元信息，让框架识别和处理。

九、常见错误速查

错误后果正确做法
同一 id 在页面用了多次 getElementById 只返回第一个；CSS 行为不可预测；HTML 验证不通过 id 全页唯一；批量选择改用 class
用 id 写 CSS 样式（#btn { ... }） 优先级过高，后续 class 无法覆盖，陷入"!important 地狱" CSS 只用 class；id 留给 JS 和锚点
class 名用中文或特殊符号 CSS 选择器需要转义，容易出错 用英文 + 连字符，如 nav-item
用 JS 直接修改 element.style.xxx 控制视觉状态 内联样式优先级最高，CSS 完全失控 用 classList.add/remove，逻辑和样式分离
