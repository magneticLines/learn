HTML · Head 元数据

title / link / favicon

这三个标签决定了页面在浏览器标签页、书签、搜索结果里的"脸面"——用户和爬虫看到的第一印象。

一、<title>：页面标题

1.1 基本用法

title 标签
<head>
  <title>深入理解 JVM 内存模型 - Shea 的技术博客</title>
</head>

title 出现在三个地方：

深入理解 JVM 内存模型 - Shea…
① 浏览器标签页标题  ·  ② 书签名称  ·  ③ 搜索结果的蓝色标题链接

1.2 怎么写好 title

规则说明
长度 50～60 个字符。搜索结果中超出部分会被截断显示为"…"
格式 关键词 - 网站名 或 网站名 | 关键词，关键词放前面权重更高
唯一性 每个页面必须不同。全站用同一个 title 是 SEO 大忌
首页特例 首页可以直接写品牌名，如 Shea 的技术博客
陷阱：<title> 里只能是纯文本，不能放任何 HTML 标签。 写 <title><strong>标题</strong></title>， 浏览器标签页会原样显示 <strong>标题</strong> 这串字符。

二、<link>：外部资源链接

<link> 是空元素，放在 <head> 里， 用来声明当前页面与外部资源的关系（relationship）。 最常见的用途是引入 CSS 文件，但它能做的远不止这些。

2.1 引入 CSS（最常用）

引入样式表
<!-- 标准写法 -->
<link rel="stylesheet" href="/styles/main.css">

<!-- 引入多个 CSS -->
<link rel="stylesheet" href="/styles/reset.css">
<link rel="stylesheet" href="/styles/main.css">

<!-- media 属性：只在特定条件下应用 -->
<link rel="stylesheet" href="/styles/print.css" media="print">
<link rel="stylesheet" href="/styles/mobile.css" media="(max-width: 768px)">
link 引入 CSS vs style 标签内联：<link> 引入外部文件， 浏览器可以缓存，多页面复用，是生产环境标准做法。 <style> 把 CSS 写在 HTML 里，每次请求都要重新下载， 仅适合单页面或关键渲染路径的内联优化。

2.2 rel 属性常用值

rel 值用途示例
stylesheet 引入 CSS 样式表 href="/main.css"
icon 网站图标（favicon） href="/favicon.ico"
canonical 声明页面的规范 URL，告诉爬虫"这是正版" href="https://example.com/page"
alternate 页面的替代版本（RSS 订阅、多语言版本） type="application/rss+xml"
preload 预加载关键资源（下一节详讲） as="font"
dns-prefetch 提前解析域名 DNS，加速跨域资源 href="//fonts.googleapis.com"
manifest PWA 应用清单文件 href="/manifest.json"

2.3 canonical：解决重复内容问题

同一篇文章可能有多个 URL（带参数、带 www、http/https 混用）， 搜索引擎会认为是重复内容并分散权重。canonical 告诉爬虫"以哪个 URL 为准"。

canonical 用法
<!-- 这三个 URL 内容相同，canonical 指向官方版本 -->
<!-- https://example.com/article?source=twitter -->
<!-- https://example.com/article?utm_campaign=email -->
<!-- https://example.com/article  ← 这是规范版本 -->

<!-- 三个页面都加上这一行，告诉爬虫合并权重到规范版本 -->
<link rel="canonical" href="https://example.com/article">

三、favicon：网站图标

3.1 什么是 favicon

favicon（Favorites Icon）是显示在浏览器标签页、书签栏、地址栏左侧的小图标。 用户识别网站靠的主要就是这个图标，它是品牌识别的重要组成。

3.2 基本写法

favicon 声明方式
<!-- 最简单：把 favicon.ico 放在网站根目录，浏览器自动读取 -->
<!-- 无需任何 link 标签，但推荐显式声明 -->

<!-- 显式声明 ico（兼容性最好）-->
<link rel="icon" href="/favicon.ico">

<!-- 现代写法：指定 PNG 和尺寸 -->
<link rel="icon" type="image/png" sizes="32x32" href="/favicon-32.png">
<link rel="icon" type="image/png" sizes="16x16" href="/favicon-16.png">

<!-- SVG favicon（现代浏览器支持，无限清晰）-->
<link rel="icon" type="image/svg+xml" href="/favicon.svg">

<!-- Apple 设备：添加到主屏幕时显示的图标 -->
<link rel="apple-touch-icon" sizes="180x180" href="/apple-touch-icon.png">

3.3 各格式对比

.ico
可包含多尺寸（16/32/48px），兼容性最好，IE 也支持
推荐始终保留一个 favicon.ico 在根目录作兜底
.svg
矢量格式，任意尺寸不失真，文件极小，支持暗色模式适配
Chrome/Firefox/Safari 现代版本均支持，推荐新项目首选
.png
透明背景，16×16 和 32×32 是标准尺寸
不支持多尺寸，需要多个 link 标签分别声明
apple-touch-icon
iOS "添加到主屏幕"时使用，必须是 PNG，推荐 180×180px
没有声明时 iOS 会截图页面，效果很差

3.4 SVG favicon 适配暗色模式

favicon.svg 内部用媒体查询适配深色模式
<!-- favicon.svg 文件内容 -->
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 32 32">
  <style>
    circle { fill: #1e293b; }           /* 浅色模式 */
    @media (prefers-color-scheme: dark) {
      circle { fill: #f1f5f9; }         /* 深色模式 */
    }
  </style>
  <circle cx="16" cy="16" r="14"/>
</svg>

3.5 完整的 favicon 组合推荐

现代项目 favicon 起手式（兼顾覆盖率）
<!-- 现代浏览器：SVG 优先 -->
<link rel="icon" href="/favicon.svg" type="image/svg+xml">

<!-- 旧浏览器降级：ico 兜底。sizes="any" 本义是"矢量可缩放"，-->
<!-- 这里是惯用 workaround：促使支持 SVG 的现代浏览器优先选上面那条 SVG -->
<link rel="icon" href="/favicon.ico" sizes="any">

<!-- iOS 主屏幕图标 -->
<link rel="apple-touch-icon" href="/apple-touch-icon.png">
实用工具：不想手动生成各尺寸，直接用 realfavicongenerator.net 上传一张 512×512 的 PNG，它会自动生成 ico、svg、png 各尺寸以及对应的 HTML 代码片段。

四、三者的协作关系

完整 head 中三者的位置
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  <!-- title：标签页 + 搜索结果标题 -->
  <title>页面标题 - 网站名</title>

  <!-- favicon：标签页图标 -->
  <link rel="icon" href="/favicon.svg" type="image/svg+xml">
  <link rel="icon" href="/favicon.ico" sizes="any">

  <!-- link：引入 CSS -->
  <link rel="stylesheet" href="/styles/main.css">

  <!-- link：SEO canonical -->
  <link rel="canonical" href="https://example.com/current-page">
</head>

五、与 Java 的类比

HTML 概念Java / 后端类比
<title> API 接口文档的 summary 字段：对外展示的简短描述，搜索/导航靠它
<link rel="stylesheet"> Maven <dependency>：声明"我要用这个外部资源"，由构建工具/浏览器负责加载
<link rel="canonical"> 数据库里的主键 / 唯一约束：多条看似相同的记录，指定哪一条是权威来源
favicon 多格式降级（svg → ico） 服务发现里的优先级降级：先找本地缓存，没有找 Redis，没有找数据库——同一套"从优到劣依次尝试"的思路
<link rel="dns-prefetch"> 连接池预热：提前建立连接，真正需要时不用等待握手
