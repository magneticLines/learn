文档结构与基本语法

每一个网页都是一个 HTML 文档。本节讲清楚它长什么样、为什么这样写。

一、HTML 是什么

HTML（HyperText Markup Language，超文本标记语言）不是编程语言， 它是一种标记语言——用标签描述内容的结构和含义，告诉浏览器「这是标题」「这是段落」「这是图片」。

浏览器读取 HTML 文件后，将其解析成一棵树形结构（DOM 树），再结合 CSS 渲染成你看到的页面。

HTML 负责结构，CSS 负责样式，JavaScript 负责行为。 三者分工明确，是前端的核心原则。

二、一个最小的 HTML 文档

最小合法 HTML 文件
<!-- 声明文档类型，告诉浏览器这是 HTML5 -->
<!DOCTYPE html>

<!-- 根元素，lang 属性指定语言 -->
<html lang="zh-CN">

  <!-- 头部：页面元信息，不显示在页面上 -->
  <head>
    <meta charset="UTF-8">
    <title>我的第一个页面</title>
  </head>

  <!-- 主体：页面可见内容放这里 -->
  <body>
    <h1>Hello, World!</h1>
  </body>

</html>

三、四个核心部分详解

① <!DOCTYPE html>

放在文件第一行，不是 HTML 标签，是给浏览器的指令： 「用 HTML5 标准解析这个文件」。

如果省略，浏览器会进入怪异模式（Quirks Mode），用旧标准渲染， 导致布局出现意料之外的问题。

必须写在第一行，前面不能有任何内容，包括空格和换行。

② <html lang="zh-CN">

整个文档的根元素，所有内容都嵌套在它里面。 lang 属性声明页面主要语言，影响：

影响场景说明
屏幕阅读器用正确的语言发音朗读内容
浏览器翻译功能识别源语言，提示是否翻译
搜索引擎了解内容语言，提升本地化搜索效果
CSS 排版:lang() 伪类可针对语言设置字体、引号样式

③ <head>

存放元信息，内容不会直接显示在页面上。常见内容：

标签作用
<meta charset="UTF-8">声明字符编码，防止中文乱码，必写
<meta name="viewport" ...>控制移动端缩放，响应式必写
<title>浏览器标签页标题，SEO 重要字段
<link>引入 CSS 文件、favicon 等
<script>引入或内联 JavaScript

④ <body>

所有可见内容放这里：文字、图片、表单、视频…… 浏览器渲染给用户看的就是 body 里的内容。

四、HTML 的树形嵌套结构

HTML 文档是一棵树，父子关系由嵌套决定：

嵌套结构示意
<html>
  <head>  ← html 的子元素
    <title>...</title>  ← head 的子元素
  </head>
  <body>
    <h1>标题</h1>
    <p>段落</p>
  </body>
</html>

浏览器把这棵树存在内存里，叫做 DOM（Document Object Model）。 JavaScript 通过 DOM API 增删改查节点，CSS 通过选择器匹配节点设置样式。 三件套都围绕这棵树工作。

良好的缩进习惯让嵌套关系一目了然。约定：每一层缩进 2 个空格。

五、文件命名与保存

规则例子
扩展名用 .htmlindex.html
文件名全小写，用短横线分隔about-us.html
网站首页固定命名index.html（服务器默认加载此文件）
编码必须 UTF-8保存时选 UTF-8（无 BOM）

六、浏览器怎么处理 HTML

浏览器拿到 HTML 文件后，经过以下步骤：

简化的浏览器渲染流程
HTML 文件
  ↓ 解析（Parsing）
DOM 树  ←→  CSS → CSSOM 树
  ↓ 合并
渲染树（Render Tree）
  ↓ 布局（Layout）
计算每个元素的位置和尺寸
  ↓ 绘制（Paint）
像素输出到屏幕
这就是为什么 CSS 放 <head>、JS 放 </body> 前（或加 defer）—— 让浏览器先解析 HTML 结构，不被脚本阻塞。

七、常见错误

错误后果
忘写 <!DOCTYPE html>浏览器进入怪异模式，样式表现异常
编码不是 UTF-8中文显示乱码
标签未正确关闭DOM 结构错乱，样式/脚本行为异常
内容写在 <head> 里页面不显示该内容
嵌套错误（如 <p><div>...</div></p>）浏览器自动修正，但结果不可预测

八、与其他知识点的联系

  • 与元素、标签、属性（下一节）：本节讲的 <html>、<head>、<body> 本身就是元素，下一节会把"元素"这个概念拆解成标签、属性、内容三部分，是从"文档骨架"过渡到"骨架的组成单位"的自然延续。
  • 与 CSS 渲染机制：本节提到的 DOM 树只是浏览器渲染流程的一半，CSS 章节会讲 CSSOM 树如何与 DOM 树合并成渲染树、再经过布局和绘制变成像素——理解 DOM 树的形成，是理解整条渲染管线的第一步。
  • 与 JavaScript / DOM API：浏览器把 HTML 解析成的 DOM 树，正是 JavaScript 通过 document.getElementById、querySelector 等 API 操作的对象——写 HTML 时的嵌套结构，直接决定了后续用 JS 查找和修改节点的路径。
  • 与 meta / title / link（后续章节）：本节只提到 <head> 是"元信息容器"，后续章节会逐一展开 <meta>、<title>、<link> 这些具体标签如何影响 SEO、社交分享和资源加载，都是在这个骨架基础上的细化。
  • 与 script 加载策略：本节提到"JS 放 body 末尾或加 defer 避免阻塞解析"，这只是一句结论，后面的 script 加载策略章节会用完整篇幅讲清楚浏览器为什么会阻塞、async 和 defer 具体差在哪。
