React.lazy + Suspense 路由懒加载

用户只想看首页,却要先下载"设置页""订单详情页""后台管理页"全部代码打成的一个巨大 bundle——这就是没做代码分割的典型症状。路由级代码分割让每个路由只在真正被访问时才加载自己的 JS,首屏体积大幅缩小。React.lazy + Suspense 是实现它最基础的方式。

为什么需要路由级代码分割

不做分割时,打包工具(Vite/webpack)默认会把所有 import 到的模块打进一个(或很少几个) JS 文件。应用越大,这个文件越臃肿,而用户打开首页时,浏览器必须先把整个应用的代码下载、解析、执行完,才能看到第一屏内容——即使这一屏根本用不到"订单详情页"的代码。

不分割
main.js 包含所有路由代码
首屏必须等全部下载完
用户可能永远不会访问的页面
也要跟着首屏一起下载
路由级分割
每个路由是独立的 chunk 文件
首屏只加载当前路由需要的代码
访问其他路由时才按需下载
首屏体积、加载时间显著下降

React.lazy 的基本用法

React.lazy(loader) 接收一个返回 import() Promise 的函数,返回一个"懒加载组件"。这个组件第一次被渲染时才会真正触发 import() 去下载对应代码;下载完成前需要用 <Suspense> 包裹并提供 fallback,告诉 React "加载过程中显示什么"。

结合 createBrowserRouter 做路由级懒加载
import { lazy, Suspense } from 'react';
import { createBrowserRouter } from 'react-router-dom';

// 只有真正导航到 /orders 时,OrderList 对应的 chunk 才会被下载
const OrderList = lazy(() => import('./pages/OrderList'));
const Settings = lazy(() => import('./pages/Settings'));

const router = createBrowserRouter([
  {
    path: '/orders',
    element: (
      <Suspense fallback={<PageSkeleton />}>
        <OrderList />
      </Suspense>
    ),
  },
  { path: '/settings', element: (<Suspense fallback={<PageSkeleton />}><Settings /></Suspense>) },
]);

为什么 lazy 组件必须是 default export

lazy(loader) 要求 loader() 返回的模块对象里必须有 default 字段,因为 lazy 内部实现是直接取 module.default 作为要渲染的组件。如果页面组件用的是具名导出(export function Settings() {}),需要手动包一层转换。

具名导出需要手动转成 default export
// Settings.jsx 里如果写的是: export function Settings() {...}
const Settings = lazy(() =>
  import('./pages/Settings').then((module) => ({ default: module.Settings }))
);
常见踩坑: 页面组件用了具名导出,却直接 lazy(() => import('./pages/Settings')),运行时会报"组件是 undefined"这类错误——因为 module.default 根本不存在。约定路由页面组件统一用 export default,能省掉这层转换代码,也是社区的通行做法。
Suspense 边界放在哪一层? 可以整个应用只放一个顶层 Suspense,也可以像上面例子一样给每个路由单独包一层。后者的好处是:切换路由时只有对应内容区显示 loading,页头、侧边栏等公共布局不受影响,体验更好——这也是为什么很多项目会把 Suspense 放在布局组件里而不是应用最外层。
和路由篇的联系: createBrowserRouter 定义的 element 只是一段 JSX,懒加载组件放进去和放普通组件没有本质区别——真正起作用的是 lazy 让这段 JSX 对应的代码被单独打包,配合 Suspense 处理"代码还没下载完"这段真空期的 UI。

小结

  • 路由级代码分割让首屏只加载当前路由需要的 JS,避免把整个应用打成一个体积巨大的 bundle。
  • React.lazy(() => import('./Page')) 定义懒加载组件,渲染时才真正下载代码;必须配合 Suspense fallback 处理加载中的 UI。
  • 懒加载组件的模块必须有 default 导出,具名导出需要手动包一层 .then() 转换。
  • Suspense 边界放的位置决定了 loading 影响的范围,通常放在路由内容区而不是整个应用外层。
