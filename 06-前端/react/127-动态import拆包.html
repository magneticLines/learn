动态 import() 拆包

上一篇用的 import('./pages/Settings') 这种写法,其实和 React 没有关系——它是 ECMAScript 标准里的动态导入语法,任何 JS 项目都能用。React.lazy 只是在这个标准能力上包了一层"配合 Suspense 渲染"的糖衣。这一篇把动态 import 本身讲清楚。

静态 import vs 动态 import()

平时写的 import Foo from './Foo' 是静态导入:必须写在文件顶层,路径在编译期就要确定,打包工具会在构建阶段把它和当前模块打进同一个产物(或按静态分析规则打包)。而 import('./Foo')(函数调用形式)是动态导入:可以写在任何地方(函数体内、条件语句里),返回一个 Promise,在运行时才决定要不要加载这个模块。

静态 import
import Foo from './Foo';
// 编译期确定依赖关系
// 默认打进主 bundle
// 语法上只能写在顶层
动态 import()
import('./Foo').then(mod => ...)
// 返回 Promise，运行时才加载
// 打包工具自动切成独立 chunk
// 可以写在函数体、条件分支里

打包工具怎么处理 import()

Vite(底层 Rollup)和 webpack 在构建时会静态分析代码,只要发现 import() 调用,就会自动把它指向的模块单独切出一个 chunk 文件,并在运行时通过插入 <script> 标签(或类似机制)按需加载这个 chunk——这一切都是自动的,不需要额外配置。

动态 import() 的基本用法(不经过 React.lazy)
async function loadChart() {
  // 只有真正调用 loadChart 时,echarts 这个大型库才会被下载
  const echarts = await import('echarts');
  const chart = echarts.init(document.getElementById('chart'));
  chart.setOption(someOption);
}

// 打包产物大致会变成:
// main.js          ← 不含 echarts,体积小
// echarts-xxxx.js  ← 独立 chunk,只在 loadChart() 执行时才下载

路由之外还能用在哪

场景说明
体积很大的第三方库如图表库 echarts、富文本编辑器、PDF 生成库——只有用到相关功能的用户才需要下载
条件渲染的重型功能比如管理员专属的后台面板,普通用户永远用不到,不该出现在普通用户的首屏 bundle 里
弹窗 / 对话框内容用户点击"导出报表"才弹出的复杂表单,没点击前不需要加载它的代码
polyfill 按需加载检测浏览器能力后,只在缺失某特性时才动态加载对应的 polyfill
非路由场景:点击按钮才加载重型库
function ExportButton() {
  const handleExport = async () => {
    // 用户不点"导出",xlsx 这个库永远不会被下载

    const sheet = utils.json_to_sheet(data);
    writeFile({ Sheets: { data: sheet }, SheetNames: ['data'] }, 'export.xlsx');
  };
  return <button onClick={handleExport}>导出</button>;
}
不一定要配 Suspense。 React.lazy 是专门用来懒加载组件并配合 Suspense 渲染的;而像上面导出按钮这种场景,加载的是普通函数/工具库,不需要渲染成组件,直接 await import() 拿到模块、手动处理 loading 状态(比如按钮转圈)即可,完全不需要 React.lazy。
常见误区:以为 import() 是 React 独有的 API。 面试里如果被问"动态导入和 React.lazy 是什么关系",容易答反。正确理解是:import() 是 JS 语言标准能力,打包工具负责拆包;React.lazy 只是在这个能力上加了一层"返回值必须是带 default 的组件模块,并且能被 Suspense 识别"的约定。没有 React,Vue、原生 JS 项目一样可以用 import() 做代码分割。
和上一篇的联系: 上一篇 lazy(() => import('./Page')) 里,真正触发拆包的就是这里的 import();lazy 只是把这个 Promise 包装成 React 能理解、能配合 Suspense 显示 loading 状态的组件。理解了这一层,就知道"懒加载"和"代码分割"其实是两个概念的组合:分割靠 import(),懒加载的 UI 处理靠 Suspense。

小结

  • 动态 import() 是 ES 标准语法,不是 React 专有能力,返回 Promise,可以写在任何位置。
  • 打包工具(Vite/webpack)看到 import() 会自动把对应模块切成独立 chunk,运行时按需加载。
  • 除了路由懒加载,还能用在按需加载大型第三方库、条件功能模块、弹窗内容等场景,不一定要配 Suspense。
  • 面试易错点:import() 和 React.lazy 不是一回事——前者是拆包机制,后者是"把拆包结果接入组件渲染流程"的封装。
