子传父:回调函数——事件向上冒泡

props 只能向下传数据,那子组件想把消息"反馈"给父组件怎么办?答案是:父组件把一个函数当作 props 传下去,子组件在合适时机调用它。这就是 React 里"数据向下、事件向上"的经典模式。

核心思路:传下去的不是数据,而是"回调"

父组件拥有 state,子组件想改它,但又不能直接碰(props 只读)。解法很自然:父组件写一个"改自己 state 的函数",把这个函数作为 props 传给子组件。子组件不需要知道函数内部干什么,只要在按钮点击、输入变化时调用它,把数据传回去即可。

父传函数,子调用
// 父组件:拥有 count,并把"如何改 count"封装成函数传下去
function Parent() {
  const [count, setCount] = useState(0);

  // 这个函数由父组件定义,操作的是父组件自己的 state
  function handleAdd(step) {
    setCount(prev => prev + step);
  }

  return (
    <div>
      <p>当前:{count}</p>
      // 把函数作为 props 传下去(注意:传函数本身,不加括号)
      <AddButton onAdd={handleAdd} />
    </div>
  );
}

// 子组件:收到 onAdd,点击时调用它,把 step 传回父组件
function AddButton({ onAdd }) {
  // 子组件不知道 onAdd 内部做了什么,只负责"在点击时通知"
  return <button onClick={() => onAdd(1)}>+1</button>;
}
父定义 handleAdd
↓ props
子拿到 onAdd
→ 点击
子调用 onAdd(1)
↑ 回到父
父 setCount
"数据向下、事件向上"(data down, events up)
这是 React 通信的核心模型:数据通过 props 向下流给子组件;子组件想反馈时,通过调用回调把事件/数据向上报给父组件,由父组件决定如何更新 state。子组件永远是"哑"的——它只描述发生了什么,不决定如何改数据。这与浏览器 DOM 事件冒泡的直觉一致:事件从内层向外层传递。

命名约定:on 开头的 prop,handle 开头的处理函数

社区约定俗成:传给子组件的回调 props 用 on 前缀(如 onAdd、onDelete),父组件内部的处理函数用 handle 前缀(如 handleAdd)。这让代码一眼能看出"哪些是对外暴露的事件接口"。

典型场景:表单输入回传父组件

子组件把输入值报给父组件
function SearchBar({ onSearch }) {
  const [text, setText] = useState('');
  return (
    <form onSubmit={e => { e.preventDefault(); onSearch(text); }}>
      <input value={text} onChange={e => setText(e.target.value)} />
      <button>搜索</button>
    </form>
  );
}

// 父组件:提交时才收到最终关键词,自己决定怎么处理
<SearchBar onSearch={kw => fetchResults(kw)} />

最高频陷阱:传函数时误加了括号

❌ 加括号:立即执行
<button onClick={onAdd(1)}>
// 渲染时就调用了 onAdd(1)!
// 传给 onClick 的是它的返回值
// 通常导致"渲染中改 state" 报错
// 或每次渲染就触发一次
✅ 包一层箭头函数
<button onClick={() => onAdd(1)}>
// 传给 onClick 的是一个函数
// 只有点击时才执行 onAdd(1)

如果回调不需要传参,可以直接写 onClick={onAdd}(传函数引用本身);需要传参时,必须用箭头函数包一层 onClick={() => onAdd(1)},否则会在渲染时立即执行。

识别信号:如果你的按钮"还没点就触发了"、或页面一加载就报 "Too many re-renders",九成是回调误加了括号,变成了渲染期间调用 setState。

小结

  • 子传父的本质:父组件把修改自己 state 的函数作为 props 传下去,子组件在事件发生时调用它。
  • 模式口诀:数据向下、事件向上。子组件只报告"发生了什么",父组件决定"怎么改"。
  • 命名约定:回调 props 用 on*,父组件处理函数用 handle*。
  • 需要传参时用 onClick={() => fn(arg)},不要写成 onClick={fn(arg)}——后者会立即执行。
