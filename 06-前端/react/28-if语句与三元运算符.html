条件渲染：为什么 JSX 里不能直接写 if

"登录了显示头像，没登录显示登录按钮"——这是最常见的界面需求。可你会发现，直接把 if 塞进 JSX 大括号里会报错。理解为什么，比背住写法更重要。

根本原因：JSX 大括号里只能放"表达式"

JSX 里的 {...} 不是任意代码块，它是一个嵌入表达式的槽位。表达式（expression）和语句（statement）是 JS 里两个不同的概念：

  • 表达式：能求出一个值的东西。如 1 + 2、a ? b : c、fn()、arr.map(...)。
  • 语句：执行一个动作，本身不产生值。如 if (...) {}、for (...) {}、switch。

为什么这个区分很关键？因为 JSX 会被 Babel 编译成 React.createElement(...) 的函数调用，你写在 {} 里的东西最终成了函数的参数。函数参数位置只能放值（表达式），你没法把 if 语句当参数传进去：

JSX 编译后：{} 里的内容成了函数参数
// 你写的 JSX
<h1>{count + 1}</h1>

// Babel 编译后——大括号里的东西成了参数，必须是"值"
React.createElement('h1', null, count + 1);

// 想象把 if 塞进去——语法根本不成立
// React.createElement('h1', null, if (x) {...})  ← 参数位置放语句，报错
一句话记住：JSX 的 {} 里放的是"能返回一个值的东西"。if 不返回值，所以不能放；三元 a ? b : c 返回值，所以可以放。这不是 React 的"规定"，而是 JS 语法的天然结果。

方案一：三元运算符（JSX 内部二选一）

三元运算符 条件 ? 真值 : 假值 是个表达式，能塞进 {}。它是 JSX 里做"二选一"渲染的首选：

三元运算符：登录状态二选一
function Header({ isLoggedIn, user }) {
  return (
    <header>
      {isLoggedIn
        ? <Avatar name={user.name} />          // 条件为真时渲染这个
        : <LoginButton />}              // 为假时渲染这个
    </header>
  );
}
陷阱：三元嵌套地狱。初学者容易写出 a ? x : b ? y : c ? z : d 这种多层嵌套三元，几乎无法阅读和维护。超过一层嵌套，就应该改用下面的"提前 return"或抽成子组件。

方案二：把 if 提到 return 外面（推荐用于复杂分支）

JSX 里不能写 if，但函数体里当然可以。把判断逻辑放在 return 之前，用变量存好要渲染的内容，或者直接提前 return：

✅ 提前 return
function Profile({ user }) {
  // 数据没到，直接返回占位
  if (!user) {
    return <Spinner />;
  }
  // 走到这里 user 一定存在
  return <div>{user.name}</div>;
}
✅ 用变量承载
function Status({ code }) {
  let content;
  if (code === 200) content = <Ok />;
  else if (code >= 500) content = <Err />;
  else content = <Unknown />;

  return <div>{content}</div>;
}

提前 return（early return / guard clause）是处理"加载中""无数据""无权限"这类边界情况的经典模式：先把这些异常/边界分支挡在前面 return 掉，主体逻辑就能保持在一条清爽的主线上，不用层层嵌套。

联系：为什么 return null 能"什么都不渲染"？React 约定：组件返回 null 表示"这次不渲染任何东西"（不是报错，也不是空白占位）。这在"条件性地整块隐藏组件"时非常有用。null、false、undefined 都不会渲染——这一点在第 30 篇会专门讲。

三种写法怎么选

场景推荐写法理由
JSX 内部二选一（简单） {cond ? A : B} 就地表达，不打断结构
只在满足时渲染，否则啥都不显示 {cond && A} 短路渲染，见第 29 篇
整个组件是否渲染（加载/权限） 函数顶部提前 return 挡住边界，主线清爽
三个及以上分支 if/else 存变量 或 switch 比嵌套三元可读得多
小结
  • JSX 的 {} 只接受表达式；if 是语句、不返回值，所以塞不进去——这是 JS 语法决定的，不是 React 的规定。
  • JSX 内部二选一用三元运算符；分支多或涉及整个组件是否渲染时，把 if 提到 return 外面，用提前 return 或变量承载。
  • 三元嵌套超过一层就该重构；return null 表示"这次不渲染"。
