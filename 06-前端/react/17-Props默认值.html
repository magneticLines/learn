Props 默认值：defaultProps 与解构默认参数

组件是给别人用的——你没法保证使用者每次都会传所有 props。当某个 prop 缺失时，子组件拿到 undefined。默认值就是在"缺货"时自动备货的机制，让组件在不完整输入下依然能正常工作。

为什么需要默认值：undefined 的危害

当父组件忘记传（或者有意不传）某个 prop 时，子组件访问到的是 undefined。不处理的话有两种后果：

没有默认值时的两种灾难
function Avatar({ size, src, alt }) {
  return <img width={size} height={size} src={src} alt={alt} />;
}

// 父组件只传了 src，没传 size 和 alt
<Avatar src="/avatar.png" />

// 结果渲染出：<img width="undefined" height="undefined" src="/avatar.png" alt="undefined">
// 灾难一：width="undefined" 不是数字，图片尺寸完全混乱
// 灾难二：alt="undefined" 出现在页面上，对屏幕阅读器用户极不友好

如果 prop 缺失时还调用了它的方法（比如 src.split('/')），那就直接报错："Cannot read properties of undefined"——页面崩溃。

方式一：解构默认值（推荐，现代写法）

在参数解构时直接用 = 指定默认值，这是标准的 ES6 语法：

解构默认值——声明与默认值在同一处，一目了然
// size 没传时用 40，src 没传时用占位图，alt 没传时用空字符串
function Avatar({
  size = 40,                        // 数字默认值
  src = '/default-avatar.png',    // 字符串默认值
  alt = '',                         // 空字符串也是合法默认值
}) {
  return <img width={size} height={size} src={src} alt={alt} />;
}

// 各种使用场景
<Avatar />                              // size=40, src='/default-avatar.png', alt=''
<Avatar size={80} />                  // size=80（覆盖默认值），其余用默认
<Avatar src="/shea.png" alt="Shea" />  // src、alt 覆盖，size 用默认 40

方式二：defaultProps（旧写法，了解即可）

React 早期的默认值机制——在组件函数外部设置一个 defaultProps 对象属性：

defaultProps——旧写法，React 18+ 已标记为过时
function Avatar({ size, src, alt }) {
  return <img width={size} height={size} src={src} alt={alt} />;
}

// 默认值放在函数外面，和函数定义是分离的
// 看函数时不知道有默认值，需要滚到文件底部才能看到
Avatar.defaultProps = {
  size: 40,
  src: '/default-avatar.png',
  alt: '',
};

// React 18 控制台会警告：
// Warning: Avatar: Support for defaultProps will be removed from
// function components in a future major release.

两种方式对比

维度 解构默认值（推荐） defaultProps（旧写法）
声明位置 函数参数处，默认值在定义旁边 函数体外部，通常在文件末尾
可读性 函数签名即文档，一眼看到所有 props 和默认值 需要翻到文件底部才能看到默认值
TypeScript 支持 配合类型标注，类型推断完美 类型定义复杂，不够优雅
React 版本状态 推荐，永远有效 React 18+ 已标记为 deprecated，未来版本将移除
适用场景 所有场景 维护旧代码时会看到，新代码不应使用

一个容易犯的错：null 不会触发默认值

null vs undefined——默认值只对 undefined 生效
function Avatar({ src = '/default.png' }) {
  // src 的值取决于父组件传了什么
  return <img src={src} />;
}

// 不传 → undefined → 用默认值 '/default.png' ✅
<Avatar />

// 传 undefined → 同上，用默认值 ✅（显式传 undefined 和不传等价）
<Avatar src={undefined} />

// 传 null → 不用默认值！null 是有意义的值，表示"明确设为空" ❌
// src 会是 null，<img src={null}> 不显示图片，也不用默认图
<Avatar src={null} />
null 和 undefined 的语义差异：undefined 表示"没有传"，null 表示"明确告诉你：这里没有值"。ES6 解构默认值只在 undefined 时触发，null 被视为合法的值传入。如果你的业务中可能出现 null，要在组件内部用 src ?? '/default.png' 做兜底（空值合并运算符对 null 和 undefined 都生效）。
本节核心：① 父组件未传的 prop 在子组件中是 undefined，不处理会导致渲染异常；② 推荐在解构参数处设默认值，声明与默认值在同一位置，可读性最好；③ defaultProps 是旧写法，React 18+ 已废弃，看旧代码会遇到，新代码用解构默认值；④ 默认值只对 undefined 生效，null 不会触发默认值，这是 JavaScript 语言层面的设计。
