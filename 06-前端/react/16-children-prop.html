children prop：组件的"插槽"

你有没有想过，<div> 为什么可以包裹任何内容？因为 HTML 天生支持"嵌套"。React 用 children prop 把同样的能力带给了自定义组件——你能造出像 <div> 一样"什么都能装"的容器组件。

children 是什么：JSX 标签之间的内容

当你写 <Card>内容</Card> 时，标签之间的"内容"会被 React 自动收集起来，以 children 这个特殊的 prop 传给 Card 组件。这不是 React 魔法，只是约定：JSX 里的"子节点"就是 children prop。

children 的传递——自动发生，无需手写
// 使用时：标签之间的内容就是 children
<Card title="用户信息">
  <p>姓名：Shea</p>
  <p>年龄：25</p>
</Card>

// React 内部等价于：
// Card({ title: '用户信息', children: [<p>姓名：Shea</p>, <p>年龄：25</p>] })
// children 是普通 prop，只是名字被 React 保留了

典型用法：定义通用容器组件

容器组件只关心"外框长什么样"，不关心"里面装什么"。children 是实现这种分离的关键——组件定义外框，使用者决定内容。

Card 容器组件——外框与内容分离
// 定义：Card 只负责外框的样式，不知道内部放什么
function Card({ title, children }) {
  return (
    <div style={{ border: '1px solid #ccc', borderRadius: '8px', padding: '16px' }}>
      <h3>{title}</h3>
      <hr />
      {/* 这里渲染"塞进来"的内容——Card 不需要知道 children 是什么 */}
      <div>{children}</div>
    </div>
  );
}

// 使用：同一个 Card，装不同的内容
<Card title="用户信息">
  <p>姓名：Shea</p>
  <button>编辑</button>
</Card>

<Card title="数据图表">
  {/* children 可以是另一个组件——Card 完全不知道，也不需要知道 */}
  <SalesChart data={chartData} />
</Card>

<Card title="公告">
  系统将于今晚 22:00 维护，请提前保存数据。
</Card>

children 可以是任何东西

children 的类型没有限制——React 能渲染什么，就能作为 children 传入：

children 的类型示例备注
字符串 <Card>纯文字内容</Card> 最简单的情况
JSX 元素 <Card><p>段落</p></Card> 最常见的情况
多个元素（数组） <Card><p/><img/></Card> 多个兄弟元素自动成为数组
另一个组件 <Card><UserList /></Card> 组合的核心——组件套组件
表达式 <Card>{condition ? <A/> : <B/>}</Card> 动态内容
undefined（无内容） <Card /> 或 <Card></Card> children 是 undefined，{children} 不渲染任何东西

进阶：JSX 作为 Prop 传入（多个插槽）

children 只是一个"默认插槽"。当组件需要多个插槽时，把 JSX 作为普通命名 prop 传入——这完全合法，因为 JSX 表达式求值后就是一个普通的 JavaScript 值。

多插槽布局——用具名 prop 传 JSX
// 一个有 header、sidebar、主内容三个区域的布局组件
function PageLayout({ header, sidebar, children }) {
  return (
    <div>
      {/* header 插槽——通过具名 prop 传入的 JSX */}
      <header>{header}</header>
      <div style={{ display: 'flex' }}>
        {/* sidebar 插槽 */}
        <aside>{sidebar}</aside>
        {/* 主内容插槽——来自 children（标签之间的内容） */}
        <main>{children}</main>
      </div>
    </div>
  );
}

// 使用：不同的"插槽"填入不同的 JSX
<PageLayout
  {/* header 和 sidebar 作为具名 prop 传入——{ } 里直接写 JSX */}
  header={<NavBar logo="MyApp" />}
  sidebar={<MenuList items={menuItems} />}
>
  {/* 标签之间的内容是 children——主内容区 */}
  <ArticleContent id={articleId} />
</PageLayout>
children 只是特殊的 prop 名，不是魔法：你可以把任意 JSX 赋给任意 prop 名（header={<Header />}）。children 的特殊之处只有一点——标签之间的内容会自动成为 children，而其他具名 prop 必须显式在 JSX 标签上写出来。本质上 <Card>内容</Card> 和 <Card children="内容" /> 完全等价（但前一种更易读）。
本节核心：① children 是 React 的特殊保留 prop，JSX 标签之间的内容自动赋值给它；② 典型场景是容器/布局组件——只管外框，不管内容；③ children 可以是字符串、JSX、数组、另一个组件，什么都行；④ 需要多个"插槽"时，把 JSX 作为具名 prop 传入，children 只是"默认插槽"；⑤ 这是 React 组件可组合性的核心机制之一。
