React · 列表渲染

key：给列表里每个元素一张"身份证"

列表渲染时那个总要写的 key，不是形式主义。它是 React 在更新列表时判断"谁是谁"的唯一依据。理解 key 的作用，就理解了 React 的 Diff 算法为什么需要它。

作用：帮 Diff 算法识别元素的"身份"

当列表数据变化（增、删、排序、修改），React 需要对比"旧的元素列表"和"新的元素列表"，决定哪些 DOM 复用、哪些新建、哪些删除——这就是 Diff。问题是：React 怎么知道新列表里的某个 <li> 和旧列表里的哪个是"同一个"？

如果没有额外信息，React 只能按位置（下标）逐个对比：新列表第 0 个 vs 旧列表第 0 个……这在"列表顺序变了"时会严重误判。key 就是解决方案——你给每个元素一个稳定的标识，React 就改用 key 来配对：

旧列表
key: a,b,c
→
按 key 配对
不是按位置
→
a→a b→b c→c
精准复用
本质类比：key 就像数据库表的主键，或者每个人的身份证号。没有它，React 只能靠"排队的第几位"认人——一旦有人插队或离开，后面所有人的身份都会被认错。有了 key，无论怎么增删排序，React 都能一眼认出"这个元素还是原来那个"，从而正确地复用它的 DOM 和内部 state。

规范一：稳定、唯一——用数据自身的 id

好的 key 有两个硬性要求：稳定（同一条数据的 key 在多次渲染间不变）和唯一（同一列表内不重复）。数据的业务 id 是天然的最佳选择：

✅ 用稳定的业务 id
{users.map(u =>
  <li key={u.id}>
    {u.name}
  </li>
)}
// u.id 跟着数据走，永远稳定
❌ 每次渲染都变的 key
{users.map(u =>
  <li key={Math.random()}>
    {u.name}
  </li>
)}
// 每次渲染 key 全变→全部重建
// 性能灾难，状态全丢
反例警示：key={Math.random()} 或 key={Date.now()}。这类"每次都变"的 key 是最坏的做法——每次渲染 React 都认为所有元素都是全新的，于是销毁全部旧 DOM、重建全部新 DOM，key 的优化意义完全丧失，还会导致输入框失焦、动画重置、组件 state 丢失等问题。key 的价值恰恰在于"稳定"。

规范二：key 只需在"兄弟之间"唯一

key 不需要全局唯一，只需在同一个父节点下的兄弟元素之间唯一即可。不同列表、不同层级可以有相同的 key，它们各自独立：

两个列表各自的 key=1 互不冲突
<div>
  <ul>
    {fruits.map(f => <li key={f.id}>{f.name}</li>)}
  </ul>
  <ul>
    {/* 这个列表的 key=1 和上面那个 key=1 不冲突，兄弟范围不同 */}
    {veggies.map(v => <li key={v.id}>{v.name}</li>)}
  </ul>
</div>

规范三：key 放在 map 直接返回的最外层元素上

key 要写在 map 回调直接返回的那个元素上，而不是它内部的子元素。用 Fragment 包裹时，需要用完整写法 <React.Fragment key=...>（简写 <> 不能带属性）：

需要包裹多个元素时用带 key 的 Fragment
{items.map(item => (
  // key 放在最外层的 Fragment 上，而不是里面的 dt/dd
  <React.Fragment key={item.id}>
    <dt>{item.term}</dt>
    <dd>{item.desc}</dd>
  </React.Fragment>
))}
缺失 key 的警告。如果 map 出来的元素没写 key，控制台会报："Warning: Each child in a list should have a unique 'key' prop." 这不是致命错误，页面还能渲染，但意味着 Diff 退化成按下标对比——在列表会变动时可能引发状态错位 bug（下一篇专门讲）。看到这个警告务必补上 key，别忽略。另外：key 不是普通 prop，子组件读不到 props.key，它是 React 保留给 Diff 用的；如果子组件也需要这个值，得另传一个 prop（如 id={item.id}）。
小结
  • key 是元素的"身份证"，让 React 的 Diff 按 key 配对而非按下标，从而正确复用 DOM 与组件 state。
  • 规范：稳定 + 唯一，首选数据业务 id；严禁 Math.random()/Date.now() 这种每次都变的 key。
  • key 只需兄弟间唯一；放在 map 直接返回的最外层元素上；key 不会作为 prop 传给子组件。
