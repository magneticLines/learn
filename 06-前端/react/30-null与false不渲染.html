哪些值不渲染：null、false、undefined、true

上两篇的三元和短路渲染，能成立的底层前提是——React 对某些值"视而不见"。这一篇把 React 渲染引擎到底忽略哪些值彻底讲清，这样前面所有条件渲染技巧你就都能推导出来了。

规则：这四个值 React 渲染为"空"

当 JSX 表达式的值是 null、false、undefined、true 之一时，React 什么都不渲染（不生成任何 DOM 节点、也不报错）：

这四个都渲染为空——页面上什么都不出现
<div>{null}</div>       // 渲染为 
，内部空
<div>{false}</div>      // 空
<div>{undefined}</div>  // 空
<div>{true}</div>       // 空（注意：true 也不渲染！）
为什么这样设计？因为这正是条件渲染需要的"逃生口"。设想 {cond && <X/>}：条件为假时表达式求值为 false，React 忽略它，于是"啥都不显示"。true 也不渲染，是为了让 {cond && <X/>} 里那个万一为 true 的中间值不至于把 "true" 字样打到页面上——虽然实践中你不会单独渲染布尔。这套约定让"用逻辑值控制渲染"变得自然。

对比：哪些值会被渲染

与"不渲染"的值相对，下面这些值 React 会当成实际内容渲染出来——上一篇的 0 陷阱就藏在这张表里：

值是否渲染说明
null❌ 不渲染最常用的"什么都不显示"
undefined❌ 不渲染组件无 return 时也是它
false / true❌ 不渲染布尔值都被忽略
0✅ 渲染 "0"陷阱！数字是有效内容
NaN✅ 渲染 "NaN"也是数字，会显示出来
''（空串）✅ 渲染空文本不报错，但生成空文本节点
[]（空数组）✅ 渲染为空数组被摊平，空数组=无内容
'hello' / 42✅ 渲染文本字符串、非零数字正常显示

实用模式：条件返回 null 整块隐藏组件

最经典的用法是让整个组件"这次不出现"。比如一个提示条，没有消息时就完全不该出现在 DOM 里：

组件返回 null = 这次我不渲染任何东西
function Banner({ message }) {
  // 没有消息时，整个组件返回 null——不产生任何 DOM
  if (!message) return null;

  return <div className="banner">{message}</div>;
}

// 父组件照常用，无需在外面再判断
<Banner message={errorText} />  // errorText 为空时 Banner 自动隐藏
返回 null vs 用 CSS 隐藏，有区别吗？有。return null 时组件不产生任何 DOM 节点；而 display: none 是节点仍在 DOM 里、只是视觉隐藏。前者更"干净"，但要注意：组件返回 null 时它仍然是被挂载的（其内部 state 在重新渲染出来后是全新的）。如果需要隐藏时保留内部状态，反而应该用 CSS 隐藏而不是返回 null。

陷阱回顾：空数组和 0

❌ 用 length 短路
{items.length &&
  <List items={items} />}
// items 为空 → length 是 0
// → 页面渲染出 "0"
✅ 转布尔或三元
{items.length > 0 &&
  <List items={items} />}
// 空数组时求值 false → 不渲染

{items.length
  ? <List items={items} />
  : <Empty />}  // 空态给占位
小结
  • React 忽略（渲染为空）的值：null、undefined、false、true。
  • 会被渲染的坑：0 → "0"、NaN → "NaN"、'' → 空文本节点。短路渲染的左边务必是布尔。
  • return null 让整个组件"这次不渲染"，不产生 DOM；需要保留内部状态时改用 CSS 隐藏。
