组件命名规范：为什么必须 PascalCase

PascalCase 不是"最好写成这样"，而是"必须这样写"——否则 React 无法区分你的组件和 HTML 原生标签。这背后是 JSX 编译规则，理解这个规则，你就彻底明白了为什么。

PascalCase 是什么

PascalCase（帕斯卡命名法）：每个单词的首字母都大写，单词之间无分隔符。

✅ PascalCase 示例
UserProfile
NavBar
LoginForm
ShoppingCart
ProductDetailPage
其他命名风格（不用于 React 组件）
userProfile    ← camelCase（变量/函数）
nav-bar        ← kebab-case（HTML 属性）
login_form     ← snake_case（Python 风格）
BUTTON         ← SCREAMING_SNAKE（常量）

为什么强制大写开头：JSX 编译规则决定的

这是本节最核心的内容。JSX 不是真正的 HTML，它会被 Babel 编译成 React.createElement() 调用。标签首字母的大小写，决定了第一个参数是字符串还是函数引用。

JSX 编译过程——字符串 vs 函数引用
// JSX 你写的：
<div className="box">内容</div>
<UserCard name="Alice" />

// Babel 编译后：
createElement('div', { className: 'box' }, '内容')
//              ↑ 字符串 → 交给浏览器创建原生 DOM 节点

createElement(UserCard, { name: 'Alice' })
//              ↑ 函数引用 → React 调用这个函数，拿到 JSX 继续处理

关键点：Babel 看的是标签名的首字母，不是你起的名字叫什么。大写 → 在当前作用域找同名变量（你定义的组件函数）；小写 → 直接传字符串给 React，让浏览器处理 HTML。这个规则写死在 Babel 的 JSX 转换逻辑里，没有例外。

如果组件小写开头会发生什么
// 定义时小写——语法上合法，但用起来出问题
function userCard() {
  return <div>用户卡片</div>;
}

// 调用时：
<userCard />

// 编译后：
createElement('userCard', null)
// React 把它当成 HTML 标签字符串，不调用你的函数
// 浏览器不认识 <userCard>，渲染成空的自定义元素
// 页面空白，且不报"找不到组件"的错误——很难排查

正确 vs 错误命名对照表

✅ 正确❌ 错误错误原因
UserProfile userProfile 首字母小写，Babel 编译成字符串，React 当 HTML 处理
NavBar nav-bar 连字符是 HTML 属性风格，在 JS 里不是合法标识符
LoginForm login_form 下划线是 Python/snake_case 风格，和 React 生态不一致
Button BUTTON 全大写容易和常量混淆，可读性差

文件命名约定：与组件名一致

社区约定（也是 Create React App、Vite 模板的默认做法）：文件名和其中导出的组件名保持完全一致。

src/components/
  UserCard.jsx    ← 文件名 = 导出组件名
  NavBar.jsx
  LoginForm.jsx
  Button.jsx
UserCard.jsx
// 文件名 UserCard.jsx，导出名也是 UserCard
// 这样 IDE 自动导入时路径和名称都能对上
export default function UserCard({ name, avatar }) {
  return (
    <div className="user-card">
      <img src={avatar} alt={name} />
      <span>{name}</span>
    </div>
  );
}

这个约定的好处：看到 import UserCard from './UserCard'，你立刻知道去哪个文件找它；ESLint 的 react/display-name 规则也能正确识别组件名用于调试。

高频问题：组件渲染出来是空白，不报错
遇到这种情况，第一反应是检查组件名是否小写开头。症状：写了组件、传了 props，页面就是不显示，React DevTools 也找不到你的组件——十有八九是组件名首字母小写了，导致 React 把它当成 HTML 未知标签，安静地渲染成空节点。这是 React 里最"静默"的错误之一，不抛异常，只是什么都不渲染。
延伸：动态组件名必须用大写变量
如果你要动态渲染组件（比如根据 type 决定渲染哪个组件），必须把它赋给大写变量：
const Component = componentMap[type]; 然后 <Component />
不能直接写 <componentMap[type] />——JSX 里不支持表达式作为标签名，必须先提取到大写变量。
小结：① PascalCase = 每个单词首字母大写，无分隔符；② 强制大写的原因是 Babel 编译规则——大写编译成函数引用，小写编译成 HTML 字符串；③ 文件名与组件名保持一致，方便 IDE 和 ESLint；④ 静默空白 + 无报错 = 先查组件名首字母。
