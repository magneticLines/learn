React · 函数组件

定义与调用组件：返回 JSX 的函数

组件是 React 的基本单元。理解"定义"和"调用"这两个动作，就理解了 React 工作的核心机制——React 本质上就是在反复调用你写的这些函数，把返回的 JSX 拼成真实 DOM。

函数组件的本质：接收 props，返回 JSX

函数组件就是一个普通的 JavaScript 函数。它没有任何魔法，满足两个要求即可：

  1. 函数名以大写字母开头（PascalCase）
  2. 返回 JSX 或 null

这和 UI = f(state) 公式完全对应：函数接收数据（props / state），输出界面描述（JSX），React 把这个描述渲染成真实 DOM。函数组件没有实例，没有 this，每次渲染就是调用一次函数。

最简单的函数组件
// props 是父组件传进来的数据对象——这里解构出 name
function Greeting({ name }) {
  // 返回 JSX，React 把它渲染成真实 DOM
  return <h1>你好，{name}</h1>;
}

三种写法：完全等价

写函数组件有三种语法，React 对它们一视同仁——选你团队统一的风格就好，不要在同一项目里混用。

三种写法对比——运行结果完全相同
// ① function 声明：最常见，有函数提升，可以在定义前调用
function Button() {
  return <button>点击</button>;
}

// ② 箭头函数 + 显式 return：灵活，可以有多行逻辑
const Button = () => {
  return <button>点击</button>;
};

// ③ 箭头函数 + 隐式返回：JSX 只有一行时最简洁
//    注意：多行 JSX 需要用括号包裹，否则 return 后换行会触发 JS 的 ASI
const Button = () => <button>点击</button>;
何时用哪种？大多数项目用 function 声明定义组件——因为函数有名字（调试 DevTools 里显示友好），而且有提升特性，在文件底部定义、顶部使用也不报错。箭头函数更适合匿名回调或者搭配 export default 一起写。

调用组件：在 JSX 里像 HTML 标签一样写

定义好组件后，你不需要手动调用函数——在 JSX 里像写 HTML 标签一样写 <MyComponent />，React 会在渲染时自动调用这个函数，并把返回的 JSX 插入到当前位置。

定义 → 调用 的完整流程
// 定义两个组件
function Logo() {
  return <img src="/logo.svg" alt="logo" />;
}

function NavBar() {
  return <nav>导航</nav>;
}

// 在 App 里调用——React 渲染时依次执行 Logo() 和 NavBar()
function App() {
  return (
    <header>
      <Logo />    {/* React 调用 Logo()，把结果放在这里 */}
      <NavBar />  {/* React 调用 NavBar()，把结果放在这里 */}
    </header>
  );
}

调用方式有两种写法，都合法：<Logo />（自闭合，没有子内容时推荐）和 <Logo></Logo>（需要传子元素时用）。

组件 vs HTML 标签：首字母大小写是语法，不只是约定

React 用首字母大小写来区分两类东西：

写法JSX 编译结果渲染结果
<div> createElement('div', ...) HTML 原生 div 元素（字符串）
<Greeting /> createElement(Greeting, ...) 调用 Greeting 函数（函数引用）

关键区别在于：小写标签编译成字符串传给 createElement，React 把它交给浏览器处理原生 DOM；大写标签编译成函数引用，React 调用它得到 JSX 后继续处理。这是 Babel 编译 JSX 的规则，写死在语言层面，不是团队约定可以改的。

✅ 大写开头：组件
// Greeting 是函数引用
// React 会调用它
<Greeting name="Alice" />

// 编译成：
createElement(Greeting, {
  name: "Alice"
})
❌ 小写开头：当 HTML 处理
// greeting 被当成 HTML 标签
// 浏览器不认识，不渲染或报警告
<greeting name="Alice" />

// 编译成：
createElement('greeting', {
  name: "Alice"
})
高频陷阱：组件小写开头导致什么都不渲染
如果你写了 function greeting() { return <p>你好</p>; }，然后在 JSX 里用 <greeting />——页面空白，控制台可能报 "Unknown prop" 警告，但不报 "找不到组件"。原因：React 把 <greeting /> 当成了一个不认识的 HTML 自定义元素，压根没去找你的函数。这种问题有时很难发现，因为不抛异常。

组件可以在哪里被调用？

任何 JSX 里都可以使用组件，包括：

  • 其他组件的返回值里（最常见——父子嵌套）
  • 条件表达式里：{isLoggedIn && <UserMenu />}
  • 数组 map 里：{items.map(i => <Item key={i.id} />)}
  • 三元表达式里：{flag ? <A /> : <B />}
小结：① 函数组件 = 接收 props、返回 JSX 的普通 JS 函数，无实例无 this；② 三种写法（function 声明、箭头显式 return、箭头隐式返回）完全等价；③ 在 JSX 里写 <MyComponent /> 就是调用——React 负责执行；④ 大写开头是语法要求，让 Babel 把它编译成函数引用而非 HTML 字符串。
