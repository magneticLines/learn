Navigate:声明式重定向组件

上一章守卫里用到了 <Navigate>。它是 React Router v6 的声明式重定向——把"跳转"表达成"渲染一个组件"。理解它,以及它和命令式 useNavigate 的分工,是路由控制的关键。

Navigate 是什么:渲染即跳转

<Navigate to="..." /> 是一个组件,但它不渲染任何 UI——一旦被渲染出来,就会立即触发导航,把 URL 换成 to 指定的地址。这契合 React 的声明式思想:你不"命令"它跳,而是"声明"在某种条件下这里应该是另一个页面。

条件重定向:满足条件时渲染 Navigate
import { Navigate } from 'react-router-dom';

function Dashboard() {
  const user = useAuth();

  // 没权限:不渲染页面,渲染一个 Navigate → 立即跳走
  if (!user.isAdmin) {
    return <Navigate to="/no-permission" replace />;
  }
  return <AdminPanel />;
}
为什么 v6 用组件而不是 v5 的 <Redirect>?
React Router v5 有个专门的 <Redirect>。v6 把它统一成了 <Navigate>——因为"重定向"本质就是"导航到别处",没必要为它单开一个 API。名字从 Redirect 改为 Navigate,也强调了它和命令式 useNavigate 是同一件事的两种形态。如果你从 v5 迁移,把 <Redirect to="x" /> 换成 <Navigate to="x" replace /> 即可(v5 的 Redirect 默认就是 replace)。

replace vs push:两种改变历史的方式

浏览器历史是一个栈。导航默认是 push(压入新记录,可以按返回键回来);replace 则是替换当前记录(返回键不会回到被替换的页面)。

push /a → /b
⇒
历史栈: /a, /b(可退回 /a)
replace /a → /b
⇒
历史栈: /b(/a 被抹掉)

<Navigate> 默认是 replace 吗?不是——默认是 push,加 replace 属性才替换。但在"重定向"语义下,几乎总该用 replace:

❌ 忘加 replace(用了 push)
// 未登录访问 /settings
// 被 push 到 /login
// 历史: /settings, /login
// 用户按返回 → 回 /settings
// → 又被拦到 /login
// → 死循环,返回键失效
✅ 加 replace
// 未登录访问 /settings
// replace 到 /login
// 历史: /login(/settings 抹掉)
// 用户按返回 → 回到进入前
//   的正常页面
// 体验顺畅
经验法则:重定向类跳转(守卫拦截、登录后跳回、旧路径迁移)一律用 replace——因为被重定向掉的那个 URL 不该出现在用户的返回历史里。而用户主动点击导航(点菜单、点链接)用默认的 push,这样返回键能回到上一页。

Navigate vs useNavigate:声明式 vs 命令式

两者做的是同一件事(改变 URL),但触发时机不同:

<Navigate />(声明式)useNavigate()(命令式)
形态一个组件,渲染即跳一个函数,调用即跳
触发时机渲染期间(根据条件)事件回调 / 副作用里
典型场景渲染时判断"这里该显示别的页"(守卫、无权限)用户操作后跳转(提交表单成功、点按钮)
写法return <Navigate to="/x" replace />navigate('/x', { replace: true })
命令式:在事件回调里跳转
function CreatePost() {
  const navigate = useNavigate();

  async function handleSubmit() {
    const id = await savePost();
    // 提交这个"动作"完成后跳转 —— 命令式的天然场景
    navigate(`/posts/${id}`);
  }
  // ...
}
别在渲染期间调用 useNavigate 的 navigate()
navigate() 会产生副作用(改 URL),在组件函数体里(渲染期间)直接调用会报警告甚至出错,应放进 useEffect 或事件回调。而"渲染期间想跳转"的场景,正是 <Navigate> 存在的理由——它把跳转封装成了合法的"渲染输出"。

一句话:渲染里判断要跳 → 用 <Navigate>;事件里操作后跳 → 用 navigate()。

Navigate 也能带 state 和相对路径

Navigate 的常用属性
// 带 state(如守卫保存来源)
<Navigate to="/login" state={{ from: location }} replace />

// 相对当前路由跳转
<Navigate to=".." relative="path" />

// 相当于历史后退(等价 navigate(-1))
<Navigate to={-1} />
