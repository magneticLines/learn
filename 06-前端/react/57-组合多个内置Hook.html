组合多个内置 Hook：把积木拼成能力

自定义 Hook 的真正威力，不在于包一个 useState，而在于把 useState + useEffect + useRef 等多个内置 Hook 组合成一个高层抽象。内置 Hook 是积木，自定义 Hook 是你用积木搭出的成品。

组合示例：useDebounce

"防抖"——值停止变化一段时间后才生效，常用于搜索框。它需要 useState（存防抖后的值）+ useEffect（起定时器）+ 清理（取消上一个定时器）。三个内置能力协作，对外只暴露一个干净的接口。

useDebounce.js —— useState + useEffect 组合
function useDebounce(value, delay = 300) {
  const [debounced, setDebounced] = useState(value);

  useEffect(() => {
    // value 每变一次就重置计时；只有稳定 delay 毫秒后才写入
    const timer = setTimeout(() => setDebounced(value), delay);
    return () => clearTimeout(timer);  // 清理：value 又变了就取消上一个
  }, [value, delay]);

  return debounced;
}

// 用起来极简：搜索框输入停 300ms 才真正发请求
function Search() {
  const [text, setText] = useState('');
  const query = useDebounce(text, 300);
  useEffect(() => { if (query) fetchResults(query); }, [query]);
  // ...
}

组合示例：useRef 参与，拿到"上一次的值"

有时需要对比"这次和上次的值"。useRef 能在渲染之间存住一个不触发重渲的值，配合 useEffect 就能实现 usePrevious。

usePrevious.js —— useRef + useEffect 组合
function usePrevious(value) {
  const ref = useRef();   // ref.current 跨渲染保留，改它不触发重渲

  useEffect(() => {
    ref.current = value;  // 渲染提交后才更新 → 下次渲染读到的还是"上一次"
  }, [value]);

  return ref.current;   // 本次渲染期间，它还是上一次提交的值
}

// count 从 3 变 4 时，prev 是 3
const prev = usePrevious(count);

还能组合别的自定义 Hook

组合是可以叠层的——自定义 Hook 内部可以调用你写的其它自定义 Hook。比如 useSearch 里同时用 useDebounce（防抖输入）和 useFetch（拉结果），把两个能力再封装成一个更高层的能力。

思想：组合优于继承

class 时代想复用逻辑，只能靠继承基类、或套一层层 HOC / render props——层级深、耦合重、命名冲突多（"wrapper hell"）。Hook 换成了组合：把小能力横向拼起来，而不是纵向堆继承。

继承 / HOC(旧)Hook 组合(新)
复用方式纵向：套壳、继承基类横向：拼装小 Hook
组件树多层 wrapper 嵌套扁平，无额外层级
数据来源props 层层注入，难追踪调用处显式，一眼看清
组合多个能力嵌套地狱并列几行 useXxx
为什么组合更好？组合是"平的"——每个能力独立、可单独测试、可自由搭配，加一个能力就多写一行 useXxx，不改结构。继承是"竖的"——耦合到父类，改一处牵连一片。这也是整个软件设计里"组合优于继承"原则在 React 的具体体现。Hook 规则（只在顶层调用）正是为了让这种组合的状态位置可预测。

社区 Hook 库：别重复造轮子

很多常见 Hook（防抖、请求、本地存储、窗口尺寸、剪贴板……）社区早已封装好，质量高、覆盖边界情况。自己写来练手很好，生产项目里直接用成熟库更省心：

  • ahooks（阿里）：面向业务，Hook 丰富（useRequest、useDebounce、useAntdTable 等），中文文档友好。
  • react-use：社区老牌，数量庞大，覆盖传感器、生命周期、状态、副作用各类。
  • usehooks-ts：TypeScript 优先，轻量、代码可直接抄读，适合学习。
建议：先自己实现一遍常见 Hook（理解原理，尤其是清理和依赖），生产中再用成熟库（省心、边界处理全）。读这些库的源码也是学 Hook 组合的好途径——它们就是"内置 Hook 怎么拼"的最佳范例。

小结

  • 自定义 Hook 的威力在组合：把 useState/useEffect/useRef 等拼成高层能力。
  • useDebounce（state+effect）、usePrevious（ref+effect）是经典组合范例。
  • 自定义 Hook 还能调用其它自定义 Hook，逐层封装出更高层的抽象。
  • Hook 体现"组合优于继承"：扁平、可测、可自由搭配，胜过 HOC 的嵌套地狱。
  • 社区库 ahooks / react-use / usehooks-ts 已封装大量常用 Hook，生产可直接用。
