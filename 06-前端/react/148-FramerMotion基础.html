Framer Motion 基础

上一篇的 CSS transition 能应付简单场景,但一旦动画要和 state 强绑定(数值随手势实时变化)、要做交错(stagger)动画、或者要支持拖拽,纯 CSS 就开始力不从心了。Framer Motion 用声明式 API 把这些复杂动画变得像写普通 JSX 一样自然。

Framer Motion 解决什么问题

纯 CSS 动画的表达能力受限于"CSS 只能声明式地写死一套规则",遇到下面这些需求会明显吃力:

  • 动画值依赖 JS 计算的动态数据——比如根据鼠标位置实时移动元素,CSS 变量能凑合但很别扭
  • 交错动画(stagger)——列表里 10 个元素依次错开 0.05 秒入场,纯 CSS 要手写 10 个 animation-delay
  • 手势交互——拖拽、长按缩放,这些本质上需要 JS 监听指针事件并实时更新样式
  • 和 React state 联动——动画的起点/终点由组件状态决定,还要在动画进行中响应新的 state 变化(比如拖到一半松手要弹回)
类比: CSS transition/keyframes 像是"预先录制好的动画脚本",播放时机固定;Framer Motion 更像"实时演出"——动画的每一帧都可以基于当前 React state 动态决定去哪、怎么去,而且写法和你已经很熟悉的 JSX props 完全一致。

motion.div:把普通元素变成可动画的元素

Framer Motion 的核心是给每个 HTML 标签提供了一个"动画版"——motion.div、motion.button 等。用法和普通标签几乎一样,只是多了几个特殊 props:initial(初始状态)、animate(目标状态)、transition(过渡参数)。

最基础的声明式动画:淡入 + 上移
import { motion } from 'framer-motion';

function Card() {
  return (
    <motion.div
      initial={{ opacity: 0, y: 20 }}   // 初始状态:透明、向下偏移 20px
      animate={{ opacity: 1, y: 0 }}    // 目标状态:完全显示、回到原位
      transition={{ duration: 0.4, ease: 'easeOut' }}
    >
      卡片内容
    </motion.div>
  );
}
// 组件一挂载,Framer Motion 自动从 initial 插值到 animate,不需要手写 CSS 类

动画值跟随 state 联动

把 animate 的值改成依赖 state,动画会在 state 变化时自动重新计算目标值并平滑过渡——这正是"和 React state 自然联动"的体现:

animate 直接绑定 state,state 一变动画自动跟随
function Toggle() {
  const [on, setOn] = useState(false);
  return (
    <motion.div
      onClick={() => setOn(!on)}
      animate={{ x: on ? 24 : 0, backgroundColor: on ? '#4ade80' : '#475569' }}
      transition={{ type: 'spring', stiffness: 400, damping: 30 }}
    />
  );
}
// on 从 false 变 true,滑块自动带弹性地滑到 24px,颜色同时过渡——一行 CSS 都不用写
注意 transition 里的 type: 'spring'。 Framer Motion 内置了物理弹簧动画模型(用 stiffness 刚度、damping 阻尼描述),比 CSS 的贝塞尔曲线更接近真实物理手感,也是很多"高级感"UI 交互的秘诀。纯 CSS 要模拟弹簧效果只能用复杂的分段 cubic-bezier 硬凑。

对比:CSS transition vs Framer Motion

不要为了炫技而全面替换 CSS transition。 简单的悬浮变色、border 变化这类交互,纯 CSS 性能更好、零依赖,继续用 CSS 就好。把 Framer Motion 留给"CSS 做起来别扭"的场景——依赖 state 联动的复杂动画、手势交互、进出场动画。技术选型的核心问题永远是"这个场景 CSS 能不能优雅解决",而不是"新工具听起来更酷"。

小结

  • Framer Motion 解决的是纯 CSS 难以优雅表达的场景:动画值依赖 JS 动态计算、交错动画、手势拖拽、与 React state 强联动。
  • 核心 API 是 motion.div 等"动画版"标签,配合 initial(初始状态)、animate(目标状态)、transition(过渡参数)三个 props,写法和普通 JSX 一致。
  • animate 的值可以直接绑定 React state,state 变化时动画自动重新计算目标值并平滑过渡,不需要手动切换 CSS 类。
  • Framer Motion 内置弹簧物理动画模型(type: 'spring'),比 CSS 贝塞尔曲线更容易做出自然的高级感交互。
  • 不是所有动画都该迁移到 Framer Motion——简单交互用 CSS 更轻量,复杂交互(尤其涉及退出动画,下一篇会讲)再引入它。
