React · 函数组件

组件可以返回什么

组件是返回 JSX 的函数——但 React 对"返回值"的要求比"必须是 JSX 元素"更灵活，也更有边界。搞清楚哪些可以返回、哪些会报错，你就掌握了条件渲染和列表渲染的底层逻辑。

合法的返回值：五种

React 能处理以下五类返回值，都是"可渲染的"：

返回值类型效果常见场景
JSX 元素 渲染为对应的 DOM 结构 最常见的正常渲染
null 什么都不渲染，不报错 条件渲染：if (!show) return null
字符串 / 数字 直接渲染为文本节点 纯文本组件、Badge 数字
数组（含 JSX） 渲染多个节点，每项需要 key 较少直接用，一般用 map + Fragment
Fragment 渲染多个根节点，不产生额外 DOM 避免多余的 div 包裹

null：条件渲染的标准写法

return null 是 React 里"让组件消失"的正式方式。它不会报错，不会留下空节点，父组件也感知不到。这是条件渲染最干净的写法：

null 的典型用途——条件渲染
// show 为 false 时，组件直接消失——不渲染任何 DOM 节点
// 和 display:none 不同，这里连 DOM 节点都不存在
function Banner({ show, message }) {
  if (!show) return null;  // 早退——不需要 else

  return (
    <div className="banner">
      {message}
    </div>
  );
}

// 使用时——父组件完全不用关心 Banner 是否可见
function App() {
  const [visible, setVisible] = useState(true);
  return <Banner show={visible} message="欢迎回来" />;
}

字符串和数字：直接作为文本渲染

组件可以直接返回字符串或数字，React 会把它渲染成文本节点。这种用法不常见，但合法：

返回字符串/数字
// 适合纯展示文本的场景
function Version() {
  return 'v1.0.0';  // 直接渲染文本，不需要包裹标签
}

function Count({ n }) {
  return n;  // 数字也可以直接返回
}

返回数组：每项需要 key

组件可以返回一个 JSX 元素组成的数组，React 会依次渲染每一项。但每项必须有 key 属性，否则 React 无法在 diff 时追踪哪个节点变了：

返回数组——每项需要 key
// 这样可以渲染多个兄弟节点，且不产生额外的 DOM 包裹
function Terms() {
  return [
    <dt key="term">React</dt>,
    <dd key="def">用于构建 UI 的 JS 库</dd>,
  ];
}

// 实际上这种场景更常用 Fragment，可读性更好（Fragment 无需 key，作为单一根节点使用时不是数组）：
function Terms() {
  return (
    <>
      <dt>React</dt>
      <dd>用于构建 UI 的 JS 库</dd>
    </>
  );
}

不合法的返回值：会报错

以下返回值 React 无法渲染，会抛出错误：

❌ 返回普通对象
function Bad() {
  // 普通 JS 对象不是 React 元素
  // 报错：Objects are not valid
  // as a React child
  return { name: 'Alice' };
}
❌ 返回 Promise
function Bad() {
  // 客户端组件不能是 async
  // 返回 Promise 会报错
  // （服务端组件 RSC 是例外）
  return fetch('/api/data');
}
React Server Components (RSC) 的例外：在 Next.js App Router 等支持 RSC 的框架里，服务端组件可以是 async 函数并返回 Promise——但这只在服务端运行，客户端组件不支持这种写法。普通 React 项目里，组件必须是同步函数。
高频陷阱：return 后换行导致 return undefined
JavaScript 有自动分号插入（ASI）机制：如果 return 后直接换行，JS 会自动在 return 后面加分号，函数返回 undefined。React 遇到 undefined 会报错（注意：null 合法，undefined 不合法）。
❌ return 后换行：返回 undefined
function Card() {
  return   // ← JS 在这里插入分号
    <div>内容</div>;  // 这行被忽略
}
✅ 用括号包裹：安全换行
function Card() {
  return (  // ← 括号防止 ASI
    <div>内容</div>
  );
}
组件空白 + 报错 "Nothing was returned from render" 或 "undefined is not a valid element"——先查 return 后面有没有漏括号。
小结：① 合法返回值：JSX、null、字符串/数字、数组（需 key）、Fragment；② null 是条件渲染的标准写法，不渲染任何 DOM，不报错；③ 普通对象和 Promise 不合法，会报错；④ return 后直接换行触发 ASI 返回 undefined——用括号包裹多行 JSX 是固定写法。
