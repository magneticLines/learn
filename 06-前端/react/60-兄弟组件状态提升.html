状态提升:兄弟组件如何共享数据

props 只能父传子,回调只能子传父。那两个互为兄弟的组件想共享一份数据怎么办?React 给出的标准答案是状态提升(lifting state up)——把 state 提到它们共同的父组件里。

问题:兄弟之间没有直接通道

React 的数据流是"树形单向"的:数据只能沿组件树向下流。兄弟组件在树上是"平级"的,没有直接连线。如果 温度输入框 和 温度显示 是兄弟,输入框改了值,显示组件怎么知道?

兄弟 A(输入)
✕ 没有直接通道
兄弟 B(显示)

解法:找共同父组件,把 state 提上去

既然兄弟不能直连,那就让它们的共同父组件持有这份 state,再通过 props 分发给两个兄弟。数据从"分散在各个子组件"变成"集中在父组件",这就是"提升"的含义。

父(持有 state)
↓ value
兄弟 B(读)
兄弟 A(改)
↑ onChange
父(setState)
经典例子:温度转换(输入框与显示共享一份温度)
// 父组件:持有唯一的一份 temperature,统一管理
function Converter() {
  const [temp, setTemp] = useState('');

  return (
    <div>
      // 兄弟 A:负责"改"——拿到 value 和 onChange
      <TempInput value={temp} onChange={setTemp} />
      // 兄弟 B:负责"读"——只拿 value
      <TempDisplay celsius={temp} />
    </div>
  );
}

// 兄弟 A:自身不存 state,值来自父,改动通知父(受控组件)
function TempInput({ value, onChange }) {
  return <input value={value} onChange={e => onChange(e.target.value)} />;
}

// 兄弟 B:纯展示,数据全部来自 props
function TempDisplay({ celsius }) {
  const f = (celsius * 9) / 5 + 32;
  return <p>华氏:{celsius === '' ? '—' : f}</p>;
}

结合前两篇:兄弟 A 用回调向上(59)通知父组件改 state,父组件把新值props 向下(58)传给兄弟 B。状态提升其实就是这两个模式的组合应用。

为什么不让每个兄弟各存一份?

❌ 反例:各存各的
function A() {
  const [temp, setTemp] = useState('');
  // ...
}
function B() {
  const [temp, setTemp] = useState('');
  // 两份 temp,永远对不上!
}
✅ 正例:父统一持有
function Parent() {
  const [temp, setTemp] = useState('');
  // 唯一数据源
  return <>
    <A value={temp} onChange={setTemp}/>
    <B value={temp}/>
  </>;
}

问题所在:两个组件各存一份,就有了两个"真相"。A 改了自己的,B 完全不知道,数据不同步。这违反了 单一数据源(single source of truth) 原则——同一份逻辑数据,在整个应用里应该只存在一个权威副本。

单一数据源:React 的核心设计哲学
状态提升的深层价值,是保证每份数据只有一个"拥有者"。当多个组件需要反映同一份数据时,把它提升到最近的共同祖先,由祖先统一持有并分发。这样数据永远一致,调试时也只需盯着一个地方。这与数据库的"避免冗余、单一主键"、Redux 的"单一 store"是同一思想的不同体现。

操作步骤(官方推荐的三步法)

  1. 找到共同父组件:两个兄弟在组件树上最近的共同祖先。
  2. 把 state 移到父组件:删掉子组件里的 useState,在父组件里声明。
  3. 通过 props 下发:父把 value 传给需要读的子,把 onChange 回调传给需要改的子。
陷阱:提得太高
只把 state 提到最近的共同父组件,不要一股脑全塞到根组件。提得越高,越多不相关的中间组件被牵连重渲染,props 也要一层层往下"钻透"(prop drilling)。如果发现要穿透很多层,才该考虑 Context(下一篇)。

小结

  • 兄弟组件无法直连,共享数据要靠状态提升:把 state 提到最近的共同父组件。
  • 父组件成为单一数据源,通过 props 下发值、通过回调接收修改。
  • 本质是"父传子 props" + "子传父回调" 两个模式的组合。
  • 只提到最近共同祖先,提太高会导致无关组件重渲染和 prop drilling。
