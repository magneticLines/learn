useCallback：缓存函数引用

上一篇的 useMemo 缓存的是"计算结果"，这一篇的 useCallback 缓存的是"函数本身"。它专门解决第 51 篇提到的痛点：每次渲染都新建函数、破坏 memo 子组件的问题。

用法：useCallback(fn, deps)

useCallback 接收一个函数和依赖数组，返回一个"被缓存的函数引用"。只要依赖不变，多次渲染返回的都是同一个函数；依赖变了才返回新函数。

缓存传给 memo 子组件的回调
const Child = React.memo(Child);  // 只在 props 变时重渲

function Parent() {
  const [n, setN] = useState(0);

  // 依赖为空 → handleClick 引用永远稳定
  // 即使 Parent 因 n 变化重渲，Child 拿到的还是同一个函数 → 不重渲
  const handleClick = useCallback(() => {
    doSomething();
  }, []);

  return (
    <div>
      <button onClick={() => setN(n + 1)}>{n}</button>
      <Child onClick={handleClick} />  // 稳定引用，memo 生效
    </div>
  );
}
关键前提：useCallback 只有配合 React.memo 才有意义。如果 Child 没被 memo 包裹，它本来就会跟着父组件重渲——你缓存不缓存 handleClick 都一样，useCallback 纯属白加开销。缓存函数引用是为了让"引用比较"通过，而只有 memo（或 effect 依赖）才会去做这个比较。

它其实是 useMemo 的语法糖

useCallback 和 useMemo 本质是一回事——前者缓存"函数"，后者缓存"函数的返回值"。下面两行完全等价：

useCallback(fn, deps) === useMemo(() => fn, deps)
// 缓存函数 fn 本身
const a = useCallback(fn, deps);

// useMemo 缓存"返回 fn 的那个函数"的结果 = fn
const b = useMemo(() => fn, deps);

// a 和 b 行为完全一致。useCallback 只是为"缓存函数"这个高频需求
// 提供的便捷写法——省得每次写 () => fn 这层包装
useMemouseCallback
缓存的是计算的结果函数本身
你传入一个"算值"的函数要缓存的那个函数
典型场景昂贵计算、稳定对象传给 memo 子组件的回调
等价关系useCallback(fn, d) === useMemo(() => fn, d)

依赖数组的坑：闭包捕获旧值

useCallback 缓存的函数会闭包捕获创建它时的 props/state。如果依赖数组漏写了函数内用到的变量，函数会一直用创建时的"旧值"，产生难查的 bug。

❌ 漏依赖 → 用到旧 count
const inc = useCallback(() => {
  setCount(count + 1)
}, [])  // 漏了 count
// 永远读到初始的 count=0，
// 点多少次都是 setCount(1)
✅ 函数式更新免依赖
const inc = useCallback(() => {
  setCount(c => c + 1)  // 用回调
}, [])  // 不再依赖 count
// setter 稳定，逻辑正确
// 且引用保持稳定
技巧：更新依赖旧状态时，用函数式更新 setCount(c => c + 1) 就能把 count 从依赖里去掉，既保证逻辑正确、又让 useCallback 依赖为空、引用永远稳定。这是 useCallback + 函数式更新的经典搭配（呼应第 22 篇函数式更新）。
同样不要滥用。给每个函数无脑套 useCallback 是反模式：没有 memo 子组件消费时它毫无收益，还增加代码噪音、依赖数组维护负担和一点点开销。判断标准很简单——这个函数会不会被传给 memo 组件、或进某个依赖数组？不会就别包。第 54 篇专讲这种过度优化的代价。

小结

  • useCallback(fn, deps) 缓存函数引用，依赖不变则复用同一个函数。
  • 只有配合 React.memo（或作为 effect/其它 Hook 依赖）时才有意义，否则白加。
  • 本质等价 useMemo(() => fn, deps)，是缓存函数的语法糖。
  • 依赖要写全，防闭包捕获旧值；更新依赖旧态时用函数式更新去掉依赖。不要滥用。
