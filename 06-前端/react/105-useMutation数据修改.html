React · TanStack Query

useMutation：处理增删改

useQuery 负责"读"，但新增、修改、删除这类会改变服务器数据的操作，用 useMutation。它不自动执行，而是给你一个 mutate 函数，由你在用户操作时主动触发。

useQuery 与 useMutation 的分工

useQuery（读）
GET，幂等
组件挂载/key 变
→ 自动执行
按 key 缓存结果
适合：列表、详情
useMutation（写）
POST/PUT/DELETE，有副作用
调 mutate() 才执行
不缓存（每次都是新操作）
带 onSuccess/onError 回调
适合：提交表单、删除
为什么写操作不能自动执行？因为"读"是幂等的、多读几次无害；而"写"会真的改数据，重复提交会造成脏数据。所以 mutation 必须由用户明确的动作（点击提交）来触发。

基本用法

AddTodo.jsx
import { useMutation } from '@tanstack/react-query';

function AddTodo() {
  const { mutate, isPending, isError } = useMutation({
    // mutationFn：接收 mutate 传入的参数，返回 Promise
    mutationFn: (newTodo) =>
      fetch('/api/todos', {
        method: 'POST',
        body: JSON.stringify(newTodo),
      }).then(r => r.json()),
  });

  return (
    <button
      disabled={isPending}            {/* 请求中禁用，防重复提交 */}
      onClick={() => mutate({ title: '写作业' })}  {/* 触发 */}
    >
      {isPending ? '提交中…' : '新增'}
    </button>
  );
}

关键：写完之后要让相关查询失效

新增一条 todo 后，列表页的 ['todos'] 缓存就过时了。最常见的模式：在 onSuccess 里调 invalidateQueries，告诉 TanStack Query"这份缓存脏了"，它会自动重新拉取，UI 随之更新。

mutate(newTodo)
→
服务器写入成功
→
onSuccess: invalidate(['todos'])
→
列表自动重取刷新
onSuccess 里失效列表缓存
import { useMutation, useQueryClient } from '@tanstack/react-query';

function AddTodo() {
  const queryClient = useQueryClient();  // 拿到全局 client

  const mutation = useMutation({
    mutationFn: addTodo,
    // 成功后：让 ['todos'] 失效 → 自动重新获取最新列表
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['todos'] });
    },
    onError: (err) => toast.error(err.message),  // 失败提示
  });
  // mutation.mutate({ ... }) 触发
}

mutate vs mutateAsync

两种触发方式
// mutate：不返回 Promise，结果通过回调处理（推荐默认用）
mutation.mutate(data, {
  onSuccess: () => navigate('/list')   // 也可在调用处传回调
});

// mutateAsync：返回 Promise，可 await / try-catch，但要自己处理错误否则会未捕获
try {
  await mutation.mutateAsync(data);
} catch (e) { /* 必须 catch，否则 unhandled rejection */ }
onSuccess / onError / onSettled 三个回调：onSuccess 成功后触发（拿到返回数据）；onError 失败触发（拿到错误）；onSettled 无论成败都触发（类似 finally，常用于关闭 loading、关弹窗）。注意：定义在 useMutation 里的回调和调用 mutate 时传的回调都会执行，前者先跑。
常见错误：忘了 invalidate。新增/删除成功后不失效列表缓存，用户看到的还是旧列表，误以为"没生效"，刷新页面才更新。凡是 mutation 改动了某份 query 数据，都要在 onSuccess 里 invalidate 对应的 queryKey（下一章详解失效策略）。
面试点：①"useQuery 和 useMutation 分工？"——读用 query 自动执行并缓存，写用 mutation 手动 mutate 触发且不缓存。②"写操作后怎么刷新列表？"——onSuccess 里 invalidateQueries 让相关缓存失效并自动重取。③"mutate 和 mutateAsync 区别？"——后者返回 Promise 可 await，但错误需自行捕获。
