React · useContext

useContext：在组件里读取上下文

Provider 把数据"广播"出去，消费端就靠 useContext 一行代码接收。它会自动找到最近的那个 Provider，拿到它的 value——不用管中间隔了多少层。

基本用法：一行读取

useContext(SomeContext) 传入的是 Context 对象本身（createContext 的返回值），不是 .Provider。返回值就是最近 Provider 的 value。

Avatar.jsx —— 深层组件直接读 theme
import { useContext } from 'react';
import { ThemeContext } from './ThemeContext';

function Avatar() {
  // 传 Context 对象本身，返回最近 Provider 的 value
  const theme = useContext(ThemeContext);

  // theme 变化时，这个组件会自动重渲——它"订阅"了这个 Context
  return <img className={`avatar-${theme}`} />;
}
别传错东西。写成 useContext(ThemeContext.Provider) 或 useContext(<ThemeContext.Provider />) 都是常见错误。useContext 只接收 Context 对象本身。

"最近的 Provider"是什么意思

同一个 Context 可以被嵌套多个 Provider。组件读到的永远是它往上找、遇到的第一个同类型 Provider 的 value。内层覆盖外层——就像 CSS 里就近的样式覆盖外层。

内层 Provider 覆盖外层
<ThemeContext.Provider value={'dark'}>
  <Toolbar />              // 这里 useContext → 'dark'

  <ThemeContext.Provider value={'light'}>
    <Dialog />            // 这里 useContext → 'light'（就近覆盖）
  </ThemeContext.Provider>
</ThemeContext.Provider>

默认值何时生效：没有 Provider 时

createContext(defaultValue) 里的默认值，只在组件往上找不到任何 Provider 时才用得上。只要外层有 Provider（哪怕它的 value 是 undefined），默认值就不生效。

有 Provider
createContext('light')

<Ctx.Provider value="dark">
  <Child />   // 读到 'dark'
</Ctx.Provider>
// 默认值 'light' 被忽略
没有 Provider
createContext('light')

// 组件直接渲染，
// 外层没有任何 Provider
<Child />   // 读到 'light'
// 此时默认值才生效
默认值的实用价值：让组件在没有 Provider 的环境下（比如单元测试、Storybook 单独渲染）也能跑起来，不会崩。很多团队会故意把默认值设成一个"报错的哨兵值"或抛异常的对象，用来提醒开发者"你忘了套 Provider"。

多个 Context 嵌套

一个组件可以同时消费多个 Context——各读各的，互不干扰。多个 Provider 嵌套时容易出现"Provider 地狱"，可以抽成一个组合组件。

同时消费多个 Context
function Profile() {
  const theme = useContext(ThemeContext);
  const user = useContext(UserContext);   // 各读各的
  return <div className={theme}>{user.name}</div>;
}

// 把多个 Provider 打包，避免顶层"Provider 金字塔"
function AppProviders({ children }) {
  return (
    <ThemeContext.Provider value={theme}>
      <UserContext.Provider value={user}>
        {children}
      </UserContext.Provider>
    </ThemeContext.Provider>
  );
}
联系已学：useContext 遵守 Hook 规则——必须在组件/自定义 Hook 顶层调用，不能放进 if、循环里。它本质上是"订阅"，React 内部记录哪些组件订阅了哪个 Context，value 变化时精准通知它们重渲。

小结

  • useContext(Ctx) 读取最近那个 Provider 的 value，跳过所有中间层。
  • 嵌套 Provider 时内层覆盖外层，就近取值。
  • 默认值只在完全没有 Provider 时生效，常用来兜底测试/独立渲染。
  • 一个组件可同时消费多个 Context；顶层 Provider 太多可抽成 AppProviders 组合组件。
