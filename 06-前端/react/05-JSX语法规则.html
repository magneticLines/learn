JSX 语法规则与限制

JSX 长得像 HTML，但它不是 HTML——它是 JavaScript 语法扩展。正因为底层是 JS，才有那些看起来"奇怪"的规则：必须有根节点、标签必须闭合、注释要用花括号……每条规则背后都有充分的理由。

JSX 不是 HTML：它是 JS 语法扩展

第一件事：把"JSX 是 HTML"这个念头从脑子里拔掉。JSX 是 JavaScript XML，是一种让你在 JS 里写"类 HTML 结构"的语法扩展。

你写的 JSX 会被 Babel（JS 编译器）翻译成 React.createElement() 调用，而 React.createElement() 返回的是一个普通的 JS 对象——即"虚拟 DOM 节点"，不是真实 DOM 元素。

你写的 JSX → Babel 编译后的 JS（完全等价）
// 你写的 JSX（可读，像 HTML）
const el = <h1 className="title">Hello</h1>;

// Babel 编译后（React 实际执行的）
const el = React.createElement(
  'h1',                    // 标签类型
  { className: 'title' },  // 属性对象
  'Hello'                  // 子节点
);

// createElement 返回的是普通 JS 对象（虚拟 DOM），长这样：
// { type: 'h1', props: { className: 'title', children: 'Hello' } }
为什么要理解这一点？JSX 的所有规则都可以从"底层是 JS 函数调用"推导出来。理解了这个，规则就不用死记——你能自己推出来。

规则一：必须有一个根节点

JSX 组件只能返回一个根元素。原因很直接：编译后是 React.createElement() 函数调用，一个函数只能有一个返回值。两个平级标签 = 两个返回值 = JS 语法错误。

❌ 报错：两个根节点
function App() {
  return (
    <h1>标题</h1>
    <p>内容</p>
    // 编译成两个 createElement 调用
    // 函数不能同时 return 两个值
  );
}
✅ 正确：Fragment 包裹
function App() {
  return (
    <>
      <h1>标题</h1>
      <p>内容</p>
    </>
    // <></> 是 Fragment 简写
    // 只有一个根，编译没问题
  );
}

用 <div> 包裹也能解决，但会在 DOM 里多出一个 div 节点，可能破坏 CSS 布局（flex/grid 父子关系错乱）。<></>（Fragment）是"透明容器"，满足规则但不产生真实 DOM 节点。

规则二：标签必须闭合

HTML 里 <img>、<br>、<input> 可以不闭合，浏览器会自动处理。但 JSX 遵循的是 XML 语法，XML 要求所有标签都必须显式关闭。

❌ JSX 里报错
// HTML 合法，JSX 不行
<img src="logo.png">
<br>
<input type="text">
✅ 必须自闭合
// JSX 要求所有标签关闭
<img src="logo.png" />
<br />
<input type="text" />

记忆技巧：没有子节点的标签，在结尾加 />；有子节点的标签，必须有对应的闭合标签 </tag>。

规则三：JSX 注释写法

HTML 注释是 <!-- 注释 -->，但 JSX 里不能用这个——JSX 最终是 JS，HTML 注释语法对 JS 引擎没有意义。JSX 注释必须放在 JS 表达式插槽 里，用 {/* 注释内容 */}：

JSX 注释写法对比
function Demo() {
  return (
    <div>
      {/* ✅ JSX 注释：花括号 + 斜杠星号 */}
      {/* 可以跨多行
          像这样写 */}

      // ❌ 下面这个 HTML 注释在 JSX 里会被当成文本渲染出来！
      // <!-- 我是注释 -->  ← 不要这样写

      <p>内容</p>
    </div>
  );
}
陷阱：如果你在 JSX 里写 <!-- 注释 -->，React 不会把它当注释——它会把这几个字符当文本节点渲染到页面上，或者直接抛语法错误。注释一定要用 {/* */}。

合法 vs 非法：完整对比示例

把以上规则放在一起，看一段综合示例：

合法的 JSX（遵守所有规则）
function UserCard() {
  return (
    {/* 规则1：只有一个根节点（这里是 div） */}
    <div className="card">
      {/* 规则3：注释用花括号包裹 */}

      {/* 规则2：img 必须自闭合 */}
      <img src="avatar.png" alt="头像" />

      <h2>张三</h2>
      <p>前端开发</p>
    </div>
  );
}
非法的 JSX（违反规则 → 报错）
function BadExample() {
  return (
    // ❌ 规则1违反：两个根节点，报错
    <h2>标题</h2>
    <p>内容</p>
  );
}

function AnotherBadExample() {
  return (
    <div>
      // ❌ 规则2违反：img 没有自闭合，报错
      <img src="photo.png">
    </div>
  );
}
推导而非死记：只要记住"JSX 编译成 JS 函数调用，返回 JS 对象"，所有规则都能推导出来——函数一个返回值（根节点唯一）、XML 语法要求标签闭合、注释要在 JS 表达式槽里。记原理，规则自然就懂了。
