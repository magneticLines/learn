文件系统路由

本系列第 90 篇学过用 createBrowserRouter 显式配置路由表——每条路径和组件的对应关系都要手写。Next.js App Router 换了一种思路:目录结构本身就是路由,app/about/page.tsx 自动对应 /about,不用写一行路由配置。

约定优于配置:目录即路由

App Router 的规则很简单:app 目录下每一层文件夹对应 URL 的一层路径段,文件夹里放一个 page.tsx 就意味着"这一层有一个可访问的页面"。

目录结构 → 自动生成的路由
app/
├── page.tsx            → /
├── about/
│   └── page.tsx        → /about
└── blog/
    ├── page.tsx        → /blog
    └── [slug]/
        └── page.tsx    → /blog/:slug  (动态路由)

[slug] 这种方括号命名的文件夹对应动态路由段,作用等价于本系列 90 篇里 createBrowserRouter 配置的 path: '/blog/:slug'——只是这里不需要手写这行配置,文件夹名字本身就声明了这是一个动态参数。

对比:文件系统路由 vs 配置式路由

配置式路由(createBrowserRouter)
const router = createBrowserRouter([
  { path: '/', element: <Home /> },
  { path: '/about', element: <About /> },
  {
    path: '/blog/:slug',
    element: <BlogPost />,
    loader: blogLoader,
  },
]);

路由表集中一处,一眼看全局
灵活:可以任意组织路径和组件的映射
但要手写每一条映射关系
文件系统路由(App Router)
app/
  page.tsx        // /
  about/page.tsx  // /about
  blog/
    page.tsx      // /blog
    [slug]/page.tsx // /blog/:slug

新增页面 = 新建文件夹 + page.tsx
不用维护单独的路由表
心智负担低,但路径完全由目录结构决定
灵活性打折(想要非常规映射较难)
怎么选? 配置式路由的优势是路由表集中、显式、灵活——尤其是路径和组件不是简单一一对应、需要复杂嵌套或权限逻辑时更好掌控;文件系统路由的优势是约定统一、新增页面零配置,团队协作时每个人都知道"新页面往哪放文件夹"。这不是谁更先进的问题,而是"配置的灵活性"与"约定的低心智负担"之间的经典取舍,在很多框架的设计里都能看到类似的权衡(比如 Vue Router 也有社区版的文件路由插件)。
陷阱:文件夹名不是路径的唯一决定因素。 App Router 里有一批特殊文件名有专门含义——page.tsx(页面)、layout.tsx(布局)、loading.tsx(加载态)、error.tsx(错误边界),初学者容易把"文件夹名"和"路径段名"简单划等号,却忘了文件夹里到底放的是哪个特殊文件名决定了它的角色,而不是路径本身。

layout.tsx:嵌套布局

每一层文件夹放一个 layout.tsx,就能让这一层及其所有子路由共享同一个外层结构(导航栏、侧边栏等),内层页面内容通过一个占位符插进来。

app/blog/layout.tsx —— 博客模块的公共布局
export default function BlogLayout({ children }: { children: React.ReactNode }) {
  return (
    <div>
      <BlogSidebar />
      {children}  {/* 子路由(blog/page.tsx 或 blog/[slug]/page.tsx)渲染到这里 */}
    </div>
  );
}
和本系列 84 篇的呼应。 这里的 children 占位符,思路和 React Router 里 <Outlet /> 的作用完全一样——都是"父级布局固定不变,子路由内容在指定位置替换"。区别只是 App Router 里这个占位符是通过组件的 children prop 隐式传入的,而 <Outlet /> 是在配置式路由体系里显式渲染的一个组件。理解了 84 篇的 Outlet,这里几乎不需要重新学一个新概念,只是换了个实现载体。

小结

  • App Router 用目录结构表达路由:app/about/page.tsx 自动对应 /about,[slug] 命名的文件夹对应动态路由段。
  • 与 90 篇的 createBrowserRouter 配置式路由相比:配置式路由表集中、灵活但要手写映射;文件系统路由约定优于配置,新增页面零配置,但灵活性有所牺牲。
  • 特殊文件名(page.tsx/layout.tsx/loading.tsx/error.tsx)决定了文件夹里内容的角色,不是文件夹名本身。
  • layout.tsx 的嵌套布局思路与 84 篇 <Outlet /> 布局路由本质一致,都是"外层结构固定 + 内层内容按路由替换"。
