useNavigate：不靠点击链接的"编程式"跳转

不是所有跳转都由用户点链接触发。"登录成功后跳到首页""表单提交后返回列表""3 秒后自动跳转"——这些要在 JS 代码里主动发起跳转。v6 用 useNavigate 这个 hook 来做，它取代了 v5 的 useHistory。

声明式 vs 编程式导航

上一篇的 <Link> 是声明式导航——跳转写在 JSX 里，由用户点击触发。但很多跳转发生在事件回调或副作用里，此时没有链接可点，需要用代码"命令"路由跳转，这就是编程式导航。

v5 → v6 的 API 更名(高频考点)。 v5 用 const history = useHistory()，然后 history.push('/home')；v6 改成 const navigate = useNavigate()，然后 navigate('/home')。名字和调用方式都变了，从旧项目迁移时这是必改项。设计上 v6 把"跳转"这个动作收敛成一个函数，更直观。

基本用法

useNavigate 返回一个 navigate 函数
import { useNavigate } from 'react-router-dom';

function LoginForm() {
  const navigate = useNavigate();  // 拿到跳转函数

  async function handleSubmit(e) {
    e.preventDefault();
    await login();
    // 登录成功后，在 JS 里主动跳转 —— 用户没点任何链接
    navigate('/dashboard');
  }

  return <form onSubmit={handleSubmit}>...</form>;
}
陷阱:navigate 只能在组件渲染期间拿到,别在渲染中直接调用。 useNavigate 是 hook，必须在组件顶层调用拿到 navigate 函数。而调用 navigate(...) 只能放在事件回调或 useEffect 里，不能在组件函数体渲染过程中直接调用（那会在渲染期间产生副作用，React 会警告甚至报错）。想在渲染时就跳转，用声明式的 <Navigate to="/x" /> 组件。

navigate 的几种参数形态

navigate 的多种调用方式
// ① 传路径字符串：跳到指定页面（默认往历史栈 push 一条）
navigate('/users/42');

// ② 传数字：在历史栈里前进/后退（像浏览器的前进后退按钮）
navigate(-1);   // 后退一步（等价于点浏览器"返回"）
navigate(1);    // 前进一步
navigate(-2);   // 后退两步

// ③ 第二参数 options：replace / state
navigate('/dashboard', { replace: true });       // 替换当前记录，见下文
navigate('/detail', { state: { from: 'list' } });  // 隐式传数据，第83篇细讲
跳转到路径
navigate('/path')
后退
navigate(-1)
替换记录
navigate('/x', {replace:true})
携带 state
navigate('/x', {state})

replace:true——什么时候要替换而不是压栈

默认 navigate('/x') 会往历史栈压入（push）一条新记录，用户能点"后退"回到上一页。但有些场景不希望用户退回去：

默认 push
登录页 → dashboard
历史栈: [登录页, dashboard]
        ↑ 用户按"后退"
        又回到登录页 😕
        （已登录还看到登录页）
replace: true
navigate('/dashboard',
  { replace: true })
历史栈: [dashboard]
        登录页被替换掉
        按"后退"不会回登录页 ✅

典型 replace: true 场景：登录成功后跳转（不该退回登录页）、重定向（旧地址 → 新地址）、表单提交完跳到结果页（不该退回去重复提交）。

对应的声明式重定向:<Navigate>。 在渲染阶段就要跳转（如"未登录访问受保护页面 → 踢到登录页"），用 <Navigate to="/login" replace /> 组件更合适——它是 useNavigate 的声明式包装，渲染即跳转，不会触发"渲染中调用副作用"的警告。

useNavigate vs Link 怎么选

触发方式用什么
用户点击一个可见的链接/按钮优先 <Link>（语义好、可访问、能新标签打开）
某段逻辑执行完后自动跳（登录、提交、定时）useNavigate
后退/前进navigate(-1) / navigate(1)
渲染时条件重定向（如鉴权）<Navigate> 组件

小结

  • 编程式导航用于"非点击链接"的跳转（登录后、提交后、定时器等）。
  • v6 用 const navigate = useNavigate() 取代 v5 的 useHistory()；navigate('/path') 取代 history.push()。
  • navigate(-1)/navigate(1) 实现后退/前进；第二参数可传 { replace, state }。
  • replace: true 替换当前历史记录，用于登录后跳转、重定向等"不该退回去"的场景。
  • 能用 <Link> 就优先用；渲染阶段的条件跳转用 <Navigate>，别在渲染中直接调 navigate()。
