useRef 存储可变值：不触发渲染的记忆盒

useRef 的第二大用途和 DOM 无关：它是一个跨渲染保留、但修改不触发重渲染的容器。理解"改 ref 不重渲、改 state 才重渲"这条分界，就能选对工具，也能解开定时器闭包之类的难题。

ref 就是一个 { current } 盒子

useRef(initial) 返回一个对象 { current: initial }。React 保证这个对象在组件的每次渲染之间是同一个（引用不变）。你随便读写 .current，React 完全不管、也不会因此重新渲染。

ref 只是个稳定的盒子
const ref = useRef(0);
// ref === { current: 0 }，每次渲染拿到的是同一个 ref 对象

ref.current = 42; // 直接改，同步生效，但不触发渲染
console.log(ref.current); // 42

state vs ref:核心区别一张表

useStateuseRef
修改后重渲染?会，触发重渲染不会
渲染间保留?会会
怎么读写只读 + setState()直接读写 .current
渲染期能读吗?能，是本次渲染的快照可以，但不推荐（值可能"不属于"本次渲染）
适合存影响 UI 的数据不影响 UI 的幕后数据
判断法则：这个值变了，界面需要跟着变吗？需要 → 用 state（让 React 重渲染）；不需要，只是幕后记账 → 用 ref。比如"点击次数要显示出来"用 state；"存个定时器 id 好清理"用 ref（用户看不到 id）。

场景一:存定时器 id

秒表:用 ref 存 intervalId
function Stopwatch() {
  const [time, setTime] = useState(0);
  const timerRef = useRef(null); // 存 id，改它不该重渲染

  function start() {
    if (timerRef.current) return; // 防重复启动
    timerRef.current = setInterval(() => setTime(t => t + 1), 1000);
  }
  function stop() {
    clearInterval(timerRef.current);
    timerRef.current = null;
  }
  // 若用 state 存 id:setState 会引发不必要的重渲染，纯属浪费
}

场景二:记住"上一次的值"

usePrevious:对比前后值
function usePrevious(value) {
  const ref = useRef();
  useEffect(() => {
    ref.current = value; // 渲染后才更新 → 下次渲染读到的是"上一次"
  });
  return ref.current; // 本次渲染读到的还是上次存的值
}

场景三:破解定时器闭包陷阱

第 38 篇的定时器 bug（读到旧 count），除了函数式更新，另一解法是用 ref 存"永远最新的值"。因为 ref 是同一个对象，任何闭包读 ref.current 都拿到最新写入的值。

用 ref 让闭包读到最新值
const [count, setCount] = useState(0);
const countRef = useRef(count);
countRef.current = count; // 每次渲染同步为最新

useEffect(() => {
  const id = setInterval(() => {
    console.log(countRef.current); // ✅ 永远是最新 count，不是锁死的 0
  }, 1000);
  return () => clearInterval(id);
}, []);
不要在渲染期读写 ref。渲染必须是纯的（第 37 篇），而 ref.current 是"渲染外"的可变状态——在渲染期读它，读到的值可能不属于本次渲染，在并发/StrictMode 下行为不可预测。规则是：ref 只在事件处理器和 effect 里读写。上面 usePrevious / 同步 count 的赋值属于官方认可的例外模式，但一般代码应遵守此规则。
别把 ref 当 state 用来偷懒。常见误区：想省一次渲染，把该显示的数据存进 ref，然后纳闷"为什么界面不更新"。记住：改 ref 不会触发渲染，界面自然不动。如果一个值变化时 UI 必须更新，它就该是 state，没有捷径。
本质对比。可以把 useRef(x) 粗略理解为 useState({current: x})[0]——一个永不改变引用的对象，但没有 setter、改它不通知 React。它填补了函数组件里"我需要一个跨渲染的实例变量，但不想触发渲染"的空白，相当于 class 组件里直接挂在 this 上的普通属性（this.timerId），而不是 this.state。
