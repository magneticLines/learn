函数式更新：setState(prev => newValue)

上一节知道了 state 是快照，在同一渲染里多次 setCount(count + 1) 只会加一次。函数式更新就是为了解决这个问题——它让每次更新都能基于"最新的真实值"计算。

语法：传函数而不是传值

setState 接受两种参数：

  • 直接传值：setCount(42)——React 用 42 作为下次渲染的 state 值
  • 传函数：setCount(prev => prev + 1)——React 把当前最新值传给函数，用函数的返回值作为下次的 state 值
函数式更新语法
// 直接传值：用固定的新值替换 state
setCount(42);                // count 下次渲染变成 42
setName('Tom');             // name 下次渲染变成 'Tom'

// 函数式更新：React 保证 prev 是当前最新的 state 值
setCount(prev => prev + 1);  // 基于最新值 +1
setVisible(v => !v);        // 翻转布尔值的标准写法
setList(prev => [...prev, newItem]);  // 基于最新数组追加

// prev 只是约定命名，你也可以叫 current、old、_：
setCount(c => c + 1);       // 简写，React 社区常见风格

解决的问题：多次更新正确累加

回到上节的问题：同一事件里连续三次 setCount(count + 1) 只加了一次。函数式更新如何解决？

❌ 直接传值——只加 1
// count 快照 = 0
setCount(count + 1); // 计划→1
setCount(count + 1); // 计划→1（还是 0+1）
setCount(count + 1); // 计划→1（还是 0+1）

// 三次都在说"把 count 设为 1"
// 批处理合并：最终 count = 1
// 点一次只加了 1，不是 3
✅ 函数式更新——加 3
// React 把这三个函数排进更新队列
setCount(c => c + 1); // prev=0 → 1
setCount(c => c + 1); // prev=1 → 2
setCount(c => c + 1); // prev=2 → 3

// 每个函数拿到上一次的结果作为 prev
// 最终 count = 3 ✅
// 这就是函数式更新的价值

本质区别：传值时，React 直接覆盖；传函数时，React 把上一次的结果传给下一个函数，形成链式计算。

c=0 → c+1=1
→
c=1 → c+1=2
→
c=2 → c+1=3
→
count = 3 ✅

实际场景里的对比

场景：点一下按钮加三次（比如三连击动画）
function TripleCounter() {
  const [count, setCount] = useState(0);

  // ❌ 错误写法：三次都基于同一个快照，只 +1
  function wrongTriple() {
    setCount(count + 1);
    setCount(count + 1);
    setCount(count + 1);
  }

  // ✅ 正确写法：函数式更新，每次基于前一次的结果
  function correctTriple() {
    setCount(c => c + 1);
    setCount(c => c + 1);
    setCount(c => c + 1);
    // React 把三个函数排队，依次传递结果，最终 +3
  }

  return (
    <div>
      <p>count = {count}</p>
      <button onClick={wrongTriple}>错误（只+1）</button>
      <button onClick={correctTriple}>正确（+3）</button>
    </div>
  );
}
经验法则：只要新值依赖旧值，一律用函数式
  • 数字累加/递减：setCount(c => c + 1)
  • 翻转布尔值：setVisible(v => !v)
  • 数组追加：setList(prev => [...prev, item])
  • 对象局部更新：setUser(prev => ({ ...prev, name: 'Tom' }))

只有在"设置一个与旧值完全无关的新值"时，才直接传值：setName('Tom')、setCount(0)（重置）。

切换布尔值的标准写法

在 React 社区里，切换布尔状态（如显示/隐藏、开/关）有一个约定俗成的写法，见到要认识：

切换布尔值——社区标准写法
const [open, setOpen] = useState(false);

// ✅ 函数式——基于当前值翻转，安全且简洁
<button onClick={() => setOpen(v => !v)}>切换</button>

// 也可以写成（效果相同，更短）
<button onClick={() => setOpen(o => !o)}>切换</button>

// ⚠️ 直接传值——单次调用完全没问题；若同一事件里多次调用则有竞态风险
<button onClick={() => setOpen(!open)}>切换</button>
// 多次调用时 !open 每次都是同一个快照值——函数式更新才能正确链式累积

什么时候用函数式，什么时候直接传值

场景推荐写法原因
累加、递减 setCount(c => c + 1) 新值依赖旧值，必须用函数式
翻转布尔 setOpen(v => !v) 新值依赖旧值（取反），必须用函数式
重置为固定值 setCount(0) 新值与旧值无关，直接传值即可
设置用户输入 setName(e.target.value) 新值来自事件，与旧值无关，直接传值
数组追加 setList(prev => [...prev, x]) 需要旧数组的内容，必须用函数式
