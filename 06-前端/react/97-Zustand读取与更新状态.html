Zustand 读取与更新:选择器与 set / get

建好了 store,接下来是日常操作:怎么读、怎么写。Zustand 读取的核心是选择器(selector)——它不只是语法,更是决定性能的关键:选对了,组件只在自己关心的数据变化时重渲。

读取:用选择器,而不是整个 store

调用 store Hook 时,传入一个选择器函数,声明"我只关心哪一部分":

选择器读取:只订阅你真正用到的字段
// ✅ 推荐:选择器只取 count,只有 count 变才重渲
const count = useCounter((s) => s.count);

// ✅ 取 action 也一样(action 引用不变,基本不会触发重渲)
const increment = useCounter((s) => s.increment);
最高频性能陷阱:不传选择器,直接拿整个 store。
const store = useCounter(); —— 这样写会订阅整个 store,store 里任何字段变化都会让这个组件重渲,哪怕你只用到 count。选择器的全部意义,就是把订阅范围缩小到你真正用到的那一块。

选择器的性能意义:Zustand 如何决定要不要重渲

每次 store 变化,Zustand 会用你的选择器算出新值,然后拿它和上一次的结果比较(默认用 Object.is,即 ===)。结果相等就跳过重渲,不等才重渲。

store 变化
→
跑一遍选择器
→
新结果 === 旧结果?
→
相等:跳过

这正是 Zustand 比 Context 强的地方:Context 是"value 变就全体重渲",Zustand 是"你选的那块变才重渲"。

陷阱:选择器返回新对象

因为比较用的是 ===,如果选择器每次都返回新对象/新数组,那 === 永远为 false,重渲优化就失效了:

❌ 每次返回新对象
// 每次渲染都是一个新 {},
// === 永远 false,每次都重渲
const { count, step } =
  useStore(s => ({
    count: s.count,
    step: s.step,
  }));
✅ 分开取原子值
// 各取原始值,== 稳定比较
const count =
  useStore(s => s.count);
const step =
  useStore(s => s.step);
确实想一次取多个字段? 用 Zustand 提供的 useShallow 做浅比较:const { count, step } = useStore(useShallow(s => ({ count: s.count, step: s.step })))。它会逐字段浅比较,而不是用 === 比整个对象引用,从而恢复优化。(旧版本里对应的是给 Hook 传第二个参数 shallow。)

更新:set 的合并 vs 替换

store 里的 action 通过 set 更新状态。set 有一个关键行为:默认做浅合并(merge),而不是替换。

set 默认浅合并:只写变的字段,其余自动保留
const useUser = create((set) => ({
  name: 'Shea',
  age: 25,
  city: 'Beijing',

  // set 只需给出要改的字段,name/city 自动保留(浅合并)
  birthday: () => set((s) => ({ age: s.age + 1 })),

  // 传对象:{ age: n } 与旧 state 浅合并
  rename: (name) => set({ name }),
}));
对比 useState:这是个好用的差异。 useState 的更新是整体替换——改一个字段也得手动 { ...prev, age } 展开。Zustand 的 set 默认浅合并顶层字段,所以顶层改动不用展开。但注意"浅"字:合并只到第一层,嵌套对象仍需手动展开,比如 set(s => ({ profile: { ...s.profile, age: 26 } }))。

需要完全替换时:set 的第二个参数

set(newState, true):替换而非合并
// 第二个参数 true = 用新对象整体替换,丢弃旧字段
// 慎用:会把 action 方法也一起丢掉!通常只在"重置整个 store"时用
set({ name: '新用户', age: 0 }, true);

get:在 action 里读当前值

create 的回调其实有第二个参数 get,用来在 action 内部读取当前最新状态——适合逻辑依赖多个字段、或不想用 set 的函数式写法时:

get(set, get) => ...:action 里读当前状态
const useCart = create((set, get) => ({
  items: [],
  coupon: 0,

  // 用 get() 拿到当前 state,做只读计算(不引发订阅)
  total: () => {
    const { items, coupon } = get();
    return items.reduce((sum, i) => sum + i.price, 0) - coupon;
  },
}));

// 也能在组件外直接读:useCart.getState().items
// 也能在组件外直接写:useCart.setState({ coupon: 10 })
get() 是"读一次快照",它不会建立订阅,所以在组件里想让 UI 随数据变化,还是要用选择器 Hook 读。get() 主要用在 action 内部,或组件外的普通函数里(通过 useStore.getState())。

速查:读与写

目的写法说明
读单个字段(响应式)useStore(s => s.x)x 变才重渲
读多个字段useStore(useShallow(s => ({...})))浅比较,避免新对象误伤
更新(合并)set({ x: 1 })浅合并顶层
依赖旧值更新set(s => ({ x: s.x + 1 }))函数式,拿到最新 state
整体替换set(obj, true)慎用,会丢 action
组件外读useStore.getState()读快照,不订阅

小结

  • 永远用选择器读取:useStore(s => s.x),声明只关心某块,只有它变才重渲。
  • 不传选择器直接 useStore() 会订阅整个 store,任何字段变都重渲——高频性能陷阱。
  • Zustand 用 === 比较选择器结果决定是否重渲;选择器返回新对象会破坏优化,多字段用 useShallow。
  • set 默认浅合并顶层字段(和 useState 的整体替换不同),嵌套对象仍需手动展开;set(obj, true) 才是替换。
  • get() 在 action 内读当前状态快照,不建立订阅;组件外可用 getState / setState 读写。
