React · Props

传递与接收 Props

Props 是父组件向子组件"喂数据"的唯一正规渠道。没有 props，组件是一块刻好内容的印章——只能印出一种东西；有了 props，它变成了活字印刷——字模一样，但能组合出任意内容。

Props 是什么：组件的"函数参数"

本质上，props 就是函数参数。React 组件是函数，调用组件（写 JSX 标签）就是在调用这个函数，JSX 上的属性就是传入的参数。

没有 props 时，组件是写死的：

没有 props——无法复用
// 这个按钮只能叫"提交"，永远叫"提交"
// 想要"取消"按钮？只能再复制一份改文字——这不是复用，是复制
function SubmitButton() {
  return <button>提交</button>;
}

有了 props，一个组件能根据数据渲染出千变万化的 UI：

有了 props——可以复用
// 同一个组件，父组件传什么 label，就渲染什么文字
function Button(props) {
  return <button>{props.label}</button>;
}

// 父组件可以渲染三种按钮，用的是同一个 Button 函数
<Button label="提交" />
<Button label="取消" />
<Button label="删除" />

传 Props：在 JSX 标签上像写 HTML 属性一样

父组件通过 JSX 属性把数据传给子组件。规则很简单：字符串直接写引号，其他所有类型（数字、布尔、数组、对象、函数）都用 { } 包起来。

传递 5 种类型的 props（父组件）
function App() {
  const handleEdit = () => console.log('编辑被点击');

  return (
    <UserCard
      name="Shea"                          // 字符串：直接写，不需要 { }
      age={25}                              // 数字：必须用 { }，否则传的是字符串 "25"
      isAdmin={true}                       // 布尔：用 { }，或简写为 isAdmin（见下方）
      skills={['React', 'Java', 'SQL']}   // 数组：{ } 内写 JS 数组字面量
      onEdit={handleEdit}                // 函数：传引用，不要加 ()，加了就变成传返回值了
    />
  );
}
常见陷阱：传函数时写成 onEdit={handleEdit()}——多了一对括号，变成立即执行并把返回值传过去，子组件点击时什么也不发生。传函数引用，不要加括号。

接收 Props：子组件函数的第一个参数

React 调用子组件函数时，会自动把所有父组件传来的属性打包成一个普通 JavaScript 对象，作为第一个参数传入。这个参数习惯命名为 props，但名字可以任意取。

接收 props——子组件（用 props.xxx 访问）
function UserCard(props) {
  // React 自动把父组件传来的属性打包成这个对象：
  // props = {
  //   name: 'Shea',
  //   age: 25,
  //   isAdmin: true,
  //   skills: ['React', 'Java', 'SQL'],
  //   onEdit: [Function: handleEdit]
  // }

  return (
    <div>
      <h2>{props.name}</h2>

      <p>年龄：{props.age}</p>

      {/* 布尔值：用短路渲染，只有 isAdmin 为 true 时才显示标签 */}
      {props.isAdmin && <span>管理员</span>}

      <ul>
        {/* 数组：用 map 渲染列表 */}
        {props.skills.map(skill => <li key={skill}>{skill}</li>)}
      </ul>

      {/* 函数：作为事件处理器，用户点击时才调用 */}
      <button onClick={props.onEdit}>编辑</button>
    </div>
  );
}

布尔值简写：省略 ={true}

当你要传 true 时，可以把 ={true} 整个省略掉——只写属性名本身。这和 HTML 的布尔属性（如 <input disabled>）行为一致。

布尔值简写规则
// 以下两行完全等价
<Modal isOpen={true} />
<Modal isOpen />          // 省略 ={true}，更简洁

// 注意：传 false 不能省略！省略就变成 true 了
<Modal isOpen={false} />  // 正确——显式传 false
// <Modal /> 也不传这个 prop——子组件拿到 undefined，需要有默认值兜底
布尔属性的记忆口诀：出现即为 true，不出现是 undefined（不是 false）。所以如果你想明确传 false，必须写 ={false}——不能靠"不写"来传 false。

为什么 props 对象不能有自定义名？

严格来说，函数参数名叫什么都行，叫 p、options、config 都可以跑通。但生态约定用 props，原因：

  • 团队协作时，所有人看到第一个参数叫 props 就知道它是干什么的
  • React DevTools、TypeScript 类型提示都围绕 props 这个语义设计
  • 实践中几乎总是直接解构（下一节），所以参数名本身几乎不出现在函数体里
本节核心：① props 是函数参数，父传子，单向流动；② 字符串用引号，其他类型用 { }；③ 子组件第一个参数是包含所有 props 的普通对象；④ 布尔值 true 可省略 ={true}，false 必须显式写；⑤ 传函数时不要加 ()，传的是引用而不是调用结果。
