React · 组件通信

父传子:Props——最基本的通信方式

组件不是孤岛,它们要互相传递数据。最基础的一种:父组件通过 props 把数据交给子组件。理解它,就理解了 React "单向数据流" 的起点。

props 是什么:父组件调用子组件时传的"参数"

把组件想成一个函数——它接收输入,返回 UI。这个输入就是 props(properties 的缩写)。父组件在 JSX 里写子组件标签时,标签上的属性就成了子组件收到的 props 对象。

父组件传,子组件收
// 子组件:声明一个参数 props,它是一个对象
function Greeting(props) {
  return <h1>你好, {props.name}!</h1>;
}

// 父组件:像给函数传参一样,在标签上写属性
function App() {
  return (
    <div>
      <Greeting name="Shea" />   // props = { name: 'Shea' }
      <Greeting name="Tom" />    // 同一组件,不同数据
    </div>
  );
}

字符串用引号,其他类型(数字、布尔、对象、数组、函数)都要用大括号 {} 包起来传:

传递非字符串:一律用大括号
<User
  name="Shea"                 // 字符串:引号
  age={25}                   // 数字:大括号
  isAdmin={true}             // 布尔:大括号
  tags={['a', 'b']}          // 数组
  profile={{ city: 'BJ' }}   // 对象(注意双大括号:外层是 JSX 表达式,内层是对象)
/>

解构写法:更简洁地接收 props

直接写 props.name 略啰嗦,实践中几乎都用解构把需要的字段拆出来,还能顺手写默认值:

解构 + 默认值(最常见的写法)
// 在参数位置直接解构,一眼看清这个组件"吃"哪些 props
function Greeting({ name, greeting = '你好' }) {
  return <h1>{greeting}, {name}!</h1>;
}

<Greeting name="Shea" />              // 你好, Shea!(greeting 用默认值)
<Greeting name="Tom" greeting="欢迎" /> // 欢迎, Tom!

核心铁律:props 是只读的

子组件绝对不能修改自己收到的 props。React 把这称为组件必须像"纯函数"一样对待 props——给同样的输入,永远返回同样的输出,不产生副作用。

❌ 反例:改 props
function Item({ item }) {
  // 直接改 props——禁止!
  item.done = true;
  return <li>{item.text}</li>;
}
✅ 正例:通知父组件去改
function Item({ item, onToggle }) {
  // 不改数据,只把"意图"报给父组件
  return (
    <li onClick={onToggle}>
      {item.text}
    </li>
  );
}

为什么这么设计? 数据的"所有权"属于创建它的组件(通常是父组件)。如果子组件能随手改 props,数据就有了多个修改源,追踪 bug 会变成噩梦——你不知道到底是谁改了它。React 强制 数据只能由拥有者修改,让数据流可预测。

单向数据流(one-way data flow)
props 只能从父流向子,像瀑布一样自上而下,不能倒流。父组件的 state 变化 → 传给子组件的 props 变化 → 子组件重新渲染。这种"单向"约束让整个应用的数据走向清晰:想知道某个值从哪来,顺着组件树往上找即可。这正是 React 可维护性的基石,也是它与"双向绑定"框架(如早期 Angular)最大的哲学差异。

props 变化会触发子组件重渲染

props 不是"传一次就固定"。当父组件重新渲染、传下来的 props 值变了,子组件会自动跟着重新渲染。这就是数据向下流动的动态过程:

父 state 变化
→
父重渲染
→
新 props 传下
→
子重渲染
常见误区:props 不是"变量赋值一次"
初学者常以为 <Child value={count} /> 只在挂载时读一次 count。其实每次父组件渲染,这行 JSX 都会重新执行,把最新的 count 传下去。子组件永远拿到的是父组件当前渲染时刻的值。

props vs state:两个最易混淆的概念

维度propsstate
来源父组件传入组件自己用 useState 创建
可变性只读,子组件不能改可变,通过 setter 修改
归属数据属于父组件数据属于组件自身
作用组件间通信(向下传)组件内部记忆

一句话记:props 是"别人给的",state 是"自己的"。给的东西不能改,自己的东西才能改。

小结

  • props 是父组件调用子组件时传入的数据,子组件通过参数(对象)接收,常用解构 + 默认值写法。
  • 字符串用引号,其余类型一律用大括号 {} 传。
  • props 只读:子组件不能修改,数据只能由拥有者(父组件)修改。
  • props 体现 React 的单向数据流——数据自上而下流动,可预测、易追踪。
  • props 变化会自动触发子组件重渲染,它是动态的,不是"传一次就固定"。
