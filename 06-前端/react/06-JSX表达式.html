JSX 中使用表达式 { }

花括号 { } 是 JSX 里最重要的语法——它是连接"静态结构"与"动态数据"的桥梁。搞清楚能放什么、不能放什么，条件渲染和列表渲染就全通了。

花括号是"JS 表达式插槽"

可以把 JSX 想象成一张填空卷：标签结构是固定的题目框架，{ } 是留给你填 JS 值的空格。花括号里必须是能产生一个值的 JS 表达式——因为 React 需要拿到这个值，决定渲染什么内容。

关键区别：表达式（expression）有值，语句（statement）没值。1 + 2 是表达式（值为 3），if (...) {} 是语句（没有值）。花括号只接受表达式。

类比：花括号就像模板字符串里的 ${}，`你好 ${name}`——你不会在 ${} 里写 if 语句吧？JSX 的 { } 是同一个道理。

能放什么：所有有值的表达式

合法的表达式——都能放进 { }
const name = 'Shea';
const age = 25;
const fruits = ['苹果', '香蕉', '橙子'];
const isVip = true;

function Demo() {
  return (
    <div>
      {/* ✅ 变量（最常见） */}
      <p>{name}</p>

      {/* ✅ 算术运算（有值） */}
      <p>明年 {age + 1} 岁</p>

      {/* ✅ 三元表达式（有值，条件渲染的正确姿势） */}
      <p>{isVip ? <span>VIP 用户</span> : '普通用户'}</p>

      {/* ✅ && 短路（有值，条件渲染的另一种姿势） */}
      {isVip && <span>专属图标</span>}

      {/* ✅ 函数调用（有返回值） */}
      <p>{name.toUpperCase()}</p>

      {/* ✅ 数组 .map()（返回数组，React 会逐个渲染） */}
      <ul>
        {fruits.map(fruit => (
          <li key={fruit}>{fruit}</li>
        ))}
      </ul>

      {/* ✅ JSX 本身也是值（可以嵌套） */}
      {<strong>粗体文字</strong>}
    </div>
  );
}

不能放什么：if/for 语句

if 语句和 for 循环是"语句"，不产生值。把它们放进 { } 会导致语法错误。解决办法：把逻辑提到 JSX 外面，或者用三元/&& 代替。

❌ 不能放语句
// 语法错误！if 没有值
return (
  <div>
    {if (isVip) {
      <span>VIP</span>
    }}
  </div>
);

// 语法错误！for 没有值
return (
  <ul>
    {for (let f of fruits) {
      <li>{f}</li>
    }}
  </ul>
);
✅ 正确替代方案
// 三元表达式代替 if
return (
  <div>
    {isVip ? <span>VIP</span> : null}
  </div>
);

// .map() 代替 for
return (
  <ul>
    {fruits.map(f => (
      <li key={f}>{f}</li>
    ))}
  </ul>
);
复杂逻辑的处理方式：如果条件很复杂，不要把 if 硬塞进 JSX。把逻辑提到 return 前面，用变量存结果，然后在 JSX 里渲染那个变量——这才是正确的组织方式。

const content = isVip ? <VipView /> : <NormalView />;
return <div>{content}</div>;

特殊不渲染值：false、null、undefined

React 对以下三个值有特殊处理——它们出现在 JSX 里时，什么都不渲染（不产生任何 DOM 节点）：

  • false
  • null
  • undefined

这个特性是条件渲染的基础——{isShow && <Modal />} 中，当 isShow 为 false 时，false 不渲染，所以 Modal 消失了。

不渲染的值 vs 会渲染的值
function App() {
  return (
    <div>
      {false}      // 不渲染，页面上什么都没有
      {null}       // 不渲染
      {undefined}  // 不渲染

      {0}          // ⚠️ 会渲染！页面上出现数字 "0"
      {''}         // 视觉上无内容（但与 false/null/undefined 不同：会渲染空文本节点）
      {NaN}        // 会渲染字符串 "NaN"
    </div>
  );
}
经典陷阱：&& 左边是数字 0

{list.length && <List />}

当 list 为空数组时，list.length 是 0。&& 短路，整个表达式的值是 0——而数字 0 会被渲染到页面上，出现一个莫名其妙的 "0"。

解法：把左边改成真正的布尔值：
{list.length > 0 && <List />}
或者：{!!list.length && <List />}
或者：{list.length ? <List /> : null}

综合示例：常见表达式用法汇总

实际开发中最常见的 { } 用法
function UserPanel({ user, messages, isAdmin }) {
  // 复杂逻辑提到 return 前面，用变量存
  const displayName = user.nickname || user.email;

  return (
    <div>
      {/* 变量 */}
      <h2>{displayName}</h2>

      {/* 三元：根据条件渲染不同内容 */}
      {isAdmin ? <AdminBadge /> : <UserBadge />}

      {/* &&：消息数量 > 0 才渲染，避免渲染 "0" */}
      {messages.length > 0 && (
        <p>你有 {messages.length} 条消息</p>
      )}

      {/* .map()：列表渲染，每项要加 key */}
      <ul>
        {messages.map(msg => (
          <li key={msg.id}>{msg.text}</li>
        ))}
      </ul>
    </div>
  );
}

与其他知识点的联系

  • JSX 语法规则（05）：05 篇讲了 JSX 的三条硬性规则，其中"注释必须用花括号包裹"直接依赖本文讲的"花括号是表达式插槽"这个机制——注释本质上是把内容藏进了一个不会被渲染的表达式里。两篇合起来才能完整理解花括号的能力边界：既能承载真实要渲染的值，也能承载"不渲染"的特殊值（如 {/* 注释 */} 本身不产生输出）。
  • 条件渲染的本质：本文反复强调"false/null/undefined 不渲染"，这正是条件渲染（&&、三元表达式）能够工作的底层机制。条件渲染不是 React 提供的专门语法，而是巧妙利用了"表达式有值、且某些值不显示"这两条规则的组合结果。
  • 列表渲染与 key：本文用 .map() 演示列表渲染时提到"每项要加 key"，这是因为 React 需要在 Diff 阶段区分列表中的每一项——如果不提供 key，React 默认用索引，一旦列表顺序变化（插入、删除、排序），会导致不必要的重新渲染甚至状态错乱。这也是为什么 .map() 生成的 JSX 数组必须配合稳定的 key（通常是数据的唯一 id，而不是数组下标）。
  • React.createElement 与虚拟 DOM 对象：花括号里的表达式，最终是作为 createElement() 的参数（children 或某个 prop 的值）传进去的。理解了这一点，就明白了"为什么放 if 语句会报错"——函数调用的参数位置在 JS 语法层面就只能是表达式，这不是 React 定的规则，是 JS 语言本身的限制。
