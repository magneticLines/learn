CSS · 视觉样式

border-radius

圆角看似简单，但椭圆圆角、胶囊按钮、圆形头像、图片裁切背后的语法逻辑值得吃透——它远不止「加个圆边」。

一、四角简写：1~4 个值

border-radius 和 margin 一样支持 1~4 个值简写，顺序是从左上角开始顺时针（左上 → 右上 → 右下 → 左下）：

写法含义
border-radius: 12px四角都 12px
border-radius: 12px 0左上/右下 12px，右上/左下 0（对角分组）
border-radius: 12px 8px 4px左上 / 右上&左下 / 右下
border-radius: 12px 8px 4px 0四角各一值，顺时针
12px
20px 0
叶子形

二、单角设置

需要只改一个角时，用对应的长属性更清晰：

单角
border-top-left-radius: 12px;
border-top-right-radius: 12px;
/* 常用：只圆上面两角，做 tab、抽屉、卡片顶部 */

三、椭圆圆角：斜杠语法（水平/垂直半径）

这是最被忽视的能力。每个角的圆其实可以是椭圆——分别指定水平半径和垂直半径，用斜杠 / 分隔：斜杠前是 4 个角的水平半径，斜杠后是 4 个角的垂直半径。

椭圆圆角
/* 水平半径 50px，垂直半径 20px → 扁椭圆角 */
border-radius: 50px / 20px;
/* 完整：四角水平 / 四角垂直 */
border-radius: 40px 10px 40px 10px / 10px 40px 10px 40px;
60px / 20px
叶尖椭圆
圆角本质是「在角上画一段椭圆弧」。普通 border-radius: 12px 是水平=垂直=12px 的正圆弧；斜杠语法让两个半径分开，于是有了椭圆弧。理解这点，各种异形圆角就不神秘了。

四、50% → 圆形与胶囊

用百分比时，半径相对于元素自身尺寸计算，于是 50% 能自适应地切出圆/胶囊：

  • 正方形 + 50% → 正圆（头像经典写法）。
  • 长方形 + 50%（或一个超大 px 如 9999px） → 胶囊/药丸按钮。
圆形 & 胶囊
.avatar { width: 64px; height: 64px; border-radius: 50%; }
.pill { padding: 6px 16px; border-radius: 9999px; }
/* 胶囊推荐用大 px 而非 50%：宽高变化时圆弧始终是半圆，更稳 */
50% 圆形
胶囊按钮
9999px
胶囊别用 border-radius: 50%：长方形用 50% 会算成「宽的一半 × 高的一半」的椭圆，按钮会变成橄榄球形。用足够大的 px（如 9999px），浏览器会自动收敛成「两端半圆」，宽高变化都稳。

五、与 overflow: hidden 配合裁切图片

圆角只裁切元素的背景和边框，不会裁切里面溢出的子元素（如 <img>）。所以圆角容器里放图片，图片的直角会露在圆角外面——必须加 overflow: hidden：

反例 → 问题 → 正例
/* 反例：只给容器加圆角 */
.card { border-radius: 12px; }
.card img { width: 100%; }
/* 问题：图片直角溢出圆角，露出四个尖角 */

/* 正例：容器裁掉溢出 */
.card { border-radius: 12px; overflow: hidden; }
/* 或直接给 img 自己加 border-radius 也行 */
小心副作用：overflow: hidden 会裁掉一切溢出内容，包括你想露出的 box-shadow、下拉菜单、tooltip。若卡片既要圆角裁图又要内部弹层，可考虑给图片单独加圆角，而非整卡 overflow: hidden。

与其他知识点的联系

  • 背景属性：background-clip 决定背景画到 border-box/padding-box 中的哪一个边界，而这个边界本身可以是圆角——圆角容器配 background-clip: padding-box 能让背景不透到虚线边框下面，两者是「形状」和「画到哪」的搭配关系。
  • 文字溢出：圆角裁切图片用的 overflow: hidden，和单行省略号三件套里的 overflow: hidden 是同一个属性的两种用法——都是「让容器裁掉超出自身形状的内容」，只是这里裁的形状从矩形变成了圆角矩形。
  • filter（下一节）：filter: drop-shadow() 会沿元素的实际不透明像素轮廓投影，而不是像 box-shadow 那样只贴矩形边框；给圆角元素加投影时，普通 box-shadow 已经能跟随圆角画出弧形阴影，这也是为什么大部分圆角卡片不需要专门上 drop-shadow。
  • 性能（重排重绘）：静态的 border-radius 只影响 Paint（重绘）阶段，开销很小；但如果用它做「动画变形」（如方形按钮 hover 变圆），比起直接动画 border-radius，用 clip-path 或 transform: scale() 配合遮罩通常在复杂形状下更省性能，这是后面性能优化章节会展开的取舍。
  • BEM / CSS 方法论：「胶囊按钮」「圆形头像」「卡片圆角」是高频复用的视觉样式，适合抽成独立的 Modifier 类（如 .btn--pill、.avatar--round），避免每个组件里重复写 border-radius: 9999px 这类魔法数值。
