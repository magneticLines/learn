Grid 网格布局

Grid 是 CSS 第一个真正的二维布局系统——同时控制行和列。Flex 解决"一条线上怎么排"，Grid 解决"一张表格里怎么摆"。

Flex 与 Grid 的本质区别

Flex 是一维的：你只能管一行或一列，换行后行与行之间互不对齐。Grid 是二维的：你预先画好一张由行列交织的网格，子项落进格子里，行列同时受控。

一个直觉类比：Flex 像把东西放进一个传送带，它们排成一条线；Grid 像把东西放进 Excel 表格，横竖都对齐。当你的设计稿天然是"表格状"（仪表盘、卡片墙、整版页面布局）时，用 Grid；当是"一条流式的列表/工具栏"时，用 Flex。

定义网格：columns 与 rows

给容器设 display: grid，再用 grid-template-columns 声明有几列、每列多宽。每个值就是一列的尺寸：

三列，宽度分别 100/自适应/100
.grid {
  display: grid;
  grid-template-columns: 100px 1fr 100px;
  gap: 10px; /* 行列间距，Grid 里 gap 最自然 */
}

fr 单位：按比例瓜分剩余空间

fr（fraction）是 Grid 专属单位，表示"剩余空间的一份"。1fr 1fr 1fr 就是三等分；2fr 1fr 是 2:1。它和 flex-grow 思路一致，但写法更直观。fr 是在减去固定尺寸和 gap 之后再分配的，所以 100px 1fr 里 1fr 拿到的是"总宽减 100px 减 gap"。

repeat / minmax / auto-fill / auto-fit

列很多时手写重复值很烦，repeat() 帮你简化：repeat(3, 1fr) = 1fr 1fr 1fr。真正强大的是它和 minmax()、auto-fit 组合，能写出无需媒体查询的自适应网格。

  • minmax(200px, 1fr)：每列最小 200px，有空间就长到 1fr。
  • auto-fill：尽可能多塞列，塞不满也保留空轨道。
  • auto-fit：塞不满时把空轨道折叠，让现有列拉伸填满。

响应式卡片网格（核心技巧）

一行代码实现自适应卡片墙
.cards {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(160px, 1fr));
  gap: 12px;
}

这行代码的含义：每个卡片至少 160px，宽度够就多放几列、不够就自动换行并拉伸填满——整个响应式逻辑不写一句媒体查询。这是 Grid 最惊艳的实用场景。下面调整浏览器窗口宽度可看到列数自动变化：

auto-fill vs auto-fit：当卡片数量少、放不满一行时，auto-fill 会保留右侧空轨道（卡片不拉伸），auto-fit 会折叠空轨道让卡片铺满整行。想让稀疏内容铺满用 auto-fit。

网格线与区域命名

Grid 提供两种"摆放子项"的思路。一种是按网格线编号定位：grid-column: 1 / 3 表示从第 1 条竖线跨到第 3 条（占两列）。另一种是更可读的区域命名 grid-template-areas，直接用文字画出整版布局：

用 ASCII 画出页面结构
.layout {
  display: grid;
  grid-template-columns: 160px 1fr;
  grid-template-areas:
    "header  header"
    "sidebar main"
    "footer  footer";
}
.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }

用 grid-column: 1 / 3（跨列）和 grid-row: 1 / 3（跨行）让某个元素占据更大面积，其余子项自动绕开它填充——这种"瀑布/拼图"效果用 Flex 几乎无法优雅实现。

该用 Grid 还是 Flex？

场景推荐理由
导航栏、工具栏、标签列表Flex一维流式，内容驱动尺寸
整页布局（头/侧/主/脚）Grid二维结构，areas 直观
等宽响应式卡片墙Gridauto-fit+minmax 免媒体查询
单个元素居中都行Grid 用 place-items:center 更短
内容多少不定、需自动换行的标签Flexflex-wrap 按内容自然流动
不是二选一：真实项目里二者常嵌套——外层用 Grid 搭页面骨架，每个区块内部再用 Flex 排工具栏。记住口诀：布局结构用 Grid，组件内部一行排列用 Flex。

与其他知识点的联系

  • 关联 Flexbox（一维布局）：Grid 和 Flex 是互补而非竞争关系——理解了 Flex 的 flex-grow 按比例分配空间的思路，再学 Grid 的 fr 单位会非常顺畅，两者本质都是"剩余空间按比例瓜分"，只是 Grid 同时管行和列。
  • 关联媒体查询：grid-template-columns 和 grid-template-areas 是媒体查询里最常被重写的属性——移动端改成单列 1fr，桌面端展开成多列或重新排布区域，这是"结构性响应式"（列数、区域换挡）与 Grid 结合最紧密的地方。
  • 关联相对单位与 clamp：repeat(auto-fit, minmax(160px, 1fr)) 这种自适应网格写法，本质是把响应式逻辑下沉到了 CSS 函数层面，不需要写媒体查询；如果再把 minmax 的下限换成 clamp() 计算出的流体值，卡片宽度也能连续缩放，两种技术经常组合使用。
  • 关联 position 与 z-index：Grid/Flex 子项和 position: absolute 一样，都能让 z-index 在没有显式 position 的情况下生效（只需 z-index 非 auto），这是很容易被忽略的一条层叠上下文触发规则。
  • 关联 React 组件布局：React 项目里的整页骨架（Header/Sidebar/Main/Footer）几乎都直接映射成 grid-template-areas，组件只需要声明自己占据哪个区域名，不必关心具体像素位置——这种"声明式布局"的思维方式和 React 组件本身的声明式风格高度一致，也是 Grid 相比传统布局最大的心智优势。
