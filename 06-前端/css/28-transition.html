transition 过渡

让属性的变化从"瞬间跳变"变成"平滑滑动"，是 CSS 交互动画最轻量的入口。

它解决什么问题

默认情况下，当一个元素的属性发生变化（比如 :hover 时背景色从灰变蓝），浏览器会立刻在下一帧把它画成新值——这是一个突兀的"跳变"。transition 的作用就是告诉浏览器："不要瞬间切换，请用 N 秒时间，按某种节奏，把旧值插值到新值"。

本质上它是浏览器替你做的补间动画（tweening）：你只声明起点和终点（两个状态的 CSS 值），中间帧由引擎自动计算。这与 JS 里手写 requestAnimationFrame 逐帧改样式相比，省心且默认走 GPU 优化路径。

四个子属性与简写

CSS
.box {
  transition-property: background-color, transform;  /* 过渡哪些属性 */
  transition-duration: 0.3s;             /* 持续时间，必填且 >0 才有效 */
  transition-timing-function: ease;      /* 缓动曲线 */
  transition-delay: 0s;                  /* 延迟多久才开始 */

  /* 简写：property duration timing-function delay */
  transition: background-color 0.3s ease 0s;
  /* 多属性不同配置用逗号分隔 */
  transition: transform 0.4s ease, opacity 0.2s linear;
}
陷阱：简写里前两个时间值，第一个总被解析为 duration，第二个才是 delay。写 transition: 0.3s 1s 表示"延迟 1 秒、持续 0.3 秒"，顺序写反是新手最常见的 bug。

建议：避免用 transition: all。all 会监听所有属性变化，可能意外触发昂贵的重排，也会让性能问题难以定位。显式列出你真正要动的属性。

悬停时背景、颜色、位移同时平滑过渡；移开时自动反向回退——这正是 transition 的优势：状态切换两端都自动补间，无需写返回动画。

哪些属性能过渡，哪些不能

能否过渡的核心标准：该属性的值是否能被连续插值。颜色（RGB 通道渐变）、长度（数值插值）、opacity、transform 都可以；而像 display 这种离散关键字（none↔block 之间没有中间态）传统上无法过渡。

类型例子能否过渡
颜色color background-color border-color能
长度/数值width opacity margin font-size能
变换transform filter能（推荐）
离散值display visibility（部分）不能 / 受限
auto 关键字height: auto不能（auto 无法插值）
关于 display：经典做法是用 opacity + visibility 模拟淡入淡出而非动 display。新规范的 transition-behavior: allow-discrete + @starting-style 已能过渡 display，但兼容性仍需注意。height: auto 不能过渡，常用 max-height 或 grid 的 grid-template-rows: 0fr→1fr 技巧绕过。

缓动函数 timing-function

缓动函数决定了动画的"节奏感"——同样是 0.3 秒，是匀速、先慢后快还是分段跳跃，观感天差地别。它本质是一条"时间 → 进度"的映射曲线（贝塞尔曲线）。

关键字等价 cubic-bezier感觉
linear—匀速，机械感，适合 loading 旋转
ease（默认）(0.25,0.1,0.25,1)慢→快→慢，最自然
ease-in(0.42,0,1,1)由慢加速，适合"离开"
ease-out(0,0,0.58,1)由快减速，适合"进入/弹出"
steps(n)—分 n 段跳变，做雪碧图逐帧动画
linear（匀速）
ease-out（减速停）
steps(6)（分段跳）

悬停此区域，对比三种曲线的运动差异。

自定义曲线用 cubic-bezier(x1,y1,x2,y2)，其中 y 可以超过 1（产生"过冲回弹"的弹性效果，如 cubic-bezier(0.34,1.56,0.64,1)）。可在 cubic-bezier.com 可视化调参。

性能：优先过渡 transform / opacity

这是动画领域最重要的一条经验法则。原因要联系浏览器渲染管线：

  • 过渡 width / height / top / left / margin → 触发 Layout（重排），浏览器要重新计算页面所有相关元素的几何位置，再 Paint、再合成，每一帧都很贵。
  • 过渡 transform / opacity → 只走 Composite（合成），可由 GPU 在独立图层上完成，跳过 Layout 和 Paint，丝滑且省电。
正例：要让元素"右移 100px"，用 transform: translateX(100px) 而不是 left: 100px；要"放大"用 transform: scale(1.2) 而不是改 width。后续 will-change 一节会深入这条管线。

transition vs animation

维度transitionanimation
触发方式需要状态变化触发（hover/class 切换/JS 改值）可自动播放，无需触发
关键帧只有起点和终点两个状态可定义任意多个中间帧（@keyframes）
循环不能循环可无限循环 / 指定次数
方向控制切换状态自动反向alternate / reverse 等精细控制
典型场景按钮 hover、展开收起、主题切换loading 旋转、脉冲、入场动画

一句话："A 状态↔B 状态的平滑切换"用 transition；"持续或多步骤的复杂动画"用 animation。两者底层共享同一套插值与缓动机制。
