overflow 与裁剪

当内容比盒子还大，多出来的部分该怎么办？overflow 决定了「溢出」的命运：显示、裁掉，还是给一个滚动条。

一、溢出从哪来

盒子的尺寸由 width/height 限定，但里面的内容不一定听话——一段超长文字、一张大图、一个写死宽度的子元素，都可能比盒子大。这时内容会溢出（overflow）盒子的边界。overflow 属性就是用来规定「溢出部分如何处理」的总开关。

二、五个取值

值行为滚动条
visible（默认）溢出内容照常显示，溢到盒子外面无
hidden溢出部分被裁掉，看不见也滚不到无
scroll裁掉但提供滚动条，始终显示滚动条（即使没溢出）始终有
auto需要时才出现滚动条（最常用）按需
clip像 hidden 一样裁切，但禁止任何滚动（包括脚本滚动），性能更好无
hidden vs clip：两者视觉上都是裁掉。区别是 hidden 仍创建滚动容器（可被 JS scrollTop 滚动、会建立 BFC），而 clip 是纯裁切、不创建滚动容器、不响应任何滚动，因此渲染更轻量。只想裁掉装饰性溢出时优先用 clip。

四、overflow-x 与 overflow-y

overflow 是简写，可分别控制横竖两轴：

CSS
.table-wrap {
  overflow-x: auto;    /* 横向超宽时给横滚动条（如宽表格） */
  overflow-y: hidden;  /* 纵向直接裁掉 */
}
陷阱：当 overflow-x 和 overflow-y 取不同值，且其中一个是 visible、另一个不是时，浏览器会把 visible 强制改成 auto。因为「一个轴裁切、另一个轴不裁」在几何上无法实现。想一轴滚、一轴溢出？做不到，这是规范规定。

五、副作用：overflow 会创建 BFC

这是 overflow 最重要、也最常被「顺手利用」的副作用：只要 overflow 不是 visible，该元素就会创建一个 BFC（Block Formatting Context，块级格式化上下文）。

BFC 是一块「布局隔离区」，内部的布局不会影响外部。这带来几个经典用途：

  • 清除浮动：父元素 overflow: hidden 能包住内部浮动的子元素，撑开高度（古早的清浮动技巧）。
  • 阻止 margin 折叠：上一节讲过，overflow: hidden 的父元素不会被子元素 margin 穿透。
  • 阻止文字环绕浮动：BFC 区域不会和浮动元素重叠。
这是双刃剑。很多人为了清浮动随手写 overflow: hidden，结果意外裁掉了本该溢出的内容（比如绝对定位的下拉菜单、tooltip、阴影）。如果你只想要 BFC 而不想要裁切，请用 display: flow-root——它是专门为「创建干净 BFC」而生的，没有任何裁切副作用。

六、滚动容器与文字省略号

给一个固定高度的容器加 overflow-y: auto，它就成了独立的滚动容器，内部内容超出时只滚这块、不滚整页——这是侧边栏、聊天列表、模态框正文的标准做法。

而单行文字截断的经典三件套，正是以 overflow 为核心：

CSS · 单行省略号
.ellipsis {
  overflow: hidden;          /* 1. 裁掉溢出 */
  white-space: nowrap;      /* 2. 不换行，逼出溢出 */
  text-overflow: ellipsis; /* 3. 溢出处显示 … */
}
这是一段非常非常非常长的文字会在容器边缘被截断并显示省略号你看不到我后面写了什么哈哈哈哈

注意：text-overflow: ellipsis 单独写没用，它依赖 overflow 先把溢出裁掉才有截断点可加省略号。多行省略号则要配合 -webkit-line-clamp（文本相关章节会展开，这里只点出 overflow 是它的地基）。

小结

  • 五个值：visible（默认露出）/ hidden（裁掉）/ scroll（恒滚动条）/ auto（按需，最常用）/ clip（纯裁切不滚动）。
  • overflow-x/y 分轴控制，但一轴 visible + 另一轴非 visible 时 visible 会被强制成 auto。
  • 非 visible 值会创建 BFC——可用来清浮动、阻止 margin 折叠，但会裁掉溢出物，要纯 BFC 请用 flow-root。
  • 滚动容器、文字省略号都以 overflow 为地基。
