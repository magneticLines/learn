display 常用值

display 是 CSS 布局的"总开关"——它决定一个元素如何参与排版、如何对待自己的子元素。理解它是理解 Flex、Grid 的前提。

为什么 display 是布局的起点

浏览器拿到一个元素时，第一个要回答的问题是：这块东西怎么摆？是独占一行，还是和别人挤在一行里？它内部的孩子又按什么规则排？display 就是回答这两个问题的属性。

这就引出了 display 的现代心智模型：每个值其实由"外显类型"和"内部类型"两部分组成。外显类型（outer）决定元素自己在父级里怎么排（像块还是像行内），内部类型（inner）决定它如何布局自己的孩子（普通流、flex 流还是 grid 流）。早期 CSS 把这两件事揉成了一个关键字，所以 block、flex 看起来像并列的值，其实不是一个维度的东西。

block / inline / inline-block

这三个是普通文档流里最基础的三种盒子，区别集中在三件事：是否独占一行、能否设宽高、能否设上下 margin/padding 撑开布局。

inline 设了 width: 200px 没反应，是初学者最常见的困惑之一。原因是 inline 盒子的尺寸由内容（文字行）决定，它属于"文本排版"的一部分，不接受你强行指定宽高。要么换成 block，要么换成 inline-block。

inline-block 解决了什么问题

inline-block 是个"混血儿"：对外像 inline（不独占一行，能并排），对内像 block（能设宽高、能设垂直 padding）。在 Flex 出现之前，它是做"一排等宽按钮""横向导航"的主力方案。

陷阱：inline-block 的空白间隙

用 inline-block 并排元素时，你会发现盒子之间莫名多出几像素的缝隙，即使没设 margin。这不是 bug：因为 inline-block 参与的是文本流，HTML 源码里元素标签之间的换行符和空格会被渲染成一个空白字符（就像单词之间的空格）。

反例：会有缝隙
<span class="ib">甲</span>
<span class="ib">乙</span>  /* 换行 = 一个空格 */

常见解法有：把标签写在同一行不换行（难维护）、父级设 font-size: 0 再在子级恢复（hack）、用注释吃掉换行 </span><!----><span>。但这些都是"打补丁"。

正解：遇到要并排排版的需求，今天直接用 display: flex。Flex 子项不参与文本流，根本不存在空白间隙问题，还能轻松控制对齐和间距（gap）。inline-block 的并排场景已基本被 Flex 取代。

还有第三种"隐藏"：opacity: 0。它占空间、可点击、可过渡，常用于淡入淡出动画。三者各有适用场景：彻底删用 display:none，占位隐藏用 visibility，要动画用 opacity。

flex 与 grid（内部布局）

display: flex 和 display: grid 改变的是内部类型：元素自己默认仍像 block（独占一行），但它的直接子元素不再按普通流排，而是变成 flex 项或 grid 项，进入一维 / 二维布局引擎。这两个是现代布局的核心，后面两节专门讲。

它们也各有 inline 版本：inline-flex / inline-grid，让容器自己变成 inline 级（能和文字并排），但内部仍是 flex/grid 布局。

display: contents（进阶）

这个值很特别：它让元素自己的盒子消失，但孩子保留。等于把这个元素"拆掉外壳"，孩子直接被提升到父级布局里。在 Flex/Grid 布局中，如果你有一层不想要的包裹 div 破坏了布局结构，可以给它设 display: contents，让孙子辈直接成为 grid 项。

注意：display: contents 早期有可访问性 bug（部分浏览器会丢失语义角色），用在 button、table 等有语义的元素上要谨慎，普通容器 div 上较安全。

新语法：display 双值写法

为了把"外显 + 内部"两个维度显式分开，CSS 引入了双关键字写法，比如 display: inline flex 等价于旧的 inline-flex，display: block flow 等价于普通 block。

新旧写法对照
.box { display: block flex;   /* = flex（外块内flex）*/
       display: inline flex;  /* = inline-flex */
       display: block flow;   /* = block（普通流）*/
}

了解即可——目前浏览器对双值的支持已较好，但实际项目里大家仍习惯写单值 flex / inline-flex，单值更短也更通用。这个新语法的价值在于帮你建立"外显与内部是两件事"的正确心智模型。

小结：display 控制两件事——对外怎么排（block/inline）、对内怎么排孩子（flow/flex/grid）。普通流时代靠 block/inline/inline-block 凑布局，现代布局直接用 flex/grid。隐藏元素时分清 none / visibility / opacity 三者的占位与动画差异。
