margin 折叠

两个 margin 相遇，结果不是相加，而是「合并取大」。这是 CSS 最反直觉、却最常考的机制。

一、什么是 margin 折叠

margin collapsing（外边距折叠）：当两个垂直方向的 margin 相遇时，它们不会叠加，而是合并成一个，取其中较大的值。注意三个限定词——只发生在垂直方向、只发生在块级元素、只折叠 margin（padding 和 border 永不折叠）。水平 margin 永远老老实实相加。

记忆口诀：「上下合并取大，左右照常相加」。折叠是 block 布局垂直流的专属行为。

二、场景一：相邻兄弟元素

最常见的情况：两个上下相邻的块，上面的 margin-bottom 和下面的 margin-top 折叠。

CSS
.top    { margin-bottom: 30px; }
.bottom { margin-top: 20px; }
/* 两者间距不是 30+20=50px，而是 max(30,20)=30px */
上块 margin-bottom: 30px
下块 margin-top: 20px

实际间距是 30px，不是 50px。两个 margin 重叠在一起了。

三、场景二：父子元素折叠

更隐蔽的情况：子元素的 margin-top 会「穿透」父元素，跑到父元素外面去，和父元素的 margin-top 折叠。

反例：你给父容器 .parent 一个白底，给子元素 .child 设 margin-top: 40px，期望子元素在父容器内部往下 40px。结果——子元素紧贴父容器顶部，反而是整个父容器往下掉了 40px。
反例
.parent { background: #eee; }
.child  { margin-top: 40px; } /* 期望在父内下移，实际带着父一起下移 */

问题根源：父元素和子元素的顶边「贴」在一起（中间没有任何东西隔开），布局引擎认为这两个 margin 在同一位置，于是合并，子 margin 顶出了父元素。

正例 · 任选一种隔开手段
.parent {
  padding-top: 1px;      /* 用 padding 隔开顶边 */
  /* 或 */
  border-top: 1px solid; /* 用 border 隔开 */
  /* 或 */
  overflow: hidden;     /* 触发 BFC，下一节细讲 */
  /* 或 */
  display: flow-root;   /* 最干净，专为创建 BFC 而生 */
}

四、场景三：空块自身折叠

一个没有内容、没有 padding/border/height 的空块，它自己的 margin-top 和 margin-bottom 会折叠成一个。所以 <div style="margin:20px 0"></div> 这种空占位块，只占 20px 高而不是 40px。

五、为什么会有折叠这种设计

这不是 bug，是刻意为之的排版优化。CSS 诞生时的主战场是文档排版（想想一篇文章）。假设每个段落 <p> 都有 margin: 1em 0：

  • 不折叠：段落之间是 上 1em + 下 1em = 2em，而文章首段顶部却只有 1em。间距忽大忽小，很难看。
  • 折叠：段落之间统一 1em，整篇节奏一致，且第一段和最后一段也自然和容器保持 1em。
折叠让「给每个块都设上下 margin」这种最自然的写法，恰好产出最和谐的垂直间距。它是为文档流量身定做的。

六、哪些情况不折叠

不折叠的情况原因
flex / grid 容器的子项它们不在 block 流里，走的是另一套布局算法
有 padding 或 border 隔开两个 margin 之间「夹了东西」，不再贴合
父元素 overflow 非 visible创建了 BFC，子 margin 被关在里面
display: flow-root / 绝对定位 / 浮动元素脱离普通流或自成 BFC
水平方向（左右 margin）折叠只发生在垂直方向

把这张表反过来看，就是「如何避免意外折叠」的工具箱：要么用 flex/grid 布局（现代项目默认，根本不折叠），要么用 padding/border 隔开，要么创建 BFC。

七、现代视角

在 flexbox 和 grid 成为主流后，margin 折叠的「坑」其实少多了——因为 flex/grid 子项之间根本不折叠，且更推荐用容器的 gap 属性来控制间距，gap 既不折叠也不会穿透父元素，行为完全可预测。能用 gap 就别用 margin 去抠间距，这是规避折叠问题的终极方案。

小结

  • 折叠 = 垂直方向相邻 margin 合并取大值，仅限 block 流。
  • 三种场景：相邻兄弟、父子（子 margin 穿透父）、空块自身。
  • 设计初衷是让文档段落间距均匀，是排版优化而非 bug。
  • flex/grid 子项、padding/border 隔开、BFC 都不折叠；优先用 gap 一劳永逸。
