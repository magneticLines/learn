背景属性

背景不只是「填个色」。理解 background 的每个子属性如何分工，你才能用一个元素叠出复杂的视觉层次，还能玩出渐变文字这种花活。

一、心智模型：背景是一摞图层

关键认知：CSS 里纯色背景在最底层，背景图叠在它之上，而且背景图可以有多张，像 Photoshop 的图层一样从上往下盖。所有 background-* 属性都是在描述「这些图层怎么画」——画在哪、画多大、要不要平铺、相对谁定位。先建立「图层」的概念，下面每个属性就都好懂了。

二、background-color：兜底色

最底层的纯色，永远在最后画。即使有背景图也建议设一个，作为图片加载失败或渐变留白时的兜底，避免文字悬空看不清。

background-color
.box { background-color: #1e2030; }

三、background-image / repeat / position / size

这四个是背景图的核心控制项，常一起出现：

属性作用常用值
background-image指定图片或渐变url(..)、linear-gradient(..)
background-repeat是否平铺no-repeat / repeat-x / repeat / space / round
background-position图片起点位置center / top right / 20px 50%
background-size图片缩放cover / contain / 100% auto

cover vs contain：最常考的一对

  • cover：等比缩放到填满容器，多出来的部分被裁掉。适合 banner、卡片封面——宁可裁也不能留白。
  • contain：等比缩放到完整放下，可能留空白。适合 logo、产品图——宁可留白也不能裁。
陷阱：background-size: cover 一定要配 background-position: center，否则裁切默认从左上角发生，人脸/主体常被切掉。这是最常见的「图片显示不全」原因。

四、background-attachment：滚动时图片跟不跟

  • scroll（默认）：背景随元素内容一起滚。
  • fixed：背景相对视口固定，页面滚动时背景不动——经典的「视差」效果。
  • local：背景随元素内部滚动条滚动。
移动端慎用 fixed：很多移动浏览器对它支持不佳，且会强制创建合成层导致滚动卡顿。视差效果优先考虑 position 或 JS。

五、background-origin / clip：定位与裁剪的「画布边界」

这两个属性都和盒模型的三个区域有关（border-box / padding-box / content-box），但管的事不同——这是个高频混淆点：

属性管什么默认值
background-origin背景图从哪开始定位（position 的原点）padding-box
background-clip背景画到哪为止（超出部分裁掉）border-box
记忆法：origin 管「起点」，clip 管「边界」。比如 background-clip: padding-box 能让背景不延伸到 border 下面，配合虚线边框时背景不会透出来。

六、简写 background：顺序与坑

简写
/* 一行搞定多个子属性 */
.box {
  background: #1e2030 url(bg.png) no-repeat center / cover;
  /*        color    image     repeat   position / size  */
}
两个坑：①size 必须写在 position 后面，用斜杠隔开（center / cover），单独写会失效。②简写会把没写到的子属性重置为初始值——如果你之前单独设过 background-repeat，又用了简写却没带 repeat，它会被悄悄重置成 repeat。

七、多重背景：一个元素叠多张图

用逗号分隔多组背景值，写在前面的在上层。颜色只能有一个，且永远在最底层。

多重背景
.hero {
  background:
    url(noise.png) repeat,          /* 上层：噪点纹理 */
    linear-gradient(#0003, #000a), /* 中层：压暗遮罩 */
    url(photo.jpg) center / cover; /* 底层图 */
}
/* 经典：图片 + 渐变遮罩，让上面的白字看得清 */
渐变图 + 压暗遮罩 → 白字清晰

八、background-clip: text → 渐变文字

把背景裁剪到「文字形状」，再让文字本身透明，背景就透过字显现出来，这就是渐变文字/图片填充文字的原理。

渐变文字三件套
.grad-text {
  background: linear-gradient(90deg, #60a5fa, #f472b6);
  background-clip: text;
  -webkit-background-clip: text; /* 兼容前缀，仍需保留 */
  color: transparent; /* 文字透明，露出背景 */
}
Gradient Text

background-clip: text + color: transparent

无障碍提醒：文字透明后，屏幕阅读器仍能读到文字（它读 DOM 而非渲染色），所以无障碍 OK。但要注意保留 -webkit-background-clip 前缀，否则部分浏览器整段文字会变透明消失。

与其他知识点的联系

  • 渐变（下一节）：渐变本质是一种特殊的 background-image，所以本节讲的 background-repeat、background-position、多重背景叠加、background-clip: text 这些能力，渐变全部能直接复用——学完背景属性再看渐变，会发现只是换了一种「图片」而已。
  • border-radius：background-clip 决定背景画到哪个盒模型边界为止，而 border-radius 决定这个边界本身是不是圆角——两者组合才能做出「圆角卡片背景不透出方角」的效果，理解 clip 的边界概念对吃透圆角裁切也有帮助。
  • filter 与 backdrop-filter：背景图和 filter: blur()/backdrop-filter 常一起出现，比如给背景图叠一层模糊做卡片底纹。但两者作用对象不同：background 系属性决定「画什么」，filter 决定「画完之后怎么再加工」，是渲染管线里前后相继的两步。
  • 性能（重排重绘）：background-attachment: fixed 之所以在移动端卡顿，是因为它强制元素脱离普通滚动合成、每次滚动都要重新计算背景相对视口的位置，本质上是拿重绘性能换视差效果。这与后面「will-change 与性能优化」一节中「合成层」的概念直接相关。
  • BEM / CSS 方法论：多重背景、渐变遮罩这类复杂背景配置往往写起来又长又难记，适合抽成一个语义化的 Modifier 类（如 .hero--overlay）而不是到处复制同一串 background 简写，这正是 BEM 提倡的「用命名封装实现细节」。
