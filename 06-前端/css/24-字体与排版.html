字体与文字排版

90% 的网页内容是文字。把字号、行高、字间距调对，页面的「质感」就上来了——这一节讲清每个排版属性该怎么用、为什么。

一、font-family 与字体栈

font-family 不是「选一个字体」，而是给浏览器一个降级列表：从左到右逐个尝试，用第一个本机装了的。最后一定要兜底一个通用族（sans-serif/serif/monospace），保证任何系统都有字可用。

字体栈（含中英文降级）
body {
  font-family:
    -apple-system, "Segoe UI",   /* 系统英文字体 */
    "PingFang SC", "Microsoft YaHei", /* 中文优先苹方/雅黑 */
    sans-serif;                  /* 终极兜底 */
}
/* 含空格或中文的字体名要加引号 */
中英混排技巧：把英文字体写在中文字体前面。因为西文字体不含汉字，渲染中文时会自动跳到后面的中文字体，于是英文用漂亮的西文字体、中文用苹方，各取所长。

二、web font：@font-face 与 font-display

想用用户机器上没有的字体，就用 @font-face 自己加载字体文件。但字体文件往往几百 KB，加载期间文字怎么显示？这由 font-display 决定，是性能与体验的关键。

@font-face
@font-face {
  font-family: "MyFont";
  src: url(myfont.woff2) format("woff2");
  font-weight: 400 700; /* 可变字体的权重范围 */
  font-display: swap;
}
font-display加载时行为
swap立刻用后备字体显示，字体到了再换。无空白期，但有「字体闪烁(FOUT)」。最常用。
block短暂隐藏文字（最多 3s）等字体，可能出现「空白(FOIT)」。
optional给极短时间，没加载到就放弃用后备字体。对性能最友好。
优先用 woff2（压缩率最高），并优先选 swap，避免用户盯着空白屏。中文字体动辄几 MB，慎做全量 web font，可用字体子集化（subset）只打包用到的字。

三、font-size / weight / style

  • font-size：正文建议用 rem（相对根字号，方便整体缩放与无障碍），别用 px 写死。
  • font-weight：100~900，400=normal，700=bold。注意：字体若没提供某个字重，浏览器会用「合成加粗」，效果较差。
  • font-style：italic（真斜体，字体专门设计）vs oblique（机械倾斜）。

四、line-height：为什么推荐无单位值

行高决定行与行的疏密，是排版可读性的核心。它能写 px、百分比或无单位数字，强烈推荐无单位：

反例 → 问题 → 正例
/* 反例：父级写死行高 px */
.parent { font-size: 16px; line-height: 24px; }
.child { font-size: 32px; }
/* 问题：child 继承的是计算后的 24px，大字行高仍 24px → 行挤在一起重叠 */

/* 正例：无单位，继承的是「倍数」 */
.parent { line-height: 1.5; }
/* child 行高 = 32px × 1.5 = 48px，自动适配 */
原理：line-height: 24px 继承的是计算后的固定值；line-height: 1.5 继承的是这个系数本身，子元素各自乘以自己的 font-size。所以无单位值是「自适应」的，几乎总是更优。正文行高 1.5~1.7 为佳。

五、letter-spacing / word-spacing

  • letter-spacing：字符间距。大写标题、字母 logo 加一点（如 0.05em）更透气；正文一般不动。
  • word-spacing：词间距（按空格算），对中文几乎无效（中文没空格分词）。

六、文本块属性：align / indent / decoration / transform

属性作用示例值
text-align水平对齐left / center / justify（两端对齐）
text-indent首行缩进2em（中文段落常用）
text-decoration下划线/删除线underline / line-through / none
text-transform大小写转换uppercase / capitalize
text-transform 只改显示，不改 DOM 里的真实文本——既能保留语义/可搜索性，又能统一视觉（如菜单全大写）。比手动敲大写更可维护。

七、字体渲染优化

  • -webkit-font-smoothing: antialiased：macOS 上让深色背景文字更细更清爽（按需用）。
  • text-rendering: optimizeLegibility：开启连字与字距微调，少量文字可用，大段慎用（影响性能）。
  • 给大段正文设定合理 行宽（约 45~75 字符，可用 max-width: 65ch），太长读起来累。
ch 单位 = 字符「0」的宽度，max-width: 65ch 是控制阅读行宽的经典手法，比写死 px 更贴合字体。
