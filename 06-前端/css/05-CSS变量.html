CSS 变量 var()

CSS 自定义属性（Custom Properties）是真正活在浏览器运行时的「变量」——它们参与层叠与继承、能被 JS 实时读写，是现代主题系统与设计 token 的核心机制。

1. 自定义属性（Custom Properties）是什么

CSS 规范把它正式命名为 Custom Properties，而非「变量」。这个名字有深意：它本质上是一个合法的 CSS 属性，和 color、font-size 处于同等地位，只是你可以自由命名、自由赋值。

定义规则：名称必须以两个连字符 -- 开头，后接任意标识符，且大小写敏感（--Color 与 --color 是两个不同属性）。用 var(--name) 引用。

CSS定义 & 引用
/* 定义：写在某个选择器块内，像普通属性一样 */
:root {
  --primary-color: #60a5fa;
  --spacing-md:    16px;
  --font-body:     "Inter", sans-serif;
}

/* 引用：用 var() 函数 */
.button {
  background: var(--primary-color);
  padding:    var(--spacing-md);
  font-family: var(--font-body);
}
为什么叫「自定义属性」而非「变量」？
因为它确实是 CSS 属性：它参与层叠（Cascade）、参与继承（Inheritance），可以被更高特异性的规则覆盖，也会沿 DOM 树向下传递给后代元素。这与编程语言中「词法作用域变量」的行为截然不同，理解这一点是避免很多陷阱的关键。

大小写敏感 — 常见陷阱

CSS大小写敏感示例
:root {
  --Color: red;    /* --Color */
  --color: blue;   /* --color  ← 两个不同的属性！ */
}

p {
  color: var(--Color);  /* 红色 */
}

2. var() 的 fallback 默认值

var() 接受第二个参数作为回退值（fallback）：当变量未定义或处于无效计算值状态时，浏览器使用这个默认值。默认值本身可以再嵌套 var()，形成链式回退。

CSSfallback 用法
.card {
  /* 基础：未定义时用 #334155 */
  background: var(--card-bg, #334155);

  /* 链式：先找 --card-border，再找 --border-color，最后用 transparent */
  border-color: var(--card-border, var(--border-color, transparent));

  /* 默认值可以是多词（逗号前的整体是变量名，逗号后到末尾是 fallback） */
  font-family: var(--font-heading, "PingFang SC", sans-serif);
}
注意：fallback 的触发条件
fallback 只在变量「未定义」或「无效计算值（invalid at computed-value time）」时触发，不会在变量值为空字符串时触发。如果你写了 --x: ;（值为空），var(--x, red) 不会使用 red，而是用空值（可能导致属性被重置为初始值）。
逗号语义：var() 只识别第一个逗号作为分隔符。因此 fallback 值中可以安全地包含逗号，如 var(--font, "PingFang SC", "Microsoft YaHei", sans-serif) 是合法的，整个字体栈都是 fallback。

3. 作用域与继承——沿 DOM 树向下传递

自定义属性完全遵循 CSS 的继承机制。把它定义在哪个元素上，就在那个元素及其所有后代上可用。子元素可以重新定义同名变量来覆盖父级的值，这个覆盖只影响该子元素及其后代，不影响兄弟节点。

CSS局部覆盖示例
.parent {
  --accent: #60a5fa;          /* 蓝色，后代默认继承此值 */
}

.child-a {
  /* 不重新定义，继承父级 --accent = #60a5fa */
  border: 1px solid var(--accent);
}

.child-b {
  --accent: #f472b6;          /* 局部覆盖为粉色，只影响 .child-b 及其子孙 */
  border: 1px solid var(--accent);
}

.child-b .grandchild {
  /* 继承 .child-b 的 --accent，仍然是粉色 */
  color: var(--accent);
}

下面是真实渲染效果：左边是父级定义，中间是子级覆盖，右边是子级覆盖后的孙级继承。

.parent
--accent: 蓝
border 蓝色
.child-a（继承）
继承 --accent: 蓝
border 同蓝
.child-b（覆盖）
--accent: 粉
border 粉色
与普通 CSS 属性继承的区别：普通属性如 color 只有「可继承」和「不可继承」两类固定属性才会继承。自定义属性默认参与继承，如果你不希望某个自定义属性被子元素继承，目前没有原生机制可以阻止（除了在子元素上用 initial 关键字重置）。

4. :root 全局变量——设计 Token 的集中管理

:root 选择器匹配 HTML 文档的根元素（即 <html>），特异性略高于直接写 html。在 :root 上定义的自定义属性等价于全局变量，整个文档所有元素都能继承使用。这是现代 CSS 管理「设计 token」的标准约定。

CSS设计 Token 集中管理
:root {
  /* === 颜色 token === */
  --color-primary:     #60a5fa;
  --color-primary-dark: #2563eb;
  --color-danger:      #f87171;
  --color-success:     #4ade80;
  --color-bg:          #0f1117;
  --color-surface:     #1e2030;
  --color-text:        #e2e8f0;
  --color-text-muted:  #94a3b8;

  /* === 间距 token === */
  --space-xs:  4px;
  --space-sm:  8px;
  --space-md:  16px;
  --space-lg:  24px;
  --space-xl:  40px;

  /* === 圆角 token === */
  --radius-sm: 4px;
  --radius-md: 8px;
  --radius-lg: 12px;
  --radius-full: 9999px;

  /* === 字体 token === */
  --font-sans: "Inter", "PingFang SC", sans-serif;
  --font-mono: "Fira Code", Consolas, monospace;
  --text-sm:   0.875rem;
  --text-base: 1rem;
  --text-lg:   1.125rem;

  /* === 阴影 token === */
  --shadow-sm: 0 1px 2px rgba(0,0,0,.3);
  --shadow-md: 0 4px 12px rgba(0,0,0,.4);
}

/* 消费 token：组件只引用变量，不再出现魔法数字 */
.btn-primary {
  background:    var(--color-primary);
  padding:       var(--space-sm) var(--space-md);
  border-radius: var(--radius-md);
  font-family:   var(--font-sans);
  box-shadow:    var(--shadow-sm);
}
设计 Token 分层命名建议：--color-primary 是语义层（表达用途），不要直接用 --blue-500（颜色层）作为组件变量。当你换主题时只需改语义层的值，组件代码完全不用动。这与 Tailwind CSS 的 CSS variables 模式、Radix UI 的 token 设计完全一致。

5. CSS 变量 vs Sass/Less 变量——运行时 vs 编译时

这是最容易被初学者忽略、面试高频考的核心区别。两者表面相似，本质截然不同。

维度 CSS 自定义属性 Sass/Less 变量
生命周期 运行时（runtime）——存活在 CSSOM 中，浏览器解析后仍然以变量形式存在 编译时（compile-time）——预处理器把值内联替换后，变量就消失了，浏览器看不见
JS 交互 可以用 getComputedStyle 读取，element.style.setProperty('--x', v) 写入，实现运行时动态修改 无法被 JS 读取或修改，因为已经变成静态 CSS 值
继承 & 层叠 参与 CSS 继承和层叠机制，可在不同元素上有不同值 静态文本替换，不参与任何 CSS 机制
媒体查询 可在 @media 块内重新赋值，响应式改变变量值 无法在运行时根据媒体查询动态切换
主题切换 只需改一个选择器上的变量值，整棵子树自动更新 需要编译两套 CSS 文件，JS 切换 <link> 标签
DevTools 可见性 在浏览器 DevTools Elements 面板中可见、可实时编辑 不可见（只能看到最终的具体值）
浏览器支持 现代浏览器全支持（IE 不支持） 编译为标准 CSS，所有浏览器支持
嵌套计算 需要 calc() 才能做数学运算 可直接用 $a + $b 进行运算
CSSCSS 变量响应媒体查询（Sass 做不到）
:root {
  --spacing: 12px;    /* 移动端默认间距 */
  --font-size: 14px;
}

@media (min-width: 768px) {
  :root {
    --spacing: 20px;   /* 桌面端自动升级，所有引用 --spacing 的组件都响应 */
    --font-size: 16px;
  }
}

/* 所有组件只需引用变量，无需各自写媒体查询 */
.card {  padding: var(--spacing);  }
.list {  gap:     var(--spacing);  }
.text {  font-size: var(--font-size);  }
实际项目中两者互补：Sass 擅长编译时的逻辑（mixin、循环、函数），CSS 变量擅长运行时的动态。现代项目常见做法：用 Sass 生成 :root { --x: ... } 这段代码，再在运行时用 CSS 变量提供动态能力。

6. 主题切换实战

核心思路：用 :root 或属性选择器（如 [data-theme="dark"]）定义两套变量值，组件只引用变量名。切换主题就是切换哪套变量值生效，组件代码一行不改。

CSS亮/暗主题定义
/* 亮色主题（默认） */
:root,
[data-theme="light"] {
  --bg:      #f8fafc;
  --text:    #1e293b;
  --accent:  #2563eb;
  --border:  #e2e8f0;
  --muted:   #64748b;
}

/* 暗色主题 */
[data-theme="dark"] {
  --bg:      #0f172a;
  --text:    #e2e8f0;
  --accent:  #60a5fa;
  --border:  #1e3a8a;
  --muted:   #94a3b8;
}

/* 组件只引用变量，不关心当前是哪个主题 */
.card {
  background:   var(--bg);
  color:        var(--text);
  border:       1px solid var(--border);
  border-radius: 10px;
  padding:      18px 20px;
}
.card h4 { color: var(--accent); }
.card p  { color: var(--muted);  }

用 JS 动态切换主题

JavaScript运行时切换主题
// 方式 1：切换 data-theme 属性（推荐，语义清晰）
document.documentElement.setAttribute('data-theme', 'dark');

// 方式 2：直接用 JS 修改某个变量的值（局部动态调整）
document.documentElement.style.setProperty('--accent', '#f472b6');

// 方式 3：在某个子元素上局部修改（不影响全局）
const card = document.querySelector('.card');
card.style.setProperty('--bg', '#1e2030');

// 读取计算后的变量值
const value = getComputedStyle(document.documentElement)
  .getPropertyValue('--accent').trim();
console.log(value);  // "#60a5fa"
结合 prefers-color-scheme：可以在 @media (prefers-color-scheme: dark) 内重定义 :root 变量，实现跟随系统主题，再用 JS 的 data-theme 方案允许用户手动覆盖。两者可以共存。

7. 陷阱与面试考点

陷阱 1：var() 只能用于属性值，不能用于属性名或选择器

CSS反例 → 问题 → 正例
/* ❌ 反例：试图用变量作为属性名 */
:root { --prop: color; }
p { var(--prop): red; }  /* 语法错误，直接无效 */

/* ❌ 反例：试图用变量构建选择器 */
:root { --cls: button; }
.var(--cls) { color: red; }  /* 无效，不是合法选择器 */

/* ✅ 正例：var() 只用于属性值位置 */
:root { --brand: red; }
p { color: var(--brand); }

陷阱 2：不能直接拼接单位，必须用 calc()

CSS反例 → 问题 → 正例
:root { --n: 16; }  /* 纯数字，没有单位 */

/* ❌ 反例：字符串拼接不等于数学运算 */
p { font-size: var(--n)px; }
/* 问题：浏览器会把 "16px" 视为无效值（var() 替换后变成 "16px"，
   中间有空格等价于两个 token，font-size 接受一个长度值，不接受这种写法）*/

/* ✅ 正例：用 calc() 附加单位 */
p { font-size: calc(var(--n) * 1px); }

/* 更常见：直接在变量值里带单位（推荐），除非你需要无单位数字做计算 */
:root { --font-size: 16px; }
p { font-size: var(--font-size); }  /* ✅ 正常 */

陷阱 3：invalid at computed-value time（无效计算值）

这是 CSS 变量最微妙的陷阱，也是面试高频考点。当一个变量被替换进去后，整体值类型不合法，浏览器不会用 fallback 的同名属性值，而是降级到属性的初始值（initial）或继承值。

CSSinvalid at computed-value time
:root {
  --bad-color: 42px;   /* px 不是合法颜色值 */
}

p {
  color: red;                          /* 先设置红色 */
  color: var(--bad-color);           /* 替换后 color: 42px → 无效 */
}
/* 结果：不是红色！浏览器不回滚到上一行的 red，
   而是使用 color 属性的继承值（或 initial 值 canvastext）。
   这与普通无效属性值的行为不同：普通情况下浏览器会忽略当前行，用上一行。
   但 var() 替换在解析时被认为是「语法上合法」的，无效是在计算阶段才发现的，
   所以层叠已经完成，无法再回退到同属性的上一条声明。 */
面试要点：var() 解析的两阶段
浏览器对 CSS 变量的处理分两步：第一步「语法解析」——含 var() 的声明在语法上是合法的，不会被丢弃；第二步「计算值替换」——真正替换变量值后才检查合法性。如果此时不合法（invalid at computed-value time），属性使用继承值（可继承属性）或初始值（不可继承属性），而不是回退到同属性的前一条规则。这是与普通 CSS 无效值处理最大的区别。

陷阱 4：自定义属性的初始值是 guaranteed-invalid value

CSS未声明变量的默认行为
/* --undefined-var 从未被定义 */
p {
  /* var() 替换失败，触发 fallback */
  color: var(--undefined-var, hotpink);  /* → hotpink ✅ */

  /* 没有 fallback，替换失败 → invalid at computed-value time */
  color: var(--undefined-var);           /* → 继承值或初始值 ❗ */
}

/* 声明了但值为空（注意：这与未声明不同） */
:root { --empty:  ; }  /* 值是空白，不是 guaranteed-invalid */
p {
  /* fallback 不触发！--empty 是「已定义的空值」 */
  color: var(--empty, red);  /* 不会用 red，而是 color: （空）→ 无效值 */
}

陷阱 5：大小写敏感与命名约定

CSS大小写敏感 + 命名建议
:root {
  --primaryColor: blue;    /* camelCase */
  --primary-color: red;   /* kebab-case（推荐，与 CSS 风格一致） */
  --PrimaryColor: green;  /* PascalCase */
}
/* 上面是三个不同的变量！团队务必统一命名风格，推荐 kebab-case */
面试高频：5 个核心考点速记
1. 变量名大小写敏感，--Color ≠ --color
2. var() 只能用于属性值，不能用于属性名或选择器
3. 无效计算值（invalid at computed-value time）不回退到同属性前一条规则，而是用继承/初始值
4. 无单位数字拼接单位需用 calc(var(--n) * 1px)
5. 已定义但值为空的变量不触发 fallback，与未定义变量行为不同

总结：核心机制一览

特性行为
定义语法--name: value;，两个连字符开头，大小写敏感
引用语法var(--name) 或 var(--name, fallback)
作用域定义在哪个元素，该元素及后代可用；:root 即全局
继承默认参与继承，子元素可覆盖同名变量
fallback 触发条件变量未定义或无效计算值时，不含「定义了空值」
与 JS 交互getComputedStyle 读，setProperty 写
响应媒体查询可在 @media 内重新赋值，实现响应式 token
vs Sass 变量运行时存活；Sass 变量编译后消失，不可动态修改
单位拼接必须用 calc(var(--n) * 1px)，不能直接 var(--n)px
