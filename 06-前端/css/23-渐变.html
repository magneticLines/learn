渐变

渐变不是颜色，是「程序生成的图片」。搞懂这一点，你就能用它画按钮、饼图、分隔线、网格，甚至代替 PNG 切图。

一、本质：渐变是 background-image

最重要的认知：linear-gradient() 等函数返回的是一张图片，所以它只能用在接受 <image> 的属性上——主要是 background-image（或 background 简写），不能写在 background-color 里。

最常见错误：background-color: linear-gradient(...) 完全无效。渐变是图片，必须用 background 或 background-image。同理它能享受 background-size、多重背景、background-clip: text 等所有背景图能力。

二、linear-gradient：线性渐变

方向与角度
/* 默认从上到下（180deg） */
background: linear-gradient(#60a5fa, #a78bfa);
/* 关键字方向 */
background: linear-gradient(to right, #60a5fa, #a78bfa);
/* 角度：0deg=向上，90deg=向右，顺时针 */
background: linear-gradient(135deg, #60a5fa, #a78bfa);
角度坑：CSS 渐变的 0deg 指向上方，顺时针增加。这和数学里 0° 指向右、逆时针不同。90deg = 向右，180deg = 向下（默认）。

色标（color stop）与硬边界

每个颜色后面可跟一个位置（百分比或长度），控制它「在哪到位」。当两个相邻色标位置相同时，颜色不再过渡而是「啪」地切换——这就是用渐变画纯色条纹/分隔的技巧。

带色标 + 硬边界
/* 平滑过渡：蓝在 0%~40%，到 100% 变紫 */
linear-gradient(to right, #60a5fa 40%, #a78bfa)
/* 硬边界：50% 处两色位置相同 → 左半蓝右半紫，无过渡 */
linear-gradient(to right, #60a5fa 50%, #a78bfa 50%)

三、radial-gradient：径向渐变

从一个圆心向外辐射。可指定形状（circle/ellipse）、大小（closest-side/farthest-corner 等）和圆心位置（at）。

radial-gradient
radial-gradient(#fff, #60a5fa) /* 默认椭圆铺满 */
radial-gradient(circle at top, #fff, #60a5fa)
radial-gradient(circle 60px at center, #fbbf24 0 50%, transparent 51%)
/* 上面这行可以画一个实心圆点（小圆 + 硬边界透明） */

四、conic-gradient：锥形渐变（饼图/色轮）

颜色绕中心旋转分布，而非辐射。这让它天生适合画饼图、色轮、进度环——几行 CSS 就能搞定，无需 SVG 或 canvas。

conic-gradient 画饼图
/* 三段扇形：用角度硬边界切出饼图 */
background: conic-gradient(
  #60a5fa 0 40%,
  #f472b6 40% 70%,
  #fbbf24 70% 100%
);
border-radius: 50%; /* 配圆角变成真饼图 */

/* 完整色轮 */
background: conic-gradient(red, yellow, lime, cyan, blue, magenta, red);

五、repeating-* 渐变：重复纹理

给渐变加 repeating- 前缀，到达最后一个色标后会从头重复，用来画条纹、网格、进度条纹理。

repeating-linear-gradient 条纹
background: repeating-linear-gradient(
  45deg, #60a5fa 0 10px, #1e2030 10px 20px
); /* 斜向蓝条纹，周期 20px */

六、常见实战用法

场景做法
渐变按钮 + hover 变深linear-gradient 背景，hover 时换更暗的两色
图片底部压暗图片上叠 linear-gradient(transparent, #000)（多重背景）
细分隔线1px 高元素铺 linear-gradient(90deg, transparent, #fff, transparent) 做渐隐线
骨架屏闪光动 background-position 的斜向渐变
渐变边框外层 linear-gradient 背景 + 内层纯色，靠 padding 露出边框
性能提示：渐变是 GPU 友好的矢量绘制，比加载位图省流量、不失真。但动画化渐变本身（如改色标）开销大，应改为动画 background-position 或 transform，让浏览器走合成层。

与其他知识点的联系

  • 背景属性（上一节）：渐变函数返回的是一张「图片」，所以它天生共享 background-image 的全部能力——多重背景叠加（渐变 + 照片做压暗遮罩）、background-size/position 控制显示范围，都是上一节背景机制的延伸应用，不是独立的新语法。
  • background-clip: text：渐变文字效果依赖的正是背景一节讲过的 background-clip: text，把渐变这张「图」裁到文字形状再让文字透明。渐变提供颜色内容，clip 提供裁剪方式，两者缺一不可。
  • border-radius：conic-gradient 画饼图必须配 border-radius: 50% 才能从方块裁成圆形——渐变负责颜色分布，圆角负责形状裁剪，这是两个独立职责的组合，理解了这点就不会疑惑「为什么饼图还要单独设圆角」。
  • animation / 性能优化：骨架屏闪光效果靠动画化 background-position，而非直接动画渐变的色标——因为改色标要重新计算渐变函数触发重绘，而移动 background-position 相对更轻。这与后面「will-change 与性能优化」一节强调的「只动便宜属性」是同一条原则的具体应用。
  • filter：渐变和 filter: hue-rotate()/saturate() 都能产出彩色视觉效果，但渐变是「画」出确定的颜色分布，filter 是对已渲染像素做「二次加工」，二者常叠加使用（比如渐变按钮再叠一层 brightness 做 hover 高亮）。
