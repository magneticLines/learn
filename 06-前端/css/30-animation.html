animation 关键帧动画

用 @keyframes 编排任意多个中间状态，做出能自动播放、循环、来回往返的复杂动画。

transition 的局限与 animation 的诞生

transition 只能描述"A→B"两个状态的过渡，且必须有触发事件。但很多动画——loading 转圈、心跳脉冲、页面入场——需要自动播放、无限循环、并经过多个中间状态。@keyframes 就是为此设计的：先用关键帧"编排"动画剧本，再用 animation-* 属性"导演"它怎么播。

@keyframes：定义关键帧

关键帧用 from / to（即 0% / 100%）或百分比标记时间点上的样式快照，浏览器负责在快照之间插值。

CSS
/* 两端写法 */
@keyframes fadeIn {
  from { opacity: 0; }
  to   { opacity: 1; }
}

/* 多帧写法：百分比可定义任意中间态 */
@keyframes bounce {
  0%, 100% { transform: translateY(0); }
  50%      { transform: translateY(-30px); }
}
技巧：相同样式的关键帧可以合并写在一起（如 0%, 100%）。没写出来的属性会保持元素原值；某一帧没声明的属性，会从相邻已声明帧插值。

animation 子属性与简写

属性作用常用值
animation-name引用哪个 @keyframesfadeIn
animation-duration一轮时长（必填）2s
animation-timing-function缓动曲线ease linear steps()
animation-delay延迟开始0.5s
animation-iteration-count播放次数infinite / 3
animation-direction播放方向normal reverse alternate
animation-fill-mode动画前后元素停在何样式none forwards both
animation-play-state运行/暂停running paused
CSS
/* 简写顺序（记忆：名字 时长 曲线 延迟 次数 方向 填充） */
animation: bounce 1s ease-in-out 0s infinite alternate both;
易错点 1 — fill-mode：默认 none，动画结束后元素瞬间弹回原始样式。想让它停在最后一帧，必须设 forwards。
易错点 2 — 时间值顺序：和 transition 一样，简写里第一个时间是 duration，第二个是 delay。
易错点 3 — alternate 与次数：alternate 下偶数轮反向播放，要看到完整来回需要 iteration-count 至少为 2（或 infinite）。

transition vs animation 怎么选

  • 用 transition：由用户交互/状态切换驱动、只有两端、需要自动反向回退（hover、展开收起）。
  • 用 animation：需要自动播放、循环、多个中间帧、精细方向控制（loading、入场、吸引注意的脉冲）。

性能法则两者通用：关键帧里尽量只改 transform 和 opacity，避免在帧里动 width/top/margin 这类触发重排的属性，否则每帧都卡。

可访问性：尊重 prefers-reduced-motion

部分用户因前庭功能障碍会被大幅动画引发眩晕。操作系统提供"减少动态效果"开关，CSS 可通过媒体查询响应它——这是专业前端的必备素养，而非可选项。

CSS
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;  /* 近乎瞬间，等于关掉动画 */
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
不要直接 animation: none 一刀切——有些动画承载了"加载中"等关键信息。更好的做法是大幅缩短或简化，保留功能性提示，去掉装饰性运动。

与其他知识点的联系

  • will-change 与性能优化（下一节）：本节反复强调「关键帧里尽量只改 transform 和 opacity」，下一节会从浏览器渲染管线的角度解释为什么——这两个属性只触发 Composite 合成阶段，跳过昂贵的 Layout/Paint，是 animation 能做到丝滑 60fps 的底层原因。
  • filter：animation 同样可以驱动 filter 的值变化（比如逐渐去色、逐渐模糊），filter 动画也会走合成层，但计算量比 transform/opacity 更大，是「能走 GPU 但不算最便宜」的一档，做大范围模糊动画时要留意帧率。
  • border-radius / 渐变：关键帧里给渐变背景做 background-position 动画（骨架屏闪光）、或让圆角在 hover 时渐变到圆形，都是 @keyframes 编排多状态能力和视觉样式属性的组合应用，animation 提供「怎么播」，具体样式属性提供「播什么」。
  • 文字溢出：入场动画常见的场景是标题文字先淡入再定位，如果标题本身需要单行省略号，要注意 -webkit-line-clamp 依赖的特殊盒模型（display: -webkit-box）在某些浏览器下和 transform 动画组合时可能有渲染层级问题，实践中需要分别测试。
  • BEM / CSS 方法论：一组关键帧动画配合触发状态（如 .spinner--active、.card--enter）适合用 Modifier 类名去挂载/移除，而不是用行内样式硬切 animation-play-state，这样动画的开关逻辑能沿用 BEM 一贯的状态命名习惯。
