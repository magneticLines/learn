position 定位

position 让元素脱离"老老实实按流排"的规则，可以精确摆放、可以叠在别人上面、可以钉在屏幕上。它是角标、弹层、固定导航、回到顶部按钮的底层机制。

五个值总览

position 决定一个元素相对谁来定位，以及 top/right/bottom/left（合称偏移量）是否生效。

值是否脱离文档流偏移量参照典型用途
static否（默认）偏移量无效正常流，默认值
relative否（占位仍在）相对自己原位置微调位置、给子级当定位锚
absolute是最近的定位祖先角标、下拉、弹层
fixed是视口(viewport)固定头/尾、回到顶部
sticky否（临界前是相对）滚动容器吸顶标题、表头

static 与 relative

static 是默认值，元素老实待在普通流里，给它写 top: 20px 完全没反应。relative 则在不脱离流（原来的位置照样占着）的前提下，允许你用偏移量把它从原位置视觉上挪开，但它留下的"坑"还在，旁边元素不会来补。

relative 真正的高频用途不是挪自己，而是给 absolute 子元素当参照系——见下文"定位上下文"。

absolute：脱离文档流

设了 position: absolute 的元素会彻底脱离文档流：它原来的位置不再为它保留，后面的元素会上来把空填上，就像它从未存在。然后它根据偏移量，相对"最近的已定位祖先"来摆放。

关键概念：定位上下文（包含块）

absolute 元素的 top/left 到底相对谁？答案是：向上找最近的、position 不为 static 的祖先，相对它的内边距盒来定位。如果一路找不到，就相对初始包含块（约等于视口）。

经典套路：父 relative + 子 absolute。给父元素加 position: relative（它自己不会有任何视觉变化），就把它变成了子 absolute 元素的"坐标原点"。这是实现角标、卡片内浮层的标准配方。
右上角红点角标
.card   { position: relative; } /* 锚点 */
.badge  {
  position: absolute;
  top: -6px; right: -6px; /* 相对 .card 右上 */
}

fixed：钉在视口上

fixed 和 absolute 类似（都脱离流），区别是它永远相对视口定位，页面滚动时纹丝不动。常用于固定顶部导航、右下角"回到顶部"按钮、全屏遮罩层。

陷阱：如果某个祖先元素设了 transform、filter 或 will-change，fixed 子元素会改为相对那个祖先定位，而不是视口——这是"我的 fixed 弹层为什么跟着滚动了"的常见元凶。原理和下一节"层叠上下文"相通：这些属性会创建新的包含块。

sticky：滚动吸附

sticky 是个"混血"：滚动到临界位置之前它表现得像 relative（正常待在流里、占位置），滚动到你设定的阈值（如 top: 0）之后就像 fixed 一样钉住，直到它的父容器滚出视野才一起离开。

sticky 不生效的常见原因

  • 没写阈值：必须指定 top/bottom/left/right 之一，否则不知道何时吸附。
  • 父容器高度不够：sticky 只在它的父容器内滚动时吸附，父容器一旦滚完就跟着走。如果父容器和 sticky 元素一样高，根本没有可滚动空间，看起来就"不吸顶"。
  • 祖先设了 overflow: hidden/auto/scroll：会改变 sticky 的滚动参照容器，常导致失效。

实战小结

  • 角标 / 卡片内浮层：父 relative + 子 absolute。
  • 固定导航 / 回到顶部：fixed + right/bottom，注意避开 transform 祖先。
  • 吸顶表头 / 章节标题：sticky + top: 0，确保父容器足够高。
  • 微调元素几像素：relative + 偏移量（不影响其他元素）。
心智模型：position 改变的是"参照谁"和"占不占位"。relative 占位、参照自己；absolute/fixed 脱离流、分别参照祖先/视口；sticky 在两种状态间切换。理解了"包含块"这个参照系，定位就不再是玄学。

与其他知识点的联系

  • 关联 z-index 与层叠上下文（下一节）：position 是触发 z-index 生效的前提条件之一——static 元素写 z-index 完全无效，而 relative/absolute/fixed/sticky 才会让 z-index 参与层叠比较。本节的"定位上下文"和下一节的"层叠上下文"是两套相似又不同的概念，容易混淆，需要对照着理解。
  • 关联 Flexbox / Grid：Flex/Grid 子项即使不设置 position，只要 z-index 非 auto 也能参与层叠排序——这是本节"position 决定 z-index 是否生效"规则的一个例外扩展，说明现代布局模型已经把层叠能力下沉到子项本身。
  • 关联 float 与清除浮动：float 和 absolute 都会让元素"脱离文档流"，但脱离方式不同：float 元素仍参与行内内容环绕，absolute 元素则完全独立于文档流之外、由最近的定位祖先决定坐标。对比这两种"脱离流"的方式，能更准确理解"文档流"这个贯穿 CSS 布局的核心概念。
  • 关联媒体查询与移动优先：移动端常见的底部导航栏、悬浮操作按钮都依赖 position: fixed，但手机浏览器地址栏的动态高度变化会影响 fixed 元素的视觉表现，这也是相对单位一节里 dvh 被引入的原因之一——两者共同解决移动端"固定定位与视口高度不稳定"的问题。
  • 关联 React 组件布局：React 里的 Modal、Tooltip、Dropdown 等浮层组件普遍依赖"父 relative + 子 absolute"或 fixed 定位，很多组件库还会用 Portal 把浮层挂载到 body 下以彻底摆脱父级定位上下文和层叠上下文的限制，这正是本节"定位上下文"概念在真实项目里最直接的应用。
