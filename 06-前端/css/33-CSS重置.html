CSS 重置与 Normalize

你还没写一行样式，页面就已经有了默认外观——那是浏览器自带的 UA stylesheet。如何让多个浏览器的起跑线一致？这就是 Reset 与 Normalize 要解决的问题。

一、问题：浏览器默认样式（UA stylesheet）

每个浏览器内置一套用户代理样式表（User Agent stylesheet），给 HTML 元素定义默认外观：h1 大且加粗、body 有 8px 的 margin、ul 有左 padding 和圆点、a 是蓝色下划线……没有它，纯 HTML 会完全没有视觉层次。

麻烦在于：各浏览器的默认值并不完全一致。同一段表单、同一个标题，在 Chrome、Safari、Firefox 里的边距、字号、行高可能略有差异。你以为是自己代码的 bug，其实是 UA stylesheet 在「背着你」设值。

经典例子：<body> 默认带 margin: 8px。新手常困惑「为什么页面四周有一圈白边、背景色铺不满」，根源就是这条 UA 默认样式。

二、两种哲学：Reset（归零）vs Normalize（抹平）

Reset：把一切清零

代表作是 Eric Meyer's Reset。思路简单粗暴：把几乎所有元素的 margin、padding、border、font-size 全部清零，让所有元素「长得一样朴素」，然后你从一张白纸开始重建所有样式。

Eric Meyer Reset（节选）
html, body, div, h1, h2, h3, p, ul, ol, li, a, table {
  margin: 0;
  padding: 0;
  border: 0;
  font-size: 100%;
  font: inherit;
}
ol, ul { list-style: none; }
/* 结果：h1 不再加粗变大、ul 没有圆点，全部回到「裸」状态 */

Normalize.css：保留有用的，抹平不一致的

Normalize.css 的哲学相反：它不清零，而是有针对性地修正各浏览器之间的差异，同时保留那些有意义的默认值。h1 依旧加粗变大、ul 依旧有圆点，只是确保它们在所有浏览器里表现一致。它还修复了一批已知 bug（如 IE 的某些渲染问题）并改善可访问性。

Normalize.css（节选思路）
/* 不归零 h1，只统一在 section 内的字号/边距，抹平浏览器差异 */
h1 { font-size: 2em; margin: 0.67em 0; }
/* 修正 Safari 中 button 字体不继承的问题 */
button, input { font-family: inherit; }
对比ResetNormalize
哲学全部清零，从白纸开始保留有用默认，只抹平差异
对 h1/ul变成无样式裸态仍保留语义化外观
工作量所有元素样式都要自己重建开箱即用，差异已被处理
体积较小稍大（含 bug 修复与注释）
适合有完整设计系统、要精确掌控每个像素普通项目、想省事又一致

三、反例 → 问题 → 正例

反例：什么都不处理，直接写业务样式。

反例：裸奔，依赖浏览器默认
/* 没有任何重置，直接写组件 */
.hero { background: navy; }

问题：body 的 8px margin 让背景色四周露白；Safari 和 Chrome 表单控件边距不一；box-sizing 默认 content-box 让加了 padding 的盒子算宽度时频频「溢出」。这些都不是你的 bug，却要你来背锅。

正例：先铺一层最小重置，统一起跑线，再写业务。

四、现代做法：最小重置（Minimal Reset）

如今很少人再用整套 Eric Meyer 全清零，而是采用一段「轻量、精挑」的现代重置。下面是被广泛采用的核心几条（思路源自 Andy Bell 等人的现代 reset）：

现代最小重置
/* 1. 统一盒模型为 border-box，宽度计算更符合直觉 */
*, *::before, *::after { box-sizing: border-box; }

/* 2. 清掉常见块级元素的默认外边距 */
body, h1, h2, h3, p, figure, blockquote, ul, ol { margin: 0; }

/* 3. 图片默认是 inline，底部会有诡异空隙；改成 block 并自适应 */
img, picture, svg, video { display: block; max-width: 100%; }

/* 4. 表单控件默认不继承字体，显式让它继承 */
input, button, textarea, select { font: inherit; }

/* 5. 给正文一个合理的默认行高 */
body { line-height: 1.5; }
第 3 条尤其值得记：img 默认 display: inline，会按文字基线排版，因此图片底部常出现几像素的「神秘空隙」。把它设为 block 就能根治这个困扰无数新手的问题。

五、何时用哪种

  • 普通项目 / 想省心：直接引 Normalize.css，或用上面的最小重置即可。
  • 有完整设计系统、要逐像素掌控：倾向 Reset 风格，从零定义所有元素。
  • 用 UI 框架（Bootstrap、Tailwind 等）：它们通常内置了自己的重置（如 Tailwind 的 Preflight），无需再额外引入。
陷阱：不要在大型项目里同时引 Reset 和 Normalize，两者哲学相反会互相打架。也别把重置代码散落在多个文件里——它应当是最先加载的一段全局基底，放在样式入口的最顶部。

与其他知识点的联系

  • BEM 命名规范（上一节）：CSS 重置和 BEM 解决的是同一类问题的不同层面——重置统一「浏览器给的默认起点」，BEM 统一「团队自己写的类名和选择器结构」。两者都是为了让 CSS 变得可预测，通常会在同一个项目的样式体系里同时出现，重置在最底层，BEM 组织业务样式。
  • 字体与排版：字体一节里讲的字体栈降级、line-height 无单位写法，很多其实是在做「手动局部重置」——弥补各浏览器 font-family 继承、默认行高不一致的问题，现代最小重置里第 4、5 条规则本质上是把这类手动修正提前到了全局层面。
  • border-radius / 背景属性：重置里 box-sizing: border-box 这条规则直接决定了 padding-box/border-box 这些盒模型区域的计算基准，而 background-clip、圆角裁切都是基于这些区域工作的——统一盒模型是所有涉及盒模型区域的视觉属性能够按预期工作的前提。
  • 文字溢出：重置第 3 条「img 默认 display: block」修的是「图片底部神秘空隙」这个和 inline 元素基线对齐有关的经典问题，这类问题和文字溢出一样，本质都是「不了解元素默认渲染行为导致的意外」，排查思路是相通的——先怀疑默认样式，再怀疑自己的代码。
  • 性能：重置代码应该内联在首屏关键 CSS 或尽早加载，因为它决定了后续所有样式计算的基准值，如果晚加载会导致页面先按浏览器默认样式渲染、重置生效后再跳一次样式，也就是可感知的「布局抖动」（layout shift），这和后面性能优化里「批量读写避免抖动」是不同场景下的同一类症状。
