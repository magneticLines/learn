文字溢出

「标题太长撑破布局」「英文长串顶出容器」——这些日常 bug 的根源都是文字溢出。这一节把单行省略、多行省略、强制换行三大处理方式一次讲透。

一、white-space：先决定要不要换行

处理溢出前，先理解 white-space——它控制空白和换行的处理方式，是其它技巧的前提：

值换行合并空白说明
normal自动是默认，多个空格合成一个
nowrap不换行是强制单行，溢出——单行省略号的前提
pre仅遇 \n否像 <pre>，保留所有空格和换行
pre-wrap自动+\n否保留空白但仍会自动换行
pre-line自动+\n是合并空格但保留换行符

二、单行省略号「三件套」

text-overflow: ellipsis 必须三个属性配合才生效，缺一不可——这是最经典的考点：

反例 → 问题 → 正例
/* 反例：只写 text-overflow，完全没效果 */
.title { text-overflow: ellipsis; }
/* 问题：文字既会换行又没溢出，ellipsis 无从触发 */

/* 正例：三件套 */
.title {
  white-space: nowrap;    /* 1. 不换行，逼出溢出 */
  overflow: hidden;      /* 2. 隐藏溢出部分 */
  text-overflow: ellipsis; /* 3. 溢出处显示 … */
}
这是一个非常非常长的标题，超出容器宽度后会显示省略号

white-space:nowrap + overflow:hidden + text-overflow:ellipsis

为什么三个缺一不可？没有 nowrap 文字会换行不溢出；没有 overflow:hidden 文字会直接溢出可见；text-overflow 只是告诉浏览器「被裁掉的地方画个 …」。它依赖前两者制造出「裁切」这件事。

三、长单词/长串换行：word-break 与 overflow-wrap

另一类问题：一长串没有空格的字符（如长 URL、Hash、邮箱）会顶破容器，因为浏览器找不到可换行的断点。两个属性解决它，但策略不同：

属性策略建议
overflow-wrap: break-word正常按单词换；只有单词太长放不下时才在单词内部断✅ 首选，最温和
word-break: break-all不管单词，到边界就断，哪怕把英文单词拦腰截断慎用，破坏英文可读性
word-break: keep-allCJK 文本不在字内换行中日韩特殊场景
处理长 URL
.content { overflow-wrap: break-word; }
/* 优先用 overflow-wrap：只在必要时断词，
   普通英文仍按单词换行，可读性最好 */
反例 https://example.com/aVeryLongPathWithoutAnySpaces

默认：长串顶破容器（看右边溢出/横向滚动）

正例 https://example.com/aVeryLongPathWithoutAnySpaces

overflow-wrap: break-word：自动断行收进容器

四、多行省略号：-webkit-line-clamp

需求升级：限制最多显示 N 行，超出加省略号（如卡片描述限 3 行）。纯 CSS 方案是 -webkit-line-clamp，虽带前缀但各大浏览器都支持：

多行省略
.desc {
  display: -webkit-box;
  -webkit-box-orient: vertical;
  -webkit-line-clamp: 3; /* 最多 3 行 */
  overflow: hidden;
}
/* 新标准属性 line-clamp 正在落地，可一起写做前瞻 */
这是一段很长的卡片描述文本，超过两行之后会被截断并在末尾自动加上省略号，常用于商品卡片或文章列表的摘要展示。

-webkit-line-clamp: 2（限两行）

两个坑：①line-clamp 必须配 display: -webkit-box 和 -webkit-box-orient: vertical，三者一组。②它对定高/混排内联元素有兼容差异，且省略号位置无法精确控制（不能保证正好在行尾文字处）。需求复杂时才考虑 JS 测量方案。

五、决策小结

需求方案
标题超长 → 单行 …三件套：nowrap + overflow:hidden + text-overflow:ellipsis
描述限 N 行 → 多行 …-webkit-line-clamp + -webkit-box
长 URL 顶破容器overflow-wrap: break-word
必须每个字符都能断word-break: break-all（慎用）

与其他知识点的联系

  • 字体与排版（上一节）：文字溢出处理的一切前提都建立在排版结果之上——line-height 决定了 -webkit-line-clamp: N 实际裁出的高度，font-size 变化会让原本刚好不溢出的文字突然触发省略号，两节内容是「先决定怎么排、再处理排不下怎么办」的因果关系。
  • border-radius：单行省略号三件套里的 overflow: hidden，和圆角裁切图片用的也是同一个 overflow: hidden——本质都是「让容器裁掉超出自己范围的内容」，只是溢出的对象一个是文字一个是子元素图片，机制完全一致。
  • 性能（重排重绘）：-webkit-line-clamp 依赖 display: -webkit-box 这种特殊盒模型来测量行数，如果容器尺寸频繁变化（如响应式断点切换），浏览器要重新计算截断位置，属于 Layout 级别的开销，不适合用在需要高频动画的场景。
  • CSS 重置：不同浏览器对 white-space、word-break 的默认继承和表单元素文字溢出的处理并不完全一致，这也是现代 CSS 重置里常会顺手统一 overflow-wrap 默认行为的原因之一。
  • BEM / CSS 方法论：「标题单行省略」「描述多行省略」是几乎每个卡片组件都要重复的样式组合，适合封装成语义化的工具类（如 .text--truncate、.text--clamp-2）并在 BEM 结构里作为 Modifier 复用，而不是在每个组件里重新拼三件套。
