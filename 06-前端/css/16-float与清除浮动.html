float 与清除浮动（进阶）

float 是 CSS 的"老兵"——它本为文字环绕图片而生，却被一代前端拿来硬撑整页布局，留下了高度塌陷、clearfix 等一堆历史包袱。读懂它，才能读懂为什么今天我们用 Flex/Grid。

float 的本意：文字环绕

float 诞生时的唯一目的，是实现报纸杂志那种图片靠边、文字绕着图排的效果。给图片设 float: left，它就贴到左侧，后面的文字会自动从它右边和下方流过去环绕它。这是 float 至今唯一仍然推荐的正经用途。

图片
这是一段会环绕图片的文字。float 的设计初衷正是这种排版：图片浮到左侧，文字自然地从它的右边和下方流过去，形成杂志般的图文混排效果。这是 float 唯一至今仍被推荐的用途。

被滥用做布局的年代

在 Flexbox（2012 起逐渐可用）和 Grid（2017 普及）出现之前，CSS 根本没有专门的布局工具。前端为了做"两栏""三栏""九宫格"，只能征用 float——让多个块都 float: left 挤成一排，再用百分比宽度凑出栅格。整个 Bootstrap 3 的栅格系统就是这么搭的。

这是把工具用在它没被设计的地方。float 不是为布局而生，于是带来了一连串麻烦，其中最著名的就是"高度塌陷"。

反例 → 问题：高度塌陷

当一个父容器里的子元素全部浮动时，这些子元素已脱离普通流，父容器会以为"我没有内容"，于是高度坍缩为 0，背景、边框全都包不住浮动的孩子：

反例：父容器高度塌陷
.parent { background:#333; } /* 期待包住孩子，实际高度=0 */
.child  { float:left; width:80px; height:50px; }

清除浮动的几种解法

1. clear 属性

clear: both 让一个元素"拒绝待在任何浮动元素旁边"，强制换到它们下方。最朴素的做法是在浮动元素后加一个空 <div style="clear:both">，但这要往 HTML 里塞无意义的空标签，很脏。

2. clearfix hack（经典方案）

用伪元素在父容器内部"末尾"自动生成一个清除浮动的隐形块，无需脏 HTML。这是 float 时代每个项目都会抄的一段代码：

正例：clearfix
.clearfix::after {
  content: "";
  display: block;
  clear: both; /* 撑开父容器 */
}

加了 clearfix 后，绿框正确地包住了浮动子项：

3. 触发 BFC（更现代的一招）

给父容器加 overflow: hidden 或 display: flow-root，会让它形成一个BFC（块级格式化上下文）。BFC 的一条规则是"计算高度时要把内部浮动元素算进去"，于是塌陷自动消失。display: flow-root 是专为此设计的现代写法，无副作用：

最干净的清浮动
.parent { display: flow-root; } /* 触发 BFC，一行搞定 */
overflow: hidden 虽然也能触发 BFC，但它会裁掉超出的内容（比如阴影、下拉菜单），是有副作用的"巧合方案"。能用 display: flow-root 就别用它。

为什么现代布局抛弃了 float

诉求float 时代现代（Flex/Grid）
水平排列float + 清浮动 + 算百分比display:flex 一行
垂直居中几乎做不到，各种 hackalign-items:center
等高列负 margin / JS 撑高Flex 默认 stretch
子项间距margin 算来算去gap
响应式列数大量媒体查询Grid auto-fit

本质区别：float 是"排版副作用"被借来当布局，Flex/Grid 是真正为布局而设计的引擎。它们不脱离流（不会塌陷）、有专门的对齐属性、有间距控制。所以今天除了"文字环绕图片"，不应再用 float 做布局。

小结：float 的正经用途只剩文字环绕。它做布局时的高度塌陷，是因为浮动元素脱离了流。清除手段从"空 div"进化到 clearfix，再到今天最干净的 display: flow-root（触发 BFC）。理解这段历史，是为了明白现代 Flex/Grid 解决了哪些真实痛点——它们不是花哨的新玩具，而是对一个时代教训的回应。

与其他知识点的联系

  • 关联 Flexbox / Grid：本节反复强调"float 是排版副作用被借来当布局"，理解了 float 时代要靠百分比宽度和 clearfix 才能勉强拼出的多栏布局后，再看 Flex 一行 display: flex、Grid 一行 grid-template-columns 就能真正体会到"专为布局设计"和"将非布局属性挪用为布局手段"之间的本质差距。
  • 关联 position 定位：float 和 absolute 都会让元素"脱离文档流"，但脱离方式完全不同——float 元素仍会被后续的行内内容环绕，而 absolute 元素彻底独立于流之外。把这两者放在一起对比，能更精确理解"脱离文档流"这句话在不同语境下的具体含义。
  • 关联 z-index 与层叠上下文：层叠顺序表中"浮动元素"被列为独立的一层（高于普通流 block、低于 inline 内容），这也是历史遗留的 float 布局里经常出现"图片浮动后遮挡关系很怪"的原因，本节的 float 机制正是那份层叠顺序表的注脚。
  • 关联媒体查询与移动优先：float 时代做响应式栅格（如 Bootstrap 3）需要给每个断点单独计算浮动列的百分比宽度，代码量巨大；而现代 Grid 的 auto-fit + minmax() 能免掉大部分媒体查询，这组对比是理解"为什么现代 CSS 更青睐声明式响应式方案"的一个具体例证。
  • 关联 React 组件布局：在 React 组件里几乎不会再用 float 做布局，唯一会保留的场景是富文本内容渲染（如文章详情页 dangerouslySetInnerHTML 出来的 HTML 里恰好有图文环绕需求），这正是本节强调的"float 唯一仍被推荐的用途"在现代前端工程中的存续场景。
