滤镜 filter（进阶）

一行 CSS 给元素套上模糊、变灰、调色、投影——还能做毛玻璃。但 filter 会改变渲染管线，用对了惊艳，用滥了卡顿。

一、filter 是什么

filter 把元素渲染成的位图，再经过一系列图像处理函数加工后显示，类似修图软件的滤镜。它作用于元素自身及其全部内容（含文字、子元素、背景），是「渲染后的二次加工」。

三、多滤镜叠加

用空格分隔多个函数，按从左到右的顺序依次作用（顺序会影响结果！）：

叠加
.photo { filter: grayscale(1) contrast(1.2) brightness(1.1); }
/* 先去色，再增对比，最后提亮 → 黑白胶片风 */

四、drop-shadow vs box-shadow

两者都做投影，但有关键区别——这是高频考点：

box-shadowfilter: drop-shadow()

三角形上：box-shadow 投出方框，drop-shadow 沿三角形边缘投影。

五、backdrop-filter：毛玻璃效果

filter 作用于元素自己，而 backdrop-filter 作用于元素背后的区域——这正是 iOS 风格毛玻璃（frosted glass）的实现：模糊的是底下的内容，元素本身清晰。

毛玻璃卡片
.glass {
  background: rgba(255, 255, 255, 0.1); /* 半透明底，能透出背景 */
  backdrop-filter: blur(10px);          /* 模糊背后内容 */
  -webkit-backdrop-filter: blur(10px); /* Safari 前缀 */
}
毛玻璃 backdrop-filter
backdrop-filter 必须配半透明背景才看得到效果（全不透明背景会盖住背后内容，无从模糊）。另外它对性能要求较高，移动端大面积使用要测试。

六、性能：filter 触发合成层

非 none 的 filter 会让浏览器把元素提升为独立合成层（compositing layer），交给 GPU 处理。这有两面性：

  • 好处：动画化 filter（如 hover 渐变模糊）走 GPU，比改 width/color 这类触发重排重绘的属性流畅。
  • 代价：每个合成层占显存；blur 半径大、面积大时 GPU 计算量陡增，低端设备会掉帧。
优化建议：①动画只动 transform/opacity 最稳，filter 次之；②大范围 backdrop-filter: blur 慎用；③静态滤镜（如固定灰度）开销很小，放心用；④需要时可用 will-change: filter 提示浏览器提前准备合成层。

七、filter vs opacity 的区别

都能影响视觉、都会创建层叠上下文，但 opacity 只能整体调透明，而 filter 是一整套图像运算（模糊、调色、投影……）。需要「半透明」用 opacity 或颜色 alpha；需要「图像加工」才上 filter。两者可叠加使用。

与其他知识点的联系

  • 背景属性与渐变：渐变本质是背景图，filter 却是对整个元素渲染结果的二次加工，两者可以叠加——比如给渐变背景加 filter: blur() 做柔光效果，理解「先画背景（含渐变），再走 filter 加工」这个先后顺序，就不会混淆两个阶段各自的能力边界。
  • border-radius：filter: drop-shadow() 跟随元素实际不透明像素的轮廓投影，这意味着给圆角元素用 drop-shadow 和用 box-shadow 效果几乎一样（都会顺着圆角画弧形阴影），但对非矩形（如三角形、抠图 PNG）差异巨大，圆角只是「矩形轮廓」这个特例。
  • will-change 与性能优化（后续章节）：本节讲到非 none 的 filter 会让浏览器把元素提升为独立合成层，这正是后面「渲染管线」一节里 Composite 阶段的直接体现——filter 和 transform/opacity 一样能走 GPU 合成，但 backdrop-filter 的模糊计算量仍然很大，是这套「便宜属性」体系里相对昂贵的例外。
  • animation：做「毛玻璃 hover 渐显」「灰度转彩色」这类效果时，通常用 transition/animation 去驱动 filter 的值变化；由于 filter 走合成层，动画帧率通常优于直接动画 background-color 等触发重绘的属性，但仍不如单纯的 transform/opacity 动画轻量。
  • BEM / CSS 方法论：「灰度遮罩」「毛玻璃卡片」这类滤镜效果适合抽成状态类 Modifier（如 .card--disabled 对应 grayscale(1)），让语义化的类名代替裸露的滤镜数值，方便多处复用同一套视觉语言。
