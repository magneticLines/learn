transform 变换

不改变文档流就能平移、缩放、旋转、倾斜元素——动画性能的关键，也是 2D/3D 视觉效果的基石。

核心心智模型

transform 改变的是元素的视觉呈现，而不是它在文档流中占据的真实位置。这意味着：你把一个 div 用 translateX(200px) 移到右边，它原来的"坑位"还在，旁边的元素不会来填补——这正是它不触发重排的根本原因（详见性能一节）。

四种基本变换函数

CSS
transform: translate(40px, 10px);  /* 平移：x, y（也有 translateX/Y/Z） */
transform: scale(1.5);            /* 缩放：1=原大小，0.5=一半，可分轴 scale(2,1) */
transform: rotate(45deg);         /* 旋转：正值顺时针 */
transform: skew(15deg, 0);        /* 倾斜：让矩形变平行四边形 */
百分比的参照：translate 的百分比是相对元素自身尺寸，不是父容器。translateX(50%) = 自身宽度的一半，这个特性是"居中技巧"的关键。

多变换叠加：顺序会影响结果

一个 transform 可以串联多个函数，但从左到右依次应用，且坐标系会被前一个变换带着走。所以 translate(100px) rotate(45deg) 和 rotate(45deg) translate(100px) 结果完全不同。

陷阱：先 rotate 再 translate，平移方向会沿着已旋转的坐标轴，元素会斜着飞出去；先 translate 再 rotate，则元素先到位、再原地转。绕原点旋转的卫星轨道效果就靠这个顺序差异实现。

transform-origin：变换基点

缩放、旋转默认以元素几何中心（50% 50%）为基点。改成 transform-origin: top left，旋转就会绕左上角转，像门轴一样。这在做"扇形展开""翻页"等效果时必不可少。

3D 变换与 perspective 透视

加上 Z 轴就进入 3D：translateZ（靠近/远离屏幕）、rotateX / rotateY（绕水平/垂直轴翻转）。但没有透视的 3D 是"假 3D"——元素会像正交投影一样平贴，看不出纵深。perspective 提供了"观察者眼睛到屏幕的距离"，值越小透视越夸张。

CSS
/* 方式一：父元素统一透视（推荐，多个子元素共享灭点） */
.scene { perspective: 600px; }
.card  { transform: rotateY(40deg); }

/* 方式二：写在自身的 transform 里（每个元素独立灭点） */
.card  { transform: perspective(600px) rotateY(40deg); }
悬停翻转

父元素 perspective 让 rotateY 产生真实的近大远小纵深。

3D 配套属性：容器加 transform-style: preserve-3d 才能让子元素保留各自的 3D 位置（做立方体、翻牌时必需）；backface-visibility: hidden 控制元素背面翻过去后是否可见（翻牌动画藏背面用）。

用 transform 做绝对居中

经典的"绝对定位居中"利器，在没有 flex/grid 的旧环境或需要相对自身偏移时仍很常用：

CSS
.center {
  position: absolute;
  top: 50%; left: 50%;          /* 左上角到达容器中心 */
  transform: translate(-50%, -50%); /* 再往回拉自身一半，元素中心对齐容器中心 */
}

妙处在于：top/left 的 50% 相对容器，而 translate 的 -50% 相对元素自身，两者配合无需知道元素的具体宽高。

为什么 transform 是动画首选

联系渲染管线：改 left/top/width 会让浏览器重新做 Layout（重排），进而 Paint、Composite，整条流水线跑满；而 transform（连同 opacity）只触发最后的 Composite（合成），浏览器把元素提升为独立 GPU 图层，仅靠图层平移/缩放矩阵运算就能完成每一帧，跳过昂贵的 Layout 和 Paint。这就是"动画只动 transform/opacity"黄金法则的底层依据。
