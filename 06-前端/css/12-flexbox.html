Flexbox 弹性布局

Flexbox 是为"一维布局"而生的——一行或一列里的元素如何分配空间、如何对齐。它终结了用 float、inline-block 凑布局和"垂直居中"的痛苦年代。

它解决了什么问题

在 Flex 之前，想做"两个元素，一个靠左一个靠右"、想做"垂直水平都居中"、想做"几个等宽且自动填满"，都要靠 float、负 margin、line-height 等各种 hack，脆弱且难维护。Flexbox 把这些诉求抽象成一套沿一条主轴分配空间、沿交叉轴对齐的统一模型。一旦理解这个模型，绝大多数日常布局都迎刃而解。

核心模型：主轴与交叉轴

这是理解 Flex 的钥匙。给容器设 display: flex 后，它就有了两条轴：

  • 主轴（main axis）：由 flex-direction 决定。默认 row，主轴是水平方向（从左到右）。
  • 交叉轴（cross axis）：永远和主轴垂直。row 时交叉轴是竖直方向。

关键点：justify-content 永远管主轴，align-items 永远管交叉轴。一旦你把 flex-direction 改成 column，主轴变成竖直方向，这两个属性的"水平/垂直含义"就会对调——这是初学者最容易绕晕的地方。记住"对轴不对方向"就不会错。

子项属性：flex 简写

容器决定整体规则，子项可以微调自己的伸缩行为。最重要的是 flex 简写，它由三个值组成：

flex 简写展开
.item { flex: 1; }  /* 等价于下面三行 */
.item {
  flex-grow: 1;    /* 有剩余空间时，按比例放大 */
  flex-shrink: 1;  /* 空间不够时，按比例缩小 */
  flex-basis: 0%;  /* 分配前的初始尺寸基准 */
}
  • flex-grow：剩余空间的分配权重。两个子项一个 grow:1 一个 grow:2，剩余空间按 1:2 分。
  • flex-shrink：空间不足时的收缩权重。设 0 表示"打死不缩"。
  • flex-basis：参与分配前的"起点宽度"，优先级高于 width。
陷阱：flex: 1 和 flex-grow: 1 不一样！flex: 1 的 basis 是 0，意思是"忽略内容固有宽度，纯按比例瓜分全部空间"，所以多个 flex:1 真正等宽。而单独写 flex-grow: 1 时 basis 是 auto，会先按内容撑开再分剩余空间，内容多的那个会更宽。要等宽列请用 flex: 1。

align-self 与 order

align-self 让单个子项覆盖容器的 align-items，比如一排都顶部对齐，唯独某个底部对齐。order 可以改变视觉顺序（默认 0，越小越靠前）而不动 HTML 结构——但要小心：它只改视觉不改 DOM 顺序，键盘 Tab 和读屏仍按源码顺序，滥用会损害可访问性。

实战布局

1. 万能居中（曾经的世纪难题）

水平垂直完美居中
.center {
  display: flex;
  justify-content: center; /* 主轴居中 */
  align-items: center;     /* 交叉轴居中 */
}

3. 等高列

Flex 子项在交叉轴上默认 stretch，意味着即使内容多少不同，几列也会自动等高。这在过去要靠脚本或负 margin 才能勉强实现，现在是免费的副作用。

小结：Flex 是一维布局之王。记住三件事：① justify 管主轴、align 管交叉轴；② flex: 1 才是真等宽（basis 为 0）；③ 间距优先用 gap 而不是 margin。当你需要"行和列同时对齐"的二维网格时，才该轮到 Grid 出场。
