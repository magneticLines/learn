选择器

选择器是 CSS 的「瞄准系统」——它决定哪些 DOM 节点会被规则命中。理解选择器，就是理解浏览器如何把样式精确投递到每一个元素。

1. 基础选择器

CSS 提供了五类基础选择器，覆盖了绝大多数「精准定位」需求。它们是其他一切复杂选择器的积木。

选择器语法含义特异性（specificity）
元素（标签）div p h1匹配所有该标签的元素0-0-1
类.card匹配 class 含该值的元素0-1-0
ID#main匹配 id 等于该值的唯一元素1-0-0
通配*匹配所有元素0-0-0
属性[href] [type="text"]匹配含特定属性/值的元素0-1-0

元素选择器

直接写标签名，命中页面上该标签的所有实例。适合设置全局基线样式，例如重置 ul 的默认 list-style。

CSS
/* 命中页面所有 <p> 元素 */
p {
  color: #94a3b8;
  line-height: 1.8;
}

类选择器（.class）

最常用的选择器。一个元素可以拥有多个 class，类选择器只要包含即命中。推荐优先使用类选择器做样式，原因在第 5 节详述。

CSS
/* 命中所有 class 含 "btn" 的元素 */
.btn {
  padding: 8px 16px;
  border-radius: 6px;
}

/* 多类组合：class 必须同时含 "btn" 和 "btn-primary" */
.btn.btn-primary {
  background: #3b82f6;
}

ID 选择器（#id）

CSS
/* 命中 id="header" 的唯一元素 */
#header {
  position: sticky;
  top: 0;
}

通配选择器（*）

命中一切元素，常用于全局 reset。但要注意性能陷阱：单独的 * 做 reset 没问题，与后代组合（如 .container *）则会触发大量匹配计算，见第 6 节。

CSS
/* 经典 box-sizing reset */
*, *::before, *::after {
  box-sizing: border-box;
}

属性选择器（[attr]）

根据 HTML 属性的存在性或值来匹配，特异性与类选择器相同（0-1-0）。共有 6 种匹配模式：

语法含义示例
[attr]有该属性即匹配（不管值）[disabled] 匹配所有禁用元素
[attr=val]属性值精确等于 val[type="checkbox"]
[attr^=val]值以 val 开头（^ = start）[href^="https"] 匹配安全链接
[attr$=val]值以 val 结尾（$ = end）[href$=".pdf"] 匹配 PDF 链接
[attr*=val]值包含 val（* = contain）[class*="icon"]
[attr~=val]值是空格分隔列表中的一项[class~="active"]
CSS — 属性选择器实战
/* PDF 链接自动加图标 */
a[href$=".pdf"]::after {
  content: " ↓PDF";
  font-size: 0.75em;
  color: #f87171;
}

/* 所有 data-* 自定义属性存在时高亮 */
[data-tooltip] {
  cursor: help;
  border-bottom: 1px dashed currentColor;
}

/* 大小写不敏感匹配（末尾加 i）*/
[lang="zh" i] { font-family: "PingFang SC", sans-serif; }

2. 组合选择器

组合选择器描述两个元素之间的结构关系。同一段 HTML，不同的组合符会命中完全不同的节点集合——这是初学者最容易混淆的地方。

组合符名称关系
A B（空格）后代B 是 A 的任意层级后代
A > B直接子代B 是 A 的直接子元素
A + B相邻兄弟B 是紧跟在 A 之后的同级元素
A ~ B通用兄弟B 是 A 之后所有同级元素

下面用同一段 HTML 演示四种组合符的命中差异（彩色背景代表被命中的元素）：

HTML 结构
<div class="s2-parent">
    <p class="s2-p">子代 p（直接子）</p>
    <div class="s2-inner">
      <p class="s2-p">孙代 p（间接后代）</p>
    </div>
</div>
<p class="s2-sib1">兄弟1（紧邻后）</p>
<span>中间隔开</span>
<p class="s2-sib2">兄弟2（后续兄弟）</p>
后代选择器（空格）：.s2-parent p — 命中所有层级的 p

子代 p ✓

[div.s2-inner 内]

孙代 p ✓

子代选择器（>）：.s2-parent > p — 只命中直接子

子代 p ✓

[div.s2-inner 内]

孙代 p ✗

相邻兄弟选择器（+）：.s2-parent + p — 只命中紧跟的兄弟

兄弟1 ✓

中间 span（被隔断）

兄弟2 ✗（不紧邻）

通用兄弟选择器（~）：.s2-parent ~ p — 命中之后所有同级 p

兄弟1 ✓

中间 span（不受影响）

兄弟2 ✓

陷阱：A + B 中间一旦有其他元素隔开，B 就不再是 A 的「相邻」兄弟，规则不命中。A ~ B 则不受中间隔挡影响，只要 B 在 A 之后的同级即可。

3. 伪类（Pseudo-class）

伪类描述元素的状态或结构位置，用单冒号 : 语法。它不改变 DOM，只是在特定条件下临时命中元素。

常用伪类速查

伪类触发条件常见用途
:hover鼠标悬停按钮高亮、卡片阴影
:focus键盘/鼠标聚焦input 边框变色，可访问性
:active鼠标按下瞬间按钮点击反馈
:visited链接已访问变色区分已读/未读链接
:first-child是父元素的第一个子节点去掉列表首项 margin-top
:last-child是父元素的最后一个子节点去掉列表末项 border-bottom
:nth-child(n)是父元素第 n 个子节点表格斑马纹
:nth-of-type(n)是同类型兄弟中第 n 个见下方详解
:not(selector)不匹配括号内选择器排除特殊项
:checkedcheckbox/radio 已选中纯 CSS 开关效果
:disabled表单元素被禁用灰化禁用控件
:empty没有子节点（含文本）隐藏空容器

nth-child vs nth-of-type —— 高频陷阱

这是面试最爱考的混淆点。核心区别：

  • :nth-child(n) —— 先数「第 n 个子节点」，再检查是否匹配前面的选择器。顺序由 DOM 节点位置决定，与标签类型无关。
  • :nth-of-type(n) —— 在同标签类型的兄弟中数第 n 个。类型隔离计数，与其他标签无关。
HTML — 演示 nth-child vs nth-of-type
<div class="s3-box">
  <h2>标题</h2>         <!-- 子节点 1 -->
  <p>段落 A</p>         <!-- 子节点 2，p 中第 1 个 -->
  <p>段落 B</p>         <!-- 子节点 3，p 中第 2 个 -->
</div>
CSS — 结果对比
/* p:nth-child(1) → 第 1 个子节点是 h2，不是 p，所以什么都不命中！ */
p:nth-child(1) { color: red; }

/* p:nth-child(2) → 第 2 个子节点是 p，命中"段落 A" */
p:nth-child(2) { color: red; }

/* p:nth-of-type(1) → p 类型中第 1 个，命中"段落 A"，不受 h2 干扰 */
p:nth-of-type(1) { color: blue; }
nth-child 的 An+B 公式：nth-child(2n) = 偶数行，nth-child(2n+1) = 奇数行。语法糖：even / odd。nth-child(3n+1) 命中 1、4、7、10…

:not() 排除选择器

CSS
/* 除了最后一个 li，其余都加分隔线 */
li:not(:last-child) {
  border-bottom: 1px solid #2d3148;
}

/* CSS4：:not() 可接受复杂选择器列表 */
input:not([type="submit"], [type="reset"]) {
  border: 1px solid #475569;
}

4. 伪元素（Pseudo-element）

伪元素用 :: 双冒号语法，它在 DOM 中不存在对应节点，而是由浏览器渲染时「凭空插入」的虚拟元素。最大价值：无需额外 HTML 就能实现装饰效果，保持结构语义干净。

单冒号 vs 双冒号的历史

CSS2 时代伪类和伪元素都用单冒号（:before/:after）。CSS3 引入双冒号（::before/::after）来区分伪类与伪元素。为了向后兼容，浏览器至今仍接受单冒号写伪元素，但新代码一律用双冒号。

伪元素作用注意
::before在元素内容最前插入虚拟节点content 属性必填，否则不渲染
::after在元素内容最后插入虚拟节点同上
::placeholder设置 input placeholder 样式继承 color 但不继承 opacity
::first-line文本块的第一行只能用字体/文字相关属性
::first-letter文本块的第一个字母/字符常做首字下沉效果
::selection用户选中文字时的高亮只支持 color/background-color 等少数属性
CSS — ::before / ::after 核心用法
/* content 可以是字符串、空字符串（装饰用）、attr() 读属性值 */
.tooltip {
  position: relative;
}

.tooltip::after {
  content: attr(data-tip);   /* 读取 data-tip 属性值 */
  position: absolute;
  bottom: 120%;
  left: 50%;
  transform: translateX(-50%);
  background: #1e2030;
  padding: 4px 10px;
  border-radius: 4px;
  white-space: nowrap;
  opacity: 0;
  pointer-events: none;
  transition: opacity 0.2s;
}

.tooltip:hover::after {
  opacity: 1;
}
悬停查看 Tooltip（::after）
  • 用 ::before 生成自动编号
  • 无需额外 HTML 标签
  • CSS counter 实现，纯样式逻辑
陷阱：::before / ::after 不能用在自闭合标签上。<img>、<input>、<br> 没有「内容区」，插入伪元素无效。这是常见 bug 来源。

5. class vs ID：使用建议

很多初学者习惯用 #id 来定位元素——毕竟 ID 是唯一的，听起来更「精准」。但在实际工程中，强烈推荐用 class 而非 ID 来挂样式。

维度classID
特异性0-1-0（低）1-0-0（高，比 10 个 class 叠加还高）
复用性一个 class 可挂多个元素一个页面应唯一，不可复用
覆盖难度低，易被后续规则覆盖极高，需要再堆更多 ID 或用 !important
JS 钩子不建议混用（行为/样式分离）ID 应专用于 JS 锚点（getElementById）
HTML 有效性可重复出现规范要求页面唯一，重复会导致 JS 只找到第一个
反例 → 问题 → 正例
/* ❌ 反例：用 ID 写样式，特异性爆炸 */
#submit-btn {
  background: blue;
}
/* 问题：当你想在某个页面覆盖这个颜色时，
   普通 class 选择器完全压不过 ID 的特异性（1-0-0），
   只能堆更多 ID 或用 !important，造成「特异性军备竞赛」*/

/* ✅ 正例：样式用 class，ID 留给 JS */
.btn-submit {
  background: blue;
}
/* 覆盖时只需加一层 class 或利用层叠顺序，干净可控 */
.form-danger .btn-submit {
  background: red;  /* 0-2-0，轻松覆盖 */
}
工程约定：BEM、Tailwind、CSS Modules 等主流方案全部基于 class。ID 只用于页面内锚点导航（href="#section"）和 JS 操作（document.getElementById）。把 ID 从样式层彻底移除，是大型项目样式可维护性的基础。

6. 浏览器如何匹配选择器：从右往左

直觉上，CSS 选择器 .nav a 看起来是「先找 .nav，再找里面的 a」——从左往右。但浏览器的实际实现恰好相反：从右往左。

为什么从右往左？

以 .nav a 为例：

  • 从左往右（直觉方式）：找到所有 .nav，再遍历它们的所有后代，检查每个是否是 a。每次 DOM 更新都要从根重新遍历，代价极高。
  • 从右往左（浏览器方式）：拿到当前 a 元素，沿祖先链向上查找，判断是否有 .nav 的祖先。多数情况下，第一步「是不是 a」就能快速排除大量元素，无需遍历整棵子树。

从右往左的本质：以当前元素为起点向上过滤，用最具体的「关键选择器」（rightmost key selector）做第一道筛。命中率越低的关键选择器，排除越快，性能越好。

CSS — 性能陷阱对比
/* ❌ 反例1：通配符作为关键选择器
   浏览器从「每一个」元素出发往上查，无法快速排除 */
.sidebar * { color: inherit; }

/* ❌ 反例2：过长后代链，每级都要查祖先
   html body .page .container .card ul li a span */
html body .page .container .card ul li a span {
  color: red;
}
/* 问题：链条越长，每个 span 元素需要向上爬的层数越多；
   且脆弱：DOM 结构改动就会失效 */

/* ✅ 正例：直接用一个语义化的 class，关键选择器特异性强、命中快 */
.card-link-text {
  color: red;
}
现代浏览器已经很聪明了。V8/Blink 内部有选择器缓存（StyleRule cache）和快速路径优化，通常选择器不是性能瓶颈。但在动画帧内频繁触发 style recalculation（例如 scroll 事件里操作 class）时，低效选择器确实会产生可测量的掉帧。原则是：不要主动制造问题。

关键选择器（Key Selector）

浏览器把选择器最右边的简单选择器称为「关键选择器」，它是匹配的第一道门。关键选择器越具体，性能越好。

选择器关键选择器性能
.nav aa（标签，命中所有 a）一般
.nav .link.link（类，范围更小）较好
.nav-link.nav-link（一步到位）最好
.sidebar **（匹配一切，无法过滤）最差

7. 常见陷阱与面试高频考点

陷阱 1：选择器列表中一个语法错误会导致整条规则失效。
div, .card, p:unknow { color: red } —— p:unknow 是无效选择器，整个规则组被浏览器丢弃。CSS3 的 :is() / :where() 可以隔离错误，只丢弃列表中有问题的项。
陷阱 2：:nth-child 与混合标签。
父元素内有 h2 + 若干 p，用 p:nth-child(1) 想取第一个 p——实际命中的是「第 1 个子节点且是 p」，而第 1 个子节点是 h2，所以命中 0 个元素。用 p:nth-of-type(1) 才能达到目的。
陷阱 3：::before/::after 在 img/input 上无效。
自闭合元素没有内容区，伪元素无处插入。想给图片加装饰，需要包一层 div 或 figure。
陷阱 4：:hover 在移动端不可靠。
触屏设备没有「悬停」状态，:hover 可能在 tap 时短暂触发并粘住。交互反馈应优先使用 :active，或用 JS 事件处理。
面试考点 1：nth-child vs nth-of-type 区别。见第 3 节详解。
面试考点 2：浏览器选择器匹配方向。从右往左，关键选择器决定性能。见第 6 节。
面试考点 3：::before/::after 的 content 不能省略。哪怕是空字符串 content: "" 也要写，否则伪元素不会被渲染到页面。
面试考点 4：属性选择器的特异性。[attr] 的特异性与 class 相同（0-1-0），不是元素级（0-0-1）。
现代新特性：:is() 接受选择器列表并取列表中最高特异性；:where() 与 :is() 相同但特异性始终为 0，非常适合编写可覆盖的基础库样式。:has()（CSS4，父选择器）可以根据子孙元素的状态来样式化祖先，是 CSS 历史上最大突破之一。
